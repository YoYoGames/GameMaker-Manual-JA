コンパイラバッチファイル/スクリプト

GameMakerコンパイラでは、Batchファイル(Windows)やShellscripts (onmacOS andLinux)をプロジェクトディレクトリと、プロジェクト内の各extension のディレクトリに配置することができます。

配置

script ファイルは、プロジェクトのディレクトリのルートで、プロジェクトの.yypファイルと一緒に配置することができます。

script ファイルは、extension のディレクトリに配置することもできます。ディレクトリは次のようになります:<project_directory>/extensions/<extension_folder>/.

extension ファイルのscript バージョンが最初に実行され、そのルートバージョンは最後に実行されます。extensions の実行順序は制御できないので、extension の開発者は、ある拡張機能のscripts が別の拡張機能のscripts に依存しないようにしなければなりません。

実行

上記のいずれかのディレクトリに置かれた特定の名前のスクリプトファイルは、コンパイルプロセスのさまざまな時点で実行されます。

Windows バッチファイル名 macOS/Linux ShellScript 名前 商品説明
pre_run_step.bat pre_run_step.sh ゲーム展開が始まる前に実行されます
post_run_step.bat post_run_step.sh ゲームが準備され、実行できる状態になったときに実行され、スクリプトの実行後、ゲームが開始される
pre_build_step.bat（プリビルドステップバット pre_build_step.sh ゲーム構築のためにasset コンパイラが起動する前に実行されます。
post_build_step.bat（ポストビルドステップ）。 post_build_step.sh ゲーム構築のためにasset コンパイラが起動した後に実行されます。
pre_package_step.bat（プリパッケージステップバット pre_package_step.sh これは最終的なパッケージングステップの前に実行されます。つまり、すべてのファイルの準備はできているが、最終的なZIPファイルまたはストアパッケージが作成されようとしているときに実行されます
post_package_step.bat post_package_step.sh パッケージングの最終ステップが完了し、最終的なZIPファイルやストアパッケージが準備された後に実行されます
post_textures.bat post_textures.sh これは、ゲーム用のtextures が生成された後に実行されます。このステップを使用して、ゲームtextures を手動で最適化することができます。

このscript はTexturesDir という環境変数を受け取り、生成されたpath にtextures を格納します。

これらのscript ファイルは、すべてのターゲット・プラットフォームでビルドする際にサポートされます。

ExtensionのScripts を無効にする

extensions を実行する複数のscripts 、たとえば Steamworks と GDK を使用している場合、GDK にエクスポートするときは Steamworks を無効にし、その逆もまた然りです。
