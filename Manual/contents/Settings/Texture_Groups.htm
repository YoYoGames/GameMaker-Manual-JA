テクスチャグループ

テクスチャグループマネージャは、IDE のツールメニューから利用できます。 ここでは、テクスチャグループの追加、削除、名前の変更ができます。

GameMakerでは、グラフィックスassets (sprites,tilesets,fonts) をそれぞれ異なるtexture グループに割り当てて、ゲーム中にハードウェアが行うtexture のスワップの回数を最適化することができます。この機能を使用するには、このウィンドウから必要なtexture グループを定義し、適切なasset タイプをそれぞれ調べ、ドロップダウンメニューから所属させたい グループを選択する必要があります。

assets グループにリストされた複数のTexture を選択し、右クリックすると、それらのassets を別のグループに移動することができます。

また、アセットブラウザの右ボタン メニューからassets をtexture グループに追加することもできます。1つまたは複数のスプライト/タイルセット/フォントassets を選択し、RMBメニューを使用して、タイルセットについて以下に示すように、assets を任意のグループに追加することができます。

は、1つのテクスチャグループが複数のテクスチャページを持つことができ、それらは等価ではないことを理解することが重要です。ゲームに各レベルのユニークなグラフィックが多数ある場合、各レベルに1つのtexture グループを 定義しても、グラフィックはグループごとに2つ以上のtexture ページを 必要とすることがあります。これは、選択したtexture ページ サイズ（ターゲットのゲーム オプションで）が、texture グループ内のすべてのグラフィックを収めることができるかどうかに依存します。

デフォルトでは、ゲーム内のすべての画像は "Default"texture グループに追加され、常に存在します。しかし、「新規追加」ボタンをクリックすれば、新しいtexture グループを作成することができ、グループ名を選択すれば、適切な名前に変更することができます。まず、グループの各セクション（sprites 、tilesets 、fonts ）に対して「このグループは空です」と表示されますので、グラフィックassets 、それらをグループに追加する必要があります。リソースの追加は、下部にある「リソースの追加」ボタンを使用するか、リソースエディタから画像、texture 、またはtile set のfont グループを選択します。「デフォルト」以外のTexture グループはいつでも削除でき、その内容は「デフォルト」グループに移動します（間違ってグループを削除した場合は、 / + "Z" で元に戻せることに注意してください）。

texture グループを定義し、グラフィックアセットを割り当てると、メインウィンドウTexture Group Editor の左側にリスト表示されます。このリストでsprites をクリックすると、右側の画像プレビューウィンドウに表示され ます。この機能は、大規模なプロジェクトで、どの画像がどのグループに割り当てられているかを確認する必要がある場合に特に便利です。ただし、グループに割り当てられた画像に複数のサブ画像がある場合（つまりアニメーション）、プレビューウィンドウには最初のサブ画像のみが表示されます。また、リソースを右クリック（ ）すると、小さなメニューが表示され、そのリソースを別のtexture グループに移動したり、そのリソースのエディタを開いたりすることができます（asset ）。

assets が別個のものとしてリストに表示されていても、すべてのsprites 、tilesets 、fonts は同じtexture ページ（またはページ）に追加され、各リソースタイプに個別のページを取得しないことに注意する必要があります。このルールの唯一の例外は、sprite asset が「SeperateTexture Page」であるとフラグが立っている場合です。この場合、それ自身の個別のtexture ページが与えられます（詳細はスプライトエディタの「テクスチャ設定」のセクションを参照してください）。

texture のページとその生成方法については、次のページをご覧ください。

グループ設定

Texture Group Editor のグループ設定は、指定されたグループのtexture ページをどのように作成するかを正確に決定するために使用され、以下のオプションが用意されています。

  • Allow Scaling- このフラグを外すと、GameMakerはいかなる状況でもtexture グループをスケールしないように強制されます。これは特にユニバーサルアプリを扱うときに便利で、fonts のような特定のリソースを独自のグループに入れ、アプリが特定のデバイスで実行されたときに使用できるようにすることができる。たとえば、fonts は Retina ディスプレイではきれいに見えますが、通常のディスプレイで描画するために縮小されるとひどく見えることがあるので、特別な非スケーリンググループに割り当てることができます。iPhone 向けに開発している場合、texture スケーリングによって画像がぼやけるためにアプリが拒否されているので、これをチェックしておくとよいでしょう。これはデフォルトでオンに 設定されています。
  • 自動クロップ-GameMakerに画像を追加する際、周囲に「スペース」がある場合（つまり透明度100％）、texture ページに追加すると自動的にクロップされ、「自動クロップ」フラグが立てられます。しかし、例えば、画像をプリミティブの非反復texture などとして使用している場合、これはtexture マッピング用の UV 座標が正しく表現されないことを意味します。このような場合、このオプションのフラグを解除すると、sprite がtexture ページに「そのまま」追加され、エッジにある透過性が維持されます。このオプションは、デフォルトでオンに 設定されています。
  • ミップマップを生成する- このオプションにフラグを立てると（デフォルトではオフ）、オリジナルのtexture ページをtexture ファイル内で複数の解像度にスケーリング・フィルタリングするようGameMakerに指示します。通常はオフですが、カメラを使ってゲームの広い範囲を拡大・縮小している場合room 、または3Dゲームを扱っている場合は、これを有効にして、ミップマップ機能を使って、ゲームが異なるミップマップレベルをどう処理するかを調整するとよいでしょう。flag このチェックボックスをオンにすると、「Border Size」の値が8に設定されます（元々8未満に設定されていた場合）。これはミップマップを使う場合、texture のページで画像間のボーダーを大きくしないと、ミップレベルが高くなるとにじんでしまうからです。デフォルトの値である 8 は、画像が元のサイズの約 8 分の 1 にまで縮小され、サブ画像が一緒ににじみ出るのを防ぎますが、必要に応じてこの値を変更することができます。Texture の「Separatesprites Page」オプションを使用している場合、ミップマップを生成するために、ここで各texture を調べてflag する必要はありません。代わりに、「General Game Options」で、「Generate mipmaps for separateflag pages」オプションをtexture することができますので、注意して下さい。
  • Border Size- ボーダーサイズは、assets グループ用に生成されたページ上のすべてのtexture のエッジのオーバーラップ (または「ピクセル倍」) 量を変更し、画像が拡大されたときに、それらの間にギャップ (またはシーム) が見えなくなるようにするものです。デフォルトは2ピクセルですが、極端に表示を拡大したり縮小したりする場合は、この値を大きくする必要があるかもしれません。タイルセットエディタには、tiles の出力ボーダーを設定するオプションがあり（これが必要な理由の説明もあります）、これはここでのボーダー設定とは別です。基本的に、tiles は一時的なtile の各texture に出力ボーダーを付けて生成され、これがtexture ページの外縁にここで指定したボーダーを付けて追加されることに注意してください。また、Mipmaps を有効にすると、高い Mip レベルでtexture の間の「にじみ」を防ぐために、textures ページのボーダーのデフォルトサイズが自動的に 8 に設定されますが、これは変更することが可能であることに注意してください。
  • 親Texture グループ- ここで、既存のグループの中から、現在選択されているグループの「親」グループを選択できます。 「子」グループは、親inherit グループに加えられたすべての変更をtexture 、主にクロスプラットフォームのゲームを扱う際のリソース管理用として使用します。親を持つtexture グループは、それ自体がtexture グループであり、他のグループと同じようにtexture ページを生成しますが、親を割り当てることで、異なるプラットフォーム用の大きな塊を除外することができます。たとえば、Android playersprites を "player" グループに、Android enemysprites を "enemy" グループに追加して、両方とも "android" 親グループを使用するようにすることができます。そして、「android」親グループを、Windows または他のプラットフォームのビルドから除外すると、子グループも除外されます。
  • Texture Group Format- ここでtexture のページのフォーマットをBZ2 + QOI、QOI、PNG から選べます。デフォルトのオプションであるBZ2 + QOI は、品質を維持しながら最も良い圧縮を提供するので、このままにしておくことをお勧めします。

    BZ2 オプションがターゲットプラットフォーム（例：Android ）での速度に影響を与える場合はQOI を、エクスポートされた PNG を手動で操作したりPNGCrush オプションを使用したい場合はPNGを選択することが可能です。

これらの設定以外にも、ウィンドウのUse Group On...のセクションで適切なボックスをチェックしたり外したりするだけで、任意のプラットフォームに任意のtexture グループをエクスポートする（またはしない）ことを選択することができます。ただし、 <Default> texture グループは常にすべてのプラットフォームにエクスポートされ、チェックを入れたり外したりすることはできません（カスタムtexture グループのみ可能です）。

テクスチャグループもコンフィギュレーション設定と連動しています。つまり、ゲームの個々の構成ごとにtexture グループを定義し、sprite 、tile set 、font のリソースの割り当てをこれらの異なるグループに設定すると、別の構成に変更したときにそれが 「記憶」されるということです。Texture Group ウィンドウで選択したエクスポートオプションも同様で、texture グループを設定し、構成ごとに特定のプラットフォームにエクスポートすることができます。

例えば、iPad用のコンフィギュレーションを作り、2048x2048pxのtexture （ゲームオプションで設定）を作成することが一つの使い方です。そして、すべてのグラフィックスassets を適切なtexture グループに割り当てます。終了後、別の構成（たとえば HTML5 用）に変更し、Texture グループウィンドウから、512x512px の新しいtexture グループを作成し、これらの新しいグループにassets を割り当てることができます。また、ページの下部にはボタンがあり、texture グループ設定をある構成から別の構成にコピーすることができます。

最後に、texture グループはビットマップ画像にのみ関係し、非ビットマップsprites はtexture グループに割り当てられません。中には、すでに独自のtexture ページ（スケルトンアニメーションsprites ）を持つものもあれば、ベクターで構成されているためtextures を必要としないだけのものもあります。詳しくは、次のセクションをご覧ください。
