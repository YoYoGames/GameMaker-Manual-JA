Ubuntuのゲームオプション

このセクションでは、Ubuntu (Linux) ゲームプロジェクトのコンパイル方法を制御するためのさまざまなオプションの概要を説明します。各セクションの内容は以下のとおりです。

一般

このタブでは、実行中に表示されるゲームの名前を設定します。また、連絡先のメールアドレスと説明（短いものと長いもの）、ゲームや開発者のホームページのURLを提供することができます。

ここでは、ゲーム内でSteam を有効にするためのオプションもあります。このオプションを有効にする場合、Steamworks SDK（ここを参照）をインストールし、ゲームの一般オプションにSteamアプリIDを追加していることを確認します。

最後に、Disable file system sandboxというオプションがあります。これをチェックすると、ファイルのセーブとロードにおけるGameMakerのサンドボックスが無効になり、ゲームを実行しているシステム上のどこからでもファイルにアクセスできるようになります。このオプションはデフォルトではチェックされていません。すべてのシステムでファイルに対するGameMakerのサンドボックスを維持することを推奨しており、チェックしてもOSレベルのパーミッションによっては一部のセーブ/ロード領域が使用できない場合があるからです。サンドボックスの詳細については、GameMakerの ファイルシステムのページをご覧ください。

グラフィック

グラフィックスオプションは、ゲームがターゲットとするUbuntu マシンのグラフィックスカードをどのように使用するかを設定するためのものです。以下のオプションが用意されています。

  • フルスクリーンで起動する。チェックすると、フルスクリーンで起動し、チェックしないとウィンドウで起動します。デフォルトではオフになっています。
  • Allow Fullscreen Switching switch : これにチェックを入れると、ユーザはUbuntu OS の標準ショートカットを使って、フルスクリーンからウィンドウズへ、そして再びウィンドウズへ戻ることができます。デフォルトではオフになっています。
  • ピクセル間の色を補間する。線形補間をオンにし、基本的にピクセルを「滑らかに」します。鮮明なピクセルグラフィックのためにはオフにすべきですが、美しいアルファブレンドや滑らかなエッジのグラフィックのためには、オンのままにしておく方が良いでしょう。
  • カーソルを表示するこのオプションを有効にすると、デフォルトのOSカーソルがゲームに表示されます。このオプションを無効にすると、マウスがゲームウィンドウ上にあるとき、ゲームプロジェクトの一部としてカーソルを作成しない限り、カーソルは表示されません（これはデフォルトでオンになっています）。
  • テアリングを避けるために同期を使用します。v-syncのオン・オフを切り替えます。room の速度が120のゲームで、プレイヤーがリフレッシュレート60のモニターを持っている場合、このオプションをオンにすると、ゲームの速度も60に固定されることに注意してください。デフォルトではオフになっています。
  • ウィンドウサイズを変更できるようにする。これをチェックすると、ユーザーがゲームウィンドウのサイズを変更することができます（この機能を使用するには、Borderless Windowオプションがオフになっている必要があります）。このオプションはデフォルトでオフになっています。
  • スケーリングここでは、アスペクト比を維持する（つまり 4:3 のroom が 16:9 で「レターボックス化」される）か、完全にスケーリングする（画像をフルスクリーンに合わせて引き伸ばす）かを選べます。

    警告! アプリケーションサーフェスのスイッチを切ると、再びスイッチを入れるまで、Windows ゲームオプションで設定されたすべてのスケーリングオプションが無効になります。詳しくは、アプリケーションサーフェスを参照してください。

最後に、テクスチャページのサイズを設定するオプションがあります。デフォルトは2048x2048ですが、256x256から8192x8192の間で選択できます。と書かれたボタンもあります。 プレビュー これは、このプラットフォーム用のtexture ページを生成し、ウィンドウを開いてその見え方を確認できるようにするものです。これは、texture ページがどのように構成されているかを確認し、texture ページが必要以上に大きく（または小さく）なることを防ぎたい場合に非常に便利です。texture ページの詳細については、こちらをご覧ください。

注意：texture のページサイズが大きくなると、スペックの低いPCではゲームの互換性が低くなりますので、ご注意ください。

画像

ここでは、ゲームのスプラッシュスクリーンを追加し、使用されるゲームアイコンを 設定することができます。アイコンは64x64ピクセルの画像で、 .png 形式でなければなりません。スプラッシュ画面は、少なくともゲームの最初のroom （またはそのビューポート）と同じ大きさで、 .png 形式である必要があります。スプラッシュスクリーンはゲームのロード中に表示され、Use Splash Screenオプションをチェックすることで有効または無効にできます（デフォルトではオフになっています）。

GameMakerには プロジェクトイメージジェネレータというツールがあり、これを使うと、ゲームをコンパイルするさまざまなターゲットプラットフォームに必要なイメージをすべて自動的に作成することができることは特筆すべきことです。このツールを使用する場合、作成されたイメージがあなたの要求するものであることを確認するために、修正する必要があります。
