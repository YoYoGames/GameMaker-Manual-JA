macOSゲームオプション

このセクションでは、macOS ゲームプロジェクトのコンパイル方法を制御するさまざまなオプションの概要を説明します。各セクションの内容は以下の通りです。

一般

製品情報 ] セクションでは、最初にゲームの表示 名を入力し、次にバージョン番号、App ID、App Output ファイル、および著作権の詳細を入力します。App Outputは、Macの開発マシン上でゲームのインストールファイルが置かれる場所です。 .app.zip を作成すると、そのファイルはどこにでも移動できますが（保存ダイアログを実行するときに指定した場所は、その .app.zip の唯一のコピーではありません）、 .pkg とその他のファイルはこのフォルダに残されます。これにより、Mac Store での購入をシミュレーションするような方法で、ローカル インストールをターミナルからテストすることが可能になります。

ここで、GameMakerが作成する最終アプリケーションの署名に使用するチームIDも指定する必要があります。ここで設定すると、macOS Preferencesで設定したTeam IDが上書きされます。Team Identifierの後にSigning Identifierを指定するオプションもあります。これはAppleが全ての非AppStoreアプリケーション（例えばSteam ゲームなど）に要求しているものです。

次のオプションは、ゲーム内でSteam を有効にするためのものです。このオプションを有効にする場合、Steamworks SDK がインストールされていること（ここを参照）と、ゲームのGeneral OptionsにSteam app IDが追加されていることを確認します。

最後に、Disable file system sandboxというオプションがあります。これをチェックすると、GameMakerのセーブ・ロードのサンドボックスが無効になり、ゲームが動作しているシステム上のどこからでもファイルにアクセスできるようになります。このオプションはデフォルトではチェックされていません。すべてのシステムでファイルに対するGameMakerのサンドボックスを維持することを推奨しており、チェックしても、OSレベルのパーミッションによっては、セーブ/ロード領域が利用できないことがあるからです。サンドボックスの詳細については、GameMakerの ファイルシステムのページを参照してください。

グラフィック

グラフィックス・オプションは、ゲームがターゲットMacのグラフィックス・カードをどのように使用するかを決定するために設定するものです。以下のオプションが含まれており、変更することができます。

  • Allow Menu And Dock In Fullscreen（フルスクリーンでメニューとドックを表示する）。このオプションをオンにすると、ゲームがフルスクリーンモードである場合にOSのメニューとドックが表示されます。デフォルトではオフになっています。
  • カーソルを表示する。チェックを入れると、通常のmacOS カーソルが表示されます。チェックを外すと、ゲームコードでカーソルを作成しない限り、カーソルが表示されなくなります。デフォルトではオフになっています。
  • フルスクリーンで起動する。チェックすると、フルスクリーンで起動し、チェックしないとウィンドウで起動します。デフォルトではオフになっています。
  • Allow Fullscreen Switching switch : これにチェックを入れると、標準のmacOS ショートカットを使って、フルスクリーンからウィンドウズへ、そして再びウィンドウズへ戻ることができます。デフォルトではオフになっています。
  • ピクセル間の色を補間する。線形補間をオンにし、基本的にピクセルを「平滑化」します。鮮明なピクセルグラフィックのためにはオフにすべきですが、美しいアルファブレンドや滑らかなエッジのグラフィックのためにはオンのままがよいでしょう。デフォルトではオフになっています。
  • テアリングを避けるために同期を使用します。v-syncのオン・オフを切り替えます。room の速度が120のゲームで、プレイヤーがリフレッシュレート60のモニターを持っている場合、このオプションをオンにすると、ゲームの速度も60に固定されることに注意してください。デフォルトではオフになっています。
  • ウィンドウサイズを変更できるようにする。これをチェックすると、ユーザーがゲームウィンドウのサイズを変更することができます（この機能を使用するには、Borderless Windowオプションがオフになっている必要があります）。このオプションはデフォルトでオフになっています。
  • Enable Retina: これをチェックすると、レティナ対応モニターでゲームを実行したときに、背面buffer が高い（実際の）解像度に設定され、チェックを外すと背面buffer が見かけの（低い）解像度に設定されます。この意味は、最初のゲームウィンドウの幅と高さが、最初のroom が設定されている部屋/ビューポートの2倍に設定されるということです。しかし、これはアプリケーションサーフェスのスケールを合わせないので、ゲームを網膜ディスプレイを最大限に活用したい場合は、アプリケーションサーフェスをウィンドウサイズと同じに設定する必要があります（これは幅と高さに沿ったピクセルが2倍になることに注意してください、したがって、ゲームを高解像度化の恩恵を受ける場合のみこれを有効にしてアプリケーションサーフェスをスケーリングしてください）。

    このオプションはデフォルトではオフになっており、これを有効にしても、VMを使用してゲームを実行したりデバッグしたりする場合には、ランナーが事前にビルドされているため、何の違いも現れません。VMで実行ファイルをビルドする場合、またはYYCでゲームのテストやビルドを行う場合にのみ機能します。
  • スケーリングここでは、アスペクト比を維持する（つまり 4:3 のroom が 16:9 で「レターボックス化」される）か、完全にスケーリングする（画像をフルスクリーンに合わせて引き伸ばす）かを選べます。

    警告! アプリケーションサーフェスのスイッチを切ると、再びスイッチを入れるまで、Windows ゲームオプションで設定されたすべてのスケーリングオプションが無効になります。詳しくは、アプリケーションサーフェスを参照してください。

最後に、テクスチャページのサイズを設定するオプションがあります。デフォルトは2048x2048ですが、256x256から8192x8192の間で選択できます。と書かれたボタンもあります。 プレビュー これは、このプラットフォーム用のtexture ページを生成し、ウィンドウを開いてその見え方を確認できるようにするものです。これは、texture ページがどのように構成されているかを確認し、texture ページが必要以上に大きく（または小さく）なることを防ぎたい場合に非常に便利です。texture ページの詳細については、こちらをご覧ください。

注意：texture のページサイズが大きくなると、スペックの低いPCではゲームの互換性が低くなりますので、ご注意ください。

画像

画像セクションは、ゲームに必要な画像を提供する場所です。Macの場合、Icon ファイル（ .png 形式で1024x1024px）とSplash Screen（ .png, .bmp, .jpg, .gif 形式のいずれか）を指定する必要があります。スプラッシュスクリーンは、ゲームがロードされる間に表示されます。

DMGパッケージの場合、DMGインストーラーの背景として表示されるインストーラーバックグラウンドイメージを提供することができます。

GameMakerには プロジェクトイメージジェネレータというツールがあり、これを使うと、ゲームをコンパイルするさまざまなターゲットプラットフォームに必要なイメージをすべて自動的に作成することができることは特筆すべきことです。このツールを使用する場合、作成されたイメージがあなたの要求するものであることを確認するために、修正する必要があります。

パッケージング

完成したゲームをApp Store Readyにしたい場合は、Build for Mac App Storeオプションをチェックする必要がありますが、これはあなたが登録デベロッパーで必要な証明書を持っている場合のみ機能することに注意してください。その下にあるアプリのアクセス許可は 、必要でないものをチェックするとAppleがアプリを拒否することがあるので、正しいものだけをチェックする必要があります。これらのオプションは、あなたのゲームが http_ と url_ 機能を使用することを許可するだけです。ゲーム内でGamePadをサポートしたい場合は、App Store ready packageを作成するオプションはオフにしておくとよいでしょう。

最後に、そのApp Categoryを選択する必要があります（詳しくは、こちらをご覧ください）。

社会

このオプションは、MacでApple Sign In Extensionを使用するためのものです。このextension はGameMaker Marketplace から入手できます。また、extension 自体に詳しい使用方法が記載されています。もし、このextension を使わないのであれば、このオプションにチェックを入れるべきではありません。
