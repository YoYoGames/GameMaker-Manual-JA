Amazon Fireゲームオプション（レガシー）

重要Amazon Fireオプションは、古いプロジェクトにのみ表示され、読み取り専用です。Amazon Fire およびその他のすべてのAndroid ターゲットは、Android Game Options で設定されるようになりました。

このセクションでは、Amazon Fire ゲームプロジェクトのコンパイル方法を制御するさまざまなオプションの概要を説明します。各セクションの内容は以下の通りです。

一般

General セクションの最初の部分は、Product Informationと Build Settings専用です。製品情報には、プロジェクトの表示名、パッケージ情報（ドメイン、会社 名、製品名）を入力する必要があります。パッケージ情報は、A-Zの文字と0-9の数字のみで構成され、記号やその他の特殊文字は一切使用できず、また大文字も使用できません。これらの情報は"com.company.myappname"というフォーマットでパッケージIDを生成するために使用されます。

注：Androidターゲットのゲームオプションを設定済みの場合、上部の「Android から同期」ボタンをクリックすると、Android ターゲットからAmazon Fire に関連情報が転送されます。

その後、Build Toolsと Target、Minimum 、Compileの各SDKのバージョンを設定する必要があります。

注：「サポートライブラリ」のバージョンを使用するかどうかのオプションもあります。GameMakerv2.2.4以降では、この設定は不要になり、ここで使用した値は無視されます。この設定は、2.2.4より前のruntime を必要とするレガシープロジェクトで使用するために維持されています。

Build Tools (および必要に応じて Support Library) は、GameMakerが Android/Amazon Fire ターゲット向けにゲームをビルドする際に使用されます。Android Studio を使用してインストールしたツールに基づき、適切な値を設定する必要があります。

  • Target SDKは、ここで指定されたバージョンでアプリをテストしたことを示します（最大で）。これは、Android OS が、OS の機能の観点からアプリをどのように扱うべきかを示すためのものです。ほとんどのアプリは、実用上、Target SDK を最新リリースバージョンの API（Compile SDK 設定で使用される）に設定することをお勧めします。これにより、アプリは最新のAndroid デバイスで可能な限り良好に表示されるようになります。なお、2019年11月よりGoogleはAPIレベル28を必須ターゲットとしました。
  • Minimum SDKのバージョンは、プロジェクトを実行するための最小のAPIレベルです。2019年11月よりGoogleはAPIレベル16を絶対必要最低限としましたが、extensions を追加する場合は、より高い最低APIレベルを要求される可能性があります。
  • コンパイルSDKのバージョンは、プロジェクトがコンパイルされるAPIのバージョンです。これは、そのバージョンのAPIに含まれる、Android API機能を使用できることを意味します。例えば、API 16の機能を使おうとして、コンパイルSDKを15に設定すると、コンパイルエラーになります。しかし、コンパイルSDKを28に設定すれば、API 26デバイス上でアプリを実行することができます（他のすべての以前のバージョンも同様です）。

選択したAPIレベルにSDK値を事前入力する]ボタンをクリックすると、事前定義されたターゲット設定の1つを選択することができます。これらのいずれかを選択すると、すべてのSDKファイルに適切な値が入力され、必要に応じて使用または編集することができます。

注：プロジェクトを正しくコンパイルするためには、Android SDK Manager またはAndroid Studio から適切な API をインストールする必要があります。

Generalセクションの第2部では、最高のユーザーエクスペリエンスを実現するために、ゲームを実行するさまざまな方向について 説明します。利用可能なオプションは以下の通りです。

  • ポートレート
  • ポートレート・フリップ
  • 風景
  • 風景を切り取ったような

最後に、以下のような雑多なオプションがあります。

  • Bluetooth/iCadeのサポートを有効にする。ゲームプロジェクトにBluetoothまたはiCadeゲームパッドのサポートを追加します。デフォルトではオンになっています。
  • Lint コード解析を実行する。このオプションにフラグを立てると、ビルドツールはビルド時にプロジェクトのlintコード解析を実行し、より詳細なエラーチェックを行い、プロジェクトのコンパイル時に出力ウィンドウに多くの情報を表示します（LOGファイルも作成され、出力ウィンドウにはコンパイル終了時にファイルへのpath が表示されます）。lint解析でエラーと判定されたものすべてが、必ずしもゲームに問題があるわけではないことに注意してください。この機能を有効にすると、オフのときは問題なくコンパイルして実行できるのに、コンパイルできなくなることがあります。基本的に、これはdebugging でのみ有効で、デフォルトではこのオプションはオフになっています。
  • インストール先。ここでは、ゲームのデフォルトのインストール先を設定することができます。自動]に設定すると、ユーザーがデバイスで選択した場所にインストールされます。[外部を優先]に設定すると、常にデバイスの外部ストレージにゲームをインストールしようとします。

また、ここでAmazon Fire Sleep Marginを設定することができます。このオプションは、特定のシステムでゲームを実行する際のスタッタリングを軽減することに関連します。基本的に、ゲームの実行速度がroom の速度より速い場合、GameMakerは残りの時間「スリープ」しますが、このスリープはかなり不正確な場合があり、必要以上に長くスリープしてしまい、フレームに時間がかかり、スタッターの原因となることがよくあります。この問題を回避するには、スリープ時間を短くして、残りの時間をloop に固定するとより正確になります。ただし、loop に固定すると CPU 使用率が高くなり、CPU 温度が上昇してデバイスのバッテリー使用量が増えるという問題があります。デフォルトでは、この値は4に設定されており、99.99%の場合はこれで問題ありませんが、ローエンドのデバイスや多くのバックグラウンドプロセスを実行しているデバイスでは、これは理想的なソリューションではなく、5から10までの値が必要な場合があります。ただし、これはデバイス固有の設定であり、あなたのビルドデバイスで動作するものが、他のユーザーにとって適切でない可能性があることに注意してください。

グラフィック

ここでは、ゲームの表示方法に関する以下の詳細を変更することができ、以下のオプションが利用可能です。

  • ピクセル間の色を補間する。線形補間をオンにし、基本的にピクセルを「平滑化」します。鮮明なピクセルグラフィックのためにはオフにすべきですが、美しいアルファブレンドや滑らかなエッジのグラフィックのためにはオンのままにしておく方が良いでしょう。これはデフォルトでオンになっています。
  • 画面の色深度。レンダリングの色深度を16ビットまたは24ビットに設定します。24bitに設定すると、一部の古いデバイスとの互換性に影響があり、また必要なashmem （AmazonFireデバイスの共有メモリ）が増えます。
  • 拡大縮小。ここで、描画キャンバスをデバイスの画面内のアスペクト比を維持して拡大縮小する（画面に合わせて縁に「パディング」を追加する）か、描画キャンバスをデバイスの画面に合わせて引き伸ばすかを選択できます。

最後に、テクスチャページのサイズを設定するオプションがあります。デフォルトは2048x2048ですが、256x256から8192x8192の間で選択できます。と書かれたボタンもあります。 プレビュー これは、このプラットフォーム用のtexture ページを生成し、ウィンドウを開いてその見え方を確認できるようにするものです。これは、texture ページがどのように構成されているかを確認し、texture ページが必要以上に大きく（または小さく）なることを防ぎたい場合に非常に便利です。texture ページの詳細については、こちらをご覧ください。

注意：texture のページサイズが大きくなると、ブラウザやデバイスによってゲームの互換性が失われることに注意してください。

アイコンについて

このセクションでは、プロジェクトで必要となる様々なストアページやデバイス用のアイコン画像を追加することができます。すべてのアイコン画像は、24bit *.png ファイルとして、指定された適切なサイズで作成する必要があります。

GameMakerには プロジェクトイメージジェネレータというツールがあり、これを使うと、ゲームをコンパイルするさまざまなターゲットプラットフォームに必要なイメージをすべて自動的に作成することができることは特筆すべきことです。このツールを使用する場合、作成された画像があなたの要求するものであることを確認するために、修正する必要があります。このツールは、画像だけでなくアイコンも生成します（以下のセクションを参照）。

画像

ここでは、全画面表示に設定されていないスプラッシュ画面の背景として使用するBorder Colourを設定できます。また、最終的なゲームファイルには、ランドスケープモードと ポートレート モードの両方でスプラッシュスクリーンを追加できます。この画面は24bit *.png ファイルで、ゲームの最初のroom (またはビュー) と同じサイズにすることをお勧めします。スプラッシュスクリーンを特定の時間表示する必要がある場合は、ここで0～10秒の範囲で設定することもできます（デフォルトの時間0は、asset のロード時間だけ表示されることを意味します）。また、スプラッシュスクリーンの表示方法を以下のオプションから設定することができます。

  • アスペクト比を維持する。デバイスの画面と比較して画像のアスペクト比を維持し、必要に応じてエッジの周りに「パディング」を追加します。
  • クロップ・トゥ・フィット(Crop To Fit)。デバイスの画面と1対1の画像を維持し、必要に応じてエッジをクロップ（またはパッド）します。
  • 引き伸ばします。デバイスの画面いっぱいに画像を引き伸ばします（表示される画像に歪みが生じる場合があります）。

最後に、バナー画像を設定します。これも24bitの*.png 。

アクセス権

このタブでは、Amazon Fire Manifest 内でゲームが要求するパーミッションの一部を変更することができます。GameMakerは、ゲームが使用する機能に基づいて、必要に応じて自動的にパーミッションを割り当てますが、正しく割り当てられていないパーミッションが必要な場合があります（例えば、カスタムextensions ）。その場合は、必要なパーミッションのボックスにチェックを入れてください。

以下のリストは、利用可能な許可の概要を示しています。

  • WRITE_EXTERNAL_STORAGE:外部ストレージへの書き込みを許可します。
  • READ_PHONE_STATE:電話の状態への読み取り専用アクセスを許可する。
  • ACCESS_NETWORK_STATE:ネットワークに関する情報へのアクセスをゲームに許可する。
  • INTERNET:ネットワークソケットを開くことを許可する。
  • BLUETOOTH:ペアリングされたBluetooth機器との接続を許可します。
  • RECORD_AUDIO:マイクからの音声入力を録音できるようにします。

注意：もし、これらが必要かどうかわからない場合は、デフォルトでチェックされていない状態にして、GameMakerにゲームのパーミッションを任せるとよいでしょう。

Amazon Fire Permissions セクションには、 Androidmanifest.xml のAndroid アプリケーションタグにパーミッションを注入できるエリアもあります。 利用可能なタグの完全なリストは、Android のドキュメントで確認できます。

  • Android開発者。アプリケーションタグ

ターゲットAPI 28以降では、以下のタグを追加しない限り、"http" を使った外部Webコールの使用は許可されません。代わりに "https" が必要です。

android:usesCleartextTraffic = "true"
