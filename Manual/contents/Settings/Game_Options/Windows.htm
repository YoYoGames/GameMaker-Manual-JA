Windowsゲームオプション

このセクションでは、Windows ゲームプロジェクトのコンパイル方法を制御するさまざまなオプションの概要を説明します。各セクションの内容は以下の通りです。

一般

Windows ゲームの一般 ページでは、ゲームがインストールされ実行されているときに が表示するために必要な情報を提供する必要があります。この情報は、Windows 製品情報セクションに記載され、完全に記入される必要があります。

このウィンドウのオプション セクションでは、ゲームパッケージのインストール先を、"%localappdata%" または "%appdata%" フォルダーのどちらかにするか、また、Copy exe to output folder を設定するかどうかを選択できます。このオプションを有効にすると、テスト時にゲームEXEをTempフォルダ に置かず、出力フォルダに保持します。

次に、Windowsのスリープマージンを設定します。このオプションは、特定のシステムでゲームを実行する際のスタッタリングを軽減することに関連します。基本的に、ゲームの実行速度がroom の速度より速い場合、GameMakerは残りの時間「スリープ」しますが、このスリープはかなり不正確な場合があり、必要以上に長くスリープしてしまい、フレームに時間がかかり、スタッターの原因となることがよくあります。これを回避するには、スリープ時間を短くして、残りの時間をloop に固定するとより正確になります。ただし、loop に固定すると、CPU 使用率が上昇し、CPU 温度が上昇し、ファンの回転が速くなるという問題が発生します。デフォルトでは、この値は10に設定されており、99.99%の場合はこれで問題ありませんが、ローエンドのマシンや多くのバックグラウンドプロセスを実行しているマシンでは、これは理想的なソリューションではないかもしれません（または非常にハイエンドなマシンでも）、1や15などの値が必要かもしれません。しかし、これはシステム固有の設定であり、あなたのビルドマシンではうまくいっても、他のユーザーには適切でない可能性があることに注意してください。

注：スリープマージンに加えて、Windows でスレッドスケジューラの解像度を調整することで、パフォーマンスと消費電力をさらに制御できます。runtime 。

次のオプションは、ゲームのためにSteamを有効にすることです。このオプションを有効にする場合、Steamworks SDK がインストールされていること（ここを参照）、およびゲームのGeneral OptionsにSteam App IDが追加されていることを確認します。GameMaker Windows をSteam とIDE のバージョンから起動している場合、およびSteam 用のゲームを開発している場合は、Use alternative launcher for Steamapplication オプションもチェックする必要がある場合があることに注意してください。このオプションは、テスト時に代替ランチャープログラムを使ってゲームを起動し、ゲーム自身のアプリケーションIDではなく、GameMakerのアプリケーションIDでSteam 、フラグが立つのを防ぐためのものです。このオプションは、Steam 向けに開発する際、ゲームの起動に問題があると判断された場合のみ有効にしてください。

最後のオプションは、「ファイルシステムのサンドボックスを無効にする」オプションです。これをチェックすると、GameMaker のファイルの保存と読み込みのサンドボックスが無効になり、ゲームが動作しているシステムのどこからでもファイルにアクセスできるようになります。このオプションはデフォルトではチェックされていません。すべてのシステムでファイルに対するGameMakerのサンドボックスを維持することを推奨しており、チェックしてもOSレベルのパーミッションによっては一部のセーブ/ロード領域が使用できない場合があるからです。サンドボックスの詳細については、GameMakerの ファイルシステムに関するページをご覧ください。

最後に、GameMakerで Windows プロジェクトのx64実行ファイルを作成するオプションがあります。このオプションを有効にすると、コンパイル時に64bitの実行ファイルを作成します。すべての新規プロジェクトでこのオプションを使用することを強くお勧めします。このオプションは、長期的にはWindows の実行ファイルのデフォルト形式となり、32ビットランナーは廃止される予定です。このオプションを有効にすると、プロジェクトで使用されているextension の dll も 64bit にする必要があることに注意してください (詳しくはここを参照)。

グラフィック

グラフィックスオプションは、ゲームがターゲットWindows PC のグラフィックスカードをどのように使用するかを設定するためのものです。以下のオプションが含まれています。

  • カーソルを表示する。チェックを外すと、ゲームコードでカーソルを作成していない限り、カーソルは表示されません。デフォルトではオンになっています。
  • フルスクリーンで起動します。チェックするとフルスクリーンで起動し、チェックしない場合はウィンドウで起動します。デフォルトではオフになっています。
  • Allow Fullscreen Switching switch : これにチェックを入れると、ユーザーは標準のWindows ショートカットを使って、フルスクリーンからウィンドウズへ、そして再びウィンドウズへ戻ることができます。デフォルトではオフになっています。
  • ピクセル間の色を補間する。線形補間をオンにし、基本的にピクセルを「滑らかに」します。鮮明なピクセルグラフィックのためにはオフにすべきですが、美しいアルファブレンドや滑らかなエッジのグラフィックのためにはオンのままにしておく方が良いでしょう。
  • テアリングを避けるために同期を使用します。VシンクのON/OFFを切り替えます。room の速度が120のゲームで、プレイヤーがリフレッシュレート60のモニターを持っている場合、このオプションをオンにすると、ゲームの速度も60に固定されることに注意してください。デフォルトではオフになっています。
  • ウィンドウサイズを変更できるようにする。これをチェックすると、ユーザーはゲームウィンドウのサイズを変更することができます（これを機能させるには、Borderless Windowオプションがオフになっている必要があります）。このオプションはデフォルトでオフになっています。
  • Borderless Window（ボーダーレスウィンドウ）。これをチェックすると、ゲームはボーダーレスウィンドウで実行され、通常の最小化、最大化、閉じるボタンとゲームタイトルは削除されます。デフォルトではオフになっています。
  • スケーリングここでは、アスペクト比を維持する（つまり 4:3 のroom が 16:9 で「レターボックス化」される）か、完全に拡大する（フルスクリーンに合わせてイメージを引き伸ばす）かを選択できます。

    警告! アプリケーションサーフェスのスイッチを切ると、再びスイッチを入れるまで、Windows ゲームオプションで設定されたすべてのスケーリングオプションが無効になります。詳しくは、アプリケーションサーフェスを参照してください。

最後に、テクスチャページのサイズを設定するオプションがあります。デフォルトは2048x2048ですが、256x256から8192x8192の間で選択できます。と書かれたボタンもあります。 プレビュー これは、このプラットフォーム用のtexture ページを生成し、その見え方を確認できるようにウィンドウを開くものです。これは、texture のページがどのように構成されているかを確認し、texture のページが必要以上に大きく（または小さく）なるのを防ぎたい場合に、非常に便利な機能です。texture ページの詳細については、こちらをご覧ください。

注意：texture のページサイズが大きくなると、スペックの低いPCではゲームの互換性が低くなりますので、ご注意ください。

画像

画像セクションは、ゲームに必要な画像を提供する場所です。Windows の場合、Iconファイル（ .ico 形式であること）とスプラッシュスクリーン（ .png 、 .bmp 、 .jpg 、 .gif 形式であること）を指定する必要があります。スプラッシュスクリーンは、ゲームのロード中に表示され、Use Splash Screenオプションをチェックすることで有効または無効にできます（デフォルトではオフになっています）。

なお、GameMakerには プロジェクトイメージジェネレーターというツールがあり、これを使うと、ゲームをコンパイルするさまざまなターゲットプラットフォームに必要なイメージをすべて自動的に作成することができます。このツールを使用する場合、作成されたイメージがあなたの要求するものであることを確認するために、修正する必要があります。

インストーラー

Windowsゲームはインストーラーで 作成することができ、ここでGameMakerが作成時に使用する様々な設定やグラフィックを定義することができます。まず最初に、インストーラの外観を決めるグラフィックオプションがあります。

  • 終了しました。インストーラーの "Finished "ページに表示されるグラフィック。 *.bmp 形式で、サイズは 164x314px である必要があります。
  • ヘッダーです。インストーラの実行中に表示されるグラフィック。 *.bmp 形式で、サイズは 150x57px である必要があります。

また、必要であれば、特定のLicense Agreementファイル（GameMaker にサンプルが付属しています）やカスタムNSIscript ファイル（詳細はこちら）を指定したり、適切なボタンをクリックしてGameMaker 付属のデフォルトファイルに戻ったりすることも可能です。
