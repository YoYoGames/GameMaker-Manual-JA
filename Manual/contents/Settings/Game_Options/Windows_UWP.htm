Windows UWPゲームオプション

このセクションでは、UWP（UniversalWindows Platform）ゲームプロジェクトのコンパイル方法を制御するさまざまなオプションの概要を説明します。各セクションは以下の通りです。

一般

UWPオプションの一般セクションは、製品情報を設定する場所です。

  • Display Name（表示名）。ユーザーに表示されるアプリのフレンドリーな名前を指定します（このstring はローカライズ可能です）。
  • パッケージ名。作成したパッケージの名前です。ゲーム用に作成したDeveloper keyに使用した名前と一致させる必要があります（以下のパッケージに関するセクションを参照ください）。
  • パッケージの表示名。これはパッケージの「短い名前」で、表示名と同じであることが理想的です。
  • パブリッシャー表示名。ゲームパブリッシャーとして関連付ける会社名または個人名です。
  • バージョン。ゲームのバージョン番号です。

設定後、最高のユーザーエクスペリエンスのために、ゲームを実行するさまざまな方向を 選択することができます。選択可能なオプションは以下の通りです。

  • 風景
  • ポートレート
  • 風景を切り取ったような
  • ポートレート・フリップ

デフォルトではすべて選択されていますので、不要なものは選択を解除してください。

Orientationオプションを設定したら、次のオプションからUWPアプリに必要な様々なPermissionを 指定することができます。

  • Enable Internet Permission:Flags このゲームはインターネットにアクセスできるとしています。デフォルトではオフになっています。
  • Enable Microphone Permission:Flags このゲームは、デバイスのマイクにアクセスできるとしています。デフォルトではオフになっています。
  • Enable IAP Sandbox:Flags In App Purchases（アプリ内課金）がサンドボックス化されたゲームとして使用されます。デフォルトではオフになっています。

デバイス・ファミリー

ここでは、実行ファイルを作成する際に、Windows UWP のエクスポートで対象とするデバイスファミリーを選択できます。デフォルトでは、「Support Universal device family」オプションがチェックされており、アプリがすべてのデバイスとファミリーをターゲットにすることを意味します。特定のデバイスファミリーにのみ対応するようにアプリを制限するには、「Universal」オプションのチェックを 外し、該当するデバイスファミリーにチェックを入れる必要があります。この場合、ユニバーサルデバイスファミリーをチェックしたまま、異なるプラットフォームのバージョンをサポートしたい他のデバイスファミリーをチェックし、そのファミリーのバージョンの詳細を適切なテキストボックスに入力します。このロジックがどのように動作するかについては、次のMicrosoftサポートページを参照してください。

  • 対象デバイスのファミリ。例

一般的に、ターゲットと最小プラットフォームのバージョンは触らず、デフォルト値のままにしておきます。しかし、ビルドに問題がある場合、特にデフォルトで設定されているWindows SDK をインストールできない場合、これらをインストールしたものに変更する必要があります。設定する値がわからない場合は、Visual Studio で新しい UWP プロジェクトを作成し、値がどのように設定されるかを確認します。

デバイスファミリーの詳細については、以下のマイクロソフト社のドキュメントを参照してください。

  • デバイス・ファミリーの概要

グラフィック

グラフィックスオプションは、ターゲットとなるUWPデバイスのグラフィックスカードをゲームがどのように使用するかを決定するために設定するものです。以下のオプションが含まれており、変更することができます。

  • ピクセル間の色を補間する。基本的にピクセルを「滑らかに」する線形補間をオンにします。鮮明なピクセルグラフィックのためにはオフにすべきですが、美しいアルファブレンドや滑らかなエッジのグラフィックのためには、オンのままにしておく方が良いでしょう。
  • カーソルを表示するこのオプションを有効にすると、デフォルトのOSカーソルがゲームに表示されます。このオプションを無効にすると、マウスがゲームウィンドウ上にあるとき、ゲームプロジェクトの一部としてカーソルを作成しない限り、カーソルは表示されません（これはデフォルトでオンになっています）。
  • フルスクリーンで起動する。チェックすると、フルスクリーンで起動し、チェックしないとウィンドウで起動します。デフォルトではオフになっています。
  • Allow Fullscreen Switching switch : これにチェックを入れると、ユーザはUbuntu OS の標準ショートカットを使って、フルスクリーンからウィンドウズへ、そして再びウィンドウズへ戻ることができます。デフォルトではオフになっています。
  • スケーリングここでは、アスペクト比を維持する（つまり 4:3 のroom が 16:9 で「レターボックス化」される）か、完全に拡大する（フルスクリーンに合わせてイメージを引き伸ばす）かを選べます。

    警告! アプリケーションサーフェスのスイッチを切ると、再びスイッチを入れるまで、Windows ゲームオプションで設定されたすべてのスケーリングオプションが無効になります。詳しくは、アプリケーションサーフェスを参照してください。

最後に、テクスチャページのサイズを設定するオプションがあります。デフォルトは2048x2048ですが、256x256から8192x8192の間で選択できます。と書かれたボタンもあります。 プレビュー これは、このプラットフォーム用のtexture ページを生成し、ウィンドウを開いてその見え方を確認できるようにするものです。これは、texture ページがどのように構成されているかを確認し、texture ページが必要以上に大きく（または小さく）なることを防ぎたい場合に非常に便利です。texture ページの詳細については、こちらをご覧ください。

注意：texture のページサイズが大きくなると、スペックの低いPCではゲームの互換性が低くなりますので、ご注意ください。

画像

Imagesセクションの最初の部分では、UWPが実行可能なさまざまなデバイスでゲームが使用するさまざまなTilesを 設定するオプションがあります。設定できるのは以下の通りです（すべて *.png フォーマットのファイルである必要があります）。

  • 小さなタイル画像は、検索バーなどのUIでアプリ名の横に表示されるロゴ画像です。44 x 44ピクセルである必要があります。
  • 中タイル画像は、71x71ピクセルの画像を使用すること。
  • 次のtile の画像は、Store のロゴtile で、50 x 50 ピクセルである必要があります。
  • ワイド画像タイルは、tile がワイド形式のときに表示される画像で、310 x 150 ピクセルである必要があります。この画像がない場合、tile は正方形のフォーマットでしか表示できず、ワイドテンプレートタイプに基づく通知を受け取ることができません。tile がどのフォーマットを使うかはユーザーが決められるので、ワイド画像tile を含めることをお勧めします。また、ワイド画像tile が提供されている場合、tile は最初ワイドフォーマットで表示される可能性があることに注意してください。
  • 大きな画像タイルは、大きなライブtile の形式で、310 x 310 ピクセルである必要があります。これはデスクトップOS上でのみ使用されることに注意してください。

次のセクションは、ゲームのロゴを定義するためのものです。これは、基本的にあなたのゲームがUWPエコシステム内で使用するアイコンです。注目すべきは、これらの画像は透明な .png ファイルであり、その場合、このセクションで指定したBackground Colorの上に描画されることです。この背景色は、アプリが所有するダイアログボックスのボタンの色や、デバイス上のStoreのApp Descriptionページにも適用されます。

ロゴを設定した後、ゲームのスプラッシュスクリーンを設定することができます。これは、あなたのゲームを起動したときに表示される画面であり、それは620×300ピクセルよりも大きくすることはできませんし、ロゴのように、それはあなたが選択した背景色がそれに使用されますその場合には、透明 .png ファイルであってもかまいません。

GameMakerには プロジェクトイメージジェネレータというツールがあり、これを使うと、ゲームをコンパイルするさまざまなターゲットプラットフォームに必要なイメージをすべて自動的に作成することができることは特筆すべきことです。このツールを使用する場合、作成されたイメージがあなたの要求するものであることを確認するために、修正する必要があります。

パッケージング

GameMakerはWindows UWP の開発用にデフォルトのDeveloper Keyを提供していますが、自分で生成してここで指定することをお勧めします。そのためには、Visual Studio で空のプロジェクトを作成し、自動生成された *.pfx ファイルのコピーを安全な場所に保存してください。そして、その *.pfx ファイルをここにリンクしてください。

ファイルへのリンク後、関連するInstall ボタンをクリックし、表示されるコマンドプロンプトの指示に従ってください。これを行わないと、Run/Debug ボタン（または F5 キー）を使用しても失敗し、UWP プラットフォームでゲームをテストできません（ただし、Create Executableを使用すると、 *.pfx がまだ登録されていない場合は、認証インストール プロセスが自動的に開始されます）。

注：アプリをストアにアップロードする場合、タイトルに適した*.pfx を取得する必要があります。これは、アプリをWindows のVisual Studio Store と関連付けることで、Visual Studio がアプリ専用の新しい*.pfx を生成することで行われます。Windows Visual Studioつまり、空のプロジェクトをWindows Store アプリに関連付け、生成された*.pfx を安全な場所にコピーするだけで、UWP Game Options で使用することができます。この方法は、YoYo Games Knowledge Baseに記載されています。

このセクションでは、署名証明書を生成するために使用された パブリッシャー名を指定する必要があります。入力の前には必ず「CN= 」を付けてください。例： CN=YoYoGames.

Xbox Live

UWPゲームオプションのXbox Liveセクションは、Xbox Oneコンソールで使用するためにUWPプロジェクトを有効にする場所です。ここでプロジェクトをXbox用にアクティベートすると、Xbox One上でプロジェクトを実行したときに以下のことが可能になります。

  • アプリ内からのXbox Liveへのサインイン
  • プロジェクトで何らかの形で Xbox Live ユーザー ID を使用する (たとえば、ゲーマータグやゲーマー写真を表示するなど)。

また、Microsoft Creators Programに参加している場合は、Creators Program Appオプションにもチェックを入れてください。Creators Programに参加している場合は、Title IDと Service Configuration IDを追加する必要があります（これらの詳細については、Microsoftから入手できます）。

現在、プロジェクトでXbox Liveの機能をテストするには、実行ではなく、実行ファイルを作成する必要があります。UWP Xboxの出力には、GameMakerに付属する一時的な証明書ではなく、パッケージ情報（上記参照）の一部として提供するWindows Store署名証明書が必要で、これは実行ボタンを押したときに使用されるものです。

XBox Liveを有効にすると、プロジェクトでXBox Liveの機能を使用することができますが、最も基本的な機能を使用するにはCreator Programに参加する必要があり、高度な機能を使用するにはID@XBoxアカウントが必要であることに注意してください。
