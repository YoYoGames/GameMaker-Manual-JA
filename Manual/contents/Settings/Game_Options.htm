ゲームオプション

ゲームオプションは、メインオプションとプラットフォーム設定の2つのセクションに分かれています。メインオプションは、一般オプションから始まり、ここでは、実行するように設計されているプラットフォームに関係なく、ゲームに影響を与える特定のものを設定することができます。

このウィンドウには、次のセクションがあります。

  • デフォルトのスクリプト言語。現在のプロジェクトのObjects イベントとScripts でGML CodeとGML Visual のどちらを使うか、あるいは新しいscript の作成時に常に尋ねるかどうかを選択します。
  • Game Frames Per Second: この値は、GameMakerがゲーム実行時に維持しようとする、1 秒あたりのフレーム数の制限を設定します。この値は、ゲームloop が 1 秒間に実行される回数を表し（1 つのゲームloop は 1 つの「フレーム」とみなされます）、デフォルトでは 60 に設定されていますが、30 もよく使用されます。どのroom の速度が理想的かについては多くの議論がありますが、実際の答えは、作っているゲームの種類、room のサイズ、インスタンスの数、コードの集中度によって異なります。たとえば、大きなroom とroom の速度が 30 の場合、見ている領域が大きければ大きいほど、低いroom 速度を知覚しやすくなるため、ゲームが「ぎこちなく」見えることがあります。一方、ゲームに集中する場合、room 速度を 120 に設定すると、処理量が多いために fps が低下してゲームが「遅延」することがあります。したがって、この値には注意を払い、常にゲームの必要性に応じて調整するようにしてください。
  • デフォルトの描画色：コードで色が設定されていない場合に、テキストやフォーラム（およびその他のもの）を描画するためのデフォルトの色です。
  • Default Draw Colourroom 領域の外側の色。ゲームroom の領域外の背景をクリアするために使用されるデフォルトの色です。
  • プロジェクト画像を生成する。これをクリックすると、プロジェクト画像生成ツールが表示されます。
  • プロジェクト開始日。プロジェクトが開始された日付。
  • プロジェクト使用時間：プロジェクトを開始してから経過した時間です。
  • プロジェクトGUID：GameMaker内で各プロジェクトに作成される一意の識別子です。変更したい場合は、Create New GUIDボタンをクリックして新しいGUIDを生成することができます。
  • Generate mipmaps for separatetexture 別々のページにミップマップを作成する。このオプションは、sprites EditorのTexture 設定でSeperateTexture Pageオプションにフラグを立てたSprite に対して、GameMakerがミップマップを作成するものです。ミップマップを有効にすると、texture のオリジナルsprite ページが、texture ファイル内で複数の解像度にスケーリング・フィルタリングされることになります。これは通常オフになっていますが、ビューを使ってゲームroom の広い範囲を拡大・縮小している場合や、3Dゲームを扱っている場合は、これを有効にして、Mipmapping Functions を使ってゲームが異なるミップマップレベルをどう扱うかを調整するとよいでしょう。sprites で個別のtexture ページを使用するフラグを立てない場合でも、Texture Group Manager からテクスチャグループ単位でミップマップを有効にすることができ ます。このオプションはデフォルトではオフになっています。
  • ソース管理を有効にする。プロジェクトをSCM(Source Control Management)下に置きたい場合は、この項目にチェックを入れる必要があります。デフォルトではオフになっています（source control の詳細についてはこちらをご覧ください）。
  • 作者。ゲーム作者（または出版社、スタジオ）の名前です。
  • プロジェクトの最終変更日：このセクションには、プロジェクトが最後に変更された日付/タイムスタンプが表示されます（これは編集できません）。
  • Spine ライセンスを承認する。ここにチェックを入れると、Spine Runtime のライセンスが表示され、承諾することができます。これで、Spine sprites をゲーム内で使用することができます（このライセンスは、Spine sprite をプロジェクトに追加しようとして、事前にここでライセンスを確認しなかった場合にも表示されます）。
  • コリジョン互換モード(Collision Compatibility Mode)。2022年バージョン以前のレガシーコリジョンシステム（floatではなくinteger値を使用）を使用するよう、GameMaker 。詳しくはCollision Compatibility Modeをご覧ください。これはデフォルトでは無効になっており、新しいプロジェクトでは無効にしておくことが推奨されます。
  • Copy on Write Behaviour for Arrays（書き込み時コピー動作）を有効にする。現在開いているプロジェクトで、非推奨のCopy on Write配列動作を有効にします。詳しくは、Copy on Writeをご覧ください。これはデフォルトでは無効になっており、新しいプロジェクトでは無効にしておくことが推奨されます。

メインオプションセクションには、「テンプレート情報」のメニューもあります。

アセットブラウザには、お持ちのライセンスで利用可能なターゲットプラットフォームに応じて、他のゲームオプションも用意されており、それらのすべてが利用可能または表示されるとは限りません。

プラットフォーム設定

以下のプラットフォーム別のゲームオプションが存在します。

  • オペラGX
  • ウィンドウズ
  • マクロス
  • ウブントゥ
  • Windows UWP
  • HTML5
  • アンドロイド
  • iOS
  • ティーブイオーエス

注：様々なコンソールゲームオプションは、異なる会社によって維持される様々な法的制限のためにここに記載されていません。コンソールライセンスをお持ちの場合は、YoYo Gamesヘルプデスクから情報を得ることができますが、まずコンソールIDの電子メールをYoYo Games に連絡し、さまざまなセクションを表示する許可を要求する必要があります。

GameMakerのプロジェクトによっては、Amazon Fireターゲットのゲームオプションが表示される場合があります。GameMakerは現在、Androidのゲームオプションを使用してFireプラットフォームを直接サポートしているため、これらはレガシーで読み取り専用のオプションです。もし、プロジェクトにまだAmazon Fire が表示されている場合は、以下のページでオプションに関する情報を確認できます。

  • Amazon Fire（レガシー）
