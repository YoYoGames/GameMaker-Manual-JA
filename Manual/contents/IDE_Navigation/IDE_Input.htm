IDE入力とナビゲーション

GameMaker IDEはマウスとキーボードの入力を受け付けており、多くの操作をどちらか一方、または両方を使って行うことができます。一般的には、マウスの左ボタン をクリックして何かを選択し、 / + を使って複数のアイテムを選択し、 を押したままアイテムを別のドックまたはworkspaces にドラッグし、さらに を使ってコンテキスト固有のメニュ ーを開くことができます。なお、GameMakerをmacOS システムで動作させていて、シングルボタンマウスを使用している場合は、 + でマウスの右クリック を使用してください。

また、さまざまなworkspace 要素やリソースエディタを操作するために、非常に多くのキーボードショートカットがあります。最も重要なものをいくつか簡単に紹介します（完全なリストはここにあります）。

  • / + "Z" : 直前の操作を取り消します。ほとんどのエディタで使用できます。取り消しにも複数のレベルがあるので、これを何度も押して変更を「ロールバック」することができます。
  • / + "Y" : 取り消した直前の操作をやり直すことができ、ほとんどのエディタで動作します。アンドゥと同様に、リドゥにも複数のレベルがあります。
  • / + "S" :GameMakerにプロジェクトを強制的に保存させます。
  • / + "T" :Goto Windowが表示され、ゲーム内の任意のasset を素早く検索して移動することができます。また、Preferencesと Game Optionsも検索できます。すべてのリストが表示され、検索語を入力すると、このリストが適切に選別されます。例えば、すべてのrooms の先頭に "rm_" がある場合、これを入力すると、この名前で始まるassets のリストだけが表示されます。

  • / +:ワークスペースの概要ウィンドウが表示され、異なるworkspaces で開かれている項目間を素早く移動することができます。 または を押している間、ウィンドウは開いたままです。 を 1 回押すと、ウィンドウ内の次のアイテムに移動します。ショートカットを離すと、選択されていた項目に移動します。

    注: このウィンドウのバインディングを、複数の修飾子 ( + + B など) または単一のキー ( B のみ) にリマップすると、ショートカットを押した後、そのキーを押したままかどうかにかかわらず、ウィンドウを閉じるかウィンドウから項目を選択するまでウィンドウは開いたままとなります。

  • :マニュアルが表示されます。GML Visual またはGML コードを使用する場合、アクションや関数（またはキーワードなど）を選択してから を押すか、マウスの中ボタン をクリックすると、該当ページのマニュアルを開くこともできますので、ご注意ください。
  • :IDE にあるドッキングされたすべてのウィンドウを折りたたんだり展開したりすることができます。

workspace を操作する他の方法としては、キーボードショートカット / + + を使用する方法があります。 <Arrow Keys>また、マウスの中ボタン（ ）を押したままマウスをドラッグすると、workspace の中を移動することができます。

様々な種類のworkspaces やウィンドウを操作する際、任意のテキストフィールド上でマウスの右ボタン を押すと、デフォルトで以下のフィールドを含むコンテキストメニューが表示されます。

フォーカスが当たっているエディターやウィンドウによっては、これらのオプションが展開される状況もあります。

このほか、ペンデバイスへの限定的な対応や、ノートパソコンを使ってプロジェクトを進める人向けの特別なモードも用意されています。いずれも以下に説明します。

ラップトップモード

ノートパソコンでGameMakerを使用している場合、IDE の上部にLaptop Modeのオプションがさらに追加されます。

デフォルトではオンになっていますが、必要に応じてこのスイッチをオフにすることができます。 ボタンをクリックします。ラップトップモードは、IDE のいくつかのツールと組み合わされ、タッチパッドを使用する際のエクスペリエンスを大幅に向上させ、3 つの主要なマウス操作を簡素化します。 パン。 ズームと スクロールそのために2つの修飾キーを使用します。左 / と左 .ノートパソコン・モードがオンのとき。 左 / と 左 は、room エディタをパンするなどのマウス操作のために「予約」されています。 左 / はズームです。 左 はスクロールとパンです。ラップトップモードでのスクロールは、パン操作と同じように動作するはずです。 直感的な操作で、スクロールバーを使わなくても十分な精度が得られます。これらのボタンを押しながらマウスを動かすと、クリックでもドラッグでもなく、単に移動するだけでよいことに注意してください。 を使用することです。 / または キーを使用します。 キーボードの右 キーで代用



タッチパネル対応

最後にもうひとつ、GameMaker用のIDE 、タッチスクリーンにも最低限対応していることをお伝えしておきます。すべてのオペレーティングシステムで、タッチスクリーンを使ってメインworkspace のアイテムをクリック、ドラッグすることができます。 ポインターを2回タップすると、右クリックができます。なお、Windows 8 以上では、GameMaker IDE 、ペンデバイスもサポートされます。
