ヘルプメニュー

GameMakerを使用する際のヘルプを得ることに関連するメニューです。以下のメニューがあります。

  • マニュアルを開く- ヘルプマニュアルを開きます。なお、メニューバーのヘルプボタン をクリックしても、 を押しても、ドキュメントを開くことができます。
  • ナレッジベース-GameMakerのナレッジベースにアクセスし、プラットフォームのセットアップ、既知の問題、GameMakerの使い方など、様々な記事を見つけることができます。
  • バグ報告-GameMakerを使用していてバグを発見した場合、または問題がある場合、ここに報告することができます。
  • Open Log In Explorer- 通常 IDE 関連のバグレポートを提出する際に必要となるui.logファイルがあるシステム上のディレクトリーを開きます。
  • GameMaker.io-GameMakerのウェブサイトを開きます。
  • GameMaker Community- ブラウザでGameMaker Communityのフォーラムを表示します。このフォーラムはコミュニティが 運営していますので、バグレポートや技術的なサポートが必要な場合は、まずオフィシャルサポートサイト（上記参照）にアクセスしてください。
  • リリースノート - GameMakerのリリースノートをブラウザで表示します。アップデートのロールバックが必要な場合は、このページのバージョン番号のリンクをクリックすると、以前のバージョンのインストーラーを入手することができます。
  • 現在のRuntime リリースノート-ランタイムはIDE と独立して更新できるため、独自のランタイムリリースノートのセットを持っています。このオプションを選択すると、ブラウザでそれらを開くことができますが、最新のruntime はあなたが使用しているものではない可能性があることに注意してください。あなたが使用しているruntime を確認したり、runtimes を追加、削除、有効化する方法については、「ランタイム・プリファレンス」セクションを参照してください。
  • 必要なSDK-GameMakerはクロスプラットフォームであるため、多くのサードパーティSDKが必要です。これをクリックすると、各プラットフォームごとに必要なものの詳細が記載されたサポート記事が表示されます。
  • エクスプローラでプロジェクトを開く- 現在ロードされているプロジェクトの保存場所をOSのファイルエクスプローラで開きます。
  • Refresh System Fonts-GameMakerが見ることのできるシステムfonts を更新します。fonts を開いている間にインストールされた新しいIDE を検出するのに便利です。
  • ライセンス-GameMakerで使用されているすべてのサードパーティーライブラリのライセンスを含むウィンドウが開きます。
  • バージョン情報 - 「GameMakerについて」のウィンドウが開き、製品のバージョン情報（バグや問題を報告するのに不可欠）やクレジットを見ることができます。
