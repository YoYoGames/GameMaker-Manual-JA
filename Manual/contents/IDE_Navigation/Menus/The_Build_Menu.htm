ビルドメニュー

ビルドメニューでは、テスト用、debugging 、または最終的な実行ファイルとしてプロジェクトをビルドすることを選択できます。利用可能なオプションは以下の通りです。

  • Run - 選択したターゲットプラットフォームで現在のプロジェクトを実行し、テストします。コンパイル時に利用できる様々なコンパイルオプションについては、コンパイルのセクションを参照してください。このコマンドは、IDE の実行ボタン（ ）を押すのと同じであることに注意してください。
  • デバッグ - debugger で現在のプロジェクトをテスト用に実行し、詳細なパフォーマンス情報を確認し、エラーや問題があればデバッグできます。debugging の詳細については、デバッグのセクションを参照してください。このコマンドは、IDE の「Debug」ボタン（ ）を押すのと同じであることに注意してください。
  • クリーン - GameMakerはテスト時のコンパイル時間を短縮するために、ファイルのキャッシュを保持しています。しかし、時には "stale"cache がゲームの中で奇妙で予期せぬエラーを引き起こすことがあります。その場合、まずcache をここからクリーンにして、再テストを行うべきです。また、ターゲットプラットフォーム用の最終的な実行ファイルを作成する前に、cache をクリーンアップするのも良いアイデアです。このコマンドは、IDE のClean Cacheボタン を押すのと同じであることに注意してください。
  • 実行ファイルを作成する - 現在のプロジェクトをコンパイルし、ターゲットプラットフォーム用の実行パッケージ（またはパッケージ）を作成します。最初にファイルエクスプローラーを開き、プロジェクトの保存先を選択してから、ゲームをコンパイルします。コンパイルターゲットの設定については、コンパイルの項を参照してください。なお、このコマンドは、IDE のCreate Executableボタン を押すのと同じです。
