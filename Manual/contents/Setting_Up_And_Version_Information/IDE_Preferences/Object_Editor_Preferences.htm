オブジェクトエディタの環境設定

Object Editor Preferencesは、Object Editorウィンドウの特定のプロパティを定義するために使用されます。メインセクションには、以下のオプションがあります。

  • Click to open Eventメソッド。このオプションは、object イベントを開くために、ダブルクリックとシングルクリックのどちらを実行する必要があるかを制御します。デフォルトでは、ダブルクリックに設定されています。
  • Inherited Events α:このオプションは、object のイベントリストにおいて、「継承されたイベント」が描画されるアルファ値を変更することができます。デフォルトでは、この値は0.5に設定されています（値は0から1の間であるべきです）。
  • イベントラベルのフォーマット。ここでは、object エディタのイベントリストでイベントラベルに適用されるさまざまなテキストを有効または無効にすることができます。デフォルトでは、"イベントの説明 "のみが有効になっています。
  • デフォルトのイベントコンテンツを追加します。このオプションはデフォルトで有効になっており、すべての新しいイベントのコードエディタに簡単なコメントを追加して、そこにコードを追加できることを思い出させるだけです。
  • デフォルトのイベント内容です。ここでは、新しいイベントのコードウィンドウにデフォルトで追加されるコメントを編集することができます。

また、Object Editorには以下のサブカテゴリが存在します。

確認ダイアログ

ここでは、Object Editor使用中に表示される可能性のあるダイアログについて説明します。以下のオプションが使用可能です。

  • GML VisualのGamemaker言語への変換に自動対応。 GML Visualworkspace 内でマウスの右ボタンを使用すると、そのイベントのGML Visual を全てGML コードに変換するオプションを選択できます。ここでのデフォルトの動作は、続行するかどうかを尋ねる警告メッセージを表示することですが、これを「OK」に設定すると、変換前にダイアログが表示されなくなります。
  • Automatic response to exceeding physicsvertex limit: このオプションを使うと、物理シェイプエディタで許容されるvertex の数を超えたときに表示されるポップアップメッセージを制御することができます。ここでのデフォルトの動作は警告メッセージを表示することですが、これを「OK」に設定するとダイアログは表示されなくなります。
  • イベント削除時の自動選択このオプションは、Object エディタのイベントリストでイベントを削除するときに表示されるポップアップメッセージを制御するために使用できます。ここでのデフォルトの動作は警告メッセージを表示することですが、これを「OK」に設定するとダイアログは表示されません。
