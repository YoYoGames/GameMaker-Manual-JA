GMLビジュアル環境設定

は GMLビジュアル環境設定は、ビジュアルエディタの外観や操作性を定義するために使用します。利用可能な異なるオプションは、それぞれが何をするかの説明とともに以下にリストアップされています。

  • GMLビジュアルツールボックスのデフォルト位置デフォルトでは、GML ビジュアルツールボックス（すべての異なるアクションライブラリが保持されている）はエディタの右側に配置されていますが、このオプションを使用すると、デフォルトで左側に配置することが可能です。
  • GML Visual ノードの概要の可視化モード。ノードの「概要」は、例えばこのDestroy Instanceアクションのように、ActionWorkspace の特定のアクションの横に表示される短い説明テキストです。

    これらを常に有効にするか、アクションが折りたたまれた場合のみ有効にするか（デフォルト設定）、完全に無効にするかを選択することができます。
  • GML VisualからGameMaker Languageへのコンバート時の自動対応。を変換するときGMLビジュアルイベントやscript をGML コードに変換するとき、デフォルトでは、これから行うことについて警告し、続行するかどうかを尋ねるメッセージが表示されます。このオプションを「OK」にすると、メッセージは表示されずに、単に変更を「OK」にします。
  • GML Visualノード削除時の自動応答について。イベントノードを削除する場合 GMLビジュアルイベント・ノードを削除する場合、デフォルトでは、これから行おうとしていることについて警告し、続行するかどうかを尋ねるメッセージが表示されます。このオプションを "OK "に設定すると、メッセージを表示することなく、削除を単純に許可します。
  • GML Visualツールボックスのアクションサムネイルの最大サイズについて。アクションツールボックスの下部に、アイコンの大きさを変更するスライダーがあります。 GMLビジュアルアイコンのサイズを変更するスライダーがあります。この設定は、最大許容サイズ（デフォルト64px）を定義するために使用されます。
  • GML Visualツールボックスのアクションサムネイルの最小サイズについて。アクションツールボックスの下部に、アイコンのサイズを変更するためのスライダーがあります。 GMLビジュアルアイコンのサイズを変更するスライダーがあります。この設定は、許容される最小サイズ（デフォルトは16px）を定義するために使用されます。
  • GMLビジュアルツールボックスのアクションサムネイルのデフォルトサイズ。この設定は、ツールボックス内のすべてのアクションアイコンのデフォルトサイズを定義するために使用されます（デフォルトは32px）。
  • 最近使用したアクションの最大表示数。アクションツールボックスの上部にあるライブラリには、最近使用したアクションが保存されています。このオプションを使用して、最近使用したアクションの最大表示数を変更できます(デフォルトは5)。
  • GML ビジュアルのアクションをつなぐ鎖の太さ。コードブロック内の異なるアクションをつなぐ鎖の太さを指定します。 GMLビジュアルの太さを指定します（デフォルトは3px）。
  • ノードのドラッグを開始するまでのマウスダウン時間(ms)。ノードをクリックしてドラッグすると、アクションコード内でノードの位置が変更されます。しかし、マウスをしばらく押したまま移動すると、workspace 内のノードの位置は移動しますが、コードの順序は変わりません。この設定では、マウスボタンを押したまま何ミリ秒待つと、ノードを移動できるようになるかを設定します（再配置はできません）。デフォルトは350です。
  • GMLビジュアルノードの概要パネルを非表示にします。ここでは、アクションコードエディタの概要パネルを非表示にすることを選択できます。デフォルトでは無効になっており、有効にするとエディタからアクションチェーンのプレーンテキストリストが削除されます。
