テキスト エディター エディターの環境設定

テキストエディタの環境設定は、さまざまなコード/スクリプトエディタの外観と操作性を定義するために使用します。メイン・ページには、以下のオプションがあります。

  • シンタックスチェックの遅延時間（ms）。シンタックスチェッカーが実行されるまでの遅延時間（ミリ秒）。このデフォルト値は2000msです。
  • コード完了ディレイ（ms）。遅延時間(ミリ秒)
  • フルスクリーンエディターでscripts を開きます。このオプションをチェックすると、script のリソースが自動的に新しいworkspace で開かれます。このオプションをオフにすると、現在のworkspace の中のウィンドウで開かれます。このオプションは、デフォルトではチェックされていません。
  • フルスクリーンエディターでshaders を開きます。このオプションをチェックすると、shader のリソースが自動的に新しいworkspace で開かれます。このオプションをオフにすると、現在のworkspace の中のウィンドウで開かれます。このオプションは、デフォルトではチェックされていません。
  • object イベントをフルスクリーンエディタで開く。このオプションをチェックすると、object イベントコードエディタが自動的に新しいworkspace で開かれます。このオプションをオフにすると、object イベントエディタに連鎖したウィンドウで開かれます。このオプションは、デフォルトではチェックされていません。
  • イベントscripts を同じウィンドウでタブとして開く。イベントにコードまたはGML Visual を追加する場合、コード・ウィンドウはイベント・ウィンドウに連鎖します。このオプションをチェックすると、編集のために開かれた以降のイベントは、同じウィンドウに上部に沿ってタブとして追加されます。このオプションをオフにすると、各イベントは独自の連鎖したウィンドウとして開かれます。このオプションはデフォルトでチェックされています。
  • リンク先のscripts を同じウィンドウでタブとして開く。リンクされたscripts が複数あり、それらを開いた場合、デフォルトでは同じウィンドウにタブとしてグループ化されます。このオプションをオフにすると、それぞれのウィンドウ/ワークスペースで開かれるようになります。このオプションは、デフォルトでチェックされています。
  • フォーカス時にテキストエディタをパンして表示します。object イベントからテキストエディタを開いたとき、このオプションを選択すると、workspace が自動的にパンしてエディタが境界内に収まるようになります。このオプションは、デフォルトではオフになっています。
  • コードウィンドウのデフォルト幅：コードウィンドウを開いたときのデフォルトの幅（ピクセル単位）を設定します。デフォルトは900です。
  • コードウィンドウのデフォルトの高さ。コードウィンドウを開いたときのデフォルトの高さ（ピクセル単位）を設定します。デフォルト値は600です。
  • コードウィンドウの最小幅：コードウィンドウで許容される最小幅（ピクセル単位）を設定します。デフォルト値は300です。
  • コードウィンドウの高さの最小値。コードウィンドウが許容する最小の高さ（ピクセル単位）を設定します。デフォルト値は200です。

上記のオプションとは別に、コードエディタとビジュアルエディタという2つのサブカテゴリがあり、以下に説明します。

コードエディタ

色」環境設定は、コードエディタとscript エディタが異なるテキストをどのように表示するかを変更するために使用されます。各項目にはいくつかのサブ項目があり、上の画像の例のように、各テキストがどのように見えるかを正確に定義することができます。一般的に、色、font 、サイズ、スタイルをそれぞれ設定することができます。さらに、コードブラケットなどの行の選択など、より一般的なスタイルを設定することもできます。

色の設定]の下に、さまざまなスタイリングオプションがあります。これらの設定は、コード/スクリプトエディタの動作を設定するために使用され、以下のオプションが利用可能です。

  • タブを保持するチェックを入れると、タブキーを押したときにタブ文字（' \t' ）としてエディターに残します。チェックを入れないと、タブはスペースに変換されます。デフォルトはoffです。
  • タブサイズ。Tab "キーを押したときに、コードを何文字分インデントさせるかを指定します。デフォルトは4です。
  • 空白を表示する。空白を表示する場合は " ." を表示します（下図参照）。デフォルトはoffです。
  • 行番号を表示する左下の行番号を表示するかどうか。デフォルトはオンです。
  • タブ線を表示する。" ...." を表示するとタブ線スペースを表示します。デフォルトはoffです。
  • 色分けを有効にする。エディターの色分けを有効または無効にします。offの場合、コードはスタイルされず、onの場合、上記の「色」セクションで指定されたフォーマットを使用してスタイルされます。デフォルトはオンです。
  • ドキュメント行数制限：任意のエディタードキュメントの最大行数を設定します。デフォルト値は0（0以下は実質的に無限大）、それ以外の値では許可される行数が制限されます。
  • 改行時のインデントを一致させる。コードの自動インデントを有効または無効にします。デフォルトはonで、新しい行が追加されたとき、その行に設定されているタブが維持されます。
  • ブレースの後を自動的にインデントする。中括弧を追加したときのコードの自動インデントを有効にするかしないか {} 。デフォルトは on で、新しいブレース { と新しい行が追加されたときに、次の行をタブで囲みます。
  • コード補完を有効にする。コード補完ウィンドウの有効・無効を設定します。コードを書いているときにこの機能を有効にすると、現在の入力テキストに基づいて使用可能な関数を示すポップアップウィンドウが表示されます。デフォルトはオンです。
  • 関数の括弧を自動追加する。オートコンプリートを使用する際に、 () の括弧の自動追加を有効または無効にします。コードを書くときにこの機能を有効にすると、IDE は関数が必要とする 2 つの括弧 () を自動的に追加し、関数が引数を取る場合はその中にカーソルを配置し、取らない場合はその後ろにカーソルを配置します。この機能は、関数がオートコンプリートから選択された場合にのみ機能します。デフォルトはオンです。
  • Undo / Redo Stack Limit: Undo/Redoスタックの上限を設定します。0以下は実質的に無限（利用可能なメモリによって制限される）であり、正の値はスタックをその数に制限し、制限に達した場合はスタックの末尾にあるすべてのアクションを削除します。デフォルトは-1です。
  • 選択範囲の一致を強調表示します。他の場所で重複しているコードの部分を軽くハイライトします。つまり、例えばある関数を選択すると、現在のコードウィンドウにあるこの関数のすべてのインスタンスもハイライトされます。デフォルトはオンです。
  • 廃止された機能のカラーコード化／コード補完。廃止されたコードの部分をハイライト表示します。デフォルトはオンです。

ビジュアルエディタ

コメント」環境設定は、GML Visual scripts に追加されたコメントの色と書式を制御します。これらのオプションは次のとおりです。

  • 色：ウィンドウに書き込まれるテキストのデフォルトの色（デフォルトはARGBの#FF53E270）。
  • フォント名。使用するfont の名前（使用可能なfonts は、システムにインストールされているものに依存します）。
  • スタイル。太字や 斜体など 、使用するfont のスタイル（すべてのウィンドウが同じスタイルを許可するわけではなく、また使用するfont にも依存することに注意してください）。
  • サイズ。指定された出力ウィンドウに対してfont を設定するサイズ。
  • アンチエイリアシングここでは、出力ウィンドウfont レンダリングのアンチエイリアスモードを設定することができます。デフォルトを使用]を選択すると、IDE が決定します。また、これらのオプションのいずれかを選択すると、強制的にOn またはOff にすることができます。
