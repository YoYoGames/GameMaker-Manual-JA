GMLコードの環境設定

GML CodePreferencesは、GameMakerの IDE 、コーディングのための言語機能のいくつかをどのように表示するかを定義するために使用されます。GMLCodeのオプションは以下の通りです。

  • 引数の種類をステータスバーに表示する。このオプションは、独自のカスタムscripts を使用していて、異なる引数などを識別するために JSDoc 表記を使用している場合に関連します。(詳しくはこちらをご覧ください)。このオプションがチェックされ、script で引数のタイプを指定すると、エディターの下部にあるステータスバーに、 {} で囲まれた引数のタイプが表示されます。これはデフォルトではオフになっています。
  • 引数の説明をステータスバーに表示する。これをチェックすると、JSDoc記法を使用してカスタムscripts に指定した説明が、エディタウィンドウの下部にあるステータスバーに表示されるようになります。デフォルトではオフになっています。
  • システム関数のUSスペルを表示する。switch デフォルトではGameMakerは関数のスペルにGB英語を使用しています、例えば draw_text_colour() 、しかしこれは正確にあなたが慣れているものではないかもしれませんので、このオプションを使って関数をUSスペルに変更することができます、例えば draw_text_color() 、となります。このオプションはデフォルトではオフになっています。
  • 表示するシンタックスエラーの最大数。この値は、一度に表示されるシンタックスエラーの最大数を設定します。もし、IDE が遅延するようであれば、この値を減らしてみてください。デフォルトでは50に設定されています。
  • 変数参照警告を有効にします。有効にすると、flag 、変数がどこかで定義されているが実際にはコードで使用されていない場合など、変数に関する潜在的な問題が発生する可能性があります。無効にすると、これらの警告は表示されなくなります。デフォルトでは、この機能は有効になっています。
  • ステートメントをリージョンとして折りたたむことができるようになりました。有効にすると、コード内のブラケットをリージョンのように折りたたむことができます。これはデフォルトで有効になっています。
