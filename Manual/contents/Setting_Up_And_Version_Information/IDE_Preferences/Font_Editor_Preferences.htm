フォントエディタの環境設定

Font Editor Preferencesは、Font Editorの動作を定義するために使用されます。 以下に、さまざまなオプションと、それぞれの動作について説明します。

  • GameMaker1 プロジェクトからインポートしたFonts を更新します。これがチェックされていると、GameMakerはfonts を更新し、そのtexture を再生成しようとします。 これは以下の場合にのみ起こります。
    • この環境設定は有効です
    • font がマシン上で利用可能である。
    • GameMakerは、font ファミリーで一致する顔を見つけることができます（インポート時に不一致が発生する最も可能性の高い原因です）。
    GameMaker1 のfont にキャッシュされた font texture がない場合にも起こることに注意してください。レガシーfont が更新できない場合、GameMaker はデフォルトでキャッシュされた を使用します。 font texture また、Font Editor ではfont の編集ができなくなり、バックアップして別のものに更新するオプションが表示されます。

    これをクリックすると、font の編集が可能になり、不足しているfont を追加するか、別のものを設定することができます。 バックアップされたfont は、font asset フォルダに、 <font name>.yy.gms1 および <font name>.png.gms1 として保存されます。これらのファイルを復元するには、手動で <font name>.yy ファイルを削除し、ファイル名から .gms1 extension を削除してこれらのファイルに置き換えてください。 重要!すべてのassets は、 .yy ファイルに名前を保存しています。 を作成し、 <name> のフィールドにも正しいリソース名を入力します（異なる場合）。
  • コンテンツ再生失敗時の自動選択ダイアログ font asset texture を再生成しようとすると、警告が表示されます。このオプションを「OK」に設定すると、警告を表示せずにfont を再生成することになります。 を表示します。
  • font 再生失敗時の自動選択ダイアログ。font asset texture の再生に失敗した場合、警告が表示されます。このオプションを "OK "にすると、メッセージの表示が省略されます。
  • インポートしたレガシーフォントのアップグレードを自動的に選択できるようにしました。レガシープロジェクトをインポートするときに、レガシーfonts についての警告とアップグレードのオファーが表示されることがあります。このオプションを "OK "に設定すると、自動的にアップデートが行われます。 というメッセージが表示されます。
