ランタイムフィードの環境設定

ランタイムフィード環境設定は、現在インストールされているランタイムと、現在使用中のruntime を表示および変更するために使用します。GameMakerは、一度に複数の異なるruntime ツールをインストールすることができ、この環境設定から使用するツールを設定することができます。理想的には最新のruntime を使いたいところですが、問題があったり、比較のために古いものを使わなければならない場合もあります。そのため、ここでGameMaker がゲームを作るのに使うruntime ツールを変更することができます。

この設定の最初の部分は、runtime ツールをどこから取得するかに関連しており、RSSフィードのURLを入力する必要があります。このフィードは、runtime ツールのアップデートをスキャンし、ダウンロードおよびインストールすることができます。このセクションにはデフォルトでMasterruntime feedのURLが表示されますが、Add ボタンをクリックして新しいRuntime Feedを作成し、RSSフィードの名前とURLを入力し、ApplyをクリックしてGameMakerにアクセスするように指示することが可能です。新しいフィードを表示させるためには、Preferencesウィンドウを一度閉じて再度開く必要があります。

Runtime フィードを作成すると、このセクションにサブカテゴリとして表示され、常に少なくとも 1 つのフィードが表示されます (デフォルトでは、これがマスター フィードになります)。どのフィードを使用しても、runtime の機能は同じです。以下、マスターフィードを例にとって説明します。

Runtime フィード環境設定は、さまざまな ツールの選択、インストール、アンインストールを行うために使用します。上部には、現在使用されている ツール、作成日、ハードディスクに保存されている場所が表示されます。これらの詳細の下には、利用可能なすべての のリストがあり、このリストから任意の 1 つを選択して、プロジェクトのコンパイルに使用する「実行中の」 に変更することができます。runtime runtime runtimes runtime

使用する別のruntime を選択するには、リストで見つけてから、 をダブルクリックするだけです。runtime がダウンロード、インストールされていない場合は、インストールするよう促されます。インストールが完了したら、新しいIDE を初期化できるようにruntime を終了し、再起動するよう求められます。
