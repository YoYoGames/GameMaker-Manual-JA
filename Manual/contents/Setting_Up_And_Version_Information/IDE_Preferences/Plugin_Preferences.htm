プラグイン環境設定

Plugins Preferencesは、GameMaker IDE に追加されたプラグインを制御するために使用されます。デフォルトでは、GameMakerのプロジェクトとGitを統合するためのSource Controlプラグイン専用のセクションが一つあります。

ソース管理 (Git)

このセクションでは、Git SCMプラグインをGameMaker で動作させるために必要な環境設定を扱います。

  • Gitの実行ファイルパス。これは、Git がすべてのGameMaker コマンドに使用するSource Control 実行ファイルの場所です。上の画像に示すように、これはGit インストールの cmd フォルダーに置かれます。

この下にあるMerge Toolと Diff ToolforSource Control 、これらの設定を行うことができます（オプション）。

  • Merge (Tool location): ここで、完全なファイルpath をGit マージツールに指定します。
  • マージ(ツールオプション):このフィールドには、マージツールを使用する際に実行する追加コマンドを追加することができます。
  • Diff（ツール位置）:ここで、path の diff ツールに、ファイルGit をフルで指定します。
  • Diff (ツールオプション):このフィールドには、diff Tool を使用する際に実行する追加のコマンドを追加することができます。

Finally there are two more options that affect how Git is set up in new projects: 

  • Add skeleton .git defaults to new projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to new projects that you create. It is enabled by default.
  • Add skeleton .git defaults to imported projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to projects that you import from a .yyz file. It is enabled by default.
