パスの環境設定

パス」 環境設定は、GameMaker paths の要素で必要となる様々なファイルIDE や、必要となる外部エディタに必要なものを制御します。IDEには、以下のオプションがあります。

  • Tempディレクトリ。すべての一時的なフォルダーを保存する場所です。Windows のデフォルトでは、これは

    \Users\<Username>\AppData\Local\Temp\GameMaker

    そして、macOS 、それは。

    /var/folders/<Path_Hash>/GameMaker

  • IDEcache ディレクトリ。IDE cache を保存する場所。Windows のデフォルトでは、この場所は

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    そして、macOS では、それは

    /Users/<Username>/.config/GameMaker/Cache

  • Assetcache ディレクトリ。各プロジェクトのasset cache を保存する場所です。Windows のデフォルトでは、これは

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    そして、macOS では、それは

    /Users/<Username>/.config/GameMaker/Cache

  • "マイプロジェクト "の場所。GameMakerが最初に新しいプロジェクトを作成する場所です。Windows のデフォルトでは、これは

    \Users\<Username>\Documents\GameMaker

    また、macOS の場合は

    /Users/<Username>/GameMaker

  • 閉じるときに一時ディレクトリを自動的に削除する。これをチェックすると、GameMakerがコンパイルなどのためにプロジェクトごとに作成するTempフォルダーを自動的に削除するようになります。この設定はデフォルトでオンになっており、チェックを外すとswitch オフになります（つまり、後で手動で一時ファイルを削除する必要があります）。
  • 閉じるときにasset cache を自動的に削除します。これをチェックすると、GameMakerを終了するときに、asset コンパイラcache フォルダが削除されます。デフォルトではオフになっており、これを有効にすると、プロジェクトをロードして実行するたびに、cache を再構築する必要があります（ゲームのサイズによっては時間がかかる場合があります）。
  • 閉じるときにIDE cache を自動的に削除します。これをチェックすると、GameMakerを終了するときに、IDE コンパイラcache フォルダが削除されます。これはデフォルトではオフになっており、これを有効にするとGameMakerを起動するたびにcache を再構築する必要があることになる。
  • Temp Folderを削除します。このボタンをクリックすると、プロジェクトのテンポラリフォルダが削除されます。
  • DeleteAsset Cache:このボタンをクリックすると、プロジェクトのコンパイラーasset cache が削除されます。
  • DeleteIDE Cache:このボタンをクリックすると、IDE のキャッシュが削除されます。

外部エディターへのpaths の設定には、以下のオプションがあります。

  • SWFファイル用の外部エディタ/ビューアへのパス。SWFフォーマットで作業している場合sprites, 好みのビューア/エディタのpath を設定することができ、Sprite EditorでEdit Imageボタンをクリックすると、指定したプログラムが開かれます。ここでのデフォルト値は、path がないことです。
  • Spine ファイル用の外部エディタ/ビューアへのパス。Spine フォーマットsprites で作業している場合、これを好みのビューア/エディタのpath に設定すると、Sprite エディタで [画像編集] ボタンをクリックしたときに、指定したプログラムが開かれます。ここでのデフォルト値は、path がないことです。
