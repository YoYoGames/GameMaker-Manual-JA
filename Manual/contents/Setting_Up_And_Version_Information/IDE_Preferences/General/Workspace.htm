ワークスペースの環境設定

ここで設定した内容は、GameMakerの各機能の中での移動の仕方に影響します。 ワークスペース.オプションは以下の通りです。

  • ワークスペースのキーボードナビゲーションの度数範囲。workspace 内では、異なるobjects とリソースのために複数のウィンドウを開くことができるので、それらの間を素早く移動するには、キーボードショートカット / + + <Arrow Keys> を使用することができます。workspace 内のウィンドウはグリッドアラインされていないため、IDE は、8 方向のいずれかを押したときに、どのウィンドウを意味するかを選択する必要がありま す。 このオプションは、使用する「ルックアングル」を設定することができます。デフォルトは40°です。
  • Workspace chain column padding (px):Workspace の要素は、しばしば互いに「連結」されています（例えば、Code Editor は Event Editor と連結され、Object Editor と連結されています）。 連鎖した2つのウィンドウ間の水平方向の距離を設定します。デフォルトは30pxです。
  • ワークスペース・チェーン行のパディング（px）。上述したように、workspace の要素はしばしば互いに「連結」されます。このオプションでは、連結された任意の 2 つのウィンドウ間の垂直距離を設定することができます。デフォルト値は20pxです。
  • ワークスペース・チェーンのレンダリングセグメント数。ここでは、レンダーチェーンの最大セグメント数を設定することができます。この値を下げると、ローエンドのマシンでのパフォーマンスが向上する場合があります。デフォルト値は 20 です。
  • ワークスペース・チェーン・リンク・カラー：workspace の要素間のチェーン表示に使用されるカラーです。デフォルトの色はARGBの$99DD00です。
  • ワークスペースのキーボード操作でズームがリセットされます。前述したように、キーボードショートカットを使ってworkspace 内のウィンドウからウィンドウへスキップすることができます。また、 / + マウスホイール を使ってworkspace をズームイン/アウトすることもできます。このオプションを設定すると、workspace を拡大表示した状態で、ショートカットを使用して別のウィンドウに移動した場合、workspace は 1:1 の表示に戻り、選択したウィンドウの中央に配置されます。このオプションは、デフォルトではオフになっています。
  • ワークスペース・チェーンが重なることがある。このオプションを有効にすると、workspace チェーンが交差したり重なったりすることを許可します。デフォルトではオフになっています。
