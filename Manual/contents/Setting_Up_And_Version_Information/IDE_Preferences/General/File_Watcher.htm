ファイルウォッチャーの環境設定

ファイルウォッチャーは、プロジェクトに関連するファイルに外部ソフトウェアによる変更（例：外部エディタでの画像編集）またはスクリプトの編集などの内部変更による変更がないか監視するIDE の部分に付けられた名前です。 asset.一般的には、これらの設定を編集する必要はありませんが、一部のシステム（プロジェクトファイルの保存ドライブとしてSSDを使用している場合など）では、ファイルウォッチャーは、単に保存するなどの操作を行ったときに変更を通知することがあります。 asset を編集するたびに、これらの設定を変更する必要があるかもしれません。

設定可能な項目は以下の通りです。

  • ファイルウォッチャーを有効にする。ファイルウォッチャーシステムの有効/無効を設定します。これを無効にすると、 IDE は外部ソースによって変更されたファイルを拾わなくなります。 この機能はファイルウォッチャーはデフォルトで有効になっています。
  • 保存後にMSでファイルウォッチャーの許容範囲を設定する IDEで。これは、ファイルを保存してから、ファイルウォッチャーが変更をチェックするまでの時間を調整します。この値を調整するのは、プロジェクトを保存しているときにファイルウォッチャーが この場合、メッセージが表示されなくなるまでの時間を長くします。デフォルトは2000msですが、1msから10000msまでの範囲で設定できます。
