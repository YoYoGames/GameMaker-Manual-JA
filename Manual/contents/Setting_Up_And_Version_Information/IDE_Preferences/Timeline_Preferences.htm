タイムラインの環境設定

Timelines PreferencesはTimeline Editor で作業する際のプロパティを定義するのに使用します。利用可能なオプションは以下の通りです。

  • GML VisualからGameMaker言語への変換に自動対応。timeline の瞬間にマウスの右ボタン メニューオプションを使用してGML Visual アクションをscript （コード）に変換すると、これは一方向の変換なので、続けるかどうかを尋ねるプロンプトが表示されます。このオプションを「OK」に変更すると、メッセージが表示されなくなり、ダイアログで「OK」をクリックしたのと同じように変換が進みます。デフォルトでは、「メッセージを表示する」に設定されています。
  • パーマネントアクション警告への自動対応。timeline 内の特定のアクションは元に戻すことができません。その場合、続行するかどうかを尋ねる警告メッセージが表示されます。このオプションを「OK」に変更すると、ダイアログで「OK」をクリックした場合と同様に、メッセージが抑制され、アクションが続行されます。初期設定は「メッセージを表示」です。
  • 作成時に新しいモーメントを開いて編集できるようにする。通常、新しいモーメントをtimeline に追加すると、GameMakerは自動的にエディタを開き、コードやGML Visual を追加することができます。しかし、このオプションをオフにすると、エディタを開かずにモーメントが追加されます (モーメントをダブルクリックすることで開くことができます ) 。デフォルトでは、このオプションはオンになっています。
