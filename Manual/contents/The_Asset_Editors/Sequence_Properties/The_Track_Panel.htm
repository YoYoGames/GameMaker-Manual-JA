  • トラックを削除する。選択した 1 つまたは複数のトラックを削除します。asset トラックを削除すると、それに関連するパラメータトラックも削除されますが、 パラメータトラックを個別に削除することができます。
  • グループトラックパネルの空きスペースにこのオプションを使用すると、グループフォルダが作成され、assets をより整然とした形式で管理するのに役立ちます。asset をクリックし、フォルダにドラッグするだけで、グループに追加することができます。すでにトラックを選択している場合、このオプションを使用すると、グループフォルダが作成されるだけでなく、 トラックが追加されます。

/ とマウスの左ボタン で任意の順序で1つまたは複数のトラックを選択し、 とマウスの左ボタン で連続したトラックを選択します。また、 / +"X", / + "C", / + "V" のキーコンビネーションでカット、コピー、ペーストすることができます。

チェンジオリエンテーション

Change Orientation ボタン は、Sequence Editor の各セクションの表示方法を変更するために使用します。デフォルトでは Cnavas View が上部に、Track Editor とDopesheet が下部に表示されますが、このボタンをクリックして異なるレイアウトを循環させるか、Options ボタン をクリックしてメニューを開き、希望のレイアウトを選択することで変更することが可能です。以下のレイアウトが利用可能です。

スプリットウィンドウ

このオプションでは、Sequence Editorが2つの別々のウィンドウに分割されます。メインのIDE ウィンドウはフルデスクトップのキャンバスビューを表示し、新しいウィンドウはDope Sheet とトラックエディタを表示します。このボタンをもう一度クリックすると、レイアウトが再び変更され、余分なウィンドウが削除されます。
スナップトップ

このオプションは、Track Editor とDopesheet をIDE の一番上にスナップさせます。
スナップライト

このオプションは、Track Editor とDopesheet をIDE の右側にスナップさせます。
スナップボトム

このオプションは、Track Editor とDopesheet をIDE の下にスナップさせます（これがデフォルトの表示です）。
スナップレフト

このオプションは、Track Editor とDopesheet をIDE の左側にスナップさせます。

また、Sequence Editor のレイアウトは、Reset オプション（ ）をクリックすることで、いつでもリセットすることができます。これにより、エディタがデフォルトの状態に戻り、閉じていたセクションが再び開かれます。

トラック検索

この検索ボックスは、sequence のさまざまなトラックをすばやく検索し、編集したいトラックを見つけるためのものです。ここに任意のテキストを入力すると、そのasset が含まれるstring のトラックの数が表示され、検索ボックスの横にある矢印も強調表示されます。これらの矢印をクリックすると、検索条件を満たす最初の、そして連続したassets に飛びます。

アセットトラック

asset に追加された各sequence には独自のトラックが与えられ、そのリストがここにあります。asset のトラックにパラメータが設定されている場合は、その下にパラメータトラックとして表示され ます。トラック名の左にある矢印アイコン（ ）をクリックすると、パラメー タトラックを拡大または縮小できます。また、トラック名を編集するには、トラックをゆっくりダブルクリック（ ） して、好きな名前を付けます。トラック名を編集するには、トラックをゆっくりダブルクリックし、好きな名前を付けます。付ける名前に制限はなく、スペース、英数字以外の文字、数字で始めることができます。

パラメータトラックの種類については、このページの下にある「パラメータトラックの追加」のセクションで詳しく説明されています。

キーコントロール

パラメータトラックを追加したら、特定のフレームにパラメータキーを追加・削除して、そのパラメータの値を設定することができます。これは、"記録キー" または "削除キー" を使って行うことができます。 ボタン（詳細は "トラックキーコントロール" の項を参照）、またはパラメータ自体のキーコントロールを使用して行うことができます。これらのコントロールを使用するには、Dope Sheet で再生ヘッドを編集したいフレームに位置させ、パラメータトラックの左側にある円形の「キー」アイコンをクリックする必要があります。フレームにキーが割り当てられていない場合、円は「空」になっており、これをクリックすると新しいキーが追加され、パラメータ値を編集することができるようになります。フレームにキーが設定されている場合、円は "filled "となり、クリックするとそのキーがフレームから削除されます。

また、上の画像では、パラメータキーマーカーの両側にある矢印ボタンを使って、dope sheet のキーフレーム間を素早くスキップすることができ、再生ヘッドを手動で配置する必要がありません。

パラメータ値

パラメータトラックが追加されたら、Dope Sheet の再生ヘッドを変更したいキーのあるフレームに設定し、異なるキャンバスウィジェットを使用してasset の位置、回転、スケール、原点を編集するだけで、いくつかのパラメータに対する値を編集できます。 これらの変更は、そのフレームのキー値に反映されます。ただし、すべてのパラメータがこの方法で編集できるわけではありませんし、必要な精度に満たない場合もあります。そのような場合は、トラックでパラメータ値をクリックして直接編集し、パラメータタイプに必要な値に設定することができます。

なお、一部のパラメータトラックには、値の横に「リンク」アイコン（ ）が表示されることがあります。これは、パラメータトラック同士をリンクさせることができることを意味しています。このアイコンをクリックすると、リンクの有効/無効が切り替わり、有効な場合は一方の値を編集すると他方の値も設定されます。

イベント情報

通常のobjects と同様に、sequences はイベントを持ち、それに割り当てられたコードを実行することができます。コードはスクリプト化された関数の形で割り当てられ、引数を取らず、イベントがトリガーされたときに呼び出されます。

イベントの追加は、「Add Event」ボタン（ ）をクリックすると、次のようなウィンドウが表示されます。

ここに掲載されているイベントは、以下の通りです。

  • 作成 - このイベントは、sequence のインスタンスが最初に作成されたときに発生します。また、sequence からroom のエディタを通してroom に入力されたときに、そのroom 内で発生する一番最初の出来事です。
  • Destroy - このイベントは、sequence のインスタンスが破棄されるときに実行され、Clean Up イベント（下記参照）の前に実行されます。
  • Clean Up- このイベントは、sequence からroom のインスタンスを削除するイベントの後に呼び出されます。したがって、sequence が破壊された場合、room が終了した場合、またはゲームが終了した場合にトリガされ、sequence インスタンスが何らかの理由でゲームから削除されたときに一旦実行する必要があるタスクを実行するよう設計されています。sequence のインスタンスを layer_sequence_destroy() などで破壊した場合、このイベントは Destroy Event (上記) の後に呼び出されます。
  • Begin Step/Step / End Step- ステップイベントは、sequence インスタンスが存在する間、ゲームの1ステップ（フレーム）ごとにチェックされるイベントで、begin, step, end の3つのパートに分かれています。ほとんどの場合、標準のステップ イベントを使用しても問題ありませんが、どのコードがどのタイミングで実行されるかをもう少し制御したい場合があります。そのために Begin と End ステップ イベントが用意されており、これらのイベントはゲームの各ステップ（フレーム）で常に同じ順番にトリガーされます。重要! sequence が一時停止している場合、これらのイベントはトリガーされません 。再生が再開されると、 sequence が再び再生を開始した次のフレームでトリガーされます。また、イベントの順番はプレイヘッドの方向には影響されません。sequence が逆向きに再生されている場合でも、イベントは Begin、Step、End の順番で実行されることに注意してください。
  • 非同期イベント。これはobject Asynchronous System Event に相当するものです。重要! sequence が一時停止している場合、このイベントはトリガーされません 。

各イベントには、イベントがトリガーされたときに呼び出される関数を1つ割り当てることができます。矢印ボタンをクリックすると、script エディターが開き、関数を編集することができます。また、下部にある「Create New」ボタンをクリックすると、「boilerplate」関数がすでに定義され、すぐに入力できる状態で新しいscript リソースを作成することができます。関数名は必要なものに変更でき、定義済みの名前を使用する必要がないことに注意してください。sequence イベントで使用される関数は、引数を取ることができないことに注意してください。

シーケンスイベントは、コードを使用して追加・編集することも可能です。詳しくはこちらをご覧ください。

ビジュアル/ロックトグル

asset トラックとパラメータトラックの両方には、トラックの可視性を切り替えるボタン と、トラックをロックするボタン があります。トラックの可視性を無効にすると、sequence をroom で実行したときに表示されなくなり、テスト中にsequence キャンバスで表示されなくなります。asset トラックの可視性をトグルすると、そのパラメータトラックの可視性もトグルされます。トラックをロックするということは、そのトラックが今後一切編集できなくなることを意味します。不可視のトラックはdope sheet 上に薄く描かれ、ロックされたトラックはオーバーレイで描かれるので、各トラックがどのような状態にあるかがすぐにわかることに注意してください。

トラックカラー

作成したasset の各トラックには、IDE 上で識別しやすいようにDope Sheet によってランダムな色が割り当てられます。この色は、トラックのカラーバーをクリックし、開いたカラーピッカーウィンドウから使用する色を選択するだけで、いつでも変更することができ ます。asset トラックに選択された色は、関連するすべてのパラメータトラックに自動的に適用されます。必要 に応じて、各パラメータトラックの色を個別に編集して、異なる色に変更することができます。asset トラックのカラーを再度変更すると、パラメータトラックに加えられたすべての変更 が上書きされますのでご注意ください。

パラメータートラックの追加

Add Parameter ボタンを使用すると、選択したasset トラックに 1 つのパラメータトラックを追加することができます。このページで前述したように、パラメータトラックは、関連するasset によって様々な異なるタイプがあります。以下は、利用可能なすべての異なるパラメータトラックとその参照先、および適用可能なassets の一覧です。

  • 位置。sequence このパラメータトラックは、4種類のasset sequence （スプライト、オブジェクト、サウンド 、シーケンス）のいずれかに追加でき、asset の空間的位置を設定するために使用します。 XとYの値をとります。
  • 回転。位置パラメータと同様に、回転パラメータはすべての シーケンスassets で利用可能で、asset の回転を度単位で設定することができます。GameMakerは回転を反時計回りに計算し、右側の 0°から始まるので、90°は上、180°は左、270°は下となることに留意してください。
  • Origin（原点）。Origin パラメータはスプライト、オブジェクト 、シーケンスアセットで 使用でき、asset の原点の X と Y の位置を設定します。原点を変更すると、asset の位置、回転、スケーリングに影響します。
  • スケール（Scale）。scale パラメータはスプライト、オブジェクト 、シーケンスアセットで 利用可能で、asset の水平および垂直スケールを設定するために使用します。これはasset がIDE とruntime でレンダリングされる方法に影響し、スカラー値として計算されます。100% はデフォルトasset サイズ、50% はハーフサイズ、200% は2倍のサイズとなります。なお、「リンク」 ボタンをクリックすると、水平と垂直のスケール値をリンクさせることができます。これは、一方の値を変更すると、もう一方の値も自動的に変更されることを意味します。
  • 画像インデックスと 画像速度：これらのパラメータは、スプライトと オブジェクトの アセットでのみ使用可能で、現在のアニメーションフレーム(のページを参照)を変更するために使用できます。 image_indexこれはFPSで設定され、sprite エディタで使用されているsprite に設定した値と同じように機能します。
  • カラーマルチプライ（Colour Multiply）。このパラメータはスプライトと オブジェクトアセットでのみ 使用可能で、描画されているsprite の現在のブレンドカラーを変更するために使用します。これは image_blendここで設定された色はsprite の色と「乗算」（つまりブレンド）されます。このタイプのトラックで作業するときは、16進数の値を手動で入力して色を設定するか、色見本をクリックしてGameMakerのカラーピッカーを開くことができることに注意してください。色はARGB形式で与えられるので、この種のパラメータトラックを使ってアルファ値を設定できることに注意してください（カラーピッカーではアルファ値の設定も可能です）。これは image_alphaをスプライトに設定します。
  • 音量です。Volumeパラメータはサウンドアセットトラックにのみ 適用され、サウンドのゲインを設定するために使用できます（実際のサウンドasset のゲインは変更されません。基本的にasset のsequences トラックに使用されるエミッタだからです）。ボリューム値はパーセンテージで計算され、0%はボリュームなし、100%はフルボリュームです。
  • ピッチ（Pitch）。ピッチパラメータはサウンドアセットトラックにのみ 適用され、サウンドのピッチを設定するために使用できます（ボリュームと同様に、実際のサウンドasset ゲインは変更されません。基本的には、asset のsequences トラックに使用されるエミッタだからです）。ピッチ値はパーセンテージで計算され、100%が音の「自然な」ピッチで、100%より小さいと低くなり、100%より大きいと高くなります。
  • Falloff: Falloff パラメータはサウンドアセットトラックにのみ 適用され、現在は実装されていません。
  • テキストトラックにのみ使用できる追加パラメータがいくつかあります。それらについては、「シーケンス中のテキスト」をお読みください。
