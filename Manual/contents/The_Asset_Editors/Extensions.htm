Extension 編集部

拡張パッケージは、GameMaker Language (GML) に多数の追加関数や定数を追加し、プロジェクト間で簡単に転送するために使用できます。Extension 関数は、外部SDKや他の言語 (Android の Java 、iOS の Objective-C など ) で書かれたソースコードにもリンクでき、ゲームの機能拡張に使用できます ( 通常、広告やアプリ内購入など、サードパーティ API の実装に使用します)。

拡張機能は、すべてのプロジェクトで頻繁に使用する関数のライブラリを作成するために使用できます。そのため、すべてのプロジェクトに同じscripts をインポートする代わりに、お気に入りの関数や定数のコレクションを保持するextension asset を追加するだけでよいのです。このような関数は、特定のプラットフォームに特化することもでき、extension がどのプラットフォームで使用できるかを簡単に選択することができます。

拡張機能のインポート

通常、サードパーティのextensions はGameMaker Marketplace にあり、購読しているものはMarketplace Libraryウィンドウにリストアップされ、そこからインストールすることができます。extension パッケージをMarketplace から入手しなかった場合でも、アセットブラウザの を右クリックして「既存のものを追加」を選択し、拡張機能のある場所をブラウズして選択すれば、プロジェクトに追加することができます。

NOTE .yymps/.yymp/.gmez のパッケージがある場合、それをIDE にドラッグしてインポートするか、「ツール」メニューから「ローカルパッケージのインポート」を選択してください。

対象プラットフォーム

extension をAsset Browser に追加したら、名前とバージョン番号をつけ、どのプラットフォームにエクスポートするか設定する必要があります。選択したプラットフォームのチェックを外すことで、そのプラットフォーム用の実行ファイルの一部としてextension をエクスポートしないようにGameMakerに指示することができる。また、extension のプロパティが、選択したターゲットプラットフォームと一致していることを確認し、extension のファイルについて、以下に基づいて互換性のあるエクスポートターゲットを選択する必要があります。

  • すべてのターゲット：GML extensions.extensions これらは、GameMaker Language (GML) を使って作られたもので、すべてのターゲット・プラットフォームに対応しています。(なお、Opera GXターゲットが現在サポートしているのは、このextension タイプだけです。)
  • Windows（Standard、UWP、Xbox）です。Windows DLL ファイル（Dynamic Link Libraries）を使用することができます。Windowsゲームオプションで「x64Windows ランタイムを使用する」をチェックした場合、プロジェクトのDLLはすべて64ビット対応である必要がありますので、ご注意ください。32ビットのDLLは、64ビットの実行ファイルでは動作しません。
  • macOS:macOS ターゲットは、標準の DyLib ファイルをextensions に使用することができます。
  • Ubuntu (Linux)。Ubuntu ターゲットでは、extension が .so ファイルを使用していることを確認する必要があります。
  • HTML5です。これらのプラットフォームでは、 JavaScript extensions を使用することができます。
  • Androidです。このプラットフォームでは、extension が Java で書かれていること、または Android Framework SDK が必要です。
  • iOS/ tvOS: このプラットフォーム用のExtensions は Objective C で書かれており、*.mm と*.h ファイル、または何らかのフレームワークを使用している場合はプリコンパイルされたネイティブ*.a ファイルが必要です。
  • PlayStation。PlayStation の場合、extensions は PRX ファイルとし、ターゲット（PS3、PS4、PS5）の仕様に合わせ、ターゲットごとに異なる PRX を使用する必要があります。PSVitaで作業する場合は、extension の代わりに SUPRX を指定してください。

注 extensions の中には、設計されたプラットフォームと CPU アーキテクチャを正しく識別するために、GameMaker の特定の命名規則に従う必要があるものがあります。詳細については、命名規則を参照してください。

extension extension この場合、メインのextension エディタで互換性のあるプラットフォームをすべて 指定し（つまり、それらの横にあるチェックボックスをオンにします）、次にextension の各部分の個々のファイルのプロパティで、それらのファイルがエクスポートされるべきプラットフォームを設定します。

特定のextension ファイルやプレースホルダーのターゲットプラットフォームを変更するには、まずextension プロパティを開き（ をダブルクリック、extension ブラウザでAsset をクリック）、extension プロパティで、編集したいリソースファイルを右から選択する必要があります。開いたウィンドウで、そのファイルがエクスポートされるプラットフォームを選択することができます。

拡張オプション

使用中のextension にはオプションがあり、Extension Editor ウィンドウの下部に表示されます。

これらは、extension によって異なり、また、すべてのものがオプショ ンを持っているとは限りません。そのオプションに関する情報は、あなたが使っているextension のドキュメントをご覧ください。

extension のオプションを設定するには、以下を参照してください。 Extension オプションの作成

ビルド時の解凍

Uncompress on build" オプションは、特定のターゲットプラットフォーム向けの圧縮された アーカイブファイルに使用できます。このオプションを有効にすると、アーカイブのコンテンツはビルドされたパッケージのIncluded Filesディレクトリに展開されますが、ターゲットプラットフォームにコピーされた後に展開されます。

たとえば、extension の .zip ファイルに対してこの機能を有効にし、Windows からmacOS の ビルドを開始すると、 .zip アーカイブはWindows からmacOS にそのままコピーされ、ターゲットプラットフォーム（この文脈ではmacOS ）でのコンパイル時に圧縮が解除されることになります。

Extension 関数の呼び出し

extension を設定したら、GameMaker Language で使用する組み込み関数や定数と同じように、その関数や定数を使用することができます。 もし、あなたのゲームがクロスプラットフォーム（例えば、Windows とAndroid ターゲットで動作するように設計されている）なら、問題を防ぐためにextension 固有の関数を呼ぶ前にOS 定数や関数を使って現在のプラットフォームをチェックすべきなんですね。

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

異なるプラットフォームでは、extensions は関数名を共有することができますので、（例えば）extension とWindows でメッセージをポップアップするための個別のファイルを持つAndroid がある場合、両方のextension ファイルは、それぞれのプラットフォームにのみエクスポートするように設定されている限り、同じ関数名を共有することができることに注意してください。

サードパーティ製公式拡張機能

GameMaker のレガシーバージョンでは、多くのサードパーティ機能が組み込まれていましたが、サードパーティによって特定の機能が変更されると、GameMaker IDEとランタイム全体を更新しなければならないため、非効率で保守が困難な状態になっていました。そこで、現在のバージョンでは、サードパーティのサポートをゲームに追加するために、extensions パッケージを使用することを採用しています。これにより、必要な機能のみを追加することができ、また、特定のニーズに合わせて変更・編集することも可能です。

これらのextensions のほとんどは、特定のゲームオプション（例えば、FacebookやGoogle Mobile Adsなど）を選択すると、ダウンロードされインストールされます。これらの詳細については、GameMaker Knowledge Baseの各プラットフォームの該当セクションから確認することができます。

  • GameMakerのプラットフォームとセットアップ

extensions 自体は、以下のMarketplace のページよりご覧いただけます。

  • 公式Marketplace 資産

なお、これらのextensions には、使い方を説明するデモとヘルプファイル（ *.pdf 形式）が付属しています。そのため、本マニュアルではこれらの機能を説明していませんのでご注意ください。
