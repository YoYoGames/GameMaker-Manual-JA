Tile Set 編集部

タイルセットは、ゲームのレベルやその他の静的コンポーネントを描画するためのグラフィックリソースです。tile set は1つのイメージで構成され、それが異なる「セル」(tiles)に分割され、各tile をroom エディタに配置することで完全なイメージを作成することができます。以下は、タイルセットとして使用できる2つの例です（sprites ）。

これらは、tile sets を作成する際に最も一般的な方法です。すべて の画像を詰め込んで作成することも、少しスペースを空けて作成することもできますが、GameMakerはどちらのタイプも扱うことができます。基本的に、すべてのtile sets は、明確に定義されたグリッド内に収まる画像で構成され、グリッドの各セルはまったく同じサイズでなければなりません（上記の画像はすべて正方形のtiles で構成されていますが、tile set は長方形のtiles からも作ることができることに注意してください）。

tile set リソースを初めて作成すると、Tile Set Editor ウィンドウが開き、以下のオプションが表示されます。

名称

ここで、あなたのtile set に名前を付けることができます。名前は英数字のみで、アンダーバー"_"記号のみ使用でき、数字で始めることはできません。

スプライト

ここでasset を作成するスプライト tile set を選択することができます。メインボタンをクリックすると、Asset エクスプローラが開き、タイルセットに使用するsprite を選択することができます。

また、New Spriteボタン ボタンをクリックして、新しいsprite asset を作成したり（編集用のasset の中に新しいworkspace のスプライトエディタを開きます）、Edit Spriteボタン を使って現在選択中のsprite を編集したり、 ボタンを押してイメージエディタで sprite 画像自体を編集することを選択することも可能です。

デフォルトでは、tile set を開くと、「タイルセットのプロパティ」ウィンドウも表示されます（下図）が、それを閉じてしまった場合は、ここのボタンをクリックして再び開くことができます。

物件紹介

タイルセットのプロパティ] ウィンドウでは、ベースとなるtile のサイズ（デフォルトは 16px の正方形）、オフセット量、およびtiles 間の間隔を指定します（最後の 2 つのオプションは、このページのさらに上の画像例で示されたsprites のように、tiles の個々のパーツ間に隙間がある場合に使用されます）。また、tile set 画像の「出力境界」の値を設定することで、画像がディスクに保存される方法を制御し、必要に応じて各tile の周囲に余分なピクセルを追加することができます。

tiles のレンダリング方法のため、エッジを「汚す」ために、それぞれの周囲にボーダーを付けて事前に準備する必要があります。下図は、tiles がデバイスグラフィックドライバによってどのようにマッピングされるかを示しています。tile の周囲の赤い線は U,V 座標がマッピングされる場所を示しています（2Dtexture のメッシュを表現するために "U" と "V" という文字が使われています。room とobject 空間座標を表現するのに "X", "Y" および "Z" はすでに使われているからです）。

room に画像を描画するときに、直接マッピングするテクセルの中心を選んでいるのがわかります。 さて、1:1 のスケールでは、これはすべて順調ですが、この を縮小するとどうなるのでしょう？縮小するときの問題は、画像を縮小すると、UVも縮小して、より圧縮された空間に収まろうとすることです。しかし、これは明らかに起こりえないので、グラフィックスドライバは利用可能なピクセル中心に基づいて、どのテクセルを描くかを選択します。tile

つまり、新しい画面ピクセルに移動するだけなら、グラフィックスドライバはピクセルの中心を使用して描画対象を決定するので、tile を囲むテキサルを選ぶことになり、tile 自体のテキサルを選ぶことにはならないのです。

上の図は、tile を 6 スクリーンピクセルに縮小して描画しています。これは、次のスクリーンピクセルに少し重なると、グラフィックドライバはそのテキスルを削除するか、有効なテキス ルで埋めようとするかを選択しなければならないことを示しています。この場合、オーバーラップしたテクセルをスクリーンピクセルに描画することに決めたとしましょう。現在、それは明らかにtile の外側なので、次のtile 、またはtiles の周りに提供したわずかなスペースからこのピクセルを取得します。つまり、縮小すると、tiles 間の亀裂や描画方法のグリッチが生じ、これが多くの混乱を引き起こすことになります...。このような場合、「liquids.com」は、「liquids.com」内の「liquids.com」内の「liquids.com」と「liquids.com」内の「liquids.com」の間にある「liquids.com」をクリックします。

ハイエンドゲームでは、単一のtexture (texture ページやtexture アトラスではありません) から描画する場合、クランプと 呼ばれるtexture モードを使用するオプションがあります。これは、最後の行のテクスルを無限に遠方に繰り返し、何があってもハードウェアに最後の行のピクセルが得られるよう強制するので、問題なくスケーリングできます。これは、出力ボーダーの 設定で行うことができます。

tile sprite tile つまり、ハードウェアがオーバーランしたときに、スペースやtile の 1 つではなく、常にtile の最後のテクセルを選択するようになりました。したがって、tile set を作成するときは、ゲーム ビューを拡大または縮小する場合、またはtiles を拡大縮小してレンダリングする場合、デフォルト値の 2 よりも高い Output Border width または height が必要な場合があることに留意してください。

次に、タイルセットリソースを所属させたいテクスチャグループを選択します。基本的に、texture グループ（Texture Group Manager で定義済み）は、特定のassets またはゲームのレベルに必要なすべてのイメージrooms を一緒に保存できるように設定するものです。texture GameMaker は、これらのグループ化されたアセットをすべて同じテクスチャページに配置し、選択した ターゲットプラットフォームでゲームを実行中にtexture ページスワップを削減しようとします。

注：この方法は必ずしも必要ではないかもしれませんし、この方法による性能向上は、ターゲットデバイスがCPUバウンドかGPUバウンドかによって異なります。

tile set プロパティの最後のオプションは、「Disable SourceSprite Export」（ソースのエクスポートを無効にする）です。flags これはデフォルトでチェックされており、プロジェクトのsprite ページが生成されるときに、ソースasset texture が省略されるようにします。これは、sprite がtile set として使用されている場合、他の用途には使用されないので、一度だけエクスポートする必要があるという前提で動作します。しかし、ベースとなるsprite リソースをtiles 以外に使用している場合は、このチェックを外して、コンパイル時にベースとなるsprite と tile set の両方がtexture ページに追加されるようにしたいものです。

編集部

エディタ] セクションの各ボタンは、特定のtile set 機能のための異なるエディタを開きま す。複数のtile set を合わせたものをベースにしたtiles ブラシ 、異なるtiles を個別のフレームとして使用したアニメーションタイルを作成できます。また、周囲にあるtiles をベースに「結合」するオートタイルを作成することも可能です。これらの各エディタについては、このページの最後にあるリンク先のセクションで詳しく説明されています。

プレビューツールボックス

これらは、sprite で選択したtile set をプレビューするために使用できるツールです。これらのボタンは現在のプレビューのズームレベルを制御し、ズームインまたはズームアウトに使用できます。 ボタンをクリックすると、プレビューが編集中のtile set sprite と 1:1 になるようにリセットされます。また、「ウィンドウに合わせる」ボタン をクリックすると、tile set 全体で現在のプレビューウィンドウに収まるようにできます（収まるように適宜ズームイン/アウトします）。 / とマウスホイール を使ってもズームイン・ズームアウトできます。また、 / + を押すと、キャンバスとtile set sprite が 1:1 になるように設定されます。

switch もう一つの特徴は、 ボタンをクリックすることで、グリッド表示のオン・オフができ、また、使用するグリッドの色も設定できます（ ボタンをクリックします）。グリッドが有効な場合、グリッド内のtiles それぞれに色のついたアウトラインが表示され、分離などの設定も反映されます。

プレビュー

このセクションは、tile set プレビューウィンドウです。プレビューを移動するには、一般的なWorkspaces と同じ基本的な操作が可能です。マウスの中ボタン とドラッグでプレビューをパンし（あるいは + の組み合わせを使用）、マウスホイール で水平方向にスクロールし、 / キーとマウスホイール でズームインとアウトを行うことができます。また、プレビューツールボックスのさまざまなボタンを使って、ズームの変更やグリッドセルオーバーレイの有効・無効を切り替えることができます。
