イベントオーダー

GameMakerのイベントを考えるとき、各ステップで発生するすべてのイベントの正確な順番は、GameMakerの内部動作に依存し、ソフトウェアの開発によって変更される可能性があるため、明言できないことに注意する必要があります。ただし、必ず 同じ順序で実行されるイベントがあります。

常に同じように発生する最初のイベントセットは、room が最初に入力されたときに発生するイベントです。異なるイベントが発生する順序は次のとおりです。

  • オブジェクト変数/インスタンス変数が初期化されます（オブジェクトエディターや ルームエディターの IDEで定義できる変数です）。 object 変数が添付されたobject がある場合、それらは他の何よりも先に作成され、変数が添付されたインスタンスは後に定義されます。 エディタを使用します。
  • 各インスタンスが 作成されると、最初にCreate イベントが実行され、次にInstance Creation Codeが実行されます。 次のインスタンスの作成に移る前に
  • Game Start Event- このイベントは、room エディタからroom に配置されたすべてのインスタンスについて、ゲームの一番最初のroom で一度だけトリガーされます (game_restart() を呼ぶとこのイベントが再びトリガーされることに注意してください)。
  • ルーム作成コード-Room が最初に入力されたときに、room Editor に書き込まれる一回限りのコードです。
  • すべてのインスタンスのルーム開始イベント- 「その他」カテゴリのイベントの1つで、永続的かどうかにかかわらず、すべてのインスタンスに対して発生します。

また、インスタンス・レイヤー・プロパティ・ウィンドウのリストを上下に移動することで、room エディタ自体で特定のインスタンスを作成する順番を設定できることも特筆すべき点です。

これらの特定のイベント以外で、何があっても同じように発生する唯一の既知の順序は、3つのステップ イベントと異なるドローイベントに 属します。これらは常に一貫しているので、もし ゲームの各ステップで特定のタイミングに依存するコードがある場合、これを使用する必要があります。

  • ステップイベント開始
  • ステップイベント（インスタンスが新しい位置に配置される直前にステップイベントが実行されることに注意）
  • ステップ終了イベント

描画に関するイベントもすべて以下のように常に同じ順序で処理されます（ただし、Window Resize イベントは異なるトリガーで発生します）。

  • 抽選会前イベント
  • 描画開始イベント
  • ドローイベント
  • 描画終了イベント
  • ポストドローイベント
  • ドローGUI開始イベント
  • ドローGUIイベント
  • GUI描画終了イベント
