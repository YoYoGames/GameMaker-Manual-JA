では、object イベントとは何でしょうか？基本的には、ゲーム中loop 、あなたがプログラムした内容に基づいて物事が起こるようにする、目立たない瞬間のことです。GameMakerはこのイベントのサイクルで動作します。room が開始された瞬間から終了する瞬間まで、ゲームloop が動作しており、ステップ ごとに一連のイベントが実行またはチェックされます。あなたはこれらのイベントに反応するコードまたはGML Visual アクションをobjects に配置することができます（ステップとはゲーム時間内の一瞬で、room スピード設定によって支配され、フレームと呼ばれることもあります） 。

これは、object が反応できるすべての基本イベントとイベントカテゴリーのリストで、各カテゴリー内には、動作をさらに細かく設定するためのサブイベントがあります。例えば、Key Press イベントカテゴリーをクリックすると、object がどの キーに反応するかを選択するためのイベントが表示されたウィンドウがポップアップ表示されます。

作成

このイベントは、object のインスタンスが最初に作成されたときに発生し、room Editor を通してRoom に入力されたときにroom に配置されたインスタンス内で最初に発生するものである。つまり、このイベントは変数の初期化、タイムラインの開始、パスの設定など、一般的に一度だけ、またはインスタンスがroom で最初に作成されたときだけ行う必要があることを行うのに理想的な場所です。object がObject Editor またはRoom Editor でオブジェクト変数またはインスタンス変数を追加した場合、これらの変数は最初に初期化されてから、Create Eventが実行されます。

インスタンスの作成イベントの後に直接実行され、インスタンス変数を作成したり、Room 変数や実際の作成イベントで追加された変数を上書きするために使用できるため、Object エディタのインスタンス作成コードから作成イベントで設定したものを変更できることを忘れないでください。

デストロイ

このイベントは、インスタンスが破壊されたときに実行されるイベントです。objects にビヘイビアを追加するときに見落とされがちですが、例えば敵が殺されたときに爆発やパーティクル効果を作ったり、object の新しいインスタンスをroom の別の場所に再スポーニングしたり、あるいはスコアにポイントを追加したりと、非常に便利なものである場合があります。

クリーンアップ

このイベントは、object のインスタンスをroom から削除するイベントの後に呼び出されます。 したがって、次の場合にトリガーされます。

  • インスタンスが破棄される
  • theroom ends
  • 試合終了

これは、ゲーム内の動的リソース（サーフェス、データ構造など）を「クリーンアップ」したり、インスタンスが何らかの方法でゲームから削除されたときに、一度 実行する必要があるタスクを実行するために使用するために設計されています。

このイベントは、トリガーとなったイベントの後に即座に 呼び出されますが、現在のイベントが終了するまで、インスタンスは実際にゲームから削除されないことに注意してください。例えば、ステップイベントで instance_destroy() を呼び出すと、Destroy イベントが呼び出され、次に Clean Up イベントが呼び出され、ステップイベントの残りの部分が実行され終わります。つまり、 instance_destroy() の呼び出しの後にあるコードはまだ実行されており、コードが必要とするデータ構造などのリソースをクリーンアップした場合、エラーの原因になる可能性があるので、このイベントを使用する場合は注意が必要です。

アラーム

アラームカテゴリは12のイベントに分割され、インスタンスに設定可能なアラームごとに1つずつ設定されます。そのため、Add Alarm カテゴリをクリックすると、このようなウィンドウが表示されます。

ここで、作成したいアラームを選択すると、イベントウィンドウに追加され、通常通りコードを追加できるようになるのがわかります。しかし、アラームとは何でしょうか？アラームが設定されていない限り何もしない特別なイベントで、アラームがカウントダウンして0になるまで待ってから、追加したアクションやコードを実行します。

アラームが0に達してコードを実行すると、その後-1までカウントダウンされ、再び設定されるまでそのままです（つまり、アラームの値が-1より大きいかどうかを確認することで、実行中かどうかが分かります）。つまり、object の create イベントで alarm[0] を 30 に設定すると、GameMaker は alarm[0] イベントに配置されたアクションやコードを実行する前に、30ゲームステップのカウントダウンを行うということです。なお、アラームを0に設定すると、イベントが発生してもアラームコードは実行されず 、すぐに-1に設定されるため、コードはスキップされます。もし、アラームが次のステップで実行される必要がある場合は、アラームを1に設定する必要があります。

これは非常に便利な機能で、正確なタイミングで物事を動かすことができますし、独自のイベントにアラームを設定することを妨げるものは何もないので、繰り返させることも可能です。例えば、モンスターが3秒ごとに右折するとします。そのモンスターの生成イベントで、room 速度 * 3 にアラームを設定します（room の速度が 30 であれば、1 秒あたり 30 ステップなので、それを 3 倍すれば 3 秒です！）そして、アラームイベントで方向を設定するコードまたはアクションと、そのアラームを再びroom 速度 * 3 に設定するアクション（またはコード）が用意されているのです。このようにして、特定の間隔でしか物事が起こらないような、シンプルなゲームloops を設定することができます。

注意すべきは、アクションやコードが何も入っていないアラームはカウントダウンされないという ことです。しかし、コメントだけでコードやアクションがない場合でも、アラームはカウントダウンを続け、通常と同じように設定や確認ができます。

ステップ

GameMakerは時間をステップに 分割し、room 、1秒間にいくつのステップがあるのかを定義します（1ステップは1フレームとも呼ばれることがあります）。1つのステップは、基本的にloop で、ゲームが動いている間、すべてのイベントがチェックされ、必要に応じてトリガーされます。したがって、ステップ イベントは、インスタンスが存在する間、ゲームのすべてのステップをチェックするイベントです。

しかし、どのコードをどのタイミングで実行するか、もう少しコントロールしたい場合もあります。そこで、Beginと Endの ステップイベントが用意されています。この3つはステップごとにチェックされますが、その順番はGameMakerエンジンのアップデートで他のイベントが変更されても変わることはありません。つまり、これは何かが必ず他の何かの前に起こることを確認する唯一の信頼できる方法なのです。

ステップイベントは何に使えるのでしょうか？連続的に実行される必要のあるアクションやコードに使うことができます。例えば、あるobject が別のものを追いかける場合、ここで、追いかけているobject に向かって動く方向を適応させ、スムーズに後ろに移動し続けることができるのです。objectsしかし、このイベントには注意が必要です。特に、ゲーム内でobject のインスタンスがたくさんある場合、room の step イベントには複雑な動作をあまり入れないようにしましょう。これは、ゲームの速度を遅くする可能性があります。特に、ゲーム内で多くのインスタンスを作成する場合は、ゲームが遅くなる可能性があるので、注意してください。

衝突

もちろん、ゲームを作るときには、2つ（またはそれ以上）のobject のインスタンスが衝突したことを知ることが非常に重要です。そのために、衝突イベントがあります。これはobject に配置し、他のどのobject に対して衝突をチェックするかを指定するイベントです。

物理をオンにしていない場合、これらの衝突は、2つのobjects のマスク（マスクはsprite プロパティで定義されますが、object プロパティで個別に割り当てることもできます）と重なるかどうかに基づいて計算されます。衝突するインスタンスの一方にマスクが割り当てられていない場合（またはsprite のマスクが何も設定されていない場合）、たとえ何かを描いていたとしても衝突は検出されませんのでご注意ください。

Physicsが オンの場合、衝突は、その物理プロパティでobject に定義した衝突形状（Fixture）のタイプに基づき、衝突に対する反応も同様となります。つまり、衝突を処理するためのコードは必要ありませんが、衝突を検出するために、このイベントには少なくともコメントが必要 です。

最後に、すべての衝突は、衝突イベントがトリガーされる前にゲームステップごとに1回計算されることに注意してください。つまり、このイベントでインスタンスを作成し、そのインスタンスで衝突をチェックしようとすると、次のゲームの繰り返しまで衝突は検出されず、解決されません。loop.

プレイヤーにゲームの様々な側面をコントロールさせることは非常に重要で、そのためにGameMakerは、3つの主要なキーボードカテゴリのいずれでも使用できる、非常に包括的なキーボードイベントのリストを提供します。一般的なキーボードカテゴリの場合、選択したキーが押されている限り、毎ステップ継続的にトリガーされますが、プレス＆リリースカテゴリのイベントは、最初にキーを押したり離したりした時に一度だけトリガーされることになります。

キーボードイベントは、キーが使われるたびにroom のすべてのアクティブなインスタンスで実際にトリガーされますが、その特定のキーに対してイベントが定義されているものだけが反応します。また、任意のobject で複数のキーボードイベントを作成でき、ゲーム実行中はそのobject のインスタンスがすべてのイベントに反応します。

object にキーボードイベントを追加すると、キーボードサブイベントメニューが表示され、チェックするキーを指定することができます。

一番上に矢印キー、次によく使うモディファイアキー、そして残りのキーボード（さらにサブセクションに分かれているので、 や など必要なキーを正確に取得できます）、最後にNo KeyとAny Key という非常に特殊なサブイベントがあります。これらは、その名前が示すように、キーが押されていない ときと、いずれかの キーが押されたときにチェックされるサブイベントです。テンキーのキーは、Number Lockが 有効な場合にのみ対応するイベントを発生させることに注意してください。

キーボードのPress とRelease イベントは、通常のキーボードイベントとほぼ同じですが、連続的にトリガーされるのではなく、一度だけトリガーされる点が異なります。キーボードが最初にキーを押したと認識すると、Keyboard Pressedイベント（通常のキーボードイベントと同様）を生成し、その後、キーが押されたと認識されなくなると、Keyboard Releaseイベントを1回だけトリガーします。

マウス

マウス」カテゴリは、選択可能な一連のイベントに分かれており、ゲーム内で起こっていることをより正確にコントロールすることができます。ここでは、これらのイベントがどのようなものであるか、正確に知ることができます。

左 、右 、中央 のボタンイベント（通常、押した 、離したに関わらず）はすべて、そのイベントを持つインスタンスのマスクで動作します。つまり、GameMakerはこれらのボタンが使われたときに、room におけるマウスの位置を、マウスイベントを持つインスタンスのコリジョンマスクと照らし合わせるということです。インスタンスのバウンディングボックスと「衝突」した場合、イベントがトリガーされます。したがって、これらのイベントを持つインスタンスは、有効なコリジョンマスクを持つsprite を持っているか、object のマスクsprite をobject のプロパティで選択していることを確認します。これらのイベントは、その名前が示すように、選択されたマウスボタンが最初に押されたり離されたりしたときに一度だけ、またはボタンが維持されている間は各ステップで連続的にトリガーされます。

マウスの入力と離脱のイベントも、インスタンスのマスクに依存して動作するという点ではボタンイベントに似ていますが、今回は、マウスが最初にインスタンスに「入る」（触れる）ときと、「離れる」（触れなくなる）ときにトリガーされます。これらのイベントは連続的ではなく、マウスがobject に入ったり出たりするたびに一度だけトリガーされます。したがって、例えば、マウスがその上に乗ったときに変化し、マウスを離すと元に戻るようなボタンを作成する場合に理想的な方法と言えます。

ジェスチャー

このイベントは、ユーザーが画面に触れたとき（モバイルの場合）、またはマウスをクリックして動かしたとき（他のすべてのプラットフォームの場合）に発生するものです。これらのイベントは、通常バージョンとグローバルバージョンがあるという点で、マウスイベントと似ています。これらのイベントの通常バージョンは、sprite （またはマスク）を持つインスタンスでタッチが発生し、そのバウンディング ボックス内でタッチが発生した場合にのみトリガーされます。一方、グローバルバージョンのイベントは、ユーザーが画面上の任意の 場所をタッチしたときにトリガーされます。

ジェスチャーイベントは、以下を検出します。

  • タップ - ユーザーがクリック/タッチしてすぐに離すとき
  • ドラッグ - ユーザーがタッチ/クリックし、それを維持したまま指/カーソルを移動させる場合。
  • フリック - ユーザーがタッチ/クリックを移動して離す動作を一度に行う場合
  • ピンチ - 画面の上に2本の指を置いた状態で、指をそろえたり離したりすること。
  • rotates - 画面の上に2本の指を置き、ある点を中心に回転させる場合。
