保存／読み込み

このイベントは、ロードに関連する特定の関数からのコールバックによってトリガーされます。 やbuffers をファイルに保存するとき、およびオーディオをメモリからロードまたはアンロードするときに使用します。このイベント自体には、内蔵の async_load DSマップには、特定の関数に必要なキーが入力されます。これらは以下のセクションにリストアップされています。

バッファ

機能を使用する場合 buffer_save_async()または buffer_load_async()を使用します。 データ転送が完了すると、非同期イベントが発生します。このイベントは、 async_load マップに以下のキーと値のペアを入力します。

  • "id": 使用する関数が返す非同期関数のID。
  • "status": データが正しく保存/ロードされた場合はtrueを、そうでない場合はfalseを返します。

これにより、保存/読み込みの進捗状況をポーリングし、完了時にメッセージを表示したり、rooms を変更したりすることができます。

オーディオグループ

オーディオグループを操作する場合、関数 audio_group_load()と audio_group_unload().load関数を使用する場合、オーディオのフルセットが揃うとこのイベントが発生します。 がメモリに読み込まれ、次のキーと値のペアがマップに入力されます。

  • "type": これは呼び出されるイベントの種類を示し、オーディオをロードするための "audiogroup_load "になります。
  • "group_id": 読み込まれたオーディオグループのIDを返します（オーディオグループエディターで定義されたものです）。

グループのオーディオがすべてロードされると、このイベントがトリガーされ、rooms を変更したり、音楽トラックを再生したりするために使用されます。
