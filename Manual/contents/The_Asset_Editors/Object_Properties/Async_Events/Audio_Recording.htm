オーディオ録音

このイベントを発生させることができるのは audio_start_recording()関数 に格納されたDS Mapを返す。 async_load, イベントをトリガーした関数からのコールバックに応じて、異なるキーと値のペアを含んでいます。

ds_map では、以下のキーが利用可能になります。

  • "buffer_id" - オーディオデータの取得に使用するテンポラリバッファのID。
  • "channel_index" - このデータが来た呼び出し関数によって返された録画チャンネルインデックス
  • "data_len" - 受信したデータの長さ(バイト)

注意：このページの冒頭で述べたように、変数async_load は非同期イベントでのみ有効であり、この変数が指す DS マップはイベントの開始時に作成されるからです。 を削除し、最後に再び削除する。ただし、すべてのオーディオがマップとともに保存されている一時的なbuffer は、イベントの終了時に削除されるので、後で使用するためにそれを保持したい場合は、カスタムbuffer にコピーしておく必要があることに注意してください。

以下の使用例では、録音したオーディオを保存するためにカスタムbuffer を作成し、入力ソース0から録音するようにゲームを設定します。

channel_index = audio_start_recording(0);
audio_buffer = buffer_create(len, buffer_fast, 1);

録音が検出されると、ゲームのステップごとに非同期オーディオ録音イベントが発生し、録音された入力を次のように処理することができます。

var channel = async_load[? "channel_index"];
if channel == channel_index
   {
   len = async_load[? "data_len"];
   buffer_copy(async_load[? "buffer_id"], 0, len, audio_buffer, 0);
   }

ここでは、単に "channel_index" キーをチェックして、それが録画開始時に返された値と一致することを確認します。一致する場合は、このイベントのために作成した一時的なbuffer の内容を、カスタムbuffer にコピーします。その後、あなたは buffer buffer のオーディオ専用機能で再生したり、加工してネットワークに送ったり、ディスクに保存したり......。録音したオーディオがbuffer にあれば、基本的に以下のことが可能です。 ということです。
