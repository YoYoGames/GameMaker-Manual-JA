HTTP

HTTPイベントは、次のようなhttp_*() 関数の1つからのコールバックによって引き起こされるものです。 http_post_string().これは実際にはこのイベント専用のDSマップを生成し、特別な変数 async_load に格納される(このイベントの使い方をさらに詳しく説明するコード例については、個々の関数を参照してほしい)。この DS マップは以下のような一般的な構造を持っている。

  • "id":コマンドから返された ID。 http_ リクエストを連続して発行した場合、どのリクエストに対する返答なのかを知る必要があります。 そのため、この値を使って、最初にリクエストを送ったときに保存した値と比較し、 正しいものを探すことになります。
  • "status" となります。エラーの場合は 0 未満、成功の場合は 0、コンテンツがダウンロードされている場合は 1 を返します。
  • "result" となります。受信したデータ（string のみ）、または、path を使用した場合は、ダウンロードしたファイルへのアクセス。 http_get_file().このキーは、ステータスが 0 の場合にのみ取得できますが、これはプラットフォーム依存です。
  • "url":要求された完全な URL。
  • "http_status":生の http ステータスコード (利用可能な場合)。これは、ほとんどのブラウザで標準的なウェブステータスコードを返します。 例: 304 は "Not Modified"、204 は "No Content" など。

を使用したときに得られるものを示しています。 http_post_string()しかし、 http_ 関数はそれぞれ微妙に異なるマップを返すことがあります。そのため、各関数のマニュアルエントリを参照して、その関数で返される正確なデータを確認してください。

注async_load はDSマップを作成するので、これらの関数は json_encode()および json_decode()関数と組み合わせると特に便利です。

また、ダウンロード用のファイルを要求した場合、このマップから追加データが提供されることもある。この場合、"status" の値は 1 となり、DS マップはこれらの追加のキーを保持することになる。

  • "contentLength" です。これは、ウェブserver が受信するはずだと言ったファイルのサイズです (server がこのデータを返さなかった場合は -1 になる可能性があります)。
  • "sizeDownloaded":すでにダウンロードされたデータのサイズです。

このイベントは、データパケットを受信するたびに発生するのではなく、メインゲームloop 内のダウンロード中に随時更新されることに注意してください。また、現在この機能は、通常のWindows ターゲットプラットフォームでのみ利用可能であることに注意してください。
