画像読み込み

このイベントは、HTML5上、もしくはURLから GameMakerに画像を読み込んだときに発生する。例えば、sprite の画像を読み込み、読み込んだ後にインスタンスの現在のsprite を新しいものに変えるだけだとする。この場合、Createイベント（または他のイベント）で以下のような処理を行うことになる。

var _url = "http://www.angusgames.com/game/background1.png";
spr = sprite_add(_url, 0, false, false, 0, 0);

これにより、デバイスやブラウザへの画像の読み込みが始まりますが、ファイルの読み込みを待っている間、GameMakerをブロックすることはありません。代わりにGameMakerは、画像がロードされ、コールバックが Image Loaded Eventをトリガーするまで通常通り動作し、そこでDS Mapが作成され、特別な変数 async_load.このマップには、以下の情報が含まれています。

  • "filename" です。リクエストされたファイルへの完全なpath 。
  • "id":読み込んだリソースのID。これは、リソースを割り当てた変数と同じになります。
  • "status" となります。エラーの場合は 0 未満の値を返します。

そして、このイベントで、新しく読み込まれた画像をsprite に割り当てます。次のコード例は、返された情報がどのように使用されるかを示しています。

if ds_map_find_value(async_load, "id") == spr
{
    if ds_map_find_value(async_load, "status") >= 0
    {
        sprite_index = spr;
    }
}

上記のコードでは、まず作成されたDSマップの id 、次にコールバックの status をチェックします。この値が 0 以上であれば (成功を意味する)、コールバックの結果が返されます。 コールバックは、新しく読み込まれた画像にsprite のインデックスを設定するために使用されます。
