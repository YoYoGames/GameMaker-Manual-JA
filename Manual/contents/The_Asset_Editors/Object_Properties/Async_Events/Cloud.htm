クラウド

クラウドイベントは、いずれかのイベントからのコールバックで発生するイベントです。 cloud_*()関数を使用します。 のように cloud_synchronise().実際にはこのイベント専用のDSマップが生成され 特殊変数 async_load (の使用方法を説明するコード例は、各関数を参照してください。 イベントをさらに詳しく説明します)。このDSマップは次のような構造になっている。

  • "status" となります。これはステータスコードを保持し、負の値はエラーを表し、その説明は "errorString" に含まれることになります。0(または正の値)は成功を意味する(詳細は以下を参照)。 resultString "には、返されたデータまたは成功メッセージが格納される。
  • "id":呼び出された関数から返された ID。一連の cloud_ リクエストを発行した場合、どのリクエストに対する返答なのかを知る必要があります。 を送信したときに保存した値から、正しいものを探します。
  • "description":最後にアップロードされたファイルの説明。
  • "resultString" です。これは、クラウドから返されたデータブロブであるstring を保持します。
  • "errorString": あらゆるエラーに対してErrorString を返します。

返された "status "マップエントリの正確な意味は、次の表で説明される。

ステータス値 errorString / resultString 商品説明
-1 errorString = "<SERVICE>にログインしていません" You have not successfully logged in to the given Cloud Service
0 resultString = 復元されたデータ New game data downloaded from the cloud (following a cloud_synchronise() 召し上げる）
1 resultString = "AlreadySynchronized "とする。 No new data since you last called cloud_synchronise
2 resultString = "ConflictDeferral"。 A conflict was encountered, but the gamer chose to ignore it
3 resultString = "GameUploadSuccess"。 data from cloud_string_save() または がクラウドに正常にアップロードされました。 cloud_file_save()
-n errorString = エラーの説明 Any other negative number means a synchronisation failure
