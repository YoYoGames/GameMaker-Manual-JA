ダイアログ

Dialog非同期イベントは、以下のいずれかからコールバックを受けたときにのみトリガーされます。 のような特殊な非同期ダイアログ関数を使用します。 get_login_async()(をご覧ください。 このイベントがどのように使用されるかの拡張コード例については、この関数を参照してください)。

これらのイベントは、名前、ログイン情報、数字、色など、何らかのユーザー入力を要求するものです。ほとんどのデバイスは、loop で返事を待つことを好まないので、非同期である必要があります。 これらの関数がダイアログを開いている間、そしてこのイベントのトリガーとなるユーザー入力があるまで、バックグラウンドで実行されつづけます。つまり、ゲームの進行中にダイアログが開かれた場合は、ゲームを一時停止するような措置を取る必要があります。 ダイアログのコールバックを受け取るまで、ゲームを続けることができます。

ここでも、特殊変数にIDを保持したDS Mapが返される async_load. このマップに保持される値は、使用する関数によって異なるので、詳しくは本マニュアルの各関数の項目を参照してください。

注意事項非同期ダイアログ関数は デバッグとテスト用にのみ使用され、リリースされたゲームでは使用しないでください。そのためには、プレイヤーからの入力を受け取るためのカスタム ユーザー インターフェイスを作成し、ダイアログの外観と動作を完全に制御できるようにする必要があります。
