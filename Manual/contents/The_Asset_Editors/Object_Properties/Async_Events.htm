非同期イベント

非同期イベントとは、GameMakerが外部からコールバックを受け取った際に発生するイベントのことで、Webやゲームを実行しているデバイスから発生することがあります。基本的には GameMakerに何かをするように指示すると、画像を読み込むなどの処理を開始しますが、その間も他の処理を続けます。そして、リクエストが完了すると、コールバックがGameMakerに送られます。 Studio 2では、このタイプのコールバックに対して定義された非同期イベントが発生します。

http_get() 非同期イベントは、キーボードイベントのように、そのイベントを持つすべての インスタンスに対して発生することに注意してください。

注：ブラウザのXSS保護により、以下のイベントのいずれかを使用する場合、ドメインをまたがるリソースへのリクエストやロードの試みはブロックされ、空白の結果を返すように見えることがあることを認識しておく必要があります。

非同期イベントカテゴリに関連するイベントには様々な種類があり、それらはすべて以下のセクションで説明されています。

注：変数 非同期ロード というのは なぜなら、この変数が指す DS マップはイベントの開始時に作成され、終了時に再び削除され、それ以外のときはこの変数の値は -1 にリセットされるからです。

  • オーディオ録音
