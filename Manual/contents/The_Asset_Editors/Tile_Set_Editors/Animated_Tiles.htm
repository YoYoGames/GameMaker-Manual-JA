アニメーテッドタイル

一般的にタイルは、ゲーム内の静的なセルと考えられていますroom しかし、GameMakerでは、sprite と同じように、タイルをアニメーション化することができます。 なぜ、sprites を使わないのか、疑問に思われるでしょう。なぜ、sprites を使わないのかというと、 には、ある種の処理オーバーヘッドがあるためです。 しかし、tiles は処理オーバーヘッドがはるかに低いため、レンダリング速度が速くなります。しかし、この代償として、tile set のアニメーションを使用する場合、次のような制約があります。 アニメーションの長さは2の累乗でなければなりません。つまり、2、4、8、16、...フレームで、tile グリッド内で作成する必要があり、 すべて同じ速度で（タイルセットごとに）アニメーション化する必要があります。しかし、たとえ このような制限がありますが、アニメーション（tiles ）は、静的な環境や背景に生命力と色彩を与える強力な機能として使用できます。

tile アニメーションを作成するには、まずtile set に必要なtiles をすべて 1 つのsprite 画像に収め（ただしsprite フレームとしてではなく、1 つの大きなsprite フレームにすべてのアニメーション画像を収める）、タイルセットエディタで「タイルアニメーション」ボタンをクリックしてアニメーションエディタを表示させる必要があり ます。

上の画像は、すべてのアニメーションフレームが含まれる1枚の画像ですが、この画像には tiles というのも、1つの巨大なtile set を使って、その中からアニメーションを作成したい部分を「選び出す」ことができるからです。たとえば、木々のある大きな風景tile set があるとします。 土と水、そしてtile set の中に、水の波紋を作るためにアニメーションさせるべき複数のtiles があります。

tile アニメーションを作成するには、まずアニメーションライブラリにアニメーションを追加する必要があり、「アニメーションの追加 」ボタン をクリックします。 ライブラリに空のアニメーションストリップを追加し、使用するフレーム数を選択するように要求します。

その後、各アニメーションフレームが表示され、最初のフレームはハイライト表示されます。

その後、エディタの左側にあるtile set の画像に戻り、任意のtile の上で をクリックすると、アニメーションに追加され、フレームが次のものに進みます。続けてtiles の上で をクリックすると、フレームがいっぱいになります。 アニメーションは、フレームの横にある「再生」ボタンをクリックすることでプレビューすることができます。

この操作を行うと、アニメーションはアニメーションライブラリの一部になり、ルームエディタに入ったときに、それを選択してroom に配置できます。アニメーションからtiles を選択してroom に配置すると、選択したtile 以降のアニメーションがtile で始まります。 フレームをroom 内に配置すると、それらがすべてアニメートされます。この場合の唯一の注意点は、アニメーション速度がライブラリ内の個々のアニメーションではなく、tile set 全体で定義されているため、常に同じになることです。

アニメーションで使用されているtiles についての注意点です。

  • tile に独自のアニメーションが指定されている場合（つまり、最初のtile として設定されているアニメーションがある場合）、そのアニメーションが使用されます。
  • もし、tile が独自のアニメーションを持っておらず、他の1つのアニメーションにのみ使用されている場合、tile がアニメーション内に存在する最初の位置からそのアニメーションを使用することになります。
  • tile が独自のアニメーションを持たず、他の複数のアニメーションで使用されている場合、それは全くアニメーションしません。これを最初のアニメーションとするアニメーションを明示的に指定する必要がありますtile 。
