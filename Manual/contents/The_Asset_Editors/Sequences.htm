Sequence 編集部

Sequenceは 基本的に、時間の経過とともにダイナミックなアニメーションを実行するassets のコレクションです。sprites 、インスタンス、サウンド、その他のものを含むことができ、これらのそれぞれは、sequence 自身で時間の経過とともに動いたり、色を変えたり、アニメーションを開始/停止するように設定することができます。追加したassets は、sequence 内の特定のトラックに 割り当てられ、これらのトラックには、キーフレームと呼ばれる異なる属性を適用することができ、静的または時間と共に変化します。sequence の編集はすべてシーケンスエディタで行い、基本的に3つのエリアで構成されています。

  • キャンバス - シーケンスを構成するassets を配置する場所です。
  • トラックパネル - アセットトラックと パラメータトラックをシーケンスに追加/削除します。
  • ドープシート- トラックのキーフレームを追加/削除/編集し、時間経過による動作を制御します。

また、sequences の中にsequences を入れて、複雑なアニメーションやエフェクトを作ることもできます。sequences はコードでアクセスして変更できるので、ゲームでの表示や使い方をできる限りコントロールすることができます。sequences のコード化された側面についての詳細は、「シーケンス機能」のセクションを参照してください。

エディターの詳細を説明し続ける前に、sequence の作成方法と、説明で使用されるいくつかのキーワードを説明し、Sequences エディターの動作をより深く理解していただきます。sequence を作成するための一般的なワークフローは次のとおりです。

  • まず、アセットブラウザから sequence を作成し、Sequences エディタを開きます。
  • 次に、asset (sprite,object, sound orsequence) をエディタキャンバスにドラッグします。
  • これによって、トラックパネルに アセットトラックが作成され、ドープシートに アセットキーが追加されます。asset トラックは、単にasset を含むトラックの名前です。一方asset キーは、asset がsequence でアニメートされるフレーム数を象徴するために描かれるバーです。
  • この時点で、通常、キャンバス内のasset をsequence の開始時の位置に配置し、スケールや回転などの初期トランスフォームを追加することになります。
  • 初期設定に満足したら、パラメータキー（またはキーフレーム）を記録します。これは、特定のasset キーがdope sheet timeline の任意の時点で持つスケール、回転、位置の値に付けられた単純な名前です。
  • 次にプレイヘッドの位置（dope sheet timeline の現在の再生位置）を変更し、asset のトランスフォームを移動または変更し、別のパラメータキーを記録することになります。
  • すべてのパラメータキーは、トラックパネルのメイントラックasset の下にあるトラックのサブセットである追加パラメータトラックに格納され、トラックエディタで各パラメータのキーデータを編集することができま す。
  • あとは、上記の作業をsequence の長さ分繰り返し、必要に応じてasset を移動、回転、拡大縮小し、必要な分だけassets を追加していくことになります。

以下、sequence エディターの各機能の概要を説明します。このページの下部には、上記の3つの主要機能をより詳しく説明するページへのリンクがあります。

シーケンスイベント

通常のobjects と同様に、sequences はイベントを持ち、そのイベントに割り当てられたコードを実行することができます。コードはスクリプト化された関数の形で割り当てられ、引数を取らないことができ、イベントがトリガーされたときに呼び出されます。 この関数は引数を取らず、イベントが発生したときに呼び出されます。

イベントの追加は、「Add Event」ボタン（ ）をクリックすると、次のようなウィンドウが表示されます。

ここに掲載されているイベントは、以下の通りです。

  • 作成 - このイベントは、sequence のインスタンスが最初に作成されたときに発生します。また、sequence からroom のエディタを通してroom に入力されたときに、そのroom 内で発生する一番最初の出来事です。
  • Destroy - このイベントは、sequence のインスタンスが破棄されるときに実行され、Clean Up イベント（下記参照）の前に実行されます。
  • Clean Up- このイベントはsequence からroom のインスタンスを削除するイベントの後に呼び出されます。つまり、sequence が破壊された場合、room が終了した場合、またはゲームが終了した場合にトリガーされます。 sequence のインスタンスが何らかの理由でゲームから削除されたときに、一度だけ実行する必要があるタスクを実行するように設計されています。sequence のインスタンスを layer_sequence_destroy() などで破壊した場合、 このイベントは Destroy イベントの後に呼び出されます。 イベント (上記参照) の後に呼び出されます。
  • Begin Step/Step / End Step- ステップイベントは、sequence インスタンスが存在する間、ゲームの1ステップ（フレーム）ごとにチェックされるイベントで、begin, step, end の3つのパートに分かれる。 終了の3つの部分に分かれます。ほとんどの場合、標準のステップ イベントを使用しても問題ありませんが、どのコードをどのタイミングで実行するか、もう少しコントロールしたい場合もあります。 これらのイベントは、ゲームの各ステップ（フレーム）ごとに、常に同じ順序でトリガーされます。重要! sequence が一時停止している場合、これらのイベントはトリガーされません 。再生が再開されると、 sequence が再び再生を開始した次のフレームでトリガーされます。 また、イベントの順番はプレイヘッドの方向には影響されません。sequence が逆向きに再生されている場合でも、イベントは Begin、Step、End の順番で実行されることに注意してください。
  • Async Event（非同期イベント）。これはobject Asynchronous System Event に相当するものです。重要! sequence が一時停止している場合、このイベントはトリガーされません 。

各イベントには、イベントがトリガーされたときに呼び出される関数を1つ割り当てることができます。イベントの入力ボックスを使って関数を割り当て、矢印ボタンをクリックすると、関数を編集するためのエディタ（script ）が表示されます。 また、下部にある「新規作成」ボタンをクリックすると、「定型」関数がすでに定義され、すぐに入力できる状態で新しいscript リソースを作成することができます。関数名は自由に変更でき、定義済みの名前を使う必要はありません。 また、不要な関数の定義を削除することもできます。sequence イベントで使用される関数は、引数を取ることができないことに注意してください。

このオプションでは、Sequence Editorが2つの別々のウィンドウに分割されます。メインのIDE ウィンドウはフルデスクトップのキャンバスビューを表示し、新しいウィンドウはDope Sheet とTrack Editorを表示します。このボタンをもう一度クリックすると はレイアウトを再び変更し、余分なウィンドウを削除します。
スナップトップ

このオプションは、Track Editor とDopesheet をIDE の一番上にスナップさせます。
スナップライト

このオプションは、Track Editor とDopesheet をIDE の右側にスナップさせます。
スナップボトム

このオプションは、Track Editor とDopesheet をIDE の下にスナップさせます（これがデフォルトの表示です）。
スナップレフト

このオプションは、Track Editor とDopesheet をIDE の左側にスナップさせます。
