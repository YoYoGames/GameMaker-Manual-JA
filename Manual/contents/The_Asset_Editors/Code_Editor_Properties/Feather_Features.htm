フェザーの特徴

FeatherはGML コードscripts において、インテリジェントなコード補完と改良された構文チェックを提供し、スマートなリファクタリングオプションも提供します。

このページでは、Feather がGML コードエディタで提供する、スクリプトと オブジェクトイベントの機能を一覧で紹介します。これらの機能は、Feather が次のような場合にのみ利用できます。 enabled.

ホバー情報

関数、sprite 、色、その他の特殊な値の上にカーソルを置くと、その情報が表示されます。以下にいくつかの例を示します。

アセットリファクタリング

asset Browser でAsset の名前を変更すると、Feather は自動的にプロジェクトのコード内のその名前に関するすべての言及を更新し、それらの参照が壊れないようにします。この機能は、Feather の設定で有効にも無効にもできます。

また、コードエディターからasset の名前を変更することができます。テキストカーソルをasset のリファレンスに置き、CTRL/CMD + SHIFT + Rを押します。

新しい名前を入力し、"Apply"を押してください。asset 自体と、プロジェクト内のすべての参照先がリネームされます。

クイックフィックス

コード中にエラーや警告が発生した場合、報告された行にテキストカーソルを置き、CTRL/CMD + Qを押して、Quick Fixesメニューを表示させます。または、ガッターにある Feather アイコンをクリックすることもできます。

このメニューでは、可能であればすぐに修正できる方法を示し、その行のルールを無効にしたり、深刻度を変更したりすることができます。

上記の例は、ターゲットが変更された後、リセットされない事象を修正するために使用されるクイックフィックスを示しています。

関数を作成する際、Quick Fixメニューで関数のJSDocコメントを自動生成することができます。

すべてのリファレンスを検索する

Feather を使用すると、プロジェクト全体で変数へのすべての参照を見つけることができます。

テキストカーソルを変数の上に置き、F3キーを押します。ワークスペースの上部に小さな検索バーが表示されます。

対応する矢印を押して前または次のリファレンスに移動するか、閉じることができます。

この検索はスマートで、単にテキスト文字列を検索するのではなく、選択された変数の宣言されたスコープからの参照を検索します。つまり、同じ名前を持ちながら、2つの異なるobjects に別々に存在する2つの変数は、マッチしないことになります。

すべてのリファレンスをリストアップ

テキストカーソルを変数の上に置き、SHIFT + F3 を押します。これにより、Search Results ウィンドウが開き、選択した変数のすべての参照がリストで表示されます。
