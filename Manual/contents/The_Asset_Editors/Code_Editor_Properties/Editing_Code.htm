編集コード

script エディタを使い始めると、script asset やイベントのコードを、正しい GMLシンタックスに従って入力するだけでよいのです。入力中は、オートコンプリートウィンドウでヘルプが表示されます。

このウィンドウは、コードを書いている最中にポップアップ表示され（通常は少し間をおいて表示されますが、環境設定で設定できます）、アルファベット順に「最適な」答えが選択できるようになっています。

コードエディタを使用中に、 をクリックし、マウスをドラッグしてコードの行を選択したり、 を押しながら、 をクリックし、ドラッグしてコードの列を選択することができます。このようにして選択したものは、標準的なキーボードショートカットの / + " C", / + "X", / + "V" を使ってコピー、カット 、ペーストする ことが可能です。

コードには様々な方法でコメントを 付けることができます。コメントすることで、物事を明確にし、コードのセクションを分離し、それぞれの部分が何をするのか、なぜそこにあるのかを明白にすることができるからです。また、チームで作業している場合、他のメンバーにメモを残すことができ、自分が何をしたのか、コードの特定の部分が何のためにあるのかを正確に把握することができるようになります。たとえあなたが一人で開発をしていても、コードにコメントを残すことは重要です。特に、一旦プロジェクトから離れ、後日またプロジェクトに戻る必要がある場合には。

行の前に "//" を置くだけで、簡単にコメントできます。また、"/* text here */" を使えば、段落全体をコメントアウトすることもできます。また、選択した行をコメントアウトしたり、コメントアウトを解除するためのキーボードショートカットも用意されています。 / + "K" でコメント、 / + + "K" でコメント解除です。次の画像は、これらの方法と、JS Docのコメント（以下で詳しく説明します）を使って書かれたコードです。

また、スクリプトには特定の JSDoc スタイルのコメントを追加して、含まれる関数のプロパティを定義し、コード内で関数を使用するときに、引数やその他の詳細とともにオートコンプリートに表示されるようにすることができます。これについては、JSDocScript コメントのセクションで詳しく説明されています。

デフォルトでは、記述されたすべてのコードには、一致する中括弧の各セットの横に、コードの折りたたみアイコン {} が表示されます。これは、小さなプラス + またはマイナス - のアイコンの形をしており、これをクリックすると、一致する中括弧の中でコードが「折りたたみ」られ、より少ないスペースで表示されるようになります。

また、コードのセクションをマークするための特別な #region タグを使用して、手動でコードの折りたたみ領域を設定することもできます。この機能は、コードのセクションの先頭に #region タグ（任意のコメントテキストと一緒に）を追加し、セクションの下部に対応する #endregion タグを追加すると、コードを「折りたたむ」、つまり、エディタで一度に表示できるコードの行数を減らすためにコードを隠すことができます。

実際のコードは次のようなものです。

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

お気づきのように、script のテキストの一部には色が付いています。script エディタは、既存のobjects 、組み込み変数や関数などを「知って」いて、それに応じて色分けしています。色分けは、名前のスペルを間違えたり、キーワードを変数として使ったり、構文エラーを起こしたときに、すぐにわかるので、間違いを防ぐのにとても役立ちます。しかし、色分けが気に入らない場合は、環境設定でswitch 、オンとオフを切り替えることができます（エディタで F10 、オンとオフを切り替えることもできます）。また、使用される色もカスタマイズすることができます。

エディタ内の任意の場所で右クリック（ ）しても、以下のメニューが表示されます。

コード行の切り取り、コピー、貼り付けなどの通常の機能の他に、選択したテキストのコメント化、インデントの追加、削除、コードウィンドウを1、2、3個のペインに分割するオプションがあります（編集中のコードでインデントを定義していない場合、 #regions に関するオプションは使用できませんのでご注意ください）。

ペインに分割する最後のオプションは、実際のウィンドウを異なる部分に分割し、script またはイベントをドラッグして他のものと一緒に表示し、比較や編集を行うことができます。

これは、2つのscripts を比較したり、接続されたscripts を同時に表示したりする場合に、別のウィンドウを必要としたり、常にタブ間でswitch する必要がなく、非常に便利です。

このメニューの他のオプションは、デバッガで使用するためにbreakpoint を切り替えることと、GML コードをGML Visual に変換すること、またはその逆を行うことです。コードをGML Visual に変換する場合、コードの各行に個別のアクションを作成するのではなく 、1 つのコードアクションを使用して、それまでイベントまたはscript にあったものすべてを保持することに注意してください。
