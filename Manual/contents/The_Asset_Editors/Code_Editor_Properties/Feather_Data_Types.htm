フェザーのデータ型

Featherはデータ型を利用して、コードを書く際に構文チェックを行い、変数や関数のパラメータに間違ったデータ型が使われていないことを確認します。

また、自作のスクリプト関数のパラメータや戻り値のデータ型を、JSDocのコメントを使って指定することも可能です。

@param および @return JSDoc タグでは、以下のいずれかのタイプを指定することができます。

注JSDoc の型は大文字と小文字を区別しない。つまり、Id.DsListとid.dslistは同じである。

Feather を使っているとIDE にAny*型が表示されることがありますが、これは識別子 (変数、パラメータ、戻り値など) の型が現時点では Feather で判別できないことを表しています。その識別子に対してさらにコードが追加されると、特定の型を仮定できるようになるかもしれません。

スペシャリスト

基本データ型の後にドット.Specifierを付加することで、そのグループのデータ型を正確に指定することができます。を使って、そのグループ内のデータの正確な型を指定します。

Id.DsList、Asset.GMObject、Constant.Colorなどの型は、指定子を使用します。

コレクションの種類

複数の値を含むデータ構造であるArrayや Id.DsListなどの型は、その内容すべてに対して単一のデータ型を指定することができる。

これは、コレクションタイプを使用して行われ、タイプには角括弧<> が付加されます。

例えば、strings を含む配列はArray<String> で、Real を含む Array を含む DS List はId.DsList<Array<Real>> となる。
