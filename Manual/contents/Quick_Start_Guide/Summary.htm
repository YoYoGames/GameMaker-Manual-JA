概要

このクイックスタートガイドも終盤に差し掛かり、GameMakerを使って自分のプロジェクトを作り始めるのに十分な知識と自信がついてきたと思います！このガイドのすべてのページを読み終えたのなら、次のことが分かっているはずです。

  • スプライトと タイル セットの作成方法とイメージエディターでの編集方法
  • オブジェクトと ルームの作成方法
  • オブジェクトと インスタンスの違い
  • GameMakerの描画方法とDrawイベントの使い方
  • Draw イベントとDraw GUI イベントの違いについて
  • 部屋の中でインスタンスを移動させる方法
  • 変数、関数 、式などのプログラミングの基本概念

あなたは短期間に長い道のりを歩んできましたが、これで旅が終わったわけではありません。このマニュアルには、IDE のあらゆる側面と、GML Visual またはGML を使ったプログラミング方法をカバーする、2つの広範な参照セクションがあります。何かわからないことや新しいことを試したいときには、このセクションに戻ることを強くお勧めします。

  • IDEリファレンス
  • GMLビジュアルリファレンス
  • GMLコードリファレンス
  • 追加情報

また、スタートページからは、ここで学んだコアなコンセプトを使って、「Space Rocks」のような小さくて楽しいゲームを作るためのGameMakerチュートリアルを多数入手することができます。

また、GameMakerでは一連の技術ブログを運営しており、GameMakerを使う上での様々な側面、最も基本的な概念から最も高度なテクニックまでをカバーしているので、誰にとっても役立つ情報が得られると思います。特に、次の記事をご覧ください。

  • GameMakerでコードを書く方法をプロに教えた6つのリソース

また、GameMakerには GMC（Game Maker Community）と呼ばれる独自のフォーラムがあり、コミュニティが運営していて、プログラミングや技術的な問題について時間を割いて助けてくれる親切なメンバーでいっぱいです。

最後に、インターネット上には膨大な量のGameMakerのリソースがあるので、行き詰まったときやチュートリアルを探しているときは、常にブラウザで情報を検索してみてください。また、Redditや Discordなどの サイトには、サインアップして参加できるアクティブなコミュニティが多数あります。

そして、もしゲームを作って公開したら、ぜひGameMaker Showcaseに投稿してください。

Happy GameMaking!
