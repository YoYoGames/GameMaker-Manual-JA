クイックスタートガイド

クイックスタートガイドへようこそこのセクションでは、GameMakerの最も重要な側面に慣れ親しみ、できるだけ早くゲームを作れるようにするための基本的なことを紹介します。このガイドでは、GML CodeとGML Visual のどちらをお使いの場合でも、できるだけ早く製品を最大限に活用できるよう、IDE の必須機能に加えて、プログラミングの簡単な紹介も行っています。

GameMakerのワークフローを説明し、すぐに制作に取り掛かれるように、GameMakerとは何か、プログラムとは何かという基本的なことから、一般的なassets 、動きやスコアなどの基本的なコーディング方法まで、このガイドブックで紹介しています。

GameMakerを使ったゲーム作りは、概念的には以下のような一般的な手順で、かなりシンプルでわかりやすいものとなっています。

  • 画像 アセット（スプライト、タイルセット、フォント）を作成し、Asset Browserに追加します。
  • ゲーム内で使いたいものを表すオブジェクトアセットを 作成し、必要な画像assets を割り当てます。
  • これらのobjects 、特定のイベントに応答して何かを行うようにプログラムします。
  • これらのインスタンス （objects ）を、プログラムを実行する部屋に 追加します。

上記で挙げた各項目の具体的な説明は次のページで行いますが、ゲームを作るために必要な動作の大まかな流れはこのようになっています。ここで重要なのは、ゲームの主な「構成要素」は常にsprites 、objects 、rooms であるということです。ただし、これらのアイテムに追加機能や動作を追加するために使用できる補助的なassets も数多くあります（例えば、以下のようなものです）。 シェーダーまたは タイムライン).

さて、それぞれの内容を詳しく説明する前に、上記の構成要素をGameMakerがどのようにゲームにしていくのか、その概要を簡単に説明します。基本的に、すべてはゲームループと呼ばれる部分で行われ、その速度は Frames Per Secondで計測され、1つの「フレーム」はGameMaker でStepと 呼ばれます。デフォルトのFrame Per Second（FPS）は60です。つまり、1秒経過するごとにGameMakerはゲーム（loop ）を60回実行し、それぞれのステップ（ループ/フレーム）でいくつかのイベントが発生し、ゲームが進行していくことになります。

これを理解するために、ゲーム内で移動させたいキャラクターがあるとします。room 。そのキャラクターのスプライトを 作成し、オブジェクトを 作成してsprite に割り当て、object にステップイベントと呼ばれるイベントを 追加することになります。このイベントで、2ピクセル分右に移動するように指示します。そして、そのインスタンス object をゲームroom に配置し、ゲームを実行すると、インスタンスは1ステップごとに2ピクセルずつ右に移動し始め、（ステップイベントはゲームステップごとに1回実行されます）1秒後には120ピクセル右に移動していることになります。

上記の例では、object に右へ移動するように指示することを話していますが、どのようにそれを行うのでしょうか。それは、GML Visualか GML Codeのどちらかを使って書いたプログラムによって 行われます。どちらを使うかはあなた次第ですが、それらを使うことでゲームをプログラミングして いることになります。詳しくは次のセクションで説明します。
