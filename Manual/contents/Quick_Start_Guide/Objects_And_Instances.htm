オブジェクトとインスタンス

これまで見てきたassets を使って、ゲームasset ブラウザに素敵な画像やサウンドを追加することができますが、ゲーム自体ではまだ何もできません！このような画像やサウンドは、ゲーム内でどのような役割を果たすのでしょうか？そのためには、GameMaker で最も重要なassets オブジェクトアセットが 必要です。Objects は特別なasset で、ゲームの側面を制御したり、特定のことを行うために使用します。ほとんどの場合、sprite に関連付けられ、ゲーム内で見ることができます。rooms しかし、時には、統計情報を追跡したり、カットシーンのタイミングを設定したりと、「裏方」のコントローラー オブジェクトとして使われることもあります。Objects には動作を設定でき、特定のイベントやお互いに反応できます。ゲームで見られるほとんどのものは、objects とユーザーやお互いとの相互作用に基づくものです。

さて、インスタンスについて語らずして、Objects は語れません。GameMakerで作るゲームでは、ゲーム中に出てくるキャラクター、モンスター、ボール、壁などなど、実はすべてインスタンスとみなされているのです。インスタンスとは本来、object の「コピー」であり、Asset Browser で見つけて、room に配置します（objects はインスタンスのテンプレートと考えて ください）。したがって、何かがインスタンスに影響を与えたり変更したりするというのは、object のroom の特定のコピーが影響を受け、残りのすべては影響を受けないということです。しかし、object に影響を与えたり変更したりするというのは、それに対して行ったことが、その時点から作成されるすべてのインスタンスに反映されるということなのです。つまり、 object はインスタンスのテンプレートであり、インスタンスはroom に配置してゲームを作るものなのです。

新しいobject asset を作成するには、他のasset と同じように、Objects asset フォルダ上で を右クリックし、「作成」（またはアセットブラウザの任意の場所で右クリックし、「オブジェクトの作成」）を選択するだけです。

これはObject Editorウィンドウで、現在のobject のプロパティを表示しています。最初にすべきことは、object に名前をつけることです。ここでも " obj_" などのプレフィックスや他の識別子を使用します。一般的に、object が何かを描いたり、他のobjects のインスタンスとの衝突を登録するには、それに割り当てられたsprite が必要です。これは ボタンを使って追加されます。このボタンを押すとAsset Explorerが開き、使用するsprite を選択できます（sprites BrowserからAsset をドラッグし、Object Editorにドロップして割り当てることもできます）。

sprite sprite asset まだ適切なasset がない場合は、新規スプライトボタン をクリックして を作成し、標準のsprite 作成オプションでobject を作成したように、好きなように編集することができます。sprite を選択したら、 の「スプライトの編集」 や「画像の編集」 ボタンをクリックして、sprite 自体の画像やsprite asset を編集することもできます。この方法は、sprite ブラウザでAsset を見つけてから、編集することを指定するより早い場合があります。

sprite を追加または作成したら、他のいくつかのobject プロパティを変更できます。たとえば、表示 するかどうかや、どのようなコリジョンマスクを持つかなどがあります。シンプルなゲームでは、ここで何も変更する必要はないでしょう。次に、object にイベントを 追加します。

イベントとは、ゲームの中で、loop 、あなたがプログラムした内容に基づいて物事が起こるようにする、目立たない瞬間のことです。GameMakerはこのイベントのサイクルで動作します。room が開始された瞬間から終了する瞬間まで、ゲームloop が動作しており、各ステップで一連のイベントが実行されます。これらのイベントに対応するGML コードまたはGML Visual アクションをインスタンスに配置できます（1ステップは ゲーム時間の一瞬で、ゲームスピード設定により支配され「フレーム」とも呼ばれま す）。

イベントを追加するには、「Add Event」ボタン（ ）をクリックします。

上に表示されているのは、さまざまなイベントカテゴリーで、その中には他のサブカテゴリーが含まれています。ここでは、すべてのイベントを紹介するのではなく、最も重要な5つのイベントを簡単に説明します。

イベント作成 このイベントは、object のインスタンスが最初に作成されたときに発生し、room エディタを通じてroom に配置されたインスタンスで、room が入力されたときに最初に発生するものである。つまり、このイベントは変数の初期化、Timelines の開始、paths の設定など、一般的に一度だけ、あるいはroom に初めてインスタンスが現れたときだけ行う必要があることを行うのに理想的な場所なのです。
ステップイベント GameMakerでは、ゲーム時間をステップに分割し、ゲームスピードで1秒間にいくつのステップがあるのかを定義しています。ステップとは、基本的にloop のことで、ゲームが動いている間、全てのイベントがチェックされ、必要に応じてトリガーされます。したがって、ステップイベントは、インスタンスが存在する間、ゲームの全てのステップをチェックするイベントであると想像できます。ステップ イベント "カテゴリには、インスタンス内のタイミングを "微調整 "するための3つのサブカテゴリがありますが、一般的にはメインのステップ イベントを最も使用することになります。
衝突イベント もちろん、ゲームを作るときには、2つ（またはそれ以上）のobject のインスタンスが衝突したことを知ることが非常に重要です。そのために、衝突イベントがあります。これはobject に配置し、他のどのobject に対して衝突をチェックするかを指定するイベントです。イベントを追加し、チェックするobject を選択したら、GML コードまたはGML Visual を追加して、衝突を解決することができます。
ドローイベント ドローイベントのカテゴリには複数の異なるドローイベントがありますが、最もよく使うのは一番上にある単に「ドロー」と呼ばれるイベントです。この一般的なドローイベントは、ステップイベントと同様にゲームフレームごとに呼び出されますが、2つの異なる方法で機能します。1つ目は、このイベントをobject に追加していない場合です ...指定された Draw Event がない場合、GameMakerはデフォルトでsprite に割り当てられたobject を描画し、object の他の場所で適用されたトランスフォーム（サイズ変更、アルファブレンディングなど）を使用します。したがって、Event list に Draw Event が表示されていない場合でも、インスタンスがroom に配置されると実行されて、画面に描画します。 2 番目の動作方法は、イベント自体を追加してGML またはGML Visual をその中に含める場合です。イベントに何かを追加することで、GameMakerに「このオブジェクトのインスタンスに対する描画をコントロールしたい」と伝え、デフォルトの描画を完全にオーバーライドします。つまり、object にsprite を割り当て、 draw イベントで text を描画するように設定すると、sprite は描画されません。
デストロイイベント このイベントは、インスタンスが破壊されたときに実行される最後のイベントです。objects object にビヘイビアを追加するときに見落とされがちですが、非常に便利です。たとえば、敵が殺されたときに爆発やパーティクル効果を作成したり、room の別の場所で新しいインスタンスを再スポーニングしたり、あるいはスコアにポイントを追加したりすることが可能です。
