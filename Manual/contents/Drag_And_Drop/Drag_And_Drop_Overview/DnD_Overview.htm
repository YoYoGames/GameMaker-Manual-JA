GMLビジュアルの概要

GML Visualは、実際にコードを入力することなくゲームを作成することができるビジュアルスクリプティングツールです。GML Visualは、テキストではなくアクションの "ブロック "を使って視覚的にコードを作成し、表示することができます。アクションとは、object のインスタンスがroom に置かれたときに、「連鎖」して何かを行うことができる簡単な式や文です。例えば、インスタンスをランダムな方向に移動させるアクションを作成イベントで行い、衝突したものを跳ね返すアクションを衝突イベントで行います（利用できるイベントの種類とその連携については、「オブジェクトイベント」のセクションを参照してください）。

上の画像は、編集対象のワークスペース上で典型的なobject を開いた状態です。イベント」ウィンドウと「コード」ウィンドウが連結されており、コードウィンドウでは、GML Visual を構築した箇所が確認できます。このウィンドウをもう少し詳しく見て、どのようなオプションが利用できるかを見てみましょう。

以下、表示されている各セクションの詳細について説明します。

イベントタブ

object にイベントを 追加すると、コードウィンドウに新しいタブが開かれ、そのイベントの内容が表示されます。複数のイベントを追加して、同時に複数のタブを開き、好きなように切り替えて編集することができます。また、いつでもタブを閉じて散らかるのを防ぐことができます（object でイベントを再度選択すると、再び開きます）。また、 をクリックしてタブをウィンドウから独自のコードウィンドウにドラッグできます。この場合、object またはコードウィンドウを閉じても、新しいウィンドウは閉じません。タブを左右にドラッグして並べ替えることもできます。

アクションの概要

イベントにアクションブロックを追加すると、「速記」形式で、何が起こっているのかをわかりやすく表示します。概要の順序は、そのイベントで各アクションブロックが実行される順序です。リスト内の任意の項目をクリックすると、アクションブロックのビューがその位置にズームされ、編集することができます。

アクションブロックのワークスペース

このworkspace エリアは、ツールボックスから アクションをドラッグして、GML Visual コードを作成する場所です。 アクションをクリックし、このエリアにドラッグすると、アクションのリストに追加され、編集することができます。ほとんどのアクションには、設定できるさまざまな引数（パラメータ）があり、キーワードや インスタンス変数、GML 関数も受け取ります。たとえば、「方向を設定」アクションを呼び出して、GML 関数を使用することができます。 random(val)でランダムな方向を設定します。

アクションブロックをコードに追加すると、視覚的に「連鎖」して、異なるアクションの発生順序を示しますが、この連鎖の作り方は、最初に使用したブロックに依存します。例えば、If Variableブロックは2つの連鎖を持ちます。1つは、チェックされた条件が true 、その次のブロックが呼び出されるブロックへの連鎖です。アクションブロックをコードに組み入れる際の詳細については、以下のリンク先のセクションを参照してください。

アクションブロックworkspace では、マウスの中ボタン を長押ししてパンし、マウスホイール でアクションリストを上下にスクロールすることができます。 を押しながらマウスホイールを動かすと、表示が左右にスクロールします。また、 / を押しながらマウスホイールを動かすと、表示がズームイン/アウトし、アクションのあるworkspace をより多く見ることができます。左側のアクションリストから をクリックすると、そのアクションにフォーカスされ、ズームと位置がリセットされます。また、マウスをブロックの端に移動し、カーソルが変わったら、必要な方向にドラッグすると、アクションブロックのサイズを変更することができます。

また、GML Visualworkspace の任意の場所で右クリックすると、RMB メニューが表示されますが、これについてはこちらで詳しく説明しています。

ツールボックス

ツールボックスには GML Visual のすべてのアクションが保存されており、異なるセクションには類似または関連するアクションのコレクション（ライブラリ）が保存されています。アクションの各コレクションは特定のテーマまたは共通の目的に関連しており、 をクリックし、必要なアクションをツールボックスからアクションブロックworkspace にドラッグして現在のイベントに追加するだけです。また、 をクリックし、ライブラリから「お気に入り」フォルダにアイコンをドラッグして、よく使うアクションをまとめておくこともできます。これは、何度も繰り返し行う特定の操作がある場合に特に便利です。お気に入りフォルダは、小さな「ビン」アイコンをクリックすることで消去できます。

利用可能なアクションの完全なリストと、それらが取る引数、およびどのように連鎖させるかについては、GMLビジュアルリファレンスセクションを参照してください。

ツールボックス内の任意のアクションの上でマウスの右ボタン 、以下のメニューを開くことができます。

ここでは、以下のオプションがあります。

  • Use <action>: マウスの右ボタンをクリックしたときにマウスの下にあったアクションを現在のイベントに追加します。workspace.
  • お気に入りを作成／ 削除します。ツールボックスの上部にある「お気に入り」ライブラリにアクションを追加または削除します。このライブラリは、定期的に使用するアクションのカスタムグループを作成する場所であり、実際のライブラリでそれらを探すのに必要な時間を削減することができます。
  • お気に入りを消去：これを選択すると、お気に入りライブラリからすべてのアクションが削除されます。
  • 最近使ったものを消去する。これを選択すると、最近使ったライブラリからすべてのアクションが削除されます。
  • すべて折りたたむ：このオプションは、すべてのGML Visual ライブラリを折りたたみ、ライブラリ・タイトルのみを表示し、アクション・ノードを表示しないようにします。各ライブラリは、それらをクリックすることで再び展開することができます。
