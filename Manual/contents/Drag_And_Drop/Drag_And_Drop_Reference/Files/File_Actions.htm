ファイル アクション ライブラリ

ファイルアクションは、Buffer ファイルと Ini ファイルの 2 種類のファイルを扱います。また、既存のファイルの名前を変更したり、コピーしたりする一般的なファイルアクションもあります。

バッファファイルは、以前に作成したbuffer からデータを保存して作成し、このデータを将来いつでも再びbuffer に読み込むことができます。buffers の詳細については、こちらをご覧ください。バッファーアクション

Iniファイルは、ほとんどのプラットフォームと互換性のある、小型で軽量なファイルです。インターフェイスの設定、ローカルハイスコア、レベルデータなど、小さな情報を保存するのに理想的で、非常に使いやすいファイルです。Iniファイルは また、存在しないIniファイルから読み込んでも、単に default （あなたが指定したもの）の返り値が返されます。 iniファイルを開き、アクションを使用する前に修正することができます。このベースとなるiniファイルは、Open Ini Fileアクションを呼び出した後、Close Ini Fileアクションを呼び出すことで作成することができます。 または、Asset BrowserのIncluded Filesに含めることができます。Asset Browserに含まれるファイルをベースIniとして使用する場合は、以下の情報もお読みください。 は、ファイルシステムの仕組みに関するマニュアルのセクションを参照してください。

注：ローカライゼーションのあるゲームでは、アクセント付きの文字が外部ファイルから読み取れることが非常に重要です。つまり、まずUTF8形式のiniファイルを作成し、それをGameMakerにインクルードファイルとして追加する必要があります。 このファイルは、ゲーム実行時にエクスポートされ、ini 関数で作成されるデフォルトの ANSI 形式の ini ファイル（事前に何も指定されていない場合）の代わりとして使用されるようになります。このようにすると、アクセント記号や非ロマン文字を正しく読み書きができるようになります。 文字が維持されます。
