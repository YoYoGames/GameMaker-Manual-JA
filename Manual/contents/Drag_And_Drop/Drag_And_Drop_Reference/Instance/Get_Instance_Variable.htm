インスタンス変数の取得

このアクションを使用すると、リストされた任意の1つを取得することができます。 内蔵という変数があります。すべてのobjects のインスタンスは、現在の x/y 位置、速度、方向、視認性などの重要な情報を保持するために、すでにいくつかの変数がその中に作成されています。インスタンス変数の値を取得することができます。

  • X Coordinate- room における X 軸に沿った位置。
  • Y座標-room のY軸に沿った位置です。
  • Direction- インスタンスの移動方向（デフォルトは0、右は0°、上は90°、左は180°、下は270°）。
  • Speed- インスタンスの移動速度（ゲームフレームあたりのピクセル数）です。
  • Horizontal Speed- インスタンスが移動している水平方向の速度（ゲームフレームあたりのピクセル数）です。負の速度はインスタンスが動いていることを意味します。 デフォルトは0です。
  • Vertical Speed- インスタンスが移動している垂直方向の速度（ゲームフレームあたりのピクセル数）です。負のスピードは上に移動していることを意味します。 デフォルトは0です。
  • Visible- 可視性のトグル変数（オブジェクトエディタで他に設定されていない限り、デフォルトは true ）。 を false に設定すると不可視になります)。不可視のインスタンスは、Drawイベント以外のすべてのイベントを実行することに注意してください。
  • Persistent- 永続性トグル変数 (オブジェクトエディタで他に設定されていない限り、デフォルトは false です)。 を true に設定すると、永続化されます)。これは、インスタンスをroom からroom まで「持続」させます。つまり、インスタンスはroom の終了時にクリーンアップされず、次のroom に持ち越されます。インスタンスが永続化されるとき、次のことに注意してください。 は、Create イベントや Destroy イベントを呼び出さず、Clean Up、Room End、Room Start イベントを呼び出します。
  • Layer- インスタンスが割り当てられているレイヤー。
  • 深度- レイヤーに割り当てられた深度の値です。
  • Sprite- インスタンスに割り当てられたsprite のリソース。
  • 画像インデックス- 割り当てられたsprite の画像インデックスです。この値は、インスタンスに割り当てられているsprite のフレームを設定します。 この値がアニメーションのフレーム数より大きい場合、5フレームのsprite をフレーム7に設定すると、実際にはフレーム2が表示されるような「ループ」が発生します。
  • 画像角度- インスタンスが設定される角度（デフォルトは0°，右は0°，上は90°，左は180°，下は は270°）。
  • Image Alpha- 0から1までのインスタンスのアルファ値（デフォルトは1）で、0は完全な透明、1は完全な不透明になります。
  • Image Blend- インスタンスとブレンドされる色sprite (デフォルトは白)。
  • Image Speed- インスタンスのアニメーション速度です。
  • In Sequence- この変数は、インスタンスが現在sequence に含まれている場合、 true を返します。 それ以外の場合は、 false 。
  • Horizontal Scale- インスタンスsprite を描画する際の水平スケール（これはインスタンスのバウンディングにも影響します）。 ボックスとコリジョン)。1 の値はスケーリングなし（1:1）を示し、小さい値はスケーリングダウン（例えば 0.5 はsprite の幅の半分）、大きい値はスケーリングアップ、負の値はsprite をミラーリングしてスケーリングを行います。 がちょうど-1である場合（この場合、sprite は単にy軸に沿ってミラーリングされ、スケーリングは行われません）。
  • Vertical Scale- インスタンスsprite を描画する際の垂直スケール（これはインスタンスのバウンディングにも影響します）。 ボックスとコリジョン)。1 の値はスケーリングなし（1:1）を示し、小さい値はスケーリングダウン（例えば 0.5 はsprite の高さを半分にします）、大きい値はスケーリングアップ、負の値はsprite を反転してスケーリングします。 がちょうど -1 である場合 (この場合、sprite は y 軸に沿って反転し、スケーリングは行われません)。
  • Mask- インスタンスのマスクインデックス。デフォルト値は-1（インスタンスは、そのインスタンスのコリジョンマスクを使用することを意味します）。 sprite しかし、他のsprite リソースに設定すると、sprite インデックスに割り当てられたsprite の代わりに、そのインスタンスのコリジョン マスクが使用されます。
  • Friction- インスタンスを遅くするために、ゲームの各ゲームフレームを使用するための摩擦。この値は単純に現在の ゲームフレームごとに速度値を設定する（速度値が負の場合は加算する）。
  • Gravity- インスタンスの重力方向（下記参照）に一定の力を加え、その速度に影響を与えます（デフォルトは 0).重力は累積的な力であり、最終的な速度に上限を設けないことを選択した場合、object が加速されることに注意してください。
  • Gravity Direction - インスタンスの重力が0より大きいときの移動方向を設定します（初期値）。 は270）。なお、GameMakerでの方向は、0°が右、90°が上、180°が左、270°が下として計算されます。
  • Score- 変数 score を設定します。
  • Lives- 変数 lives を設定します。
  • Health- 変数 health を設定します。

取得する組み込み変数を選択したら、次に戻り値を保持するターゲット変数を設定します。もし、この変数を一時変数としてflag にすると、アクションの中で作成され イベントまたはアクションscript 。また、同じアクションで追加の変数の値を取得するには、プラスアイコン アクションの横にあるインスタンス変数を選択し、返された値を格納するための付随する変数を指定します。
