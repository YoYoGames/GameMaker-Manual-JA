仮想キーボードを表示する

このアクションは、ゲームを実行しているデバイスに仮想キーボードを表示するために使用することができます。このアクションを呼び出すと、最初の3つの引数にそれぞれ以下の定数のいずれかを指定するよう求められます。

  • キーボードタイプ。この定数は、仮想キーボードで利用可能なキーセットを設定するために使用されます。利用可能なキーボードタイプは以下の通りです。
    キーボードタイプ 説明
    kbv_type_default 現在のシステムのデフォルトのキーボードの種類です。
    kbv_type_ascii ASCIIのみのキーボード。
    kbv_type_url URL入力用に最適化された通常のキーボードです。通常、「.com」などのドメインキーや「/」、「.」キーがあります。
    kbv_type_email 電子メールの入力に最適化された通常のキーボード。通常、"@"や"。"キーを備えています。
    kbv_type_numbers 数字のみのキーボードで、通常はテンキーとして表示されます。
    kbv_type_phone 電話帳キーボード。通常、「*」と「#」キーを備えた数字のみのキーボードです。
    kbv_type_phone_name 電話番号と名前の両方を入力するのに最適なキーボード。通常、ASCIIキーボードに似ていますが、選択できる特殊文字が限定されています。
  • リターンタイプ。この定数は、仮想キーボードのリターンキー／アクションキーに表示される内容を設定するために使用されます。利用可能なリターンタイプは以下の通りです。
    リターンタイプ 説明
    kbv_returnkey_default 現在のシステムのデフォルトのリターンキータイトルです。
    kbv_returnkey_go リターンキーのタイトルを "Go "に設定する。
    kbv_returnkey_google リターンキーに "Google"、または一般的な検索アイコンを設定することもある。
    kbv_returnkey_join リターンキーに "Go "を設定する。
    kbv_returnkey_next リターンキーに "Next "を設定します。
    kbv_returnkey_route リターンキーのタイトルを「Route」に設定します。
    kbv_returnkey_search リターンキーに「検索」を設定します。場合によっては一般的な検索アイコンを設定します。
    kbv_returnkey_send リターンキーのタイトルを「送信」に設定します。
    kbv_returnkey_yahoo リターンキーのタイトルを「Yahoo」に設定します。また、汎用検索アイコンを設定する場合もあります。
    kbv_returnkey_done リターンキーに「完了」を設定します。
    kbv_returnkey_continue リターンキーに「Continue」を設定します。
    kbv_returnkey_emergency リターンキーに「緊急連絡」を設定します。
  • 自動大文字入力のタイプ。この定数は、仮想キーボードで入力された単語がどのように自動大文字入力されるかを決定するために使用されます。利用可能な自動大文字入力のタイプは以下の通りです。
    オートキャピタリゼーションタイプ 説明
    kbv_autocapitalize_none 自動大文字入力は無効です。
    kbv_autocapitalize_words 単語は自動大文字入力されます。
    kbv_autocapitalize_sentences 文は自動大文字入力されます。
    kbv_autocapitalize_characters すべての文字が大文字になります。

最後の引数は、予測入力の有効・無効を設定するもので、許可する場合は true に、許可しない場合は false に設定します。 を使用します。予測入力モードでは、仮想キーボードは通常のGameMakerのキー押下イベントを発生させません。その代わり、最後に押された文字とキーボードstring の変数のみが更新されます。これは、以下の理由によるものです。 キー押下を検出するために使用される内部テキストフィールドの変化が、実際の仮想キーボードキーによるものか、テキストサジェストによるものかを検出できないためです。このような場合、 keyboard_string の入力を読みたいと思うでしょう。 を読み取ることができます。

プラットフォームOSによって、ユーザーが手にするキーボードは異なる機能を持つことになり、ターゲットごとに使用する場合は以下のような注意点があることも知っておく必要があります。

  • UWP: UWPの実装では、 return_key_type, autocapitalization_type, predictive_text_enabled をサポートしていません。これらのパラメータを関数に渡しても、エラーにはなりません。 キーボードに影響を与えます。
  • Android / Amazon Fireの場合。Android では、キーボードタイプ kbv_type_phone_name はサポートされておらず、代わりに ASCII キーボードが表示されます。また、リターンキータイプは kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next のみとなります。 kbv_returnkey_send または kbv_returnkey_done 。そのプラットフォームで他のリターンキーが使用されている場合、システムは以下の代替リターンキーにデフォルトで対応します。
    使用できないリターンキー 交換用キー
    kbv_returnkey_google
    kbv_returnkey_yahoo    		 kbv_returnkey_search
    kbv_returnkey_join
    kbv_returnkey_route
    kbv_returnkey_emergency    		 kbv_returnkey_go
    kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV / FireTVの場合。ASCIIキーボードでは、カスタムリターンキータイプはサポートされていません- デフォルトのリターンキーが常に表示されます。
  • tvOS: Predictive text and auto-capitalization is not supported ontvOS, and the height of the keyboard returned by system events and function keyboard_virtual_height() は、画面全体に広がるため、また、tvOS でキーボード長方形機能が無効になっているため、画面の高さがデフォルトとなります。また、物理キーボード（Bluetooth）のイベントは、仮想キーボードを使用しない限り、ブロードキャストされないことに注意してください。 が開かれました。

この関数を呼び出すと、システム非同期イベントが発生し、 async_load  DS マップに以下のキー/バリューペアが入力されます。

  • "event_type" - 仮想キーボード操作によってトリガーされた場合、これは "仮想キーボードの状態" になります。
  • "screen_height" - 仮想キーボードの高さ (ピクセル単位)です。キーボードが見えない場合は、0 になります。
  • "keyboard_status" - キーボードの現在のステータスで、以下の文字列のうちの1つとして返されます。
    • "隠蔽"
    • "非表示"
    • "表示"
    • "可視"

アクションのシンタックス。

論拠となる。

論証 商品説明
keyboard_type 仮想キーボードで使用できるキーセットを決定します。
return_key_type 仮想キーボードのリターンキー/アクションキーに表示される内容を決定します。
autocapitalization_type バーチャルキーボードで入力された単語を自動大文字にするかどうかを設定します。
predictive_text_enabled true/false に設定すると、予測入力の有効／無効が切り替わります。
