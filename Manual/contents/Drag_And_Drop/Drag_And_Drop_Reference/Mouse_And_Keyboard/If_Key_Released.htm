キーが解放された場合

このアクションは、キーボードのキーがいつ離されたかをチェックします。キーが離された1つのゲームステップ（フレーム）で true 、それ以外の時は false を返します。 not モディファイアを有効にすると、このアクションは次のようになります。 キーがリリースされていない場合は、逆に、リリースされたゲームステップ（フレーム）で false を返し、それ以外の場合は true を返します。

なお、「if」ブロックにアクションを追加するには、下の画像のように、アクションの横にドロップする必要があります。

これらのアクションは、"if" の評価値が true の場合に実行されるようになりました。 一方、他の場所でドロップされたアクションは、"if "ブロックの後で実行されます。

上記のアクションブロックのコードは、ステップごとにキーボードの状態をポーリングし キーが押されている場合、キーが最初に押されたかどうかをチェックする。初回押下時のチェックが true であった場合、音を鳴らし、キーが押されている間、sprite のブレンドを赤に変更します。キーが押されていない場合 ダウン時にチェックを行い、ブレンドカラーをホワイトにリセットします。
