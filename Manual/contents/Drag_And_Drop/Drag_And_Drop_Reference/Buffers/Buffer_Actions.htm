バッファアクションライブラリ

バッファとは、データをある場所から別の場所に移動したり、何らかの方法で操作している間、一時的にデータを保持するために使用する物理メモリの領域です。例えば、buffer 、ゲームの状態データを保持するために使用することができます。GameMakerには、buffers の作成、変更、ロード/セーブに関連する特定のアクションがいくつかありますが、それらを使用する前に、以下の点に注意してください。

  • buffer を作成すると、自動的にクリアされ、0に埋められます。
  • buffer 関数の引数 "size "は，常にbuffer のバイト単位の大きさを指す。
  • GML Visualbuffers は常に1バイトアライメントです。この「アラインメント」とは、buffer 内のデータの格納方法を指し、GMLbuffer Functions を使用する際に使用されますが、GML Visual では関係ありません。
  • buffers のデータは、順次読み書きが行われます（例えば、Write Bufferアクションを 3 回呼び出すと、最初のデータを書き込み、前に進み、2 番目のデータを右に進み、前に進み、最後のデータを書き込みます）。「seek」値というものがあり、次の読み書きがbuffer のどこで行われるかを教えてくれます。
  • すべてのデータは、buffer に書き込まれ、buffer から文字列として返されます。したがって、実数を保存した場合は、データ型アクションのいずれかを使用して、string から値に変換する必要があります。

buffers の作成に使用されるメモリはシステムメモリなので、ゲームにフォーカスがない場合でもbuffer に保存されたデータは安全です。ただし、アプリを閉じたり再スタートすると、データは失われます。buffers をファイルとして保存し、ファイルアクションを使用してゲームに読み込むこともできます。
