バッファの作成

このアクションは、ゲーム内でメモリの一部をbuffer として割り当てるために使用できます。この関数は、変数に格納され、その後のbuffer に対するすべての関数呼び出しに使用される、一意のバッファ ID値を返します。 グローバル変数、インスタンス変数、またはbuffer ID を格納するための一時的なローカル変数を作成するようにアクションに指示します。buffers を追加で作成するには、プラスアイコンの の横にある を選択し、buffer ID を格納する別の変数を選択します。

注：このように動的に作成されたリソースは、メモリリークを防ぐため、不要になったらメモリから削除することが重要です。作成したbuffer が終了したら、再び解放してください。 Delete Bufferアクション（Other - Clean Upイベント内など）を使用します。これは、一時的なローカル・バッファを使用する場合に特に重要です。 イベントまたはアクションbuffer が終了する前にscript を再度削除しないと、アクセスできなくなり、後で問題が発生する可能性があるからです。
