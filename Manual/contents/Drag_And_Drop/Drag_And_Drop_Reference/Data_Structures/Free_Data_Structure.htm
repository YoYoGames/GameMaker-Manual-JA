フリーデータ構造

このアクションはデータ構造をメモリから解放し、データ構造が不要になったときにいつでも呼び出す必要があります。例えば、インスタンスのDestroyやRoom Endイベントなどです。データ構造を解放するということは、それ以上のアクションがないことを意味します。 は、もはや存在しないので、それを参照するために使用することはできませんが、If Data Structure Existsアクションを使用してこれを確認することもできます。

データ構造のインデックスがグローバルスコープの変数に割り当てられていない限り、room から別のものに引き継がれないことに注意してください。たとえば、プレーヤーのインスタンスを作成し、リストのデータ構造がそのインスタンスの変数に割り当てられている場合。 プレーヤーが破壊された場合、またはプレーヤーインスタンスを含むroom が変更された場合、リストを解放する必要があります (プレーヤーインスタンスが永続的であるとフラグ付けされていない限り)。 の両方が発生します。メモリリークによるパフォーマンスの低下やゲームのクラッシュを防ぐため、作成したデータ構造は不要になった時点でこの方法で破棄することが重要です。
