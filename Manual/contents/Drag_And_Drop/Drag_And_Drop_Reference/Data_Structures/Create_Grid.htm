グリッドの作成

このアクションは、新しいグリッドデータ構造を作成し、インデックス値を返すので、後で他のデータ構造アクションでグリッドにアクセスすることができます。グリッドのインデックスは、指定したターゲット変数に返されます。 Assign VariableまたはDeclare Temp を使用して作成するか、flag "Temp" チェックボックスを使用して、一時的なローカル変数に名前を付けて作成することができます。 を使用して、script またはイベントの終了まで値を保存します。他のデータ構造とは異なり、グリッドは「空」で作成されるわけではなく、新しく作成されたグリッドの各セルは値0を持つことに注意してください。セルのデータを変更するかどうかは、あなた次第です。 または、グリッドをクリアするアクションを使用して、自分でグリッドを別の値にクリアします。プラスアイコンをクリックすると、さらにDSグリッドを作成できます。 を選択し、グリッドIDを格納する別の変数を選択し、幅と高さの値を再度指定します。

重要!データ構造を作成すると、ターゲット・プラットフォームのメモリを消費します。 メモリリークにより、ゲームのパフォーマンスが低下したり、クラッシュしたりする可能性があります。これは、データ構造のインデックスを格納するために一時的なローカル変数を使用している場合に特に関連します。 だからといって、データ構造が削除されるわけではありません。データ構造はまだ存在しますが、それを参照する方法がないため、インスタンス変数を使用して後で構造体を解放するか、または イベントや関数のインデックスが一時変数に格納されている場合。
