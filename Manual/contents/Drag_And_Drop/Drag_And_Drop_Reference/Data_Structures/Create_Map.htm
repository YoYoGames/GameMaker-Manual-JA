マップの作成

このアクションは新しいマップデータ構造を作成し、インデックス値を返すので、後で他のデータ構造アクションでマップにアクセスすることができます。マップインデックスは、指定されたターゲット変数に返されます。 または、flag "Temp" チェックボックスを使用して、一時的なローカル変数に名前を付けて作成することもできます。 は、script またはイベントが終了するまで値を保存します。新しく作成されたマップデータ構造は「空」、つまりマップキー/バリューのエントリーを含んでいないとみなされます。DSマップを追加作成するには、プラスアイコン の横にある アクションを選択し、マップIDを格納する別の変数を選択します。

重要!データ構造を作成すると、ターゲット・プラットフォームのメモリを消費します。 メモリリークにより、ゲームのパフォーマンスが低下したり、クラッシュしたりする可能性があります。これは、データ構造のインデックスを格納するために一時的なローカル変数を使用している場合に特に関連します。 だからといって、データ構造が削除されるわけではありません。データ構造はまだ存在しますが、それを参照する方法がないため、インスタンス変数を使用して後で構造体を解放するか、または イベントや関数のインデックスが一時変数に格納されている場合。
