スタック作成

このアクションは新しいスタックデータ構造を作成し、インデックス値を返すので、後で他のデータ構造アクションでスタックにアクセスすることができます。スタックインデックスは、指定したターゲット変数に返されます。 または、flag "Temp" チェックボックスを使用して、一時的なローカル変数に名前を付けて作成します。 変数に格納し、script またはイベントの終了まで値を保存します。新しく作成されたスタックデータ構造は、「空」、つまりエントリを何も含んでいないとみなされます。DS スタックを追加で作成するには、プラスアイコン の横にある アクションを実行し、スタックIDを格納する別の変数を選択します。

重要!データ構造を作成すると、ターゲット・プラットフォームのメモリを消費します。 メモリリークにより、ゲームのパフォーマンスが低下したり、クラッシュしたりする可能性があります。これは、データ構造のインデックスを格納するために一時的なローカル変数を使用している場合に特に関連します。 だからといって、データ構造が削除されるわけではありません。データ構造はまだ存在しますが、それを参照する方法がないため、インスタンス変数を使用して後で構造体を解放するか、または イベントや関数のインデックスが一時変数に格納されている場合。
