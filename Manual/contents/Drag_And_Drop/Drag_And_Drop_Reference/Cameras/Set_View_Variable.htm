ビューバリアブルの設定

roomこのアクションでは、ビューポートがカメラのビューを表示する方法に関する特定の組み込み変数を設定することができます。 という変数があります。ビューポートは、指定されたカメラビューを描画するために使用される画面の領域です（room エディタで設定）。この方法で設定できるさまざまな変数の概要は、以下の表のとおりです。

バリアブル 商品説明
Camera 別のカメラビューを表示するようにビューポートを設定するためのものです。変更するカメラの一意のID値を指定する必要があります( を参照してください。）
Visibility これはビューポートの可視性トグルで、 true または false の値が必要です。 true に設定すると、ビューポートが可視になり（つまり、画面に描画されます）、 false に設定すると、描画されません。 を表示します。
View Port X Coordinate ゲームウィンドウ内のビューポートのX位置です。ただし、本当に必要なのは複数のビューポートがアクティブなときだけで、たとえば画面分割の効果を出すときなど、1つのポートだけがアクティブなときに使うと は予期せぬ結果をもたらします。
View Port Y Coordinate ゲームウィンドウ内でのビューポートのY位置を指定します。ただし、本当に必要なのは複数のビューポートがアクティブなときだけで、たとえば画面分割の効果を出すときなど、1つのポートだけがアクティブなときに使うと は予期せぬ結果をもたらします。
View Port Width ビューポートの幅（ピクセル）です。ビューポートの幅は、ゲーム開始時のゲームウィンドウや背景キャンバスの幅を定義します（複数のビューポートがアクティブになっている場合は、それらを組み合わせた幅）。 GML 関数と一緒に呼び出されない限り、ゲームが開始されても、ゲーム・ウィンドウのサイズに目に見える影響はありません。 window_set_size(). また、カメラに割り当てられているポートサイズよりも大きいまたは小さいポートサイズがある場合、カメラビューはそれに合わせて縮小（または拡大）されることに注意してください。
View Port Height ビューポートの高さ（ピクセル）です。ビューポートの高さは、ゲーム開始時のゲームウィンドウや背景キャンバスの高さを定義します（複数のビューポートがアクティブになっている場合は、それらを組み合わせた高さ）。 GML を呼び出さない限り、ゲームウィンドウのサイズに目に見える影響を与えることはありません。 window_set_size(). また、カメラビューに割り当てられているポートサイズよりも大きいまたは小さいポートサイズがある場合、カメラビューはそれに合わせて縮小（または拡大）されることに注意してください。
View Port Surface ID この変数により、与えられたビューポートの内容をサーフェスに描画するよう設定することができ、その値はビューポートの描画に使用するサーフェスのものとなります。より詳細な情報は、こちらをご覧ください。
