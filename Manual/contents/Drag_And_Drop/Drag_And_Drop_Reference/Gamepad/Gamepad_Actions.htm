ゲームパッドアクションライブラリ

GameMakerには、接続された複数のゲームパッドからアナログとデジタルの両方のコントロールを検出するための専用アクションがいくつか用意されています。これらのアクションでは、ゲームパッドのインデックス値を指定する必要があります。これは、選択中のゲームパッドを表す数値（0からカウント）です。ゲームパッドがデバイスに接続される（または取り外される）と、非同期のシステムイベントがトリガーされることに注意してください（ただし、これを処理するためにはコードが必要な場合があります）。

ゲームパッドのスロットは0から始まり、0〜3は Xinputゲームパッド（Xbox360のコントローラーと互換機）専用です。ただし、スロット4～11では、DirectInputゲームパッドもチェックできますので、これらのスロットに接続すれば、他の多くのモデルのコントローラーも検出できます。

入力定数

ゲームパッドのアクションを扱う場合、入力は軸、ボタン、帽子から得ることができ、GameMakerは以下の組み込み定数に割り当てます（一般的に「帽子」は非標準のコントローラーでのみ検出されることに注意してください）。

定数 商品説明
gp_face1 上ボタン1（Xbox 360コントローラの「A」、PSコントローラの「十字」に対応します。）
gp_face2 上ボタン2（Xbox 360コントローラの「B」、PSコントローラの「○」に対応します。）
gp_face3 上ボタン3（Xbox 360コントローラの「X」、PSコントローラの「四角」に対応します。）
gp_face4 上ボタン4（Xbox 360コントローラの「Y」、PSコントローラの「三角」に対応します。）
gp_shoulderl 左肩ボタン
gp_shoulderlb 左肩のトリガー
gp_shoulderr 右肩ボタン
gp_shoulderrb 右肩のトリガー
gp_select セレクトボタン（DS4コントローラの場合、タッチパッドを押し下げるとトリガーされる）
gp_start スタートボタン（PS4コントローラーの「オプション」ボタンです）
gp_stickl 左スティックが押された状態（ボタンとして）
gp_stickr 右スティックが押された状態（ボタンとして）
gp_padu Dパッドアップ
gp_padd Dパッド下
gp_padl Dパッド左
gp_padr Dパッド右
gp_axislh 左スティック水平軸（アナログ）

gp_axisrh 右スティック水平軸（アナログ）
gp_axisrv 右スティック縦軸（アナログ）

アクション

ゲームパッドで使用可能なアクションは、すべて以下のとおりです。

ゲームパッド軸を取得する
ゲームパッドトリガーを取得
ゲームパッド数の取得
ゲームパッドコネクト
ゲームパッド軸のデッドゾーンを設定する
ゲームパッドボタンのしきい値を設定する
ゲームパッドボタンが押された場合
ゲームパッドボタンが押された場合
ゲームパッドボタンを離した場合

以下のリストは、プラットフォーム間の現在の互換性を示しています（将来のアップデートで変更される可能性があることに注意してください）。

  • Windowsに完全対応しており、同時に最大11台まで接続可能です。Windows では、最初の4つのゲームパッドスロット（0-3）はXInput dllを使用して処理されます。つまり、XBoxコントローラのみが100％互換性があり、他のコントローラタイプについては、残りのゲームパッドスロット（4-11）を確認する必要があります。
  • macOSは 最大4台まで接続可能で、接続可能なデバイスはPlaystation3またはXbox360に限られます。Macゲームオプションの"Build for Mac AppStore"オプションがオフであることが、パッドサポートの動作に必要な条件です。
  • Ubuntuは GamePadの入力にも対応していますが、Ubuntu レポジトリから追加のライブラリをインストールする必要があるかもしれません。コマンドライン・ターミナルを開いて以下を入力すれば、ジョイスティック本体だけでなく、ジョイスティックのGUIサポートもインストールされます。

sudo apt-get install jstest-gtk
sudo apt-get install joystick.

  • HTML5 ゲームは、Safariを除く主要なブラウザでゲームパッドに対応しています。
  • ゲームパッドのサポートはiCadeキャビネットでiOSにも 及んでいます。左軸はスティックコントローラー（ただし、入力はアナログではなくデジタル）、4つの「顔」ボタンは筐体前面のボタン、4つの肩ボタンは筐体背面のボタンにマッピングされています。
  • Androidは NYKOコントローラーと一般的なBluetoothコントローラーをサポートしていますが、それらが有効になっている場合のみ、Androidゲームオプションの 一般セクションでiCade/Bluetoothオプションにチェックを入れる必要があることを意味します。これらは完全に動作するためにはAPIレベル12が必要で、実際に接続されているかどうかに関わらず、GameMakerはデバイスがペアリングされているBluetoothデバイスを接続済みとして登録することに注意する必要があります。そのため、"スロット "を割り当てたりチェックしたりする際には、この点を考慮する必要があります。
  • PS4では、タッチパッドのトラッキングを使用する場合は、タッチパッド用の関数を使用する必要があります。 device_mouse_* ボタンがあります（このためのGML Visual アクションはありません）。
