パーティクルエミッターの作成

パーティクルを表示するには、まずパーティクルシステムを作成し、次にパーティクルの種類を定義し、最後にこのアクションを使用してパーティクルエミッターを作成する必要があります。エミッターを作成するには、このアクションを呼び出し、エミッターを事前に作成したパーティクルシステムに割り当て（パーティクルシステムを参照）、ターゲット変数にその の一意なID値で、以降のアクションで使用できます。この変数を一時的なローカル変数としてflag にすることもできますが、その場合、アクションscript やイベントの終了時に破棄されることになり、エミッタはまだ存在しますが 一般的にエミッターを一時的なローカル変数に格納する必要はありませんが、パーティクルを表面に描画する場合などには必要な場合があります）。

重要!パーティクルシステムを永続化する場合、各room の終了時にDestroy Particle Emitterを使用してすべてのエミッタを破壊し、必要に応じて次のroom で再作成する必要があります。 または、それらの ID 値をグローバル変数に保存して、どのroom でもいつでもアクセスできるようにするか、永続的なobject でそれらを作成し、object が永続的にインスタンス変数を通してそれらにアクセスできるようにする必要があります。また、次のことにも注意してください。 Restart Gameアクションを呼び出す場合は、事前にパーティクルシステムを破壊し、システムおよび割り当てられたエミッタをメモリから削除してから再スタートする必要があります。エミッタを正しく削除しなかった場合 をシステムメモリから削除すると、メモリリークが発生します。これは、あるリソースがメモリに追加された後、「参照解除」されたときに起こる現象で、GameMakerがそのリソースとこれ以上やりとりする方法がないことを意味しています。そのため にアクセスできないリソースによってメモリが詰まり、ゲームに問題が発生する可能性があります。
