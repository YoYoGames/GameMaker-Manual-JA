パーティクルシステムの作成

このアクションは、パーティクルシステムを初期化し、パーティクルタイプやエミッターを作成・使用できるようにします。レイヤーの名前をstring で指定します。たとえば、"Particle Layer" (room editor に存在する必要があります) それから、flag 。 システムを、Persistent（持続的）か否かを指定します。もしシステムがPersistentであるとフラグが立てば、作成された後のすべてのrooms でアクティブになり、その後のrooms で最初のレイヤーが存在しなければ、管理されたレイヤーが割り当てられる。 永続化のフラグが立っていない場合、システムは作成されたroom の終了時に自動的に破棄され、システムに関連するエミッタもすべて破棄されます。もし、手動でメモリからシステムを削除したい場合は、次のものを使用する必要があります。 アクションは、Destroy Particle Systemです。

また、パーティクルシステム ID を格納する target 変数も必要です。このID値は、他の関数でシステムのプロパティを設定したり、パーティクル作成時にどのシステムを使うかをGameMakerに指示するために使われるものである。 この変数を一時的なローカル変数としてflag にすることもできますが、その場合、アクションscript やイベントの終了時に破棄されるため、システムはまだ存在しますが、アクセスする手段がない状態になってしまいますので、注意が必要です。 を使用する場合は注意が必要です（一般的にシステムを一時的なローカル変数に格納する必要はありませんが、表面にパーティクルを描画するような特定のものについては、それが必要になる場合があります）。

注：パーシステントパーティクルシステムは、一度作成するとゲームの間、システムメモリに常駐するため、再作成する必要がある場合（例えば、Restart Gameアクションを呼び出す場合）、まず現在のシステムを破壊しなければならず、そうしないとメモリリークが発生します。メモリリークとは、メモリに追加されたリソースが「参照解除」されることで、GameMakerがそのリソースとやりとりする手段がなくなることです。 そのリソースはもうない。このため、アクセスできないリソースによってメモリが詰まり、ゲームに問題が発生する可能性があります。不要になったパーティクルシステムを破棄するには、Destroy Particle Systemアクションを使用します。
