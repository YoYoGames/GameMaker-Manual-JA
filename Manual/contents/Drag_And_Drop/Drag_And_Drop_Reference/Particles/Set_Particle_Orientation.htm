粒子の向きを設定する

このアクションは、パーティクルの向き（パーティクル画像が描画される角度）を設定するために使用します。サイズを設定するタイプのユニークなID値を指定します（as returned when アクションCreate Particle Typeでタイプを作成した場合)、最初の開始方向の範囲を指定し、次に増加量、最後に "wiggle "を指定します。パーティクルが作成される方向 で、0が右、90が上、180が左、270が下と計算されるので、最小値0、最大値360に設定すると、各パーティクルは360°の周りにランダムな量で回転して放出されます（これはパーティクル方向には影響しないことに注意しましょう）。

パーティクルの向きを増やすオプションでは、パーティクルが生きている間、ゲームフレームごとに基本的な向き角度に追加される値を入力することができます。例えば、この値を0.1に設定すると、各フレームの画像角度に0.1が追加されます。 ゲームフレーム負の値を使用すると、パーティクルは反時計回りに回転し、負の値を使用すると時計回りに回転するようになります。最小の方向 の減少が-0.01で、増加の最小値は0.01です。

また、粒子の向きに "wiggle "オプションを設定することができます。これは0から指定した値の間でランダムに生成され、パーティクルの寿命の各ステップでパーティクル画像の角度に加算または減算される値です。 は、描画された画像の回転を変動させる。明らかに大きな値は小さな値よりも顕著であり、この値は最大範囲が-20から20でマイナスになることもある。

最後に、向きをパーティクル方向に対して相対的にするかしないかを選択することができます。つまり、パーティクルに方向（と速度）が設定されている間、パーティクル画像はそのベクトルに基づいて方向づけられるということです。注意点としては パーティクルの速度を小さくする設定にしていて、パーティクルが生きている間に0になると、速度がないために方向がデフォルト値の0°になり、相対的な方向によってパーティクル画像が異なる方向に「ジャンプ」してしまいます。 の方向角度を指定します。

アクションのシンタックス。

論拠となる。

論証 商品説明
Type の向きを設定するパーティクルタイプのユニークなID値です。
Min Angle パーティクルの種類に応じた最小の向き（-360～+360、デフォルト0）
Max Angle パーティクルの種類に応じた最大方向（-360～+360、デフォルト360）
Increase ゲームフレームごとにパーティクルの種類の向きに追加される値（負の値も可能、最小値は+/-0.01）
Wiggle 粒子の向きを1ステップごとにランダムに加減する最大値(+/-20)
Relative To Direction パーティクル画像の角度をパーティクルの進行方向に対して設定する（有効）か、しない（無効）かを指定します。

例

上記のアクションブロックのコードは、新しいパーティクル タイプを作成し、そのユニークな ID値をグローバル変数に格納する。その後、パーティクルタイプのすべてのプロパティを設定します。
