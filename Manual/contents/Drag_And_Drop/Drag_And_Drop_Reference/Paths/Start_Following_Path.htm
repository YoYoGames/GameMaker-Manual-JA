スタートフォローパス

このアクションは、path ブラウザで作成されたAsset に沿って、インスタンスが追従を開始します。このアクションを使用して、どのpath を追いかけるか、path を追いかける速度（ゲームフレームあたりのピクセル数で測定）、 どのように追いかけるかをインスタンスに指示することができます。 は、path の末尾に到達したときの動作、およびpath の絶対位置と相対位置のどちらに従うかについて説明します。速度を設定するとき、インスタンスがpath を逆にたどることを示すために負の値を使用することができ、以下の点に注意する必要があります。 path ここで設定したスピードは、path エディタで各ポイントに設定したスピードモディファイアによって変更されます。したがって、path スピードを2に設定し、path でポイントの1つに200%のスピードモディファイアを設定すると、そのポイントでの実際のスピードは次のようになります。 1ゲームフレームあたり4ピクセル。

path の初期速度を設定したら、インスタンスがpath の終わりに到達したときの動作を次のオプションから設定できます（インスタンスがpath の終わりに到達すると、Other -Path Ended Event もトリガーされることに注意してください）。

  • Stop:path を終了し、最後のpath ポイントでインスタンスを停止します。
  • 再スタート path がpath Editor で閉じたとマークされていない場合、Path を最初から続行し、再び開始位置にジャンプします。
  • ループする。現在の位置から続けて、path を再度実行します。
  • リバース path に沿って再び逆進し (path のフォロー速度を逆にすることで実現)、path の始点に到達すると、再び逆進する、などなど。

最後に、flag path 、Absolute（絶対）か どうかを選択できます。デフォルトでは、インスタンスは現在の位置（相対位置）からpath を追い始めますが、flag の Absolute チェックボックスをオンにすると、インスタンスは現在の位置から を追い始めます。 path Path Editorでデザインして配置したのと全く同じ位置（絶対位置）です。下の画像はこれを示しています。
