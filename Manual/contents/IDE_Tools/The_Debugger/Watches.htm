Debugger の全体的なアイデアは、ゲームの細部まで「見る」ことを許可することです。これは「ウォッチウィンドウ」を使って行われ、デフォルトでは出力ウィンドウにドッキングして開かれます。 IDE そのうちのいくつかは、リアルタイムで更新することができます（debugging ツールバーからリアルタイムDebugger を有効にした場合）。また、他のものはdebugger でゲームを一時停止しておく必要があります。

どのウィンドウでも変数をウォッチしているときは、複数のデータ型のうちの1つであることがあります。しかし、GameMakerの保存方法によっては、Debugger がその変数を正確に知っているとは限りません。 変数に格納されます。これは、データ構造やインスタンスの内部ID値が整数 であるため、Debugger 、変数に格納されている整数値がデータ構造のものなのか、インスタンスのものなのか、あるいは、データ構造のものなのかを常に知っているわけではありません。 単なる整数値です。そのため、任意の変数値を右クリック（ ）すると、利用可能なデータ型が一覧表示されるポップアップウィンドウが表示されます。 この変数がなりうるものであること。

また、データ構造がインスタンス変数に格納されている場合でも、そのID番号 はグローバルなので、4つのインスタンスがあり、それぞれがds_listを作成する場合（例）、各リストは0から3までのIDを持ち、それらはすべて0になることはないでしょう。

データ型を選択したら、変数名の横にある小さな をクリックするだけで、構造内のエントリーが表示されます。これにより、構造体が展開されます。 また、 をダブルクリックすることで、表示されている変数の値をいくつか変更し、新しい値を与えることができます。

注：この方法で配列やデータ構造を再割り当てすることはできませんが、その内容の一部を変更することはできます（マップの「キー」値や優先度queues の優先度値を変更することはできません）。

レンダリング状態

この時計は、利用可能なすべての情報を表示します。 描画アルファや塗りつぶし色など、レンダリングされている現在のフレームの状態についてです。これはリアルタイムで更新できないので、更新してフレームを取得するには、ゲームを一時停止する必要があります。 の詳細をご覧ください。


サーフェス／テクスチャー

このウォッチにより ゲーム内のあらゆるサーフェス （アプリケーションサーフェスを含む）に描画されているもの、およびメモリ内のテクスチャページを正確に確認することができます。ウォッチの下部にあるメニューから、textures またはサーフェスを選択し、texture またはサーフェス ID にマウスオーバーすると、ポップアップウィンドウに表示されます。

なお、これらの更新を行うには、debugger を一時停止 する必要があります。
