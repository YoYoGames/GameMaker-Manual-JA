リポジトリのクローン

プロジェクトが外部リポジトリに保存されている場合、変更をプッシュすることと、新しいものをプルすることの両方を覚えておく必要があります。コミットした時点で変更をローカルに保存し、インターネットに接続されていない場所でも作業ができるようにし、準備ができたら同期させ、同期させるために変更をリモートリポジトリにプッシュするからです。変更をプルするというのは、他の人が行った変更をリモートリポジトリから自分のローカルリポジトリに引き込むということです。しかし、この場合、同じファイルを編集している人がいる可能性があり、マージの競合が発生する可能性があります。このようなコンフリクトの対処法を知っておく必要があります。そこで、わざとコンフリクトを発生させて、その対処法を紹介しましょう。

注：ローカルのプロジェクトリポジトリを共有することは（Dropboxのようなファイル同期サービスであっても）推奨されません。一般的にコミットは直接リポジトリに書き込まれるため、競合の保護がほとんどないからです。

競合を発生させるには、まず2つのローカルリポジトリを用意する必要があります。1つはマスターリポジトリ、もう1つはコピー用です。これを作成するには、リポジトリのクローンを作成する必要があります。

  • ソース管理>リポジトリのクローン]を選択します。
  • URLフィールドに、上記の「ファイルを元に戻す」ガイドで設定したプロジェクトへのpath を入力します（デフォルトでは、これは <DRIVE>:\Users\<Username>\Documents\GameMaker\<ProjectName>) となります）。
  • 次に、クローン先としてpath を設定する必要があります。
    Ok」ボタンをクリックすると、「non-bare」リポジトリをクローンしようとしているという以下の警告が表示されます。
    ここでもう一度「OK」をクリックすればいいのです。
  • ここで、「裸の」プロキシリポジトリ（これは、例えばDropboxに安全に保存できるリポジトリです）を作成するためのディレクトリエクスプローラが表示されます。このプロキシリポジトリを作成したい場所を参照し、使用するフォルダを選択（または作成）してください。

そうすると、GameMakerは元のプロジェクトをプロキシにプッシュし、新しいプロジェクトをクローンリポジトリのウィンドウで設定したpath にプルして、自動的にファイルブラウザを開き、保存されているプロジェクトを開くことができます。ソース管理>履歴を見るを選択すると、元のコミットと新しい "Push For Clone" コミットが表示されます。

これでリポジトリをクローンしたので、ローカル版のプロジェクトで作業を続けることができます。
