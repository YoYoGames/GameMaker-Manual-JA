マージツール

他の人と一緒にプロジェクトに取り組む場合、複数の人が同じファイルを編集するため、競合が発生する可能性が常にあります。ほとんどのsource control システムでは、競合するファイルをうまくマージすることができますが、同じ場所に変更がある場合、何をすべきか知るために少し手助けが必要です - ここで、変更を表示し、あなたに判断させるマージツールの 出番です。

マージツールはそれぞれ異なるため、ファイルを指定するのに役立つ4つのマクロがあります。これは、コンフリクトウィンドウで 「マージツールを選択する」 というボタンをクリックして設定します。

このボタンをクリックすると、Git のプラグインで環境設定ウィンドウが表示されます。

ここでは、選択したマージツールにpath を提供し、次に、何を行うかを決定する際に使用するマージツールオプションマクロ（またはマクロ）を提供します。

  • ${scm_mine}:path あなたのバージョンに
  • ${scm_theirs}:path to their version
  • ${scm_base}:path 衝突した変更が発生する前のバージョンに戻ります。
  • ${scm_merged}:path マージツールの保存先を指定します。

セットアップのヘルプとして、特定のマージツールにpath を与えると、GameMakerは選択したツールに応じた、開始時のデフォルトをいくつか埋めてくれます。

  • KDiff3です。 "${scm_theirs} "${scm_mine}" -o "${scm_merged}"
  • メルド "${scm_mine}" "${scm_merged}" "${scm_theirs}"
  • メモ帳++です。 ${scm_merged}
  • トータスマージ /base:"${scm_base}" /theirs:"${scm_theirs}" /mine:"${scm_mine}" /merged:"${scm_merged}"
  • DiffMerge: "${scm_mine}" "${scm_merged}" "${scm_theirs}"

なお、マージツールを持っていない（あるいは使いたくない）場合は、script をIDE で開き、そこから修正を試みることができます。

さて、コンフリクトのセクションで述べたコンフリクトの例に戻りましょう。マスターからのプルを行い、ファイルのひとつがコンフリクトしています。Source Control Conflicts ウィンドウが開きますので、コンフリクトしているscript をクリックし、マージツールを設定しましたので、マージをクリックします。マージ」をクリックすると、コンフリクトを処理するためのツールが表示され、提供されたマクロを使用して必要な変更を行うことができます。マージツールを終了して閉じると、GameMaker IDE に以下のような確認ウィンドウが表示されます。

マージツールを使って自分で行ったように、このプロンプトに対して "Yes" と答えるだけです。そして、マージ操作の終了を意味するためにこの変更をコミットする必要があります(Source Control > Commit Changes)。コミットウィンドウにマージ中であることが表示され、ステージされていないファイルもいくつか表示されます。

これらのステージングされていないファイルは、私たちのscript ファイルに、 .base 、 .ours 、 .theirs が加わったものです。これらを安全に削除するために、一番上の と、一番下の + をクリックし、すべて選択します。次に、選択したファイルを右クリックして、パスの削除を選択すると、削除されます。これはマージであり、競合があったことを示すコミットメッセージもあらかじめ入力されています（希望に応じて、これに追加したり編集したりすることができます）。

今なら「コミット 」をクリックできます。

他のファイルで問題が発生する可能性があるため、リモートリポジトリや共有リポジトリを使用する際のワークフローは、以下のようにする必要があるということです。

Make Changes > Save > Commit > Pull > Merge > Push

ディフ・ツール

リポジトリへのコミット時に使用するDiff ツールも設定する必要があります。これは Merge ツールと同じようにPreferences から設定します。

Diff ツールは Merge ツールとほぼ同じ方法で設定されますが、 ${scm_base} と ${scm_theirs} オプションのみを使用します。この場合、 ${scm_base} は変更されていないファイルを表し、 ${scm_theirs} はファイルの現在の状態を表します。ツールオプションには、Merge ツールと同様に、Meld、KDiff3 、TortoiseMerge のデフォルトオプションがあらかじめ入力されているはずです。GameMakerは単一ファイルの差分パッチの出力をサポートしていない ため、Notepad++ のデフォルトはないことに注意してください。

Diff ツールを使って現在のリビジョンと以前のリビジョン間の変更点を表示するには、Commit ウィンドウを開いて、ファイルをダブルクリックするだけです。ツールが正しく設定されていれば、ツールが起動し、ファイル間の変更点を見ることができるようになります。
