マイクロウェブサーバー

この小さなツールの主な機能は、GameMakerで利用可能ないくつかのターゲットプラットフォーム（HTML5ターゲットなど）に接続し、ファイルを提供することである。また、オフラインで使用する場合は、GameMakerのマニュアル用のファイルも提供する。

Micro WebServer を必要とするターゲットでゲームを実行すると、選択したデバイスにゲームが提供される際に、新しいウィンドウが開くのがわかります。

注Micro WebServer を必要とするターゲットで初めてゲームを実行すると、ファイアウォールの警告が表示されることがあります。このような場合は、Webserver がファイアウォールを介して通信することを許可する必要があります。

Micro WebServer を実行すると、ローカルのどのデバイスからでも（ポートフォワーディングが適切に設定されていればリモートでも）ゲームにアクセスすることができます。このウィンドウに表示されているポートを介してコンピュータのローカルIPアドレスにアクセスするだけで、携帯電話などの他のデバイスや他のコンピュータからゲームをプレイすることができるようになります。

例えば、あなたのコンピュータのローカルIPアドレスが192.168.0.110（見つけ方はこれを読んでください）、Micro WebServer ウィンドウに表示されているポートが51264だとします。コンピュータでゲームを実行した後、携帯電話のブラウザから192.168.0.110:51264に接続し、ゲームをプレイすることになります。

WebServer 、配信されたファイルに関連する一定のデバッグ情報が表示され、下部にこれをクリアするボタンがあります。クリア "ボタンの横には、プロジェクトのルートJSファイルへのpath 。その下には、"ネットワーク速度"のオプションがあります。これはデフォルトで「No Limit」に設定されていますが、クリックするとさまざまなネットワークオプションのリストが表示されます。

この設定を変更すると、パッケージサイズの オプションも右側に変更されます。また、手動で編集することも可能で、その場合は「ネットワーク速度」が「カスタム」に設定されます。最大値以外に設定すると、ネットワーク転送速度が効果的に調整され、低帯域幅環境でのゲーム テストが可能になります。

また、使用されているポート 番号と一緒に許可IPのリストも見ることができますが、これらはMicro WebServer で編集することはできません。これらは、指定されたデバイスターゲットのデバイスマネージャの設定、または「デフォルト」デバイスを使用する場合はHTML5 Preferencesで設定する必要があります。
