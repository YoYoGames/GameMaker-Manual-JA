ローカルAsset パッケージ

Toolsメニューには、LocalAsset Packageを作成、インポートするための2つのオプションがあります。ローカルAsset パッケージは、 *.yymp ファイル形式を使用して保存されたリソースのグループで、既存のプロジェクトに簡単にインストールすることができます。つまり、script ライブラリ、スプラッシュスクリーンイントロ、メニューシステムなど、よく使うリソースのパッケージライブラリを、GameMaker マーケットプレイスを使わずに、自分で作成することができるのです。

Asset パッケージの作成

asset パッケージを作成するには、まず、プロジェクトに保存したいリソースを作成し、ツールメニューから「ローカルパッケージの作成」オプションを選択する必要があります。すると、以下のようなウィンドウが表示されます。

このウィンドウでは、オプションで発行者名、必須で表示名とパッケージIDを指定することができます。パッケージIDはパッケージが保存されるファイル名を生成するために使用され、作成時に自動的に *.yymp ファイルextension が追加されます。また、パッケージのバージョン番号を設定し、署名するかどうかを選択することができます（デフォルトではオフになっています）。パッケージに署名するには、署名証明書を作成する必要があります（Marketplace Preferencesから作成できます）。また、署名されたasset は、作成後に改ざんされた場合、インポートすることができなくなることを意味します。

パッケージの詳細を入力したら、右側の「Project」セクションから入れたいassets を選択します（マウスの左ボタン または / と を組み合わせて使用します）。選択したリソースは、「Add 」または「 Add All」ボタンを使って「Asset Package」ビューに移動し、「Remove 」または「Remove All」ボタンを使って再び削除することが可能です。

リソースの選択と必要事項の入力が完了したら、Ok ボタンをクリックするとファイルエクスプローラが開き、パッケージの保存場所を選択することができます。選択するとパッケージが作成され、確認ウィンドウが表示され、OSのファイルエクスプローラでパッケージの保存場所を開くオプションが表示されます。

Asset パッケージをインポートする

LocalAsset パッケージをインポートするには、2つの方法があります。

  • 1つ目は、 *.yymp ファイルをエクスプローラからGameMakerIDE にドラッグする方法です。
  • もう一つは、ツールメニューから「ローカルパッケージのインポート」を選択することです。

ツールメニューを使用する場合は、エクスプローラーウィンドウが開きますので、パッケージの場所をブラウズして、それを選択してインポートすることができます。いずれの方法でも、GameMakerで以下のようなウィンドウが表示されます。

左側がパッケージの内容で、右側がインポートしたい部分です。パッケージから1つまたは複数のリソースを選択し（マウスの左ボタン またはマウスの左ボタンと一緒に / と の組み合わせを使用）、Add ボタンを使用してインポート用に追加するか、Add All ボタンを使用してインポート用にすべてを追加することが可能です。

すべての準備ができたら、Import ボタンをクリックして、選択したリソースをプロジェクトに追加することができます。新しいプロジェクトを作成し（保存する場所を指定するよう促されます）、パッケージリソースをそのプロジェクトに追加します。

GameMaker.3以前のバージョンで作成されたasset パッケージは、2.3アップデートでプロジェクトファイル形式が変更されたため、変換が必要であることに注意してください。このため、assets を他のプロジェクトに頻繁にインポートする場合は、プロジェクト全体をインポートしてすべてを変換し、新しいパッケージとして新しい場所にエクスポートした方がよい場合があります。
