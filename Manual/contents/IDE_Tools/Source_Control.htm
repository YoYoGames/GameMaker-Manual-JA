ソース管理

SCM（Source Control Management）とは、リポジトリとローカルソースを介して、プロジェクトのサブバージョンやバックアップを扱う手法に付けられた名称です。基本的に、SCM ソリューションは独立したソフトウェアパッケージであり、作業中にプロジェクトのバージョンを維持、変更、比較するためのあらゆる側面を制御します。これは、特にチームで作業する場合に便利で、誰が何をするかを管理でき、データの損失や後日取り消す必要のある変更を心配する必要がありません。

GameMakerはGit をSCM として利用しています。SCM をGameMaker で利用するためには、Git を別途インストールし設定する必要があります。Git に関するより具体的な情報はGit Documentationを参照してください。

以下に、このプラグインを設定し、プロジェクトでSCMツールを使用する方法についての小さなチュートリアルへのリンクがあります。

重要! GameMaker でsource control を使用する前に、その仕組みについてある程度知っておくことをお勧めします。

  • Git プラグインを設定する
  • リポジトリオプション
  • 標準的なワークフロー
  • ファイルを元に戻す
  • リポジトリのクローン
  • コンフリクト
  • 外部Merge/Diffツールのセットアップ

GameMakerは、Source Control をプロジェクトで有効にすると、メニューバーにSource Control のドロップダウンメニューが表示され、以下のオプションが入力されます（詳細は上記の「 Git プラグインのセットアップ」のセクションを参照してください）。

ここでは、利用可能な各オプションの概要を説明します（ほとんどのオプションは、上記のチュートリアルのセクションでより深く説明されています）。

  • プロジェクトリポジトリを作成する。このオプションは、プロジェクトファイルと同じディレクトリにローカルリポジトリを作成することを許可します。
  • リポジトリをクローンする。このオプションを使用すると、ソースから新しい宛先にリポジトリをクローンすることができます。
  • 変更をコミットする。このオプションでは、プロジェクト内で変更されたファイルをステージングし、リポジトリにコミットすることができます。
  • 変更をプッシュします。コミットまたはマージを行った後、このオプションは変更をマスターリポジトリにプッシュするために使用されます。
  • 変更のプル。このオプションを使用すると、マスターリポジトリから変更されたファイルをプルすることによって、ローカルリポジトリを更新することができます。
  • 履歴を見る。このウィンドウでは、プロジェクトのすべてのバージョン履歴を見ることができ、特定のファイルpaths またはリビジョン全体をロールバックすることを選択できます。
  • コンフリクトを表示する。コンフリクトウィンドウを開き、プロジェクト内に存在するコンフリクトファイルをリストアップし、GameMaker IDE または特定の Merge Tool を使って対処することができます。
