リモートワーカー

ゲームをコンパイルするときは、通常、GameMakerがインストールされているコンピュータである「ローカル」ワーカーを使用します。しかし、状況によっては、他のコンピュータを使ってプロジェクトを構築する必要があります。例えば、以下のような場合です。 Mac で作業していて、Windows プロジェクトをビルドしたい場合、リモートワーカーを設定し、Windows PC にビルドをさせることになります。基本的に、リモートワーカーを使うと、GameMakerはすべてのリモートワーカーを送信します。 必要なファイルを実行中の（ローカル）マシンからリモートワーカーマシンに送り、そのマシンを使ってプロジェクトの最終実行ファイルを構築します（リモートマシンに保存するか、その後ローカルマシンに戻すか）。 ただし、この機能を使用する前に、ビルドマシンに特定のツールをインストールし、いくつかのセットアップを行う必要がありますので、ここで説明します。

重要!ローカルのWindows PC とリモートのWindows PC、またはローカルの Mac PC とリモートのWindows PC の間でリモートワーカーを設定できますが、現在ローカルのWindows (または Mac) PC とリモートの Mac PC を設定することはできません。 Macのリモートワーカーを持つことができます。Windows 。

初期設定

リモートワーカーを設定するためには、まず、使用するPCにリモートワーカー構築ツールアプリケーションをインストールする必要があります。このツールは、ローカルマシンのGameMakerの ビルドメニューから、"Show Remote Worker Installer"を選択することで見つけることができます。

または、ターゲットマネージャの「目」アイコンをクリックし、ターゲットマネージャから リモートワーカーインストーラー

リモート設定

設定 ページには、リモートワーカーアプリケーションのさまざまなpaths とビルドの設定が表示されます。設定の上部には、リモートワーカーアプリケーションで使用されるさまざまなpaths 。 必要であれば、これらを修正し、代替のpaths に設定する必要があります（一般的にはデフォルトのpaths で問題ないでしょう）。リモートワーカー設定ページには、次のいずれかの方法でアクセスできます。

  • リモートマシンのブラウザで開くリモートワーカーページの上部にある「設定」リンクをクリックします。
  • リモートワーカートレイアイコンのRMBを使用して、リモートマシンのブラウザで設定を開くことができます。
  • ローカルマシンでターゲットマネージャを開き、「歯車」のアイコンをクリックすると、ローカルマシンのウィンドウにワーカーの設定が表示されます。

このページでは、プロジェクト構築のためのpaths を設定するためのさまざまなオプションを確認（および設定）することができます。

コンパイル

リモートワーカーをセットアップし、ローカルビルドの設定（上記のセクションを参照）を完了したら、ゲームのコンパイルに使い始めることができます。この場合、Target Managerからリモートワーカーを選択するだけです。 のリストで、テスト/コンパイルするプラットフォームと、その他のオプション（コンパイルの種類や設定など）を選択し、Play または Create Executable を押してください。

注：ターゲットマネージャで利用可能なプラットフォームは、リモートワーカーのコンピュータの機能によって変更され、すべてのターゲットが利用できるわけではありません。また、リモートワーカー用の「テスト」ターゲットは存在しないことに注意してください。

単にプロジェクトを再生するだけなら、リモートワーカーマシンがコンパイルして実行しますが、実行ファイルを作成する場合は、リモートワーカーでプロジェクトをコンパイルして、 *.zip ファイルを返します。 ローカルマシンに保存する。

リモートワーカーを使用する場合、Debug ボタン と Clean ボタン のどちらも使用できないことに留意してください。また、現在一度に実行できるリモートビルドは1つだけであることに注意してください。


リモートワーカーをリモートマシン上にインストールして実行すると、タスクバーにアイコンとして表示され、その上で右クリックすると、以下のオプションが表示されます。

  • 開くデフォルトのブラウザでリモートワーカーアプリケーションを開きます。
  • ログを表示する。リモートワーカーのログファイルをブラウザで開きます。
  • ネットワーク設定。ここでは、リモートワーカーマシンにビルドを送信できるようにするマシンをフィルタリングすることができます。各フィルタには、CIDR形式のフィルタ、または許可するマシンのホスト名を指定できます。複数のマシンを1台のリモートワーカーに接続する場合、または接続の種類を混在させる場合（有線と無線LANなど）、ここでIPの許可範囲を編集する必要があることを認識しておいてください。
  • 設定を編集する。リモートワーカー設定メニュー（上記「設定」セクションで説明）を開きます。
  • 終了する。リモートワーカーアプリを終了します。リモートワーカーアプリを再起動するまで、ローカルマシンからアプリに接続できなくなります（コンパイルもできなくなります）。
