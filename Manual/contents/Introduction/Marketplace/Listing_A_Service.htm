assets でMarketplace を持つ以外に、サービスを作成することもできます。サービスのオファーを送信する前に、以下の関連ページに記載されている手順を完了する必要があるため、続行する前に、必ずお読みください。

  • パブリッシャーアカウントの作成

サービスは、GameMaker Marketplace のサブセクションで、個人や企業がサードパーティのサービスを提供することができる。例えば、あなたが広告APIなどを持っていて、GameMakerのユーザーに提供したい場合、ここでサービスを提供することになる。同様に、あなたがミュージシャンやアーティスト、プログラマーで、人に見せるためのポートフォリオを持っていて、仕事を依頼したい場合、ここでもサービスを提供することになる。

重要！ YoYo Games は、契約者と被契約者の間のいかなる問題にも責任を負わず、紛争が発生しても調停やその他の方法で関与することはありません。

マーケットプレイスのメインページ、上部の「カテゴリー」メニューから、または右側のメニューから選択することで、サービスパネルにアクセスすることができます。

緑色の矢印をクリックすると、利用可能なサービスのカテゴリ（マネタイズや 広告など）が表示されるサブメニューが表示されます。

サービス概要には、提供されているサービスに関するすべての情報と、プロバイダに連絡したり、サービスに登録したりするためのリンクが含まれています。また、サービスに必要なリソースへの追加リンク、例えば、リーダーボードとアチーブメントサービスを提供するサイトのSDKasset へのリンクもあります。

Marketplace で発表されているサービス プロバイダーのいずれかに連絡する前に、他のユーザーが与えたレビュー （ある場合） を確認し、提供されるEULAを読み、指定されたサービスの条件を理解していることを確認します。YoYo ゲームは、エンド ユーザーと任意のサービスの間の取引に責任を負いませんし、人々 の広告のための便利な場所を提供するだけです。

サービス一覧を設定するためには、まず以下のことを行っている必要があります。

  • Marketplace Publisherのアカウントページを設定します。
  • 出版社名や電子メールなど、出版社に関する基本情報を入力し、提示されたasset タイプのリストから「その他」を選択してください。

完了すると、新しいアカウントを有効にするためのリンクが記載された確認メールが送信されます。確認メールを受け取ったら、サービスリストのセットアップの準備は完了です。パブリッシャーウィンドウの左側にあるサービス メニューをクリックすると、サービス利用規約が表示されますので、同意の上、次に進んでください。同意すると、現在のサービス一覧を表示するページが表示されます。新しいサービスを作成するには、"新しいサービスを追加する "をクリックしてください。

このページでは、追加するサービスの種類、提供するサービスの詳細、およびその他の関連情報を定義します。必要な情報は以下の通りです。

  • プライマリーカテゴリー。これは、提供するサービスの種類です。プライマリーカテゴリーを設定したら、さらにサブカテゴリーを選択することができます（可能な場合）。これにより、Marketplace を検索する際に、適切なサービスを見つけることができます。
  • 表示名。サービスページに表示されるサービス名です（最大255byte）。
  • 説明文。サービス内容、機能、使用方法、ドキュメントへのリンクなどを説明する短い説明文（2000字以内）。
  • キーワード。Marketplace であなたのサービスを見つけるために検索を絞り込むために使用できる1つ以上の単語。 これらのキーワードは、提供されるサービスに関連するものでなければなりません
  • 関連資産。あなたのサービスがさらにassets （GMZ、SDKなど）を必要とする場合、そのために通常のMarketplaceAsset Listingを作成し、ここにリンクさせることができます。asset に名前を付けて、表示されるリストから選択してください。asset は、以前に公開されている必要があります。
  • サポートメール。提供されているサービスについて問い合わせをしたい場合に、サポートを受けるためのメールアカウントです。
  • サポートサイト。ユーザーがサービスのサポートを受けることができるURL（例：FAQ、フォーラム、お問い合わせ先など）
  • カスタムEULA：提供されるサービスに対して書かれた特定のEULAを読むことができるURL（これが指定されていない場合、あなたのサービスはここにある一般的なYoYo Games EULAに該当することになる）。
  • プライバシーポリシーのURL。あなたのサービスがデバイスやユーザーのデータを必要とする場合、またはサービスが子供向けに設計されている場合、プライバシーポリシーへのリンクを提供する必要があります。
  • マーケティングURL。ユーザーがサービスやパブリッシャーに関する詳細情報を得ることができ、ビデオやスクリーンショット、その他の関連マーケティング資料を見ることができるURLです。

Marketplaceアイコン（200x200 PNG ）と特徴的なグラフィック（1140x360 PNG ）、それから少なくとも1枚のスクリーンショット（ PNG であるべきですが、以下のサイズでオーサリングすることが可能です。1280×720、1366×768、1600×900、1920×1080）、最後にYouTubeまたはVimeoのビデオを追加するオプションがあります。リスティングの設定方法に関する詳細な情報と推奨事項については、assets 、マーケットプレイスのスタイルガイドを参照してください。

これらの情報を入力したら、"保存 "をクリックして、パブリッシャーダッシュボードの一般サービスリストに戻ることができます。これで、新しいサービスが「審査中」のステータスで表示されていることが確認できます。

サービスにSDKがある場合、またはそのSDKのためにextension を作成したい場合は、マーケットプレイスアセットを作成する方法を参照してください。これが完了したら、サービス一覧の「関連資産」セクションに、asset パッケージの URL を追加してください。

お客様がMarketplace に初めてサービスオファーを提出される際には、それが最低限のエントリーレベルの基準を満たしていることを確認するため、YoYo Games による審査が必要です。これは、次のことを意味します。

  • スクリーンショット、アイコン、フィーチャーグラフィックは、提供するサービスに関連する明確なものである必要があります。
  • サービス内容の説明は、連絡を受けた後に行うサービスと一致している必要があります。
  • サービスについての説明は、一貫性があり、詳細で、よく書けている必要があります。
  • 説明文または掲載ページ内のすべてのリンクが正しく、関連性があること。
  • 提供するサービスが、許容される素材のルールとガイドラインに該当するものであること（これにはスクリーンショットとアイコンが含まれます）。

審査には1～2日かかりますが、審査が完了すると、Marketplace サービスページであなたのサービスが公開されます。審査に落ちた場合、メールでお知らせします。審査に落ちた理由や審査に通るための方法が分からない場合は、ヘルプデスクにご連絡ください。
