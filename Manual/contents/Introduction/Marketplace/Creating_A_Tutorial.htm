Marketplace チュートリアルを作成する

チュートリアルの作成は通常のMarketplace asset パッケージの作成と似ていますが、特定のインクルードファイルを追加し、最初のチュートリアルページ(およびオプションのビデオページ)を含める必要があることだけが異なります。チュートリアルworkspace は次のようになります。一般的なMarketplace Assets で説明したように、パブリッシャーを選択し、チュートリアルに名前と一意の ID (逆 URL 形式) を与え、「タイプ」ドロップダウンでチュートリアルを選択する必要がありま す。これにより、チュートリアルを開始するためのベース HTML ファイルと、（オプションの）ビデオファイルおよび URL を追加するオプションが表示されます。ビデオファイルは、チュートリアルの他の部分と同様に、 Marketplace から提供されたテンプレートを使用する必要があります。ユーザーがチュートリアルをロードすると、ビデオはIDE の中の別のフレームにロードされます。スタートページとビデオページに指定したページは、Included Files の中のpaths に相対することに注意し、エクスプロラを開いてGameMakerが入力するデフォルト以外のものを選ぶことができます。

実際にチュートリアルを作成するには、チュートリアルプロジェクトにインクルードファイルを追加する必要があります。これらは、チュートリアルに使用されるファイルをIDE が適切に検出できるように、非常に特殊な方法で追加する必要があります。基本的なフォルダ構成は、以下の画像のとおりです。

そこで、大きく分けて2つのフォルダーを用意しました。

  • TutorialContent。チュートリアルのすべてのファイル（ HTML ）、スタイル設定用のファイル（ CSS ）、画像（リソースに必要な画像ではなく、チュートリアル内で表示する画像）、および必要な場合はビデオテンプレートがここに格納されます。チュートリアルを章に分けたい場合などは、サブフォルダを作成することもできますが、チュートリアルのスタートページとして使用するベースとなる HTML ファイルを少なくとも1つ用意する必要があります。
  • TutorialResourcesです。このフォルダには、さらに2つのサブフォルダがあり、画像と サウンドの アセットが格納されています。チュートリアルで、ユーザーがsprite を追加する必要があり（例）、それをチュートリアルに含める場合、画像リソースは Images フォルダに配置します。これらのフォルダはチュートリアルプロジェクトにインポートされ、ユーザーがsprite エディタを開いて画像を追加しようとすると、エクスプローラウィンドウのデフォルトのpath がこの場所になり、ユーザーは必要なassets を簡単に見つけることができるようになります。

チュートリアルをIDE で動作させるには、このフォルダ構造と、スタイル設定用の CSS ファイルを含むベースとなる HTML のスタートページを用意する必要があります。チュートリアルプロジェクトでは、ビデオの一部をスキップしたり、特定のセクションや機能に関するマニュアルを開いたりするための特別なリンクがいくつか用意されており、これらはすべてMarketplace の公式チュートリアルテンプレートに記載されています。

重要!独自のチュートリアルを作成する前に、マーケットプレイスからチュートリアル・テンプレートをダウンロードすることを強くお勧めします。これらのテンプレートには、HTML の重要な部分の例、チュートリアル内のボタンやメニューを再現するための CSS や画像が含まれており、表示方法よりもコンテンツについて自由に悩むことができます。

含まれるファイルを設定し、必要な情報を提供したら、[OK]をクリックするとチュートリアルがMarketplace にアップロードされます。終了するとworkspace にasset が表示されるので、編集してアイコンやテキスト、スクリーンショットなどを追加することができます。チュートリアルasset が公開されると、ユーザーはスタートページのチュートリアル セクションでそれを見つけることができ、そこでダウンロードしてインストールすることができます。
