スタートページ

GameMakerを初めて起動すると、YoYo または Opera Account にログインするよう求められます（まだ登録していない場合は、ここを参照してください）。登録しログインすると、GameMaker IDEがスタートページに表示されます。

GameMakerを初めて起動したとき、スタート画面の各要素にカーソルを合わせると、その説明が表示されるようになっています。

ここで、「SKIP to Setup Wizard」ボタンをクリックすると、GameMakerで初めてゲームを作成するために必要な手順を、セットアップウィザードの助けを借りて、最初のプロジェクトを作り始めることができます。

また、「閉じる」をクリックするとウィザードを閉じて手動でGameMakerを使い続けることができます。オプションとして、GameMakerを再び起動したときにウィザードを表示したくない場合は「二度と表示しない」のチェックボックスを有効にしてください。

スタートページ要素

スタートページには、以下の要素があります。

  • IDEのメニューバーです。IDE 全体で使用できるメニューが表示されます。詳細は以下のセクションで説明します。
  • プロジェクトのオプション。プロジェクトセクションでは、プロジェクトの作成、オープン、インポートを行うことができます。これについては、このページの下の方で詳しく説明しています。
  • 最近のプロジェクト。ここでは、開くことができる過去のプロジェクトの一覧が表示されます。また、プロジェクトのフルネームとpath 、マウスの左ボタン をクリックすると、そのプロジェクトを開くことができます。switch また、右上のボタンを押すことで、このセクションの表示をtile モードとリストモードの間で切り替えることができます。
  • バージョンの詳細。このセクションでは、現在使用されているIDE のバージョンと、現在のRuntimeのバージョンに関する詳細を説明します。また、IDE またはRuntime のいずれかに変更があった場合、このセクションで通知されます。
  • ユーザーの詳細です。YoYo アカウントのユーザー名と、最後にログインした時間が表示されます。
  • 有用なリソース。GameMakerの公式チュートリアルのリソースにアクセスするための様々なtiles が表示されます。

IDEメニュー

上部には、一般的なIDE メニューがあり、マニュアルの次のセクションで説明されています。

  • ファイルメニュー
  • 編集メニュー
  • ビルドメニュー
  • Windows メニュー
  • ツールメニュー
  • Marketplace メニュー
  • レイアウトメニュー
  • ヘルプメニュー

上部のメニューバーには、現在フォーカスしているウィンドウに応じて、 コンテキスト固有のメニューオプションが表示されることに注意してください。例えば、画像編集ウィンドウにフォーカスがある場合、ここに「画像」、「表示」、「効果」の追加メニュー項目が表示されます。これらの追加メニュー項目については、workspace またはそれらを生成するウィンドウのマニュアルの関連セクションで説明されています。

New」 ボタンで新規プロジェクトを作成、「Open」 ボタンで既存プロジェクトを開く、「Import」 ボタンで圧縮された YYZ GameMakerプロジェクトファイルやレガシーGameMakerを開くことができます。Studio 1.4 GMX プロジェクトファイルを開くことができます。開く」と「インポート」のどちらも、ファイルエクスプローラを開いて、必要なプロジェクトファイルをブラウズすることができます。

注：GameMakerとの完全な後方互換性はありません。しかし、インポートした1.4プロジェクトは、互換性スクリプトを使用して古い機能が自動的に再現されるため、そのまま実行することができます。考えられる問題とGML に加えられた変更の詳細については、ヘルプセンターの記事「GMS 1.4 プロジェクトをGameMaker に移植する」と、マニュアルの「廃止された機能」のセクションを参照してください。

新しいプロジェクトを作成するには、「新規作成」ボタンをクリックし、「テンプレート」メニューを開きます。
