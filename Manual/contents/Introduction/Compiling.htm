コンパイル

ゲームのコンパイルには、テスト用にコンパイルする場合と、特定のターゲットプラットフォーム用に最終的にコンパイルする場合の2通りがあります。 実行形式パッケージを作成するためにコンパイルすることです。このページでは、この2つのオプションについて詳しく説明することを目的としています。

テスト用コンパイル

テスト用にゲームをコンパイルするには、IDE の上部にある Play ボタン を押すだけで、指定したターゲットを使用してテスト用にゲームを起動できます。また、Debug ボタン（ ）を使用してテストすることにより、ゲームをDebug モードで実行することができます。これにより、ゲームが実行されるだけでなく、デバッグウィンドウが開き、ゲームのパフォーマンスを監視して問題を確認することができます（詳細は、デバッグのセクションを参照してください）。

ターゲット設定

デフォルトでは、GameMakerは内蔵の仮想マシン（VM）を使って実行し、デバッグします。これは、使用しているデスクトップOS上で実行するのとほぼ同じです。しかし、GameMakerは クロスプラットフォームエンジンであり、あなたのプロジェクトの実行可能パッケージを多くの異なるターゲットプラットフォームでテスト、デバッグ、コンパイルすることができます（利用できる正確なプラットフォームは、あなたのライセンスの詳細によって決まります）。 現在のターゲットプラットフォームを変更するには、Targetsボタン（ ）をクリックし、Targets Windowを開きます。

上部の[Targets]ボタンの横には、実際に使用されているプラットフォームと特定の設定を示す現在の設定があり、ウィンドウの残りの部分は、代わりに使用するために選択できるすべての利用可能なターゲットの詳細とオプションで占められています。このウィンドウの各セクションを以下に説明します。

プラットフォーム

このセクションでは、利用可能なすべてのターゲットプラットフォームをリストアップしています。このリストの内容は、お持ちのライセンスによって異なりますが、少なくとも「Opera GX」ターゲットが常に含まれています。ターゲットを選択するには、 をクリックするだけです。これにより、残りのオプションウィンドウが更新され、選択されたプラットフォームに応じて異なる詳細が表示されます。

各プラットフォームでのコンパイルの詳細は、後述の「最終的な実行可能パッケージの作成」の項で説明します。

出力

各ターゲットプラットフォームは1つ以上の出力形式を持つことができ、主なものは以下の通りです。

  • VMです。VM (Virtual Machine) ターゲットは、各プラットフォーム用の汎用ランナーを 使用し、ゲーム用にコードを解釈します。一般にこのオプションはビルド時間が早いのでテスト用に使われますが、YYCオプション（利用可能な場合）を使った場合のようなパフォーマンスの向上は望めません。このオプションは、小規模なゲームやパフォーマンスが問題にならないようなゲームの実行形式パッケージをコンパイルするために使用できます。
  • YYCYYC(YoYo Compiler) は、GML のコードから C++ コードを生成し、ターゲットプラットフォームの C++ コンパイラを使ってターゲットプラットフォームのネイティブコードにコンパイルする前に使用します。不要な関数を削除し、その他の最適化技術を多数実行することで、より小さく、パフォーマンスが向上した実行可能なパッケージを作成します。

    これにより、特にロジックを多用するゲームでは、ゲームのパフォーマンスが少なくとも2～3倍向上し、大規模なプロジェクトやCPU負荷の高いプロジェクトに理想的です。コンパイルに時間がかかる場合があります。ターゲットプラットフォーム用に最終的な完全なcache パッケージを構築する前に、必ずコンパイラのasset をクリアする必要があります。YYCターゲットでは、選択したプラットフォーム用に追加のツールをインストールする必要がある場合があります。
  • JavaScriptです。JavaScriptターゲットは、HTML5ターゲットのような特定のターゲットでのみ使用でき、ゲームを純粋なJavaScriptにコンパイルするように設定します。これは出力するJSにECMAScript 2015(またはES6)を使用します。

For 2024.800+ beta versions of GameMaker, some platforms may also include the "GMRT" and "GMRT VM" outputs as part of the GMRT (GameMaker Runtime) open beta.

デバイス

一部のプラットフォーム（iOS やAndroid など）では、GameMakerに1つまたは複数のデバイスを関連付けることができ、ゲームを選択的にコンパイルすることができます。 初期状態では、デバイスのリストは空っぽで、デバイスエディタを開くには、鉛筆アイコン をクリックする必要があります。

このウィンドウでは、新しいデバイスを追加したり、接続されているデバイスの接続をGameMakerにテストさせることができます。このウィンドウの正確な内容は、プラットフォームの仕様によって異なります（プラットフォームの詳細については、デバイスマネージャのセクションを参照してください）。デバイスが見つかったり、追加されたりすると、このウィンドウに表示されます。

デバイスのセットアップやトラブルシューティングの正確な手順や要件は、GameMaker Knowledge Baseの該当セクションに記載されています。

最終的な実行可能パッケージの作成

概要

IDE にある実行ファイルの作成ボタン をクリックしてコンパイラのビルドを開始するか、ビルドメニューから 実行ファイルの作成を選択するだけです。どちらを選んでも、選択したターゲットプラットフォームによってビルドプロセスが開始されます。

Opera GXターゲットでは、特別なウィンドウが開き、ゲームをコンパイルしてGX.gamesにアップロードできます。その他のターゲットでは、ファイルエクスプローラウィンドウが開き、ゲームの実行ファイルに使用したい最終的な名前を付けて、保存を クリックしてコンパイルとビルドのプロセスを開始することができます。これが終わると、必要なファイルが生成され、好きなように配布することができます。

特定のプラットフォームでは、ゲームプロジェクトをコンパイルするために、正しい設定をする必要があります。 ビルドツール(ここを参照) を設定し、適切なPlatform Preferences (プラットフォーム設定) を記入する必要があります。

ライセンスによっては、コマンドラインでゲームをビルドするオプションがあり、プロジェクトの継続的統合を設定し、ビルドとテストのプロセスを合理化することができます。コマンドラインでのビルドの詳細については、このページを参照してください。

注意プロジェクトの最終ビルドをリリースする前に、必ずAsset CompilerCacheをクリアしてください（IDE の上部にある「ほうき」アイコン を使用します）。GameMakerはコンパイル速度を最小限にするために多くのゲームファイルをcache しますが、これらは時々破損することがあるので、リリース用ビルドを行う前にcache をクリアしておくと安全です。

最終的なゲームパッケージの最大サイズは4GBです。ただし、32ビット版（Windows ）の場合は2GBとなります。最終的なゲームサイズを小さくするためのヒントをご覧ください。

対象フォーマット

各ターゲットオプションは、以下に示すプラットフォーム固有のフォーマットで保存されます。

  • Opera GX- "Opera GX "ターゲット用にコンパイルする場合、"Opera GX Packaging "ウィンドウが表示され、最初にOperaアカウントを使ってサインインするよう求められます。なお、すでにOperaアカウントを使ってGameMakerにサインインしている場合は、ここでサインインする必要はありません。

    サインイン後、コンパイルが開始され、GX.gamesにアップロードされます。その後、「Edit Game on Opera」をクリックすると、GX.games DevCloud上で編集し、パブリッシュすることができます。
    Opera GXのアップロード手順についてはこちらのページを、YYCやMobileなどの各種エクスポート設定についてはこちらのページをご参照ください。
  • Windows - 一般的なWindows OS用にコンパイルすると、まずインストーラーか Zip パッケージのどちらを作成するかを選択するよう求められます。インストーラーはゲームをインストールするための実行ファイル、Zipファイルはゲームファイルをすべて保存した *.zip 形式の圧縮ファイル（ゲームを実行するにはファイルを展開する必要があります）です。パッケージングオプションを記憶する"にチェックを入れると、GameMakerはその選択を記憶して次回以降のコンパイルに使用します（Windows環境設定からリセットや変更が可能です）。詳しくはYoYo Games Help Centerを参照してください。Windows ターゲットは、一般ゲームオプションで選択した内容によって、32bitまたは 64bitの実行ファイルをコンパイルできることに注意してください。 32bitRuntime は将来的に非推奨となり削除される予定ですので、全ての新規プロジェクトは64bit実行ファイルを作成するためにx64runtime を使用することを強くお勧めします。
  • Ubuntu (Linux)-Ubuntu 用にコンパイルすると、x64 またはARM の実行ファイルを含む.AppImageまたは.zipファイルを作成するかどうか尋ねられます（ただし、AppImage はARM をサポートしていません）。詳しくは、YoYo Gamesヘルプセンターでご確認ください。
    .AppImageはSteam 以外のディストリビューターでパッケージとして共有するために使用され、.zipはSteam にアップロードするためにのみ使用されます（特にSteam Runtime を使用するため）。このため、作成された.zipは、通常のzipファイルとして開くことができません。
  • HTML5 - HTML5のビルドを選択した場合、 index.html ファイル（これはデフォルトの名前ですが、Platform Preferencesで独自の名前を付けることもできます）と、ゲームファイルを含むフォルダが作成され、指定された場所に保存されます。ゲームを動作させるには、この2つがサーバーにアップロードされている必要があります。 index.html をカスタマイズして、背景色を変えたり、位置を変えたりすることもできます。詳しくは、YoYo Gamesヘルプセンターでご確認ください。
  • Android - Android デバイスの場合、ゲームを保存するために表示されるウィンドウから、 *.apk または *.aab (Android App Bundle) ファイルを構築するよう選択することができます。

    選択するファイルの種類は、対象とするストアによって異なり、Google Playでは *.aab 、その他のストアでは *.apk ファイルを使用する必要があります。詳しくは、YoYo Gamesヘルプセンターでご確認ください。
  • iOS - iOS にコンパイルすると xarchive ファイルが作成され、Xtools で最終的なiOS パッケージを作成するために使用されます。 iOS のコンパイルには、OSX以上のApple Macコンピュータと、関連する証明書と権限が必要であることに注意してください。詳しくは、YoYo Gamesヘルプセンターでご確認ください。
  • macOS - macOS ターゲット用にコンパイルすると、最初にDMGか Zip ファイルのどちらを作成するか選択するよう要求されます。
    「Package as DMG "を選択すると、Macコンピュータにゲームをインストールするために使用できる.dmgインストーラパッケージがビルドされます。このビルドプロセスの間、Mac上にインストーラが表示されますが、ビルドプロセスが完了すると消えますので注意してください。

    "Package as Zip" は、後で Mac App Store にアップロードするかどうかによって、 *.app ファイルか *.pkg ファイルをビルドします。iOS と同様に、OSX 以上を搭載した Apple Mac コンピュータと、関連する証明書および権限が必要です。詳しくはYoYo Gamesヘルプセンターでご確認ください。
  • Windows UWP-Windows UWP の場合、GameMakerは *.appxbundle パッケージを作成し、Windows Store にアップロードするか、他の場所に配布することができます。Create Executable ボタンをクリックすると、GameMakerに作成したいパッケージの種類（ARM 、x64、x86アーキテクチャ用）を伝えるプロンプトが表示されるので、ゲームがサポートされるターゲットを選択します（最低1つは選択しなければなりませんが、3つすべて、または任意の組み合わせを選択することができます）。

    Package For Store Upload のボックスをチェックすると、最終的に作成されるパッケージは *.appxupload ファイルになります。これは、こちらの記事で説明されているように、Microsoft がストアにアプリを提出する際に使用するよう指定しているものです (*.appxbundle の場合と同様に、選択したすべてのアーキテクチャが含まれます)。UWPプラットフォームへのセットアップとコンパイルについては、YoYo Gamesヘルプセンターで詳しく説明されています。

実行可能なasset パッケージを作成したら、ファイルを他の人に渡したり、ウェブサイトに置いてダウンロードしたり、これらのファイルをさまざまなホスティングサービスにアップロードして個別に配布したり、あるいはオンラインストア（Google Play、Apple App Store、Microsoft Store など）に置いて一般に配布・販売したりすることができます。

なお、GameMakerで作成したゲームは、販売も含めてどのように配布してもかまいません。もちろん、作成に使用したsprites 、画像、サウンドも同様に配布・販売可能であり、すべてのassets について法的権利を有していることが前提です。また、そのゲームがYoYo Games のGameMaker 用 EULA に準拠していることが前提です。
