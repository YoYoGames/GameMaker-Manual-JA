出力ウィンドウ

GameMakerで初めてプロジェクトを開くと、画面の下にOutput Windowが ドッキングして表示されます。このドックされたウィンドウには、特定の状況に応じて、プロジェクトのさまざまな出力情報を表示するタブがあります。ドックはIDE の下部にあるボタンをクリックすることで閉じることができます。また、 をクリックしてドッキングされたウィンドウのタブを別のドックにドラッグして位置を変更したり、ドックのコンテンツバーで をクリックしてworkspace にドラッグしてスタンドアロンのウィンドウを作成したりすることもできます。

また、 をクリックし、ドッキングされた出力を別の出力にドラッグすると、2つのエラー出力が同じタブ内に配置された下の例のように、分割表示出力ウィンドウを作成することができます（タブ上で をスロークリックして、その名前も変更することができます）。

レイアウトウィンドウから「レイアウトのリセット」を選択すれば、いつでもIDE のデフォルトの状態に戻すことができます。また、閉じたタブはウィンドウメニューから再度開くことができます。また、「出力ウィンドウの環境設定」を編集することで、各出力ウィンドウの内容の表示方法を変更することができます。

デフォルトのドッキングタブの説明は以下の通りです。

出力

一般出力 ウィンドウには、すべてのコンパイラ出力が表示されます。 を使用してゲームに追加したメッセージも表示されます。 show_debug_message()関数を使用してゲームに追加したメッセージも表示されます。初期に表示される情報のほとんどは、ゲームがどのように構築されているかというデバッグ情報であり、通常は無視できます。 そのため、通常は無視することができます。しかし、ターゲットプラットフォーム用にプロジェクトをビルドする際に問題が発生した場合、原因を突き止めるのに役立つだけでなく、サポートに連絡する際の情報にもなります。ここに表示される情報の量は、コンポジットから変更することができます。 ここに表示される情報の量は、環境設定のコンパイルセクションで変更できます。

検索結果

キーボードショートカット / + + "F" を使って「検索と置換」ウィンドウを開くか、「編集」メニューから開くことができます。検索語を入力すると、その結果がこのウィンドウに書式付きで表示されます。

[object] - [event] - [Line Number]: [search string]

検索語がscript で見つかった場合は、単純にそうなります。

[script] - [Line Number]: [search string]

次に、返されたエントリーのどれかをダブルクリックすると、与えられたasset が編集のために正しい位置で開かれます。

このウィンドウは、Featherの「全参照文献の検索」機能の結果も表示されます。

ソース管理

このウィンドウには、SCMプラグインに関するすべての出力が表示されます。詳しくは Source Control の設定について、詳しくはこちらをご覧ください。

ブレークポイント

ブレークポイントとは、ゲームコードやGML Visualの中で、GameMakerにプロジェクトの実行を一時停止するように指示した場所のことです。 デバッグモードでプロジェクトの実行を一時停止するように GameMaker に指示した場所です。ゲームbreakpoint の任意の場所に "F9" キーでloop を追加することができ、追加するとこの出力タブに表示されます。のチェックボックスをクリックすることで、有効/無効を切り替えることができます（削除はできません）。 マウスの右ボタン（ ）を使って、breakpoint を削除したり、 があるコード/GML Visual ウィンドウを開くことができる小さなメニューが表示されます。 breakpoint ウィンドウを開くことができます。

シンタックスエラー

シンタックスエラーとは、GameMakerの シンタックスチェッカーが検出したコード上のエラーのことです。 が検出したコードのエラーです。シンタックスエラーが発生する理由は様々で、例えば、": "が必要なのに"; "が入っているなど、ゲームのテストやコンパイルをする前に修正する必要があります。出力ウィンドウの各シンタックスエラーの項目は をダブルクリックすると、フラグが立った位置のウィンドウが開き、解決することができます。

構文エラーは、コードを書くと更新され（未完成のコードに対してエラーが報告されるのを防ぐため、何かをタイプしてからこのウィンドウにエラーが表示されるまでに短い間があります）、その形式に従って表示されます。

[object] - [event] - [Line Number] - [Position In Line]: [error string]

あるいは、エラーがscript にある場合、そうなります。

[script] - [Line Number] - [Position in line]: [error string]

シンタックスエラーは通常、ゲームのコンパイルを妨げますが、コンパイルを妨げないものの、解決すべきエラーもフラグされることに注意してください。そのエラーとは

  • どこにも使われていない宣言された変数に対して
  • まだ宣言されていない、どこかで使われている変数について

一般的には、変数名のタイプミスや、その他の間違いが起こりやすいので、修正するようにフラグが立てられますが、意図的な場合もあります。 しかし、意図的な場合もあります。将来のために変数を宣言しておきながら、実際にはまだ使っていない場合もあるからです。このため、このようなエラーがあっても、ゲームはコンパイルできます。考えられる構文エラーメッセージの完全なリストについては、こちらを参照してください。

コンパイルエラー

Compiler Errorは、シンタックスチェッカーが検出できない何らかのエラーが発生した場合、またはコンパイルオプションの設定に関連したエラーが発生した場合に起こります。このような場合、コンパイラーエラーが発生し、ゲームの実行が停止します。コンパイラーエラーは一般出力ウィンドウにも表示されますが、ここに別途表示されます（他の一般出力の中に紛れ込んでしまう可能性があるため）。

コンパイラのエラーメッセージは、すべて同じフォーマットで表示されます。

[object] - [event] - [Line Number]: [error string]

もし、script でエラーが見つかった場合は、単純にそうなります。

[script] - [Line Number]: [error string]

コンパイラーエラーエントリーのいずれかをダブルクリックすると、エラーが発生したとフラグが立った位置で、指定されたasset を開くことができます。考えられるエラーメッセージの完全なリストは、ここを参照してください。

フェザーメッセージ

Feather Messages ウィンドウには、Feather から報告されたエラーや警告、提案などがすべて表示されます。リスト内の任意の項目をダブルクリックすると、対応するコードウィンドウに移動することができます。

詳しくは、「フェザーメッセージ（フェザー有効）」をご覧ください。
