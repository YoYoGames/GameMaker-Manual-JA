インスタンスプレイス

この関数を使用すると、チェック用のコードを実行したインスタンスのコリジョン マスクを使用して、ある位置で他のインスタンスまたはobject のすべてのインスタンスとの衝突をチェックできます。この関数を使うと、GameMakerにインスタンスを新しい位置に移動させ、衝突をチェックし、戻って衝突があったかどうかを教えてくれるように頼むことになります。

これは正確なコリジョンに対して機能しますが、チェックされるインスタンスとobject の両方が正確なコリジョンマスクを選択している場合のみで、それ以外の場合はバウンディングボックスコリジョンのみが適用されます。

この関数は、一意なインスタンスを返します。 idのobject が衝突しますが，それが不要な場合は，この関数を使った方が若干速いです。 place_meeting().この関数は、特別なキーワード all と other も受け付け、衝突が発生しなかった場合はキーワード noone を、衝突が発生した場合は見つかったインスタンスのユニークなインスタンス ID 値を返します。

構文です。

instance_place(x, y, obj);

論証 タイプ 商品説明
x リアル The x position to check for instances.
y リアル The y position to check for instances.
obj オブジェクトのアセットまたはインスタンスID The object to check for instances of.

リターンです。

Instance ID or noone

例

var _inst = instance_place(x, y, obj_Enemy);
if (_inst != noone)
{
    hp -= _inst.dmg;
    instance_destroy(_inst);
}

上記のコードは、「obj_Enemy」のインスタンスとの衝突をチェックし、衝突があった場合は、衝突したインスタンスの「dmg」変数に格納されている量だけ「hp」変数を減らし、衝突したインスタンスを破壊します。
