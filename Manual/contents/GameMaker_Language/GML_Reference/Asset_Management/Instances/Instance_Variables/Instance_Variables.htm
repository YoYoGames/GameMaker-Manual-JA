インスタンス変数

新しいobject を作成すると、特定の変数がデフォルト値で初期化された状態で提供されます。これらの変数はインスタンス変数と呼ばれ、ゲームobject にrooms を配置するたびに作成され、一度インスタンスが作成されると、これらの変数の値はそのインスタンスに固有のものとなり、そのインスタンスだけになります。これらの変数の中には、 x や y のようにたくさん使うものもあれば、やりたいことによってはあまり必要ないものもありますが、一般的にこれらは非常に便利なので、可能であれば独自のカスタム変数を作成するよりも、これらの変数を使用したほうがよいでしょう。

インスタンス変数のほとんどは、読み込むだけでなく設定することもでき、特定の変数の値を調整するだけでインスタンスのプロパティや動作を変更することができます。たとえば、組み込み変数 visible を false に設定するだけで、インスタンスの描画を阻止できます。以下では、ゲーム内のすべてのインスタンスobjects に対して初期化されるさまざまな変数について説明します。

重要インスタンスが sequenceという場合、image_xscale /image_yscale / image_angle /x /y のように、これらの変数のいくつかは、sequence が再生開始後の各ステップを更新する際に上書きされます。

これらの変数は、一般的なインスタンスプロパティを扱う。

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

これらの変数は、インスタンスの位置と動きを扱います。

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

オブジェクトプロパティ

次の変数は、インスタンスの作成元であるobject の一意のインデックス ID を保持します。

  • object_index

これらの変数はすべてインスタンスに割り当てられたsprite に関連しており、何をどのように描くかを変更するために使用することができます。

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

これらの変数は、コリジョンマスクを扱います。

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

これらの変数は、paths 、割り当てられた場合、インスタンスがどのように相互作用するかに関連しています。

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

これらの変数は、インスタンスがタイムラインを使用するように設定するためのものです。

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

シーケンス

以下の変数は、任意の時点でインスタンスを制御している可能性があるSequence に関する情報を格納します。

  • in_sequence
  • sequence_instance
