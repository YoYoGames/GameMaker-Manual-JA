depth

object を作成するときに、初期深度を割り当てることができます。これは、ゲームをプレイしているときにobject のインスタンスがroom にどのように描画されるかを定義します。この変数は、ゲームの実行中にこの深度値を取得したり変更したりするのに使用できます。通常、インスタンスは個別のレイヤーに追加され、そのレイヤーは特定の深さに設定されるので、この変数を使用する必要はありません。しかし、この値を使用してインスタンスの深さを変更したい場合があります。同じ深さのインスタンスがない場合、この一時的なレイヤーはメモリから削除されることに注意してください（通常のレイヤーとは異なり、何もない状態でも残ります）。

重要!変数 depth を変更し、GameMakerがレイヤーを管理する場合、内蔵された変数はレイヤーIDではなく-1を返す。 layerは-1を返し、レイヤーIDを返さない。これは、管理されたレイヤーはコードで操作できないからだ。

GameMakerでは、あるインスタンスの深度値が低いほど「カメラに近い」インスタンスが描かれ、深度値が高いほど「カメラから遠い」インスタンスが描かれます。例えば、-1000は-100の上に描かれ、0はその上に描かれるというように。

object エディタでは、同じ深さの のインスタンスが、他のインスタンスの上に配置されているように見えても、互room いの上や下に描かれることがあることに注意してください。何かが他のものの上または下に描かれることを保証したい場合は、常に深さ（またはレイヤー）を明示的に設定する必要があります。また、深さの最小値 (-16000) と最大値 (+16000) があり、これを超えるとインスタンスは描画されませんが、インスタンスはまだ存在し、イベントを処理します。

注意 インスタンスのDrawイベントでインスタンスの深さを設定することはできません（他のイベントはすべて問題ありません）。

深度をFXレイヤーの深度より低く変更すると、インスタンスに適用されるフィルタとエフェクトが変更される可能性があることに留意してください。

構文です。

depth

リターンです。

Real (single precision floating point value)

例

if y != yprevious
{
    depth = -y;
}

上記のコードでは、y位置が変更されたかどうかを確認し、変更されていれば、それに対応するように深さも設定されます。
