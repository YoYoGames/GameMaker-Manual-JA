インスタンスの非活性化

GameMakerでは、インスタンスを「スイッチオフ」して、一切処理されないようにすることが可能です。技術的には、非アクティブ化処理自体のポインタとして存在する以外、実際にはもう存在しないので、非アクティブ化されたインスタンスは、再びアクティブ化されるまで、いかなる方法でも操作や変更ができないことを意味します。このため、これらの関数は適切に使用しないと問題が発生する可能性があり、特に永続的なobjects 、非アクティブ化された永続的なobject は、最初に再アクティブ化しない限り次のroom に移動しないので、十分に注意してください（事実上ゲームからそれを削除することになります）。また、アクティブ化と非アクティブ化は瞬間的なものではなく、この方法で状態が変更されたインスタンスは、関数が呼び出されたイベントが終了するまでアクティブ/非アクティブと見なされないことに注意してください。

通常、ゲームの各ステップでインスタンスを非アクティブにする必要はなく、実際にゲームのパフォーマンスの問題を引き起こす可能性があります。その代わり、アラームの数ステップごと（例えば、カメラビューの位置が変わったときなど）にこれらの関数を実行することをお勧めします。特に、描画イベントでこれらの関数を使用しないことが重要で、これはゲームに深刻なエラーを引き起こす可能性があります。

無効化されたインスタンスは事実上ゲーム内に存在しなくなりますが、個々のインスタンスはまだその変数にアクセスすることができます。ただし、 with(object) や with(instance) を使用することはできません。インスタンスに直接アクセスするには、そのインスタンスに固有の ID を以下のような形式で使用する必要があります。

val = inst.variable;

上記の例では、"inst "は非アクティブ化されたインスタンスのIDを保持する変数となり、これは以前にアクティブ化されていない非アクティブ化されたインスタンスを参照する唯一の方法である。

注意事項room の開始時にインスタンスを非アクティブにした場合（つまり、room の作成コード、またはroom 内のインスタンスの作成イベントから）、room エディタからroom 内に配置されたすべてのインスタンスは、非アクティブになる前に作成イベントをまだ実行します。

警告： 物理が有効になっているインスタンスを非アクティブにしても、そのフィクスチャが物理シミュレーションの中で相互作用することは止められません。インスタンスをアクティベート/ディアクティベートするときに、phy_active変数を true もしくは false に設定する必要があります。

インスタンスの停止には、以下の機能があります。

  • instance_activate_all
  • instance_activate_object
  • instance_activate_region
  • instance_activate_layer
  • instance_deactivate_all
  • instance_deactivate_object
  • instance_deactivate_region
  • instance_deactivate_layer
