シェーダー定数

OpenGL ES Shading Language (GLSL ES)Reference Pages で定義されているshader 関数と定数とは別に、GameMaker 独自のshader 定数も多数用意されています。

shader 配列定数gm_Matrices を使用する場合、以下のディスプレイマトリクス定数を配列のインデックスとして使用することができます。

定数 商品説明
MATRIX_VIEW この配列インデックス定数は、現在のビューマトリックスへのインデックスを保持します。返されたインデックスは、shader コード内で gm_Matrices 定数を呼び出す際に、配列の値として使用されます。
MATRIX_PROJECTION この配列インデックス定数は、現在の投影行列のインデックスを保持します。返されたインデックスは、shader コード内で gm_Matrices 定数を呼び出す際に、配列値として使用されます。
MATRIX_WORLD この配列のインデックス定数は、現在のワールドマトリクスへのインデックスを保持します。これは、ワールド空間に光の情報がある場合、照明などに使用することができます。返されたインデックスは、shader のコード内で gm_Matrices 定数を呼び出す際に、配列値として使用されます。
MATRIX_WORLD_VIEW この配列のインデックス定数は、ワールドマトリクスとビューマトリクスを掛け合わせた結果のインデックスを保持します。これは、フォグなどによく使用されます。返されたインデックスは、shader コード内で gm_Matrices 定数を呼び出す際に、配列値として使用されます。
MATRIX_WORLD_VIEW_PROJECTION この配列のインデックス定数は、ワールド、ビュー、プロジェクションの各マトリックスを掛け合わせた結果のインデックスを保持します。これは、vertex の位置に使用される通常の変換行列です。返されたインデックスは、shader コード内で gm_Matrices 定数を呼び出すときに、配列値として使用されます。
MATRIX_MAX これは，配列のインデックスではなく，vertex shader にある行列配列のサイズを返します．
MAX_VS_LIGHTS これは配列のインデックスではなく、vertex shader にある照明の数を返します。

照明用には以下の定数も用意されています。

定数 商品説明
MAX_VS_LIGHTS シェーダーで使用可能なポイントライトとディレクショナルライトの最大数

以下の定義済みマトリックスユニフォームと定数をshader で使用することで、GameMaker固有の値にアクセスすることができます。

定数 商品説明
gm_Matrices[matrix] この配列定数は変換行列を返すもので、GameMakerが shader コードエディター内で使用できるように作成する、利用可能な事前定義されたユニフォームの1つである。配列のインデックスは、上記の定数のいずれかから選択されます。
gm_BaseTexture これは2Dサンプラー定数で、GameMakerで設定された現在のtexture のobject を返します。つまり、sprite が呼び出されないで描画するときに通常使用される現在のtexture 、サーフェス、shader になります。
gm_LightingEnabled 3D使用時にGameMakerのライティングを取得・設定するのに使用します。
gm_FogStart ポリゴンがフォグ色になじみ始める距離を得るために使用できます。
gm_RcpFogRange これは、霧が最大になり、何も見えなくなる距離を求めるのに利用できる。
gm_PS_FogEnabled GPU でピクセルフォグが有効かどうか、 true または false を返します。
gm_FogColour GameMakerで使用するフォグの色を取得するために使用することができます。
gm_VS_FogEnabled これは、GPU がvertex フォグを有効にしている場合は true を、そうでない場合は false を返します。
gm_AlphaTestEnabled これは、shader でアルファテストを取得するために使用することができます。 gpu_set_alphatestenable()はアルファテストの詳細です。
gm_AlphaRefValue これは、現在のアルファテストの基準値を取得するために使用することができます。参照 gpu_set_alphatestref()は、アルファテストリファレンスの詳細についてです。
