in_sequence

これは、ゲーム内の各インスタンス（object ）に作成されるインスタンス変数の一部であるビルトイン変数です。インスタンスがsequence で制御されている場合。 この変数は true を返し、それ以外の場合は false を返します。この変数は読み取り専用で、変更することはできません。

なお、この変数は、制御しているSequence が終了すると、そのSequence 要素がまだ存在していてもfalse になり、そのSequence 要素が再び再生されると true になる。

この変数をプレイヤー（または CPU）が制御するobjects で使用すると、Sequence にいないときだけコードで移動するようにできます。これはカットシーンで、同じインスタンスをSequence で使用する必要があるときに便利です（使用方法は sequence_instance_override_object() ）、そのカットシーンがアクティブな間は、通常のコードで移動できないようにする必要があります。

構文です。

in_sequence

リターンです。

例

if (!in_sequence)
{
    x += move_x;
    y += move_y;
}

上記のコードでは、変数 in_sequence をチェックし、それが false であれば (インスタンスがsequence で制御されていないことを意味します)、インスタンスの move_x と move_y を追加しています。 の中にいないときだけ動くようにする。Sequence 。
