レイヤークリエイト

この関数は、現在のroom 内に新しいレイヤーを作成するために使用されます。新しいレイヤーが占める深さを指定すると、この関数は一意なレイヤー ID 値を返し、この値を用いて以降の関数でそのレイヤーにアクセスすることができます。また、オプションの「name」引数を与えることで、レイヤーに特定の名前を付けることができます（string ）。レイヤーにアクセスするためにレイヤーIDを使う代わりに、この名前string 。ただし、名前strings を使うと、ID値だけを使うよりも性能への影響が大きいことに注意しなければいけません。レイヤーの名前を指定しない場合、作成されたレイヤーには "_layer_XXX" という形式の名前が割り当てられます。"XXX" はレイヤーにユニークな名前を付けるために使用する16進数値です。

深度値を指定する場合、カメラから「遠い」レイヤーほど高く、「近い」レイヤーほど低くなります。深度 -100, 0, 100 の 3 つのレイヤーがある場合、レイヤーは 100, 0, -100 の順番で描画され、「トップ」レイヤー（つまりカメラビューに最も近く、他のすべてに重ねて描画）は、深度 -100 のレイヤーになります。

次の画像は、レイヤーの深さがどのように機能するかを示す概略図です。

なお、1つの深度に複数のレイヤーを割り当てることはできますが、同じ深度のレイヤーがどのような順番で描画されるかは保証できませんので、任意の深度に1つのレイヤーだけを割り当てるのがベストです。また、レイヤーの最小および最大深度は-16000から16000であり、これらの深度以外のものはレンダリングされないことに注意してください。この範囲外の深さが必要な場合は、関数 layer_force_draw_depth().

構文です。

layer_create(depth, [name])

論証 タイプ 商品説明
depth リアル The depth that the layer has to be created at.
name 文字列 OPTIONAL レイヤーに付ける名前（ ）。string

リターンです。

Layer ID

例

global.Bullet_Layer = layer_create(-100);

上記のコードでは、深度値-100の新しいレイヤーを作成し、そのID値をグローバル変数に代入しています。
