layer_set_target_room

この関数を呼ぶと、それ以降のレイヤー関数はすべて与えられた部屋に適用されることをGameMakerに伝えることになります。このようにして、現在のroom 以外のroom で、レイヤーやレイヤーの内容をプロシージャルに変更したり、生成したりすることができます。レイヤーやレイヤー要素をroom に追加し終えたら、関数 layer_reset_target_room()を使用してroom ターゲットをリセットします (またはroom の引数に -1 を指定してこの関数を再度呼び出します)。この関数は、現在の部屋以外では使用できません。また、現在実行中のrooms 以外のroom にレイヤーやレイヤー要素を追加/削除できるように設計されていることに注意してください。

警告! room別のレイヤーをターゲットにしている場合、通常のレイヤー機能はすべて使用できますがinstance_create_layer()または instance_create_depth()また、レイヤー関数 layer_add_instance()を使用することができます。

構文です。

layer_set_target_room(room)

論証タイプ 商品説明
room The room to target for all further layer functions

リターンです。

例

layer_set_target_room(rm_Game);
var l = layer_get_id("SpriteAssets");
repeat(50)
{
    var _x = irandom(1000);
    var _y = irandom(1000);
    layer_sprite_create(l, _x, _y, spr_Trees);
}
layer_reset_target_room();

上記のコードでは、ターゲットroom をroom "rm_Game" に設定し、その中の "SpriteAssets" というレイヤーのレイヤー ID を取得しますroom.このレイヤー ID を使用して、レイヤーのターゲットが現在のsprite にリセットされる前に、ランダムな 50 個のassets room がレイヤーに追加されます。
