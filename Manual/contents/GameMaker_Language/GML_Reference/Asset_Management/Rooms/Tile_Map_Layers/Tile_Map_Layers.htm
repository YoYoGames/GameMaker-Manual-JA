地図レイヤーをタイル状に表示

GameMaker ルームエディタでは、レイヤーを使って tiles を任意のroom に追加することができます。その名の通り、tile マップレイヤーは、tile set 内の同じ深さにあるとみなされるtiles room を集めたもので、レイヤーを重ねることで、あるものを他のものの上に、または下に描画させることができます。また、コードからレイヤーの特定の側面を制御することができ、ものを追加または削除したり、実行時にレイヤーまたはそれが含むものの特定のプロパティを変更することができます。room エディターでは、1つのレイヤーにつき、1つのtile マップに制限されていることに注意してください ( タイルレイヤー) が、コードで作業する場合は、1つのレイヤに複数のtile マップを割り当てることができ、それぞれのマップは固有の ID とプロパティを持つので、そのようにします。

  • タイルセットは、tile レイヤーとtile マップを作成するために使用するリソースです。
  • タイルレイヤーは、Room Editor で作成し、tile sets に追加するものです。
  • タイルマップは、tiles エディタまたはコードによって、そのレイヤーの単一の要素として追加されるroom のコレクションを呼び出すものです。

tile マップを設定する上記の関数とは別に、個々のtiles のデータを操作するために使用できる関数も多数あります。tile データの「ブロブ」は、tile のインデックスと、tile が回転、反転、ミラーリングされているかどうかを示す数ビットの追加で構成されています。tile tile インデックス用に予約されているビットをマスクすることになるため、上級者向けですが、tile データに独自のビットマスクを追加することもできます。このマスクを使用して、tile のカスタム衝突を作成したり、コードでチェックすることができます。しかし、通常、独自のtile データブロブを作成する必要はなく、代わりに関数 tilemap_get()でtile のデータを取得し、それを以下の関数で操作し、再び tilemap_set().
