オーディオクリエイトプレイキュー

この関数は、buffer queue をオーディオ用に準備します。データ形式（現在サポートされているのは buffer_u8 または buffer_s16 のみ）、サンプルレート（1000hz から 48000hz まで）、オーディオが必要とするチャンネルを、以下に示す定数のいずれかから渡します。

定数 商品説明
audio_mono モノラル（1チャンネル）オーディオ。
audio_stereo ステレオ（デュアルチャンネル）オーディオ。
audio_3D 3D（5.1）オーディオ。


この関数で作成されたオーディオqueues は、関数によるメモリリークを防ぐため、不要になった時点で解放する必要があることに注意してください。 audio_free_play_queue().

この関数は、queue が終了すると、オーディオ再生非同期イベントを発生させる。このイベントでは、特別なDS マップが変数 async_loadを、以下のキーと値のペアで指定する。

  • "queue_id" - データの再生が終了したqueue
  • "buffer_id" - 再生されなくなったbuffer 。
  • "queue_shutdown" - 通常の再生時には0に設定され、イベントを受信したときに1が設定されるのは、以下の理由からです。 audio_free_play_queue()が呼び出されました。これが1に設定されているときは、queue 、それ以上のデータをアップしないようにします。

構文です。

audio_create_play_queue(queueFormat, queueRate, queueChannels);

論証 タイプ 商品説明
queueFormat バッファデータ型定数 The format of the buffer data to use (buffer_u8 または )。 buffer_s16
queueRate リアル The sample rate of the data in the buffer queue.
queueChannels オーディオ・チャンネル・タイプ The channels to use from one of the constants listed above.

リターンです。

Audio Queue ID

例

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

上記のコードでは、新しいオーディオqueue を作成し、その ID を今後の参照用に変数に割り当てています。
