オーディオフリープレイキュー

この関数は、与えられたオーディオqueue に関連するメモリーを解放するために使用される。queue インデックスは，この関数を使ってqueue を作成したときに返された値です。 audio_create_play_queue()メモリリークを防ぐため、この関数はqueue が不要になったときに呼び出す必要があります。queue を解放すると、再生中のサウンドはすべて停止します。また、サウンドがストリーミングされているbuffer は、それが割り当てられているqueue が解放されるまで削除することができません。この関数は、Audio Playback Asynchronous Event をトリガーし、このイベントで特別なDS マップを変数 async_loadを、以下のキーと値のペアで作成する。

  • "queue_id" - データの再生が終了したqueue
  • "buffer_id" - 再生されなくなったbuffer 。
  • "queue_shutdown" - 通常の再生時には0に、イベント受信時には1に設定されます。これは、この関数でqueue を解放したためです。これが1にセットされているときは、queue 、それ以上データをアップしないようにします。

構文です。

audio_free_play_queue(queueIndex);

論証 タイプ 商品説明
queueIndex オーディオQueue ID The index of the queue to free.

リターンです。

N/A

例

var q_id = async_load[? "queue_id"];
if (q_id == audio_queue)
{
    audio_free_play_queue(audio_queue);
    buffer_delete(audio_buffer);
}

上記のコードは、非同期オーディオ再生イベントに入り、再生が終了したオーディオが「audio_queue」のものであるかどうかをチェックし、もしそうならqueue とそれに関連するbuffer を解放します。
