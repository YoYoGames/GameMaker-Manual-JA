audio_get_recorder_info

この関数は、与えられたレコーダーソースのインデックスに関する情報を持つDSマップを返す。利用可能なレコーダーソースの数は、関数 audio_get_recorder_count()また、返されるマップは以下のキーと値のペアを含んでいます。

  • "name" - デバイスを説明するための名前
  • 「index" - 記録に使用するインデックス。
  • "data_format" - データが返される形式 (現在は常に buffer_s16 ですが、将来は他の形式もサポートされるかもしれません)
  • "sample_rate" - 返されるデータのサンプルレート（単位：Hz）（現在は16000hzに固定されていますが、将来的には変更される可能性があります）。
  • "channels" - 定数 audio_mono (ステレオおよび3D用のさらなる定数が将来的にサポートされる可能性があります)

この関数は DS マップを作成してくれますが、後でそれを再び削除することはないので、メモリリークを防ぐために、不要になったら自分でマップを破棄しなければならないことに注意してください。

注意 ほとんどのプラットフォームは何らかの形で音声の録音をサポートしていますが、プラットフォームが許可している場合でも、すべてのデバイスが許可しているわけではありませんので、他の録音関数を使う前に必ず audio_get_recorder_count()関数が 0 より大きな値を返すことを常に確認し、残りの録音関数を使用する前に録音デバイスが使用可能であることを検証する必要があります。

構文です。

audio_get_recorder_info(recorder_index);

論証 タイプ 商品説明
recorder_index リアル The index of the recorder source to get the information from.

リターンです。

DS Map ID

例

len = async_load[? "data_len"];
audio_buff = buffer_create(len, buffer_fast, 1);
buffer_copy(async_load[? "buffer_id"], 0, len, buff, 0);
audio_queue_sound(audio_queue, audio_buff, 0, len);
audio_play_sound(audio_queue, 10, 0);

上記のコードは、非同期オーディオ録音イベントで呼び出され、録音されたデータをbuffer に割り当て、それをオーディオqueue に追加します。そして、これが再生されます。
