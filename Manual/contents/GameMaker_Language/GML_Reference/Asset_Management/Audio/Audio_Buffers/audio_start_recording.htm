オーディオスタート・レコーディング

この機能は、インデックスされたレコーダーソースから音声の録音を開始します。レコーダーソースの数は関数 audio_get_recorder_count()録音を開始すると、音声は一時的にbuffer に保存され、Audio Recording Asynchronous Event のトリガーが開始されます。このイベントは、録音が行われるたびにトリガーされ、以下のキーと値のペアを持つ特別なDSマップを変数 async_loadに以下のキーと値のペアを持つ特別なDSマップを作成します。

  • "buffer_id" - 音声データの取得に使用する、一時的なbuffer の ID。
  • "channel_index" - このデータが記録されたチャンネルのインデックス（この関数によって返されます）です。
  • "data_len" - 受信したデータの長さ(バイト)

非同期イベントが発生した後、一時的なbuffer のオーディオは消去されますので、事前に作成したカスタムbuffer にデータをコピーする必要があることに注意してください。

注意 ほとんどのプラットフォームは何らかの形で音声の録音をサポートしていますが、プラットフォームが許可している場合でも、すべてのデバイスが許可しているわけではありませんので、他の録音関数を使う前に必ず audio_get_recorder_count()関数が 0 より大きな値を返すことを常に確認し、残りの録音関数を使用する前に録音デバイスが使用可能であることを検証する必要があります。

構文です。

audio_start_recording(recorder_index);

論証 タイプ 商品説明
recorder_index リアル The index of the recorder source to use.

リターンです。

Buffer ID

例

audio_record = audio_start_recording(0);

上記のコードでは、レコーダーソース0から録音を開始し、録音したチャンネルのインデックスを変数「audio_record」に格納し、非同期のオーディオ録音イベントで使用します。
