audio_play_sound

この機能を使えば、ゲーム内のあらゆるサウンドリソースを再生することができま す。サウンドインデックスを提供し、プライオリティを割り当てます。このプライオ リティは、再生中のサウンドの数が関数で設定された上限を超えた時に、どのようにサウン ドを処理するかを決定するのに使われます。 audio_channel_num().優先度の低いサウンドは停止され、優先度の高いサウンドが優先されます。優先度の値は任意の実数です（実際の値は任意で、0から1でも0から100でも、GameMakerは同じように優先順位を付けます）。優先順位を扱う場合、数字が大きいほど優先順位が高くなり、優先順位100のサウンドが優先順位1のサウンドよりも優先されることに注意してください。第3引数は、サウンドをloop 。これを true に設定すると、停止するまでサウンドをloop 。これを false に設定すると、サウンドを1回だけ再生します。

この関数は、再生中のサウンドのユニークなインデックス番号を返し、それを変数に格納することで、適切な関数でサウンドを一時停止したり停止したりすることができるようになります。つまり、同じサウンドを複数回再生している場合、オーディオ関数を使用することで、そのサウンドの1つのインスタンスだけを対象とすることができます。

構文です。

audio_play_sound(index, priority, loop, [gain], [offset], [pitch], [listener_mask]);

論証 タイプ 商品説明
index サウンドアセット The index of the sound to play.
priority リアル Set the channel priority for the sound.
loop ブーリアン Sets the sound to loop or not.
gain リアル OPTIONAL 利得を得るための価値。
offset リアル OPTIONAL 開始点を設定する時間（秒単位）。与えられた音の長さより長い値は無視される。
pitch リアル OPTIONAL ピッチ乗算器（初期値1）。
listener_mask リアル OPTIONAL サウンドに設定するビットマスクデータを指定します。HTML5 ターゲットでは、複数のリスナーをサポートしないため、この値は影響しません。

リターンです。

Sound Instance ID

例1．

if health <= 0
{
    lives -= 1;
    audio_play_sound(snd_PlayerDead, 10, false);
}

上記のコードは、グローバル変数 "health "をチェックし、0以下であればグローバル変数 "lives "から1を取り除き、サウンドを再生します。

例2．

if bbox_left > room_width
{
    audio_play_sound(snd_Goodbye, 10, false, 1.1, 0, 2);
}

上記のコードでは、インスタンスがroom の右端を越えているかどうかをチェックし、ゲイン1.1、ピッチ2（通常のピッチの2倍）のサウンドを再生しています。
