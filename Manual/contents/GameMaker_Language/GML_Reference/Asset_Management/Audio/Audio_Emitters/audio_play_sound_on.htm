オーディオプレイサウンドオン

この機能を使えば、任意のサウンドリソースをエミッタ経由で再生することができ、エミッタのゲイン、ポジション、ピッチ、ベロシティを変更すると、最終的に再生されるサウンドをユーザーがどのように聴くかに影響します。使用するエミッタのインデックス、再生するサウンドのインデックス、そしてサウンドをloop にするかどうかを指定します。次にプライオリティ（優先順位）を設定します。このプライオリティは、関数 audio_channel_num().優先度の低いサウンドは停止され、優先度の高いサウンドが優先されます。優先度の値は任意の実数です（実際の値は任意で、0から1でも0から100でも、GameMakerは同じように優先順位をつけます）。優先順位を扱う場合、数字が大きいほど優先順位が高く、優先順位100のサウンドは優先順位1のサウンドより優先されることに注意してください。

この関数は、再生中のサウンドのユニークなインデックス番号を返し、それを変数に格納することで、適切な関数でサウンドを一時停止したり停止したりすることができるようになります。つまり、同じサウンドを複数回再生している場合、オーディオ関数を使用することで、そのサウンドの1つのインスタンスだけを対象とすることができます。

構文です。

audio_play_sound_on(emitter, sound, loop, priority, [gain], [offset], [pitch], [listener_mask]);

論証 タイプ 商品説明
emitter オーディオエミッターID The index of the emitter to use.
sound サウンドアセット The index of the sound to use.
loop ブーリアン Flags the sound as looping or not.
priority リアル Set the channel priority for the sound.
gain リアル OPTIONAL 利得を得るための価値。
offset リアル OPTIONAL 開始点を設定する時間（秒単位）。与えられた音の長さより長い値は無視される。
pitch リアル OPTIONAL ピッチ乗算器（初期値1）。
listener_mask リアル OPTIONAL サウンドに設定するビットマスクデータを指定します。HTML5 ターゲットでは、複数のリスナーをサポートしないため、この値は影響しません。

リターンです。

Sound Instance ID

例1．

if global.SFX
{
    audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

上記のコードでは、グローバル変数 "SFX" をチェックして、 true を返したら、変数 "snd_Explode" でインデックスされたサウンドが、変数 "s_emit" でインデックスされたエミッタを通じて、低い優先度で、ループせずに再生されます。

例2．

if hit == true
{
    audio_play_sound_on(s_emit, snd_Hit, false, 1, 1.3);
}

上記のコードは、コードを実行しているインスタンスがヒットされたかどうかをチェックします。もし true 、エミッタ "s_emit "でサウンド "snd_Hit "を再生します。ゲインは1.3とやや高めに設定されています。ここで設定したゲインとエミッタのゲインが掛け合わされ、最終的にサウンドが再生されるゲインが決定されます。
