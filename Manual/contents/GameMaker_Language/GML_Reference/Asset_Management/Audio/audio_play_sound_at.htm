audio_play_sound_at

この機能により、オーディオ空間内の任意の位置にあるサウンドリソースを再生することができます。サウンドインデックスを指定し、3D空間内の位置を指定します。リスナーのデフォルトの位置は(0, 0, 0)なので、リスナーが移動しておらず、音を左から鳴らしたい場合（例えば）、x 位置を負の値に設定する必要があります（リスナー位置の設定に関する詳細は、以下を参照してください）。 audio_listener_position()).

また、フォールオフ距離（0は無音、デフォルトは100）を設定すると、リスナーの位置から遠くなるにつれて音がフェードアウトします。フェードアウトがどのように聞こえるかは、フォールオフ基準（ソースの音量が通常半分になる距離）とロールオフ係数（フォールオフ基準距離を過ぎた音にのみ影響する）に依存します。デフォルトのファクターは通常1です。異なるフォールオフ値の効果は、選択したモデルによって異なります（異なるフォールオフモデルとこれらの値の使用方法に関する完全なガイドについては、関数 audio_falloff_set_model()).

次の2つの引数は、サウンドをloop にするかどうかの設定と、サウンドに優先順位を割り当てるためのものです。このプライオリティは、再生中のサウンドの数が関数 audio_channel_num().優先度の低いサウンドは停止され、優先度の高いサウンドが優先されます。優先度の値は任意の実数です（実際の値は任意で、0から1でも0から100でもGameMakerは同じように優先順位をつけます）。優先順位を扱う場合、数字が大きいほど優先順位が高く、優先順位100のサウンドは優先順位1のサウンドより優先されることに注意してください。

この関数は、再生中のサウンドのユニークなインデックス番号を返し、それを変数に格納することで、適切な関数でサウンドを一時停止したり停止したりすることができるようになります。つまり、同じサウンドを複数回再生している場合、オーディオ関数を使用することで、そのサウンドの1つのインスタンスのみを対象とすることができます。

構文です。

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority, [gain], [offset], [pitch], [listener_mask]);

論証 タイプ 商品説明
index サウンドアセット The index of the sound to play.
x リアル The x position.
y リアル The y position.
z リアル The z position.
falloff_ref リアル The falloff reference relative to the listener (clamp).
falloff_max リアル The maximum falloff distance relative to the listener.
falloff_factor リアル The falloff factor (default 1).
loop ブーリアン Flags the sound as looping or not.
priority リアル Set the channel priority for the sound.
gain リアル OPTIONAL 利得を得るための価値。
offset リアル OPTIONAL 開始点を設定する時間（秒単位）。与えられた音の長さより長い値は無視される。
pitch リアル OPTIONAL ピッチ乗算器（初期値1）。
listener_mask リアル OPTIONAL サウンドに設定するビットマスクデータを指定します。HTML5 ターゲットでは、複数のリスナーをサポートしないため、この値は影響しません。

リターンです。

Sound Instance ID

例1．

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);
}

上記のコードは、グローバル変数 "SFX "をチェックし、 true を返した場合、変数 "snd_Waterfall "でインデックスされたサウンドが、そのroom 位置で、フォールオフ基準 100、フォールオフ距離 300、フォールオフ係数 1、低優先でループ再生されます。

例2．

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1, 1, 2);
}

上記のコードでは、グローバル変数 "SFX" をチェックして、 true を返したら、変数 "snd_Waterfall" でインデックスされたサウンドが、与えられたフォールオフ設定とゲインの変更なしに、そのroom 位置でループ再生されることになります。サウンドはすぐに2番目の位置から再生されます。
