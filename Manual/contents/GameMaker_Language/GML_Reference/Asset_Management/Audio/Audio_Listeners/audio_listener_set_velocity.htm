オーディオリスナーセットベロシティ

この関数は、与えられたリスナーにドップラー 効果を与え、与えられた相対的な x、y、z 位置から分解されるベクトルに基づいて、オーディオの動きをシミュレートするために使用することができます。デフォルトのリスナーインデックスは 0 ですが、関数 audio_get_listener_info()を使用して、ターゲットプラットフォームで利用可能なさまざまなインデックスを取得することができます。

しかし、たとえば、プレイヤーが下から上へ、敵が上から下へ一定の動きをするスクロール・ゲームを作る場合、リスナーとエミッタの速度（エミッタの場合は、次のように設定します）を設定する必要があります。 audio_emitter_velocity()を適切なベクトルに設定することで、プレイヤーインスタンスを通過する際の正しいドップラー効果をシミュレートすることができます。

下の画像は、上記のゲームの例でどのように設定されるかを示しています。

構文です。

audio_listener_set_velocity(index, x, y, z);

論証 タイプ 商品説明
index リアルまたはオーディオリスナーID The listener to change the velocity of (default 0).
x リアル The x velocity of the listener (default 0).
y リアル The y velocity of the listener (default 0).
z リアル The z velocity of the listener (default 0).

リターンです。

N/A

例

var num = audio_get_listener_count();
for(var i = 0; i < num; ++i;)
{
    var info = audio_get_listener_info(i);
    var data = audio_listener_get_data(info[? "index"]);
    if data[? "vx"] != 0
    {
        audio_listener_set_velocity(info[? "index"], 0, 0, 0);
    }
    ds_map_destroy(info);
    ds_map_destroy(data);
}

上記のコードは、利用可能なリスナーの数をチェックし、loops 、もし彼らのX速度が0でなければ、彼らの速度値を0、0、0にセットします。
