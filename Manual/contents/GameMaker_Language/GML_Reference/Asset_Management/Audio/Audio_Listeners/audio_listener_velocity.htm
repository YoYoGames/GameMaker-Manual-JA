オーディオリスナーベロシティ

この関数は、リスナーにドップラー 効果を与え、与えられた相対的な x、y、z 位置から分解されるベクトルに基づいてオーディオの動きをシミュレートするために使用することができます。リスナー自体が動かない場合や、動きが一定でない場合は、通常これらの値を設定する必要はありませんが、例えば、プレイヤーが下から上に一定の動きをし、敵が上から下に一定の動きをするスクロールゲームを作っている場合、リスナーとエミッターの速度（エミッターには audio_emitter_velocity() ）を適切なベクトルに設定し、プレイヤーインスタンスを越えて動くときに正しいドップラー効果をシミュレートすることができます。

注意 複数のリスナーがいる場合は、関数 audio_listener_set_velocity().

下の画像は、上記のゲームの例でどのように設定されるかを示しています。

構文です。

audio_listener_velocity(vx, vy, vz);

論証 タイプ 商品説明
vx リアル The x velocity component of the listener (default 0).
vy リアル The y velocity component of the listener (default 0).
vz リアル The z velocity component of the listener (default 0).

リターンです。

N/A

例

if speed > 0
{
    audio_listener_velocity(abs(hspeed), abs(vspeed), 0);
}

上記のコードは、プレイヤーインスタンスの速度が0を超えているかどうかをチェックし、0であれば適切な絶対値 hspeed と vspeed 、リスナーの速度を設定するコンポーネントを使用します。
