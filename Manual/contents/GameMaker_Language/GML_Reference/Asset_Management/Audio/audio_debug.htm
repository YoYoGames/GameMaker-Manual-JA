オーディオデバッグ

switch この機能は、オーディオシステムのデバッグ情報を表示するために使用することができ、 true でオン、 false でオフとすることができます。この方法でdebugging オーディオの場合、オーバーレイには次の情報が表示されます。

  • Handle - OpenAL のソースハンドル（再生される音ごとにユニークです）．
  • Data Handle- OpenALbuffer のハンドル（同じソースハンドルを持つ複数のサウンドに同じ値を表示します。）
  • Processed Buffers- 再生されたバッファの数buffers
  • Queued Buffers-buffers がキューに入った数（再生待ちの数）。
  • Gain - オーディオのゲイン値（0～65536）．
  • 再生オフセット- 現在のバッファ内のサンプル位置（現在の再生位置）です。
  • QueuedBuffers Bars- 処理された/キューに入れられたbuffers の小さなボックス図で、再生オフセット（現在の位置）が青い線で描かれています。

オーディオの同期グループについては、次のような機能もあります。 audio_sync_group_debug().

注意事項 この機能は、HTML5ターゲットプラットフォームでは動作しません。

構文です。

audio_debug(enable);

論証タイプ 商品説明
enable Enable (set to true) または無効化 (false) オーディオdebugging.

リターンです。

例

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

上記のコードは、ゲームがデバッグモードで実行されているかどうかに応じて、switch オーディオデバッグオーバーレイをオンまたはオフにします。
