フォントテクスチャーのページサイズ

texture この組み込み変数を用いて，関数 font_add().その関数を使ってfont を追加すると、GameMakerはこの変数で定義されたサイズまで、必要なグリフのtexture ページcache を作成する。その流れは以下の通り。

  • 使用する各文字は、font からtexture ページにキャッシュされます。
  • texture のページがいっぱいになると (大きなキャラクターやたくさんの種類のキャラクターを使うため)GameMakerはcache からキャラクターを削除し、必要な新しいキャラクターと置き換えます。
  • GameMakerはtexture を大きくしすぎないようにしています。これは、すべてのデバイスが大きなtextures に対応できるわけではないからです。また、texture を小さくしすぎないようにしています。これはtexture への文字の削除と再追加に CPU 時間がかかり、ゲームの実行速度が遅くなるからです。

texture もし、GameMakerが使うデフォルトの1024pxより大きい、または小さいページサイズが必要な場合は、この組み込み変数を使って値を設定することができる。 font_add() 機能を使っていない場合、この設定はゲームのパフォーマンスに影響を与えない。fonts で追加されたIDE は通常のtexture ページにキャッシュされるからである（これはFont EditorとターゲットプラットフォームのGame Optionsから制御できる）。

構文です。

font_texture_page_size;

リターンです。

Real (maximum width/height in pixels)

例

font_texture_page_size = 512;
newfont = font_add("Arial", 24, true, true, 32, 128);

上記のコードでは、font cache texture のページを最大サイズ512pxに設定し、新たにfont を追加しています。
