font_add_sprite_ext

この機能を使うと、「スプライトストリップ」（sprite 自体はsprite ブラウザのasset Asset か、 sprite_add を使ってゲームリソースに追加したsprite である必要があります）を使って、新しいfont asset を作成することができます。通常の font_add_sprite()この関数は、sprite のサブイメージに特定の順序を持たせていますが、この関数は、関数の引数 "string_map" に基づいて、sprite のサブイメージをマッピングしています。この引数はstring で、sprite font のサブイメージの順番をGameMakerに伝えると、テキストを書くときに自動的にマッピングしてくれます。例えば、"AaBbCcDdEeFfGgHh "という文字列マップがある場合、sprite font のサブイメージはこの順番で並んでいなければなりません。スペース」を任意の文字、例えばスペースにしたい大きさの一行として定義すると、テキスト内のすべてのスペースがその幅でレンダリングされます（ただし、選択された画像は決してレンダリングされません）。スペースにsprite を指定しない場合は、代わりにsprite font で最も幅の広い文字の幅が使用されます。

また、font の間隔をプロポーショナル (true) にするかしないか (false) に設定できます。プロポーショナルfont は実際の文字の幅に基づいて間隔が決められ (従って、例えば「i」の文字は「w」の文字よりもroom が小さくなります)、非プロポーショナルfont は等幅font のようにサブイメージ幅に基づいて間隔が決められま す。最後に、書くときに各文字の間に空けるスペースを定義できます。これは任意の整数値で、0はスペースなし（プロポーショナルなら文字が「接触」する）です。

この関数はインデックス値を返す。この値は，今後このfont を参照するすべてのコードで必要となるため，変数に格納しておく必要がある。また，何らかの理由で関数が失敗した場合は-1を返す。

注目すべきは、font のアライメント関数が機能すること（例えば draw_set_halign()) その場合、font sprite の原点は左上隅に設定されている必要があります。他の値を使用する場合は、font を使用してテキストを描画する際に、原点オフセットを考慮する必要があります。

注意事項 GameMakerに font を読み込んだら、忘れずに再度削除する必要があります。 font_delete()さもないと、メモリリークを起こし、ゲームが遅くなり、最終的にはクラッシュしてしまいます。

構文です。

font_add_sprite_ext(spr, string_map, prop, sep);

論証 タイプ 商品説明
spr スプライトアセット The sprite to add a font based on.
string_map 文字列 String from which sprite sub-image order is taken.
prop ブーリアン Set as proportional font or not.
sep リアル The space to leave between each letter.

リターンです。

Font Asset

例

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

上記のコードでは、変数 "spr_CalcFont" にインデックスされたfont から新しいasset sprite を作成し、変数 "global.Font" に新しいfont のインデックスを格納します。sprite font のサブイメージは、指定されたstring にマッピングされます。
