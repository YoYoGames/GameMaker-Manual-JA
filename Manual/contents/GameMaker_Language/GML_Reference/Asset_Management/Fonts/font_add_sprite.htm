font_add_sprite

この関数を使うと、「スプライトストリップ」を使って、新しいfont asset を作ることができます。ここで、それぞれのサブイメージは、個々の記号や文字になります。インデックスは、使用する最初の文字のUTF8マップ番号で、使用する範囲は、通常のfont 、"!"で始まるものと同じですが、必要なものだけを含めるべきで、そのためには ord()関数を使って、開始文字に対応する正しい UTF8 値を取得します。作成されたsprite font にはfont にある個々のグリフがサブ画像として設定され、「最初の」グリフの image_index は 0 になります。「スペース」を任意の文字、たとえばスペースにしたいサイズの 1 行として定義すると、テキスト内のすべてのスペースがその幅でレンダリングされます (ただし選択した画像は決してレンダリングされません) あるいはsprite でスペースを指定していない場合はsprite font で一番幅の広い文字が代わりに使用されます。

また、font の間隔をプロポーショナル (true) にするかしないか (false) に設定できます。プロポーショナルfont は実際の文字の幅に基づいて間隔が決められ (従って、例えば「i」の文字は「w」の文字よりもroom が小さくなります)、非プロポーショナルfont は等幅font のようにサブイメージ幅に基づいて間隔が決められま す。最後に、書くときに各文字の間に空けるスペースを定義できます。これは任意の整数値で、0はスペースなし（プロポーショナルなら文字が「接触」する）です。

この関数はインデックス値を返す。この値は，今後このfont を参照するすべてのコードで必要となるため，変数に格納しておく必要がある。また，何らかの理由で関数が失敗した場合は-1を返す。

例えば、'.'と'0'は使うが'/'は使わない場合、'/'を含めるか、使用する2文字の間に空白のサブイメージを残す必要があることに注意する必要があります。

sprite 自体は、sprite ブラウザからasset Asset を読み込むか、最初に sprite_add を使ってファイルから読み込み、読み込んだsprite のインデックスを取得します（これが関数内で使用されます）。font アライメント関数（例えば draw_set_halign()) が期待通りに動作するには、sprite font を使用する場合、sprite の原点を左上隅に設定する必要があります。他の値を使用する場合は、font を使用してテキストを描画するときに、原点オフセットを考慮する必要があります。

注意事項 GameMakerに font を読み込んだら、忘れずに再度削除する必要があります。 font_delete()さもないと、メモリリークを起こし、ゲームが遅くなり、最終的にはクラッシュしてしまいます。

構文です。

font_add_sprite(spr, first, prop, sep);

論証 タイプ 商品説明
spr スプライトアセット The sprite to add a font based on.
first リアル The index of the first available character.
prop ブーリアン Set as proportional font or not.
sep リアル The space to leave between each letter.

リターンです。

Font Asset

例

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

上記のコードは、font asset から新しいsprite asset を作成し、後で使用するために新しいfont のインデックスを変数 "global.Font" に格納します。
