イメージブレンド

この変数はインスタンスsprite の「色合い」を制御し、デフォルト値は -1 (ただし c_white も可能) です。その他の値 ( c_red, c_aqua のような内部色定数を含む) は、指定された色と元のsprite をブレンドします。この変数の変更が見えるようにするには、インスタンスが draw イベントを持たないか（つまりGameMakerがデフォルトでsprite を描画する）、または次のような拡張描画関数のいずれかを使用して描画される必要があることに留意してください。 draw_self()または draw_sprite_ext().

HTML5 プラットフォームでは、（WebGL を使用していない限り）ブレンドはできるだけ制限するようにしてください。ブレンドされたsprite はそれぞれ個別にキャッシュされるため、ブレンドされたsprites が多数あるとパフォーマンスに悪影響を及ぼす可能性があります（cache サイズは、関数 sprite_set_cache_size()).

構文です。

image_blend;

リターンです。

Colour Constant

例

image_blend = make_colour_hsv(255, 255, random(255));

上記のコードは、インスタンスに定義されたsprite の image_blend をランダムな色に設定するものです。
