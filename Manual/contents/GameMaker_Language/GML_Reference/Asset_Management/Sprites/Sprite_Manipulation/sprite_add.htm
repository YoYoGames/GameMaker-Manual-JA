この関数では、画像をsprite として追加し、外部ソースから読み込みます。読み込む画像ファイルは、常に *.png 、 *.gif 、 *.jpg/jpeg 、 *.json のいずれかの形式でなければなりません（*json ファイルは、Spine で作成したスケルトンアニメーションスプライトの読み込みに使用されます）。この関数は，新しいsprite のインデックスを返します．このインデックスは，今後sprite に関連するすべてのコードで使用されなければなりません． この関数を HTML5 で使用する場合，あるいは URL から画像を取得する場合，この関数はImage Loaded非同期イベントも発生させます．詳しくはこちらのページをご覧ください。

sprites アニメーション化される画像は、 *.gif （下記参照）を除き、すべて「帯」の形式でなければならず、スプライト幅＝帯幅／サブ画像という規則に従って、指定された数のサブ画像に分割されます。

上の画像でわかるように、sprite は濃い紫色の背景に配置されています。これを取り除くには、引数 "removeback" を true に設定します。これはsprite の左下のピクセルにある色をチェックし、それを取り除く色として使用することで動作します。たとえば、上の画像では、左下のピクセルの色を緑にすると、sprite のすべての緑の部分が削除され、残りの紫色の背景は無視されることになります。

removeback」を選択した場合、GameMakerの「smooth」引数に true を指定し、sprite のエッジを滑らかにすることもできます。これは、背景を取り除いた後のsprite の端に、半透明の縁取りを作るだけです。

最後に、sprite の x と y の原点を指定することもできます。これは、sprite を使用するインスタンスに「固定」される点で、常にsprite のサブイメージの左上隅 0,0 からの相対値として計算されます。したがって、たとえば、これらの値を (16,16) に設定した 32 x 32 ピクセルのsprite は、その原点を中央に持つことになる。

*.gif 形式の画像を追加する場合、ファイル名と画像番号（最初のフレームは1であるべき）を指定するだけで、他の引数はすべて0のままでよく、GIFの最初の画像だけが読み込まれることに注意する必要があります。 *.json sprites の場合、画像番号は 1 であるべきで、"removeback" と "smooth" の設定は無視されます。

デフォルトでは、すべての新しいsprites のバウンディングボックスは自動的に計算されます（正確なバウンディングボックスはsprite のサイズと透明度に依存します）が、これをカスタマイズしたい場合は、関数 sprite_collision_mask().

この方法でsprite を追加する場合、使用するメモリが予想以上に大きくなることに注意する必要があります。これは、GameMakerがsprite をtexture のページとして保存し、さらに GPU メモリにも保存するため、同じsprite の画像ファイルよりも総メモリ量が大きくなるためです。

注意：sprite をGameMakerに読み込んだら、忘れずに再度削除する必要があります( sprite_delete()さもないと、メモリーリークが起こり、ゲームのスピードが落ち、最終的にはクラッシュしてしまいます。

注：iOSでは、インクルードファイルをフォルダから読み込む場合、つまり、path のような "Backgrounds/background1.pngpath 「ファイル名を指定するだけで十分です。他のすべてのプラットフォームでは、フォルダをpath の一部としてファイルに含める必要があります。

注：HTML5では、安全なsprite からserver を読み込む場合、cross-origin type を設定し、絶対的なpath の代わりに相対的なpath を使用する必要がある場合があります。参照 http_set_request_crossorigin()をご覧ください。

構文です。

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

論証 タイプ 商品説明
fname 文字列 The name (a string file path) of the file to add.
imgnum リアル Use to indicate the number of sub-images (1 for a single image or for a *.gif).
removeback ブーリアン Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth ブーリアン Indicates whether to smooth the edges if transparent.
xorig リアル Indicate the x position of the origin in the sprite.
yorig リアル Indicate the y position of the origin in the sprite.

リターンです。

Sprite Asset

例

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

上記のコードでは、sprite をゲームにロードし、そのインデックスを変数 "spr" に格納しています。
