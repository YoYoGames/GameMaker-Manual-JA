表面からのスプライト作成

この関数を使うと、あらかじめ初期化されたサーフェスからsprite を作成することができます（サーフェスインデックス ID 値は、サーフェスを surface_create()).関数に入力するx,y座標は、ゲームウィンドウ（アクティブな場合はビュー）ではなく、サーフェイスの位置（左上隅）（0,0）を基準にしてください。幅と高さの引数はピクセルで、使用するサーフェスの部分の幅と高さを定義します。

引数 "removeback" を true に設定すると、sprite から色を削除します。sprite の左下のピクセルにある色をチェックし、それを削除する色として使用します。

removeback」を選択した場合、GameMakerの「smooth」引数に true を指定し、sprite のエッジを滑らかにすることもできます。これは、背景を取り除いた後のsprite の端に、半透明の縁取りを作るだけです。

最後に、sprite の x と y の原点を指定することもできます。これは、sprite を使用するインスタンスに「固定」される点で、常にsprite のサブイメージの左上隅 0,0 からの相対値として計算されます。したがって、たとえば、これらの値を (16,16) に設定した 32 x 32 ピクセルのsprite は、その原点を中央に持つことになる。

デフォルトでは、すべての新しいsprites のバウンディングボックスは自動的に計算されます（正確なバウンドボックスは、sprite のサイズと透明度に依存します）。 sprite_collision_mask().

注意事項この方法でGameMakerに sprite を作成した場合、忘れずに再度削除する必要があります( sprite_delete()さもないと、メモリーリークが起こり、ゲームが遅くなり、最終的にはクラッシュします。

構文です。

sprite_create_from_surface(index, x, y, w, h, removeback, smooth, xorig, yorig);

論証 タイプ 商品説明
index サーフェスID The index of the surface to create from.
x リアル The x position to copy from.
y リアル The y position to copy from.
w リアル The width of the area to be copied (from the x position).
h リアル The height of the area to be copied (from the y position).
removeback ブーリアン Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth ブーリアン Indicates whether to smooth the edges.
xorig リアル Indicates the x position of the origin in the sprite.
yorig リアル Indicates the y position of the origin in the sprite.

リターンズ

Sprite Asset

例

var surf;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);

上記のコードでは、サーフェイスを作成し、そのインデックスをローカル変数「surf」に格納しています。次に、そのサーフェスをターゲットにしてクリアし、sprites をいくつか重ねて描画する。最後に、サーフェスに描かれた合成画像から新しいsprite を作成し、そのインデックスを "spr_Custom" に割り当ててから、サーフェスが使用していたメモリを解放する。
