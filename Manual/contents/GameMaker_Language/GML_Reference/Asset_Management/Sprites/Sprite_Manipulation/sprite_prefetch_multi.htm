sprite_prefetch_multi

この関数は、指定されたtexture を含むtexture ページを多数プリフェッチ（sprites メモリに配置）するために使用することができます。sprite のインデックス（sprite asset を作成したときに定義したもの）を配列で指定すると、そのインデックスが含まれるtexture ページがメモリにロードされます。なお、この関数は、選択したリソースでプリフェッチがサポートされていない場合やターゲットプラットフォームが HTML5 の場合は -1 を返し、すべてが正しく動作した場合は 0 を返します。

注意: ほとんどのデバイスでtexture がtexture メモリにアップロードされるため、パフォーマンスが低下します。 sprite_prefetch_multi()が必要なグラフィックスは、ストールを回避するために、レベルの開始時に実行します。

構文です。

sprite_prefetch_multi

論証 タイプ 商品説明
array スプライトアセットの配列 Array with the sprite indices to fetch

リターンです。

Real (-1 or 0)

例

spr_a[0] = spr_Player_Aura1;
spr_a[1] = spr_Player_Aura2;
spr_a[2] = spr_Player_Aura3;
spr_a[3] = spr_Player_Aura4;
sprite_prefetch_multi(spr_a);

上記のコードでは、各要素がsprite のインデックスを保持する配列を作成しています。この配列を使って、sprite textures をメモリに配置します。
