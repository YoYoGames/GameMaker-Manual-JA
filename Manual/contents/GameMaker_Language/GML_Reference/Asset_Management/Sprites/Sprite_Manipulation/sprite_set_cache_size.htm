sprite_set_cache_size

画像のブレンドを行う場合、HTML5では実行ファイルと同じように動的に行うことができません。そのため、GameMakerはブレンドされた画像のコピーを一時的に保存し、それを読み込む必要がある。この関数は、ブレンドされた画像のコピー数を設定する。 は、古いものが上書きされる前に、与えられたsprite のキャッシュが可能です。デフォルトは 4 です。これはsprite のすべてのサブイメージに適用されます。

構文です。

sprite_set_cache_size(ind, max);

論証タイプ 商品説明
ind The index of the sprite to change the cache size of.
max The maximum number of cached copies of the sprite that can be stored.

リターンズ

例

sprite_set_cache_size(sprite0, 2);

これは、sprite0 のすべてのサブイメージのsprite cache を 2 コピーに設定します。
