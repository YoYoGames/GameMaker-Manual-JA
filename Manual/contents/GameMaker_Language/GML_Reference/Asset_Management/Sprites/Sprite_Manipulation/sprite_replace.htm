とほぼ同じ仕組みで動作します。 sprite_add()ただ、インポートするsprite のインデックスを返す代わりに、以前に作成したsprite のインデックスを上書きします。この関数を使用する場合は、sprite のインデックスを作成し、他の関数で変数に格納したものを使用する必要があります。 sprite_add()または sprite_create_from_surface()または、さらに sprite_duplicate()リソースツリーではなく、sprite asset 。この関数を使って、ゲームsprite からassets を置き換えることはできますが、そうすると game_restart() を呼び出すかどうかにかかわらず、置き換えるsprite の参照 ID を失うことになるので、推奨されません。置き換えられるsprite に関係なく、この関数はスプライトのために新しいtexture ページを作成します。そのため、レンダリングに必要なtexture のスワップ回数が増え、パフォーマンスに悪影響を及ぼす可能性があるので、使用する際には注意が必要です。

読み込む画像ファイルは常に *.png 形式でなければなりません。また、アニメーションsprites にする画像はすべて「strip」形式でなければなりません（下図を参照してください）。画像は「スプライトの幅＝ストリップの幅／サブ画像の幅」という規則に従って、指定された数のサブ画像に分割されます。

上の画像でわかるように、sprite は濃い紫色の背景に配置されています。これを取り除くには、引数 "removeback" を true に設定します。これはsprite の左下のピクセルにある色をチェックし、それを取り除く色として使用することで動作します。たとえば、上の画像では、左下のピクセルの色を緑にすると、sprite のすべての緑の部分が削除され、残りの紫色の背景は無視されることになります。

removeback」を選択した場合、GameMakerの「smooth」引数に true を指定し、sprite のエッジを滑らかにすることもできます。これは、背景を取り除いた後のsprite の端に、半透明の縁取りを作るだけです。

最後に、sprite の x と y の原点を指定することもできます。これは、sprite を使用するインスタンスに「固定」される点で、常にsprite のサブイメージの左上隅 0,0 からの相対値として計算されます。したがって、たとえば、これらの値を (16,16) に設定した 32 x 32 ピクセルのsprite は、その原点を中央に持つことになる。

デフォルトでは、すべての新しいsprites のバウンディングボックスは自動的に計算されます（正確なバウンドボックスは、sprite のサイズと透明度に依存します）。 sprite_collision_mask().

注：選択されたターゲットプラットフォームによっては、ファイルの保存先や読み込み先に制限があります。詳細は、「ファイル操作」を参照してください。

注：HTML5 ターゲットゲームでこの関数を使用して外部server からリソースを読み込む場合、ブラウザの XSS 対策により、ドメインをまたがるリソースの読み込み試行がブロックされ、空白の結果が返されるように見える可能性があることを認識しておく必要があります。

構文です。

sprite_replace(ind, fname, imgnumb, removeback, smooth, xorig, yorig);

論証 タイプ 商品説明
ind スプライトアセット The index of the sprite to permanently replace.
fname 文字列 The filename of the image to make the new sprite.
imgnumb リアル The number of frames the sprite will be cut up into horizontally. Use 1 for one single image or *.gif.
removeback ブーリアン Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth ブーリアン Indicates whether to smooth the edges.
xorig リアル The x coordinate of the origin, relative to the sprite's top left corner.
yorig リアル The y coordinate of the origin, relative to the sprite's top left corner.

N/A

例

sprite_replace(spr_banner, "gravemaker.png", 1, false, false, 0, 0);

上記のコードは、"spr_banner "にインデックスされている画像asset を、外部から読み込んだ別の画像に置き換えるものです。
