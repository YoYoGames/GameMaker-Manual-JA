スプライトマージ

この関数は、引数1のインデックス("ind2")と引数0のインデックス("ind1")を持つsprite をマージします。画像そのものがマージされるのではなく、画像のインデックスがマージされます。 sprite "ind2" のサブ画像がsprite "ind1" のサブ画像の上に追加され、つまり最後に追加される。sprites のサイズが異なる場合、追加されたsprites は、画像サイズに合うように引き伸ばされることに注意。 ind1 "の場合。

この変更は永久的なもので、この機能を使った瞬間からゲームが終了するか、sprite が削除されるまで、マージされるsprite は変更されますが、マージされるsprite はそのまま残ります。

注：マージされる画像asset とマージするための画像asset のいずれも、オリジナルのゲームリソースの1つであってはなりません。必ず sprite_duplicate()最初 を使用して2つの新しいリソースを作成し（含まれるゲームassets をマージする場合）、それらをマージします。

注：この機能はビットマップ sprites に対してのみ有効であり、SWF や JSON (Spine)sprites では動作しません。

構文です。

sprite_merge(ind1, ind2);

論証タイプ 商品説明
ind1 The index of the sprite to merge.
ind2 The index of the sprite that ind1 is to be merged with.

リターンズ

例

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

上記のコードでは、sprite をローカル変数にロードし、複製された別のsprite とマージしています。最後に、マージされたsprite がインスタンス sprite_index に代入され、ロードされたsprite は削除されます。 を使用しているメモリを解放する。この操作の後、Destroy やRoom End イベントのように、ある時点で sprite_delete()を削除するために、マージされたスプライト もメモリから削除する。
