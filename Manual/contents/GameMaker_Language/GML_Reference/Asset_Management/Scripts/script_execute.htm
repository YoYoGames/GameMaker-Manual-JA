script_execute

この機能は本来、script アセットを動的に呼び出すために設計されたものですが、ユーザー定義関数の導入により、2通りの使い方ができるようになりました。

  • 最初の方法は、レガシーな scripts をサポートすることです。ここでは、各script が1つの関数で、script のタイトルが関数の名前でもありました。この方法で使用すると、以前と同様に、ユーザーが作成したscript asset を実行し、与えられた引数をそこに渡します。この場合、実際にスクリプトを呼び出しているのではなく、script 内の同名のスクリプト関数を呼び出していることに注意してください。
  • 2つ目の方法は、複数の関数を含むscript asset から関数を呼び出す方法です。この場合、script asset の名前ではなく、関数名とその関数が必要とする引数を指定します。script script この方法はスクリプトの関数にのみ有効で、メソッドには有効でないことに注意してください。

この関数は主にレガシーサポートのために設計されましたが、ネットワークやリモートプロシージャコールの場面ではまだ使えるので、非推奨ではありません。また、この関数は低速であり、多用するとパフォーマンスに悪影響を及ぼす可能性があることも知っておく必要があります。

NOTE内蔵の機能とは併用できません。

構文です。

script_execute(scr, arg0, arg1, arg2..., etc...);

論証 タイプ 商品説明
scr スクリプトアセットまたは スクリプト機能 The function/script that you want to call
arg0, arg1, arg2..., etc... バリアブル The different arguments that you want to pass through to the function/script

リターンです。

Variable (Will depend on the return value from the script/function being called)

例

script_execute(choose(move_up, move_down, move_left, move_right), irandom(5));

上記のサンプルコードでは、 script_execute 、4つのユーザー定義関数（この場合は移動に関するもの）の中から1つを選び、その第1引数としてランダムな整数を渡しています。
