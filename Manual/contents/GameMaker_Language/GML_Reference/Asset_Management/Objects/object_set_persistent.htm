object_set_persistent

この機能により、特定のobject のパーシステンスを設定することができます。パーシステントなobject は、room からroom に「引き継がれ」、常にそのプロパティを維持します（パーシステンスの詳細については、「オブジェクトエディタ」のセクションを参照してください）。object のパーシステンスを設定すると、パーシステンスを変更した後に作成されるこのobject のインスタンスはすべてパーシステントに作成されますが、room にすでにあるインスタンスは影響を受けない場合があります。

room で永続性を有効にしたインスタンスを作成すると、レイヤーまたは深さが割り当てられます（object のインスタンスを作成するために使用した関数に依存します）。room が変更され、次のroom に、割り当てられたものと同じ名前または深さのレイヤーがない場合、rooms にわたって持続するインスタンスのために新しいレイヤーが作成されます。インスタンスの作成時にレイヤー名を付けた場合、新しいレイヤーも元のレイヤーと同じ名前になり、インスタンスに深さを割り当てた場合は、新しいレイヤーが "_layer_XXX" という名前になります。" XXX "はレイヤーに一意な名前を付けるための16進数値です。これらのレイヤーはroom からRoom End で削除されます。レイヤーと永続的なインスタンスに関するもう一つの注意点は、作成時にインスタンスに名前付きのレイヤーを割り当てた場合、次のrooms に同じ名前の別のレイヤーがあると、永続的なインスタンスはレイヤーの深さに関係なく同じ名前のレイヤーに割り当てられることです。最後に、永続化されたインスタンスが、インスタンスが作成されたのと同じ深さのレイヤーを持つroom に移動した場合、このレイヤーには割り当てられず、代わりに同じ深さに新しいレイヤーが作成されます（前に説明した命名規則に従っています）。

これはインスタンス機能ではありません。を使って、個々のインスタンスの persistentflag を設定することができます。 persistentobject flag IDEまた、この関数でflag の persistentobject を true に変更しても、room にあるインスタンスはすべてそのままで、関数を呼び出した後に作成したインスタンスだけが persistent を true にした状態で起動します。

構文です。

object_set_persistent(index, pers);

論証 タイプ 商品説明
index オブジェクト・アセット The index of the object to change.
pers ブーリアン Whether the object should be flagged as persistent (true) か、そうでないか (false) です。

リターンです。

N/A

例

if (room == rm_final)
{
    object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

上記のコードでは、インスタンスが現在どのroom にあるかを確認し、指定された "rm_final" にあれば、object "obj_Player" のパーシステンスを false に設定し、インスタンスを生成しています。
