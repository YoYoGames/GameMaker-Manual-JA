パスクリア点

この関数を使うと、path からすべてのポイント定義を削除し、効果的に「空」のpath を作成することができます。これは、path を削除するものではありません。 path_delete()しかし、この関数は実際のpath アセットを変更するため、この関数を使用した瞬間からゲーム終了まで、ゲーム内のすべてのインスタンスがpath を使用する方法に永続的に影響を与えることに注意する必要があります。これが必要でない場合は、 path_duplicate() のような関数を使用して、まずpath のコピーを作成し、複製されたasset に対してこの関数を呼び出します ( path_delete() が不要になったらasset に対して呼び出すのを忘れないでください)。

構文です。

path_clear_points(index);

論証タイプ 商品説明
index The index of the path to clear.

リターンです。

例

if path_get_number(mypath) > 0
{
    path_clear_points(mypath);
}

上記のコードは、変数「mypath」でインデックスされたポイントがpath 上にあるかどうかを確認し、あればpath をクリアしています。
