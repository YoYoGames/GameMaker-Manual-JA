パスアペンド

このコードを使用すると、1つのpath を別のものに追加して、効果的に結合することができます。これが機能するためには、両方のpaths が以前に存在する必要があります。一般に、この関数は path_add() を使って作成したpaths に対して使いたいものです。なぜなら、この関数をpath asset に対して使うと、その関数を使った瞬間からゲームの終わりまで、ゲーム内のインスタンスのpath に永久に影響を与えるからです。

注意この操作では、ゲームに追加されたpath は削除されません。これはまだ存在しますが、ポイントを持っていないだけです。 path_delete().

構文です。

path_append(index, path);

論証タイプ 商品説明
index The new index of the path.
path The index of the path that will be appended to 'index'.

リターンです。

N/A

例

path_append(mypath, mppath);
path_delete(mppath);

上記のコードは、変数「mppath」にインデックスされたpath のデータを、変数「mypath」にインデックスされたpath に追加し、「mppath」をメモリから削除しています。
