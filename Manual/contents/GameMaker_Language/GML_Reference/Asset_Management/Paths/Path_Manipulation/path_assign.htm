パスアサイン

この機能を使うと、path のデータを、path から別のデータへコピーすることができます。コピー先の path は、まずクリアされ（path のポイントがある場合）、コピー先のpath で完全に上書きされます。 からコピーされます。path はどちらも削除されず、結果として2つのインデックスを持つ2つのpaths ができますが、形とポイントは全く同じです。一般に、この関数は path_add() を使って作成されたpath に対して使いたいものです。 path asset に使用すると、この関数を使用した瞬間からゲーム終了まで、ゲーム内のすべてのインスタンスのpath に永久に影響を与えるからです。

構文です。

path_assign(index, path);

論証タイプ 商品説明
index The index of the path to be overwritten.
path The index of the path that will overwrite 'index'.

リターンです。

例

mypath = path_add();
path_assign(mypath, choose(path_1, path_2, path_3));

上記のコードは、変数 "mypath "にインデックスされた新しい空のpath を作成し、利用可能な3つのリソースpath のうちの1つからpath データをコピーオーバーします。
