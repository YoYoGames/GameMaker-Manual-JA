パスリバース

この機能により、個々のpath ポイントの番号付けの順序を逆にすることができます。例えば、path が 5 ポイントの場合、ポイント 0 はポイント 4、ポイント 1 はポイント 3、ポイント 2 は変更されません。実際の位置は を処理する順番が変わるだけで、ポイントは変わりません。この関数は実際のpath アセットを変更するため、この関数を実行した瞬間からゲーム内のすべてのインスタンスがpath を使用する方法に永続的に影響を及ぼします。 関数が使用され、ゲームが終了するまで。もし、これが必要でない場合は、 path_duplicate() のような関数を使用して、まずpath のコピーを作成し、それからこの関数を呼び出します。 を複製されたasset に対して呼び出します (不要になったらasset に対して path_delete() を呼び出すことを忘れないでください)。

構文です。

path_reverse(index);

論証タイプ 商品説明
index The index of the path to change.

リターンです。

例

path_reverse(pth_AI);

これは、与えられたパス上のすべての点の処理順序を逆にすることになる。
