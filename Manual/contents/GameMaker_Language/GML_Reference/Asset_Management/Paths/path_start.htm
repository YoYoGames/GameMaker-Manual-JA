パススタート

path は、一連の定義点から作成され、それがリンクして、インスタンスの動きを計画するために使用されます。これらはコード、または Path Editor で作成することができ、ゲーム内で使用するインスタンスに割り当てることができます。この関数を使用して、どのpath に従うか、どの速度でpath に従うか（1 ステップあたりのピクセル数で測定）、path の終わりに達したときにどう動作するか、そして絶対位置か相対位置path のどちらに従うかをインスタンスに伝えることができ ます。この最後の部分は、パスエディタで設計し配置したとおりにpath を開始し追従する（絶対）か、またはインスタンスが作成された位置からpath を開始し追従する（相対）かを意味します。

上の画像でわかるように、2つのインスタンスは path_start() で始まる同じpath を辿っていますが、一方のインスタンスは絶対的なpath を辿っており、もう一方のインスタンスは相対的なpath を辿っていることがわかります。relativeがfalse（絶対）に設定されている場合、インスタンスはpath のどこに配置されていても、room の開始点に「ジャンプ」することに注意する必要があります。

このページでは、インスタンスがpath の末尾に到達したときの挙動を定義するために使用する定数を示しています。

構文です。

path_start(path, speed, endaction, absolute);

論証 タイプ 商品説明
path パスアセット The path index to start.
speed リアル The speed of which to follow the path in pixels per step, negative meaning going backwards.
endaction パスエンドアクション定数 What to do when the end of the path is reached (see the constants listed above).
absolute ブーリアン Whether the calling instance should follow the absolute path as it is in the editor (true) or a relative path to its current position (false).

リターンです。

N/A

例

path_start(path, 4, path_action_reverse, 0);

これは、呼び出したインスタンスが、変数 "path" でインデックスされたpath を、1ステップあたり4ピクセルの速度で追いかけるようにするものである。これは、自分自身の位置を基準にしてpath を追いかけ、最後に到達すると方向を反転して来た道を戻る。
