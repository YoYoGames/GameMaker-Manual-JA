エンコードとハッシュ化

外部ファイルを扱う場合、エンドユーザーがそのファイルを開いて情報を変更し、ゲームを変更する可能性が常にあります。そのため、GameMakerでは、基本的なエンコード関数と、strings やファイルに対してハッシュチェックを行い、整合性が保たれていることを確認してから使用する関数を用意しています。また、JSON形式（strings ）のエンコード、デコードのための関数もいくつか用意されています。

注意事項 エンコーディングは暗号化ではありません。base64エンコーディングは、ファイルを肉眼で読めなくするもので、デコードにはユーザーの努力が必要ですが、ハッキングからは安全ではありません。これらの関数と独自の暗号化を混ぜることをお勧めします（インターネット上には多くの暗号化の形式や、script のGameMaker 関数が公開されています）。

  • json_encode
  • json_decode
  • json_stringify
  • json_parse
  • base64_encode
  • base64_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_file
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_file
  • zip_unzip
  • load_csv

なお、Bufferのエンコードやハッシュ化に関連する機能も多数ありますので、以下のリンク先から検索してください。

  • buffer_md5
  • buffer_sha1
  • buffer_crc32
  • buffer_base64_encode
