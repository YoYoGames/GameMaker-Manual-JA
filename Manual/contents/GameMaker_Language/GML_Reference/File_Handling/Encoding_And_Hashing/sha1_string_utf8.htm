sha1_string_utf8

暗号技術では、SHA-1は米国国家安全保障局によって設計された暗号ハッシュ関数で、広く使われているいくつかのアプリケーションやプロトコルで採用されており、有名なGitではファイルの変更をチェックするのに使われています。 この関数は、入力のUTF-8string （1文字あたりのバイト数は可変です）を受け取り、例えばASCII形式で160ビットのメッセージダイジェストを返します。

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

このようにして、安全な鍵を生成し、それを保存して、外部のserver （例）に送信（または受信）される情報の整合性をチェックするために使用することができます。

注：sha1エンコーディングには、UTF-8とunicodeの2つの形式があります。どちらも異なるserver セットアップでの通信を容易にするために提供されていますが、最も一般的に使用されるのは unicode です。

注意：SHA-1は完全に安全ではなく、破られる可能性があります。詳しくはこのページをご覧ください。

構文です。

sha1_string_utf8(string)

論証タイプ 商品説明
string The string to hash.

リターンです。

例

var hash, str;
str = json_encode(hiscore_map);
hash = sha1_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

上記のコードでは、DSマップをJSON（string ）にエンコードしています。そして sha1 ハッシュが生成され、ini ファイルに保存される。これは、後でserver から同じ情報を受け取ったときに、JSON の整合性をチェックするために使用できるようにするためである。 が送信される。
