sha1_string_unicode

暗号技術では、SHA-1は米国国家安全保障局によって設計された暗号ハッシュ関数で、広く使われているいくつかのアプリケーションやプロトコルで採用されており、有名なGitではファイルの変更チェックに使用されている。 この関数は、入力のunicodestring 、160ビットのメッセージダイジェストをASCII形式で返します。例えば、以下のようになります。

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

このようにして、安全な鍵を生成し、それを保存して、外部のserver （例）に送信（または受信）される情報の整合性をチェックするために使用することができます。

注：sha1エンコーディングには、UTF-8とunicodeの2つの形式があります。どちらも異なるserver セットアップでの通信を容易にするために提供されていますが、最も一般的に使用されるのは unicode です。

注意：SHA-1は完全に安全ではなく、破られる可能性があります。詳しくはこのページをご覧ください。

構文です。

sha1_string_unicode(string)

論証タイプ 商品説明
string The string to hash.

リターンです。

例

var hash, str;
str = base64_encode(game_data);
hash = sha1_string_unicode(str);
http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?hash=" + hash); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?data=" + str);

上記のコードでは、string を base64 でエンコードし、sha1 ハッシュを生成している。最後に、そのハッシュとエンコードされたstring の両方がserver に送られる。
