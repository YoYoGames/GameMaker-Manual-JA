テキストファイル

テキストファイルは、ゲームの外部に大量のデータを保存するのに便利な方法です。iniファイルとは逆に、テキストファイルは一度に32個まで開くことができ、大きなデータの塊も読み書きが可能です。ただし、*.ini ファイルのように「セクション」や「キー」の構造が決まっているわけではないので、データをどのように構成するかはあなた次第です。

ローカライゼーションのあるゲームでは、アクセント付きの文字が外部ファイルから読み取れることが非常に重要であることに注意してください。つまり、まず UTF8形式のtxtファイルを作成し、それをGameMakerにインクルードファイルとして追加することで、ゲーム実行時にエクスポートされ、GameMakerのファイル機能で事前にファイルが提供されていない場合に作成されるデフォルトのANSI形式のtxtファイルの代わりに使用されます。こうすることで、アクセントや非ロマン文字を維持したまま、正しく読み書きができるようになります。

ファイルを扱う機能として、以下のようなものがある。

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
