working_directory

Working_directory は、実際には使用目的に応じて2つの異なる値を返すことができます。

ディスクにファイルを書き込む場合、 working_directory はターゲットデバイスのゲーム用に確保されたローカルストレージの領域を指します（Windows では "%LOCALAPPDATA%/gamedir/" で、"gamedir" はゲーム名の入ったディレクトリです）。

ただし、ディスクから読み込む場合、 working_directory は、ローカルストレージ（game_save_id ）または インクルードファイルが格納されている領域のいずれかになる可能性があります。

NOTE変数 working_directory は、最後のバックスラッシュを含む path を返します。

例えば、ゲームにデフォルトの *.txt ファイルが含まれていて、それを working_directory を使ってメモリに読み込んだ場合、そのファイルは Included Files が格納されている領域から取得されます。 working_directory その情報を working_directory を使ってテキストファイルに書き込むと、そのファイルはターゲットプラットフォームのローカルストレージに書き込まれます。

なお、一般的にはこれを使用する必要はなく、単にファイル名そのものを指定する（さらに、path の情報を追加する）ことで十分です。例えば、これ。

file_text_open_read(working_directory + "my_file.txt");

は、このようにした方が良いと思います。

file_text_open_read("my_file.txt");

警告 GameMakerはサンドボックス化されているため、この機能は期待通りに動作しない可能性があります!詳しくは「ファイルシステム」の項をご覧ください。

構文です。

working_directory

リターンです。

String

例

ini_open(working_directory + "temp_ini.ini");

ゲームの作業ディレクトリにあるiniファイルを開きます（まだ存在しない場合は作成します）。これは、ローカルストレージか、インクルードファイルがある領域で、探したいファイルがどちらかに存在するかどうかで決まります。
