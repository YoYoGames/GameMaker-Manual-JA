ファイル操作

ゲームを作っていると、ある時点で情報を外部に保存し、その情報を取り出せるようにすることが必要になります。例えば、ログイン情報、ハイスコア、プレイヤーのレベル経過などを保存したい場合があります。どのような理由であれ、GameMakerで作業する際に利用できる様々なファイル機能の使い方を知っておく必要があります。

GameMakerはデフォルトでSandbox化されて いるため、ファイルシステム機能を使用する際に多くの制限や小さな複雑さがあることを理解しておくことは非常に重要で、対象とするプラットフォームによってファイルシステムの動作が若干異なることも知っておくとよいでしょう。この点については、次のセクションを参照してください。

  • ファイルシステム

デスクトップターゲット（Windows,macOS,Ubuntu (Linux) ）は、選択したプラットフォームのゲームオプションからサンドボックスをオフにすることができます。つまり、OSのパーミッションの範囲内で、どこからでもセーブとロードが可能になります。

Steam からゲーム専用APIの使用許可を得ている場合、新たにファイル関数のセットも利用できることがわかります。これらについては、本マニュアルの「Steam API」のセクションで説明しています。

最後に、ファイル機能を使用する際に考慮すべきターゲット・プラットフォーム間の明確な違いがあることにも留意してください。これらは以下の通りです。

  • iOSです。iOS で、フォルダからインクルードファイルを読み込む場合、例えばpath で "Backgrounds/background1.png" のような場合、path の一部としてフォルダを含める必要はなく、単にファイル名を与えるだけで十分です。他のすべてのプラットフォームでは、ファイルへのpath の一部として、フォルダーを指定する必要があります。
  • macOSおよびUbuntu (Linux):macOS とUbuntu 、これらのプラットフォームではファイル名とpaths は大文字と小文字を区別するため、潜在的な問題を防ぐために、GameMakerは自動的にすべての名前/パスをデフォルトで小文字を使用するようにします。
  • HTML5です。HTML5では、ローカルストレージへの保存と読み込みは、それぞれの機能ページで説明されているように動作します。しかし、ファイルをserver から読み込む場合は、同期読み込みはほとんどのブラウザで非推奨となっており、いずれ廃止になるため、 buffer_load_async()の機能のみを使用する必要があります。また、ブラウザのローカルストレージにディレクトリを作成したり、破棄したりすることはできません。
