スプライトとタイル

room エディタで背景を設定し、sprites をインスタンスに割り当てることができても、作っているゲームに対して十分な柔軟性がないことがよくあります。例えば、メニューを描画するインスタンスがある場合、様々なボタンsprites を一度に表示したいかもしれませんし、スクロールする視差の背景を持つゲームがある場合、何らかのタイプのコントローラインスタンスでコードを通して設定できるようにする必要があります。これらの状況では、sprite と背景assets をroom の任意の場所に描画できる以下の関数を使用する必要があります。また、いくつかの関数では、特殊効果を得るためにsprite の形状、サイズ、色、アルファを操作することができます。

インスタンスにsprite を割り当てると、デフォルトではGameMakerが様々なスプライト変数で与えられたプロパティで描画します。しかし、draw イベントに何かを追加すると、「コントロールする」ことになり、何を描画するかGameMakerに正確に伝えなければなりません。つまり、以下の関数を使って、インスタンスが何を描画し、room のどこに描画するのかを正確に定義する必要があるのです。

  • draw_self
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • draw_sprite_part
  • draw_sprite_part_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

SWF 形式sprites を使用している場合、描画時に使用するアンチエイリアスを制御するための追加オプションもあります（これらのタイプの詳細については、非ビットマップスプライトのインポートを参照してくださいsprites ）。これらの関数はグローバルレベルで動作し、呼び出された後に描画されたすべてのベクター SWFsprites の AA を設定することに注意してください。

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

上記の関数とは別に、スケルトン・アニメーションのために設計された2つの特別な関数があります（これらのタイプのsprites の詳細については、非ビットマップ・スプリットのインポートを参照し、これらのタイプのsprites に関連する特定の関数については、スケルトン・アニメーションのセクションを参照 してください）。

  • draw_skeleton
  • draw_skeleton_instance
  • draw_skeleton_collision
  • draw_skeleton_time

タイルは、それが含むデータ、使用されるtile set およびtile マッププロパティに基づいて自動的に描画されます。しかし、以下の関数を使用することで、個別のtiles またはtile マップを描画することができます。

  • draw_tile
  • draw_tilemap
