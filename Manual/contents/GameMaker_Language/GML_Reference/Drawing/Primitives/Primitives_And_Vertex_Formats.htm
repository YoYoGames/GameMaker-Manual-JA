プリミティブとVertex フォーマット

プリミティブとは、画面に表示されるものを定義するために、点を使って行われる描画の一種です。これらは、1つの点、線、または三角形で構成され、色付け、アルファブレンディング、さらにtexture 、2Dゲームで非常にエキサイティングな効果を作り出すために使用できることを意味します！プリミティブを使用することで、2Dゲームでは、いくつかの非常にエキサイティングな効果を作り出すことができます。texture 例えば、複数の三角形プリミティブを使用して長方形を作成し、flag sprite 、インスタンスの描画イベントで、そのflag が風に「波」、プリミティブを形成するポイントを変更するだけで、使用したsprite を歪ませることができます。

警告!こ れらの機能は、ゲームオプションでOpenGLを有効にしていない限り、HTML5モジュールでは動作しません。

描画プリミティブの情報は、以下のページでご覧いただけます。

  • draw_primitive_begin
  • draw_primitive_begin_texture
  • draw_primitive_end
  • draw_vertex
  • draw_vertex_colour
  • draw_vertex_texture
  • draw_vertex_texture_colour

GameMakerでは、独自の頂点フォーマットを定義することができ、そこから独自のプリミティブを作成することができます。これにより、shader 操作が大幅に高速化されたり、機能を拡張して驚くような効果を生み出すことができます。これに関する情報は、以下のセクションにあります。

  • vertex_format_begin
  • vertex_format_add_colour
  • vertex_format_add_position
  • vertex_format_add_position_3d
  • vertex_format_add_texcoord
  • vertex_format_add_normal
  • vertex_format_add_custom
  • vertex_format_end
  • vertex_format_delete

vertex フォーマットを作成したら（上記のセクションを参照）、それを使って、以下の veretxbuffer 関数を使って、描きたいプリミティブを構築することができます。

重要!構築するプリミティブは、Vertex Format 関数で設定したフォーマットに従う必要があります。したがって、vertex フォーマットで位置データのみを定義した場合、カラーデータでプリミティブを構築しても意味がありません。構築するプリミティブにプロパティを追加する順序は、vertex フォーマットを作成するときに追加した順序で定義されることに注意してください。したがって、位置、色、texture 座標という順序でベクトルフォーマットを定義した場合、構築中のプリミティブにも同じ順序でこれらのプロパティを追加しなければならず、そうしないとエラーになります。
