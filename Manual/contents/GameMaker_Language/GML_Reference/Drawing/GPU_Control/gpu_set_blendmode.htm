gpu_set_blendmode

GameMakerがピクセルを描画しようとするとき、ソースカラー（描画しようとするピクセルの色）とデスティネーションカラー（描画先のピクセルにすでにある色）があります。したがって、ピクセルの新しい色を決定するとき、ソースカラーとデスティネーションの色は選択したブレンドモードに従って計算されます。色の各要素は0から1の間の浮動小数点値として保存され、新しい色は、ソースカラーの各要素に何らかの係数を掛け、デスティネーションカラーの各要素に何らかの係数を掛けて、その結果を要素ごとに加算することによって計算されます。

この機能は、以下の定数を用いて、4つの基本的な方法でブレンドモードを設定することができます。

定数 商品説明 拡張ブレンドモード
bm_normal 通常のブレンド（デフォルトのブレンドモード）。 (bm_src_alpha, bm_inv_src_alpha)
bm_add 加法混色。明るい部分の輝度値が加算されます。 (bm_src_alpha, bm_one)
bm_subtract 減算式ブレンディング。明るい部分の輝度値が減算される。 (bm_zero, bm_inv_src_colour)
bm_max マックスブレンディング加算ブレンディングに似ている。 (bm_src_alpha, bm_inv_src_colour)

上の表からわかるように、この4つのブレンドモードは、実際には、以下のページで紹介されている拡張ブレンドモードを合成したものです。 gpu_set_blendmode_ext().

ブレンドモードを最大限に活用し、その仕組みと画面に描画される最終画像への影響を理解するために、以下のガイドをお読みになることをお勧めします。

  • Blendmodesの使い方ガイド

構文です。

gpu_set_blendmode(mode);

論証 タイプ 商品説明
mode ブレンドモードコンスタント The blend mode to use (see the table above)

リターンです。

N/A

例

gpu_set_blendmode(bm_add);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

これにより、黒が透明になり、白が外側で強く、円の中心から徐々に弱くなる「グロー」効果が生まれます。ブレンドモードは、円が描かれた後にリセットされ、それ以降のすべてのものに加法的ブレンドが適用されないようにします。
