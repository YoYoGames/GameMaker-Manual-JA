gpu_set_blendmode_ext

GameMakerがピクセルを描画しようとするとき、ソースカラー（描画しようとするピクセルの色）とデスティネーションカラー（描画先のピクセルにすでにある色）があり、ピクセルの新しい色を決定するとき、選択したブレンドモードに従って、ソースカラーとデスティネーションカラーが計算されます。色の各要素は0から1の間の浮動小数点値として保存され、新しい色は、ソースカラーの各要素に何らかの係数を掛け、デスティネーションカラーの各要素に他の係数を掛け、その結果を要素ごとに加算することによって計算されます。

この機能により、カスタムブレンドモードを作成するために考慮すべきさまざまな構成要素を設定することができます。ソースとデスティネーションはともに赤、緑、青、アルファの各成分を持ち、下図ではソースのRGBAを（Rs、Gs、Bs、As）、デスティネーションのそれを（Rd、Gd、Bd、Ad）と見なします。使用可能な11個の定数は、この関数で使用する場合、ソースとデスティネーションのどちらか（または両方）にすることができます。

定数 ブレンド係数（赤、緑、青、アルファ）
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs、1-Gs、1-Bs、1-As)
bm_src_alpha (アス、アス、アス、アス)
bm_inv_src_alpha （1-As、1-As、1-As、1-As）
bm_dest_alpha （アド、アド、アド、アド）
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) ここで、f = min(As, 1-Ad)

重要!WebGLを有効にしていないHTML5では、以下のモードが正しく表示されません。

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

この関数には、2 つの個別の引数を指定することも、引数の配列を指定することもできます (これは、関数 gpu_get_blendmode_ext()のように）。配列で指定する場合は、次の2つの要素を持つ必要があります。

  • [0] = ソースブレンドモード (デフォルトは bm_src_alpha)
  • [1] = デスティネーションブレンドモード (デフォルトは bm_inv_src_alpha)

ブレンドモードを最大限に活用し、その仕組みと画面に描画される最終画像への影響を理解するために、以下のガイドをお読みになることをお勧めします。

  • Blendmodesの使い方ガイド

構文です。

gpu_set_blendmode(src, dest);

論証 タイプ 商品説明
src ブレンドモードファクター定数 Source blend mode factor (see constants above).
dest ブレンドモードファクター定数 Destination blend mode factor (see constants above)

リターンです。

N/A

例

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

これにより、黒が透明になり、白が外側で強く、円の中心から徐々に弱くなる「グロー」効果が生まれます。ブレンドモードは、円が描かれた後にリセットされ、それ以降のすべてのものに加法的ブレンドが適用されないようにします。
