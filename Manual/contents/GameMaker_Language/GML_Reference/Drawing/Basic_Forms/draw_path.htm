draw_path

この関数を使うと、GameMakerに path を画面に描画させることができる。path は単純な線として描かれ、呼び出したインスタンスからの相対位置か、path エディタやコードで作成した絶対位置のどちらかを指定できます。この機能はdebugging ダイナミックなpaths (例えば、インスタンスに対して mp_grid_path()関数)を使用します。

注：shader を使って図形を描く場合、ほとんどのshaders は次のような入力を期待していることに注意する必要があります。vertex texture, 色.しかし、この関数を使用する場合、vertex とカラーデータのみが渡されるため、shader は何も描画しない（または、何か描画するが正しく描画しない）可能性があります。このように図形を描画する必要がある場合は、shader をこの点を考慮してカスタマイズする必要があります。

構文です。

draw_path(path, x, y, absolute);

論証タイプ 商品説明
path The path to draw
x The x coordinate of where the path is drawn
y The y coordinate of where the path is drawn
absolute Whether the path is drawn at the absolute position (true) または相対位置 (false)

リターンです。

例

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
    draw_path(path, x, y, false);
}

上記のコードでは、 mp_grid_path 関数を使用してpath を生成し、変数 "path" に格納します。path が正常に生成されると、コードを実行しているインスタンスからの相対位置で画面上に描画されます。
