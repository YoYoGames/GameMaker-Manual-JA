draw_set_circle_precision

円を描く際のパフォーマンスを最適化するために、GameMakerでは基本的に円形に見えるだけの辺を持つポリゴン形状を描きます。しかし、円の大きさや大きさによっては、この値を変更することで、円形に見えるようになります。 この値によって、ゲームのパフォーマンスが向上したり、円の見栄えがよくなったりします。この関数に入力できる精度の値は、円形ポリゴンが持つ辺の数で、この値は4から64の間の任意の値です。 ただし、4で割り切れる数でなければならず、デフォルト値は24である。

注：shader を使って図形を描く場合、ほとんどのshaders は次のような入力を期待していることに注意する必要があります。vertex texture, 色.しかし、この関数を使う場合、vertex と色のデータだけが渡されます。 shader は何も描画しない（あるいは、描画はするが正しく表示されない）かもしれません。このように図形を描画する必要がある場合は、shader をこの点を考慮してカスタマイズする必要があります。

構文です。

draw_set_circle_precision(precision);

論証タイプ 商品説明
precision The precision of all circles drawn after this function is called.

リターンです。

例

draw_set_circle_precision(12);

上記のコードでは、すべての円を12角形のポリゴンで描画します。
