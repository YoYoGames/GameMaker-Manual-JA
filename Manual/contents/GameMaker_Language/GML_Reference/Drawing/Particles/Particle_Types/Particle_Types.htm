粒子の種類

パーティクルとは、ゲーム内でパーティクルシステムを使用することによって生み出されるグラフィック効果のことです。通常、ゲーム開始時にすべてのパーティクルを定義し、そのインデックス値をグローバル変数に格納します。このグローバル変数を使って、ゲーム中いつでもパーティクルのバースト、エミット、生成を行うことができます。また、メモリの問題でパーティクルの作成、使用、破棄が必要な場合や、ゲーム中の特定のポイントにのみ多くの種類のパーティクルを使用したい場合にも対応できます。パーティクルは非常に汎用性の高いリソースです。

パーティクルを作成し、そのインデックスを変数に格納することで、以降のすべてのパーティクル ルーチンが正しくアクセスできるようにする必要があります。以下の関数はパーティクル タイプの作成、チェック、クリア、破壊に使用されます。

  • part_type_exists
  • part_type_create
  • part_type_destroy
  • part_type_clear

パーティクルの種類を作成したら、それに形状を与える必要があります。この形状は、64x64 ピクセルsprites でアルファブレンディングされた 14 個の組み込み形状のうちの 1 つにするか、または独自のsprites を使用することができ、アニメーション、静止、ランダムサブイメージを使用することができます。以下の関数は、各パーティクルの視覚的な基本形状を定義します。

  • part_type_shape
  • part_type_sprite

パーティクルの種類（デフォルトの形状または独自の形状）をsprite で選択したら、そのサイズとスケールを指定することができます。サイズは範囲指定が可能で、すべてのパーティクルは指定された範囲内で選ばれたサイズで作成されます。scale は、サイズのバリエーションが追加される前のパーティクルの x 軸と y 軸のスケールの「基本」値で、これらの値を制御するために以下の関数が使用されます。

  • part_type_size
  • part_type_scale

また、粒子は向きや動きを持つことができます。多くのパーティクル関数と同様に、速度、方向、向きは与えられた範囲から選択されるランダムな値として解決され、以下の関数で設定することが可能です。

  • part_type_speed
  • part_type_direction
  • part_type_gravity
  • part_type_orientation

パーティクルのスタイルと動きが決まったら、次に色とアルファ値を設定します。この2つの値は、パーティクルの寿命の間、絶対値として設定することも、1つの値から徐々に他の値に移行するスケールとして設定することもできます。また、パーティクルとその周囲の混合を設定する特別な機能があり、これは通常または加算のどちらでも可能で、いくつかの素晴らしい効果をもたらします。

  • part_type_colour_mix
  • part_type_colour_rgb
  • part_type_colour_hsv
  • part_type_colour1
  • part_type_colour2
  • part_type_colour3
  • part_type_alpha1
  • part_type_alpha2
  • part_type_alpha3
  • part_type_blend

パーティクルは画面上に限られた時間だけ存在し、これを「ライフタイム」と呼びます。粒子の寿命はステップで測定され、（これまでの関数と同様に）これは値の範囲として表現され、そこから特定の寿命が与えられます。また、パーティクルが他のパーティクルを生成するような面白い関数もいくつかあります。ステップごとや、パーティクルが "死ぬ"（つまり、スクリーンから消える）ときに生成されますが、これらの関数はすぐに何千ものパーティクルを生成してゲームをクラッシュさせることがあるので、十分注意してくださいライフタイムやステップ、デスなどを設定する関数は以下の通りです。

  • part_type_life
  • part_type_step
  • part_type_death

コードによるパーティクル作成は、変更・調整可能なパラメータが多いため、手間と時間のかかる作業となることが多いということは知っておいて損はありません。そのため、パーティクルデザイナーを利用することをお勧めします（自分で作ることもできます）。
