パート_システム_レイヤーを作成する

この関数は、指定されたレイヤーに新しいパーティクルシステムを作成します。この関数が返すユニークなレイヤー ID を与えます。 layer_create()またはstring として使用するレイヤーの名前 - 例えば "instance_layer" - そしてflag システムが永続的であるかどうかを指定します。もし、システムが永続的であるとフラグが立っていなければ、作成されたroom の終了時に自動的に破壊されます（これは、あなたが関数 part_system_destroy()また、システムに関連付けられているエミッタもすべて破壊されます）。しかし、persistentとフラグを立てた場合、room が変更されると、システムは次のroom に運ばれます。次のroom に、割り当てられたレイヤーと同じ名前または深さのレイヤーがない場合、rooms にわたってpersistentであるシステム用に新しいレイヤーが作られ、それは元のレイヤーと同じ名前にされます。rooms を変更するとき、次のrooms に同じ名前の別のレイヤーがあれば、レイヤーの深さに関係なく、その名前を持つレイヤーに永続化インスタンスが割り当てられます。最後に、永続化されたシステムが、システムが作成されたレイヤーと同じ深さのレイヤーを持つroom に移動した場合、そのレイヤーには割り当てられず、代わりに同じ深さに（元のレイヤーと同じ名前を持つ）新しいレイヤーが作成されます。

この関数はパーティクルシステムのユニークな ID 値を返します。この値は、今後システム ID を指定する必要があるすべての関数呼び出しで使用する必要があります。

重要! flag パーティクルシステムを永続化する場合、パーティクルシステム（およびそれに割り当てられたエミッター）は、使用していないときに適切なdestroy関数を使用して手動でクリーンアップする必要があり、そうしないとメモリリークが発生して最終的なゲームに悪影響を及ぼす危険性があります。

構文です。

part_system_create_layer(layer_id, persistent);

論証 タイプ 商品説明
layer 文字列またはレイヤID The layer ID (or name) to assign the particle system to (can be any layer type)
persistent ブーリアン Flag the particle system as persistent (set to true) か、そうでないか ( false に設定)

リターンです。

Particle System ID

例

global.p_sys = part_system_create_layer("effects_layer", true);

上記のコードでは、指定されたレイヤーに新しいパーティクルシステムを作成し、flag 以降の永続化としてrooms 。パーティクルシステムの ID は、将来参照できるようにグローバル スコープ変数に保存されます。
