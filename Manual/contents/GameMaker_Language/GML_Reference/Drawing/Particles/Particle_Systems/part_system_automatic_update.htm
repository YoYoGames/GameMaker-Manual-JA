パートシステム自動更新

この関数は、GameMaker がパーティクルシステムを自動的に更新するかどうかを制御します。通常は、デフォルトの値である true （自動更新がオン）が希望通りの動作になるので、この関数を使用する必要はないでしょう。しかし、特殊効果やゲームを一時停止させるために、これを false に設定すると、関数を使って明示的に命令しない限り、指定したパーティクルシステムの更新が行われないようになります。 part_system_update()または、再度この機能を使って自動更新をオンにします。更新しないとは、このシステムで作成されたパーティクルは位置や色などのパラメータが変化しなくなり、エミッタも動作しなくなり、自動更新を false に設定した時点のシステムで「凍結」することを意味します。

注意点としては、関数で作成したシンプルなエフェクトを使う場合は effect_create_above()または effect_create_below()の場合、パーティクルシステムのインデックスとして 0（以下の効果）または 1（上記の効果）の値を使用し、これらの自動更新を切り替えることができます（これはGML Visual particle effect の更新を切り替えるためにも機能します）。

構文です。

part_system_automatic_update(ind, automatic);

論証 タイプ 商品説明
ind パーティクルシステムID The index of the particle system to change.
automatic ブーリアン Whether automatic updating is on (true) か、そうでないか (false)。

リターンです。

N/A

例

if global.Pause
{
    part_system_automatic_update(global.Sname, false);
}

上記のコードは、グローバル変数 "Paused" が true の場合、switch パーティクルの更新を停止します。
