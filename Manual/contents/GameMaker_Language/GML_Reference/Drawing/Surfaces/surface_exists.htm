サーフェスエクジスト

サーフェスは揮発性であるため、この機能はサーフェスを扱う上で必要不可欠です。サーフェスは常にtexture メモリに保持され、ある瞬間から次の瞬間まで破壊される可能性があります（たとえば、Windows でスクリーンセーバーが起動したときや、Android デバイスで最小化したときなど）。したがって、何かする前に（スクリーンに描くことも含めて）サーフェスが存在するか常に確認する必要があります。以下のコード例では、インスタンスの draw イベントでこのコマンドを使用して、サーフェスをチェックし、サーフェスが削除されている場合は再作成しています（サーフェスはobject の create イベントで最初に作成されていることに注意してください）。

構文です。

surface_exists(surface_id);

論証タイプ 商品説明
surface_id The ID of the surface to check.

リターンです。

例

if !surface_exists(surf)
{
    surf = surface_create(1024, 1024);
}
else
{
    if (view_current == 1)
    {
        draw_surface(surf, 0, 0);
    }
}

上記のコードでは、変数surfでインデックスされたサーフェイスが存在するかどうかをチェックし、存在しない場合は再作成しています。存在する場合は、現在描画されているビューをチェックし、それがview[1]であればサーフェスを描画します。
