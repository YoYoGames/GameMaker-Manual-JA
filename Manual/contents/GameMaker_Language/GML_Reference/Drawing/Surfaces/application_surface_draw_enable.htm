アプリケーションサーフェス描画を有効にする

アプリケーションサーフェイスの自動描画の有効／無効を設定することができます。デフォルトでは、これは有効になっていますが、多くの場合、サーフェスがいつ、どのように描画されるかを引き継ぎたいと思うでしょう（たとえば、shaders を使用する場合）、そのような場合 この関数を使用して false に設定し、必要なときに必要な方法で自分で描画できるようにします。switch 自動描画をオフにして、アプリケーションの表面を自分で描画する場合、次のような問題が発生する可能性があることに注意してください。 sprites のアルファコンポーネントとサーフェス自体に影響を与えます。これは、自動描画がオンの場合、GameMakerが アルファブレンディングを行わずにアプリケーションサーフェスを描画するためです。もし、switch 自動描画をオフにした場合は、次のことが必要である。 は、以下のコード（例）を使って、自分で処理する必要があります。

gpu_set_blendenable(false);
draw_surface_ext(application_surface, 0, 0, 1, 1, 0, c_white, 1);
gpu_set_blendenable(true);

構文です。

application_surface_draw_enable(flag);

論証タイプ 商品説明
flag The flag will be either true (有効、デフォルト値)または (無効)。 false

リターンです。

例

application_surface_draw_enable(false);

上記のコードは、switch アプリケーション表面の自動描画をオフにします。
