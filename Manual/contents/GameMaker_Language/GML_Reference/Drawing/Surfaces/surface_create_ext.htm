surface_create_ext

この機能を使うと、Webページ内に既に存在するcanvas要素にサーフェスを貼り付けることができます。つまり、ゲームの一部をページ内のさまざまな場所に効率よく分割して描画することができるのです。そのためには、ゲームの *.html ページ内で、作成したいサーフェイスに対応する正しいサイズと名前を使用して、canvas 要素を正しく定義しておく必要があります。つまり、「メイン」キャンバスと、「セカンダリ」キャンバス要素があり、この機能で正しいサーフェイスに割り当てます。次の画像は、3つのキャンバス要素を持つページのセットアップの例です。

ページレイアウトは、特にすべての要素を正しく並べたい場合、厄介です。しかし、レイアウトの作成という大変な作業が完了したら、ゲームオプションの HTML5タブで、この html ファイルをゲームのデフォルトページファイルとして追加できます。次にやるべきことは、room とビューの設定です。各サーフェイスは、ゲーム画像を「キャプチャ」するために、特定のビューと関連付ける必要があるからです（ビュー変数 view_surface_id[0...7]).以下の画像は、上記のキャンバスの例でゲームroom をどのようにセットアップするかを示しています。

最後に、この関数を使用して表面を作成します。名前はキャンバス要素に使用したものと同じにし、サイズは同じキャンバスのサイズに対応させます。この関数はサーフェスのインデックスを返すので、将来関数を呼び出すときのために変数に格納しておきます。サーフェスが最初に作成されたとき、それは基本的にこの目的のために確保された単なるメモリ領域であるため「ノイズ」を含むことがあります（そしてそのメモリはまだ情報を含んでいるかもしれません）。 draw_clear_alpha().

注：この機能は、HTML5モジュールでのみ使用可能です。

構文です。

surface_create_ext(name, w, h);

論証タイプ 商品説明
name The name of the canvas element to link the surface to.
w The width of the surface to be created.
h The height of the surface to be created.

リターンです。

Surface ID

例

s1 = surface_create_ext("surface1", 192, 550);
s2 = surface_create_ext("surface2", 608, 186);
view_surface_id[1] = s1;
view_surface_id[2] = s2;

上記のコードでは、サイズの異なる2つのサーフェスを作成し、それぞれを異なるキャンバス要素に割り当て、それらのサーフェスを2つのビューに割り当てて、room の正しい部分がキャプチャされるようにしています。
