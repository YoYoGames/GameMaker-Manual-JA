surface_depth_disable

この機能は、深度buffer の自動生成を無効にした後に作成されたすべてのサーフェスについて、その生成を無効にします。通常、すべてのサーフェスは深度buffers を持っているので、それらに 3Dobjects を描画すると、深度によって適切にソートされます。 表面へのレンダリングと背面へのレンダリングの間の機能的なパリティを維持するためにbuffer 、しかし深度を割り当てるbuffers 実質的に表面に必要なメモリ量を2倍にし、これは過剰で不必要なオーバーヘッドになる可能性があります。 特にメモリを大量に消費するゲームの場合、2Dゲームでは一般的に必要ありません。

この機能を使用するには、サーフェスを作成する前にこの機能を呼び出し、（必要であれば）作成後に再び有効にすることができます。深度buffer が無効になっているときに作成されたすべてのサーフェスは、描画されるときに深度buffer を使用しなくなり、この結果 は、サーフェスに描画されるものが深さのパリティを持つようになります。以下の画像は、これを実際に説明したものです。

上の画像では、vertex buffer を使って、長方形と三角形を3Dで描いています。このとき の座標は緑の長方形の後ろにありますが、三角形を描くコードは長方形のコードの後で呼び出され、深度buffer を有効にすると、三角形はまだ長方形の後ろに描画されます。 3Dの奥行きを尊重しているためです。しかし、サーフェスの深度buffer を無効にすると、コードの順序のみが尊重されるため、三角形は四角形の上に描画されます。

注：サーフェス使用時に3Dの深度を適切に保持するには、深度buffer を有効にするだけでなく、z-writing と z-testing も有効にする必要があります。関数 gpu_set_ztestenable() と gpu_set_zwriteenable()をご覧ください。

注：あなたのゲームが100％2Dの場合、または表面深度buffer が必要ないことがわかっている場合は、ゲーム開始時にこの関数を1回呼び出すだけでbuffer を無効にすることができます。

構文です。

surface_depth_disable(disable);

論証タイプ 商品説明
disable If set to true 作成されたサーフェスの深度バッファが無効になります。

リターンです。

例

var _depth_setting = surface_get_depth_disable();
surface_depth_disable(false);
surface_with_depth = surface_create(room_width, room_height);
surface_depth_disable(true);
surface_without_depth = surface_create(room_width, room_height);
surface_depth_disable(_depth_setting);

上記のコードでは、現在の深度buffer の設定を変数に保存し、深度バッファリングを有効にしたサーフェスとそうでないサーフェスの2つを作成してから、元の設定を再度復元しています。
