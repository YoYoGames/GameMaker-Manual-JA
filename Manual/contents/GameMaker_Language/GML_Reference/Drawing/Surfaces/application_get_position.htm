application_get_position

ターゲットプラットフォームのゲームオプションで「アスペクト比を維持する」にチェックを入れると、GameMakerは自動的にアプリケーションサーフェイスの描画位置を設定し、正しくセンタリング、スケーリングされて表示されるようにします。 指定されたディスプレイしかし、もしあなたがこのサーフェスを操作していて、自分で描画したい場合、この関数を使えば、サーフェスが描画されているディスプレイやウィンドウの場所を正確に知ることができますので、次のことが可能になります。 自分で描画したり、GUI画像やポストドロー画像をそこに整列させることができる。

この関数は，4つのキーを持つ配列を返す。キー0と1は表面の左上隅のXとYの位置，キー2と3は表面の右下隅のXとYの位置であり，キー0と1は表面の左上隅のXとYの位置，キー2と3は表面の右下隅のXとYの位置である。 は、すべてディスプレイまたはウィンドウのサイズに対する相対値である。

構文です。

application_get_position();

リターンです。

例

var a = application_get_position();
xx = a[0];
yy = a[1];
ww = a[2] - a[0];
hh = a[3] - a[1];

上記のコードは、アプリケーションサーフェスの位置と、ディスプレイウィンドウに対する絶対的な幅と高さを取得し、将来使用するために4つの変数に格納します。
