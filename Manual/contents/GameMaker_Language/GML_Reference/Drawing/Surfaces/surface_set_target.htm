表面目標設定

この関数を使うと、それ以降の描画をすべてスクリーンではなくターゲット面に設定し、特定のものだけを指定した面に描画するようにGameMakerに指示することができる。を呼ばない場合はご注意ください。 surface_reset_target()を終了させると、それ以降のすべての描画は (他のインスタンスでも) サーフェイス上で行われるため、スクリーンには何も描画されなくなります。また、インスタンスの draw イベントでサーフェス自体が画面に描画されないと、何も表示されないことを認識する必要がある。この関数の戻り値も、サーフェスターゲットが設定されたかどうかをチェックするためのデバッグツールとしてチェックすることができる。

注意点としては、サーフェスはスタックされているので、ターゲットからターゲットへジャンプして、最後に通常の描画ターゲットにリセットすることはできず、レンダリングターゲットを開いたり閉じたりする必要があることです。例えば、このような場合、正しく動作しません。

surface_set_target(surf1);
draw_text(32, 32, "surface1");
surface_set_target(surf2);
draw_text(32, 64, "surface2");
surface_reset_target();

その代わり、コードブロックに開閉括弧{}を使うように、設定した面ごとにターゲットを再設定する必要があります。つまり、上記は次のどちらかで記述する必要があります。

surface_set_target(surf1);
draw_text(32, 32, "surface1");
surface_reset_target();
surface_set_target(surf2);
draw_text(32, 64, "surface2");
surface_reset_target();

またはこのように

surface_set_target(surf1);
draw_text(32, 32, "surface1");
surface_set_target(surf2);
draw_text(32, 64, "surface2");
surface_reset_target();
surface_reset_target();

注：サーフェスを使用する場合、texture のメモリーに保存されているため、サーフェスがいつでも存在しなくなる可能性があります。を使用して、サーフェスが存在するかどうかを常に確認する必要があります。 surface_exists()直接参照する前に

構文です。

surface_set_target(surface_id);

論証タイプ 商品説明
surface_id The ID of the surface to set as the drawing target.

リターンです。

例

if view_current = 0
{
    surface_set_target(surf);
    with (obj_effect)
    {
        draw_self();
    }
    surface_reset_target();
}
else
{
    draw_surface(surf, 0, 0);
}

上記のコードは、現在どのビューが描画されているかをチェックし、それが view[0] であれば、描画ターゲットをサーフェスに設定し、object "obj_Effect" のインスタンスをすべて描画してから描画ターゲットを再度リセットしています。ビューが view[0] でない場合、サーフェスはスクリーンに描画されます。
