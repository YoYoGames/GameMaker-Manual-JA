バッファアシンクグループオプション

この機能により、保存/読み込み中のbuffer グループに対して、プラットフォーム固有のオプションを設定することができます。利用可能なオプションは以下の通りです。

  • "subtitle" または "slottitle" - このオプションの値は、string 、OSでセーブデータを管理する際にユーザーに表示されます。これはデータを保存するときのみ重要で、再度ロードするときは重要ではありません。
  • "showdialog" - このオプションの値はブール値です true または false 。 true に設定すると、GameMakerはセーブまたはロード時にシステムUIを表示し、それ以外の場合はUIを表示せずにクイックセーブ/クイックロードを行います。通常、複数のセーブスロットをサポートし、ユーザーがスロットを選べるようにする場合にのみ必要ですが、ユーザーごとに1つのスロットをサポートするだけなら、これを false にセットしてください。
  • 「savepadindex" - このオプションでは、保存またはロードするプレイヤーのパッドインデックスを指定し、システムはこのプレイヤーの保存フォルダにデータを書き込み、そこからデータを読み取ります。
  • "saveslotsize" - このオプションでは、保存する実際のサイズをバイト単位で指定する必要があります (そのため、このオプションでは buffer_seek()と buffer_tell()で取得するなどの方法があります)。すべての保存は最小限のスペースがあらかじめ割り当てられており、それは通常ターゲットプラットフォームによって異なるため、この値を指定することは必須ではないことに注意してください。
  • "ps_create_backup" -(PlayStation のみ)- これを true に設定すると、保存ディレクトリがアンマウントされたとき (すなわち buffer_async_group_end() 関数が呼ばれたとき) に保存データがバックアップされるようになります。詳しくはコンソールのドキュメントをご覧ください。

重要!この機能は現在、PSVita、PS4、XBox One のターゲットモジュールに対してのみ有効です。それ以外のターゲットでは何もしません。

構文です。

buffer_async_group_option(option, value);

論証 タイプ 商品説明
option 文字列 The option to set.
value リアルまたは 文字列 The value to set (can be a real or string, depending on the option).

リターンです。

N/A

例

buffer_async_group_begin("save_folder_name");
buffer_async_group_option("showdialog", false);
buffer_async_group_option("slottitle", "Catch The Haggis Save");
buffer_async_group_option("subtitle", "All your haggis are saved here!");
save = buffer_save_async(buff, "Player_Save.sav", 0, 16384);
buffer_async_group_end();

上記のコードでは、buffer グループを起動し、グループオプションを設定してから、4つのファイルを非同期で保存するように設定しています。その後、グループ定義を終了します（この時点で保存が開始されます）。
