バッファアシンクグループ開始

この関数は、複数のbuffers から複数のファイルへの保存出し（あるいは複数のbuffers の読み込み）を開始したいときに呼び出されます。groupnameはstring で、ファイルが保存されるディレクトリ名として使用されます。また、後でファイルをpath に読み込む際には、ファイルIDE の一部として使用する必要があります（いずれかの buffer_load()関数)を使用します。この関数は buffer_save_async()と buffer_load_async() 関数を呼び、非同期グループも終了させる必要があります。 buffer_async_group_end()関数を呼び出さないと、ファイルの保存/読み込みは行われません。

なお、コンソールプラットフォーム（例えばPS4など）では、グループ名がセーブスロットの説明として使用され、この機能を使用することで、セーブアウトされるファイルごとにUIが表示されることを回避することができます。また、UWPを使用する場合、例えばERA Xboxのグループと同じグループを作成することで、ネイティブXboxとMicrosoftストアのウィンドウ間でセーブゲームをローミングすることができることに注意してください。

if os_type == os_uwp
{
    buffer_async_group_begin("root/yourgroupname");
}
else
{
    buffer_async_group_begin("yourgroupname");
}

構文です。

buffer_async_group_begin(groupname);

論証 タイプ 商品説明
groupname 文字列 The name of the group (as a string).

リターンです。

N/A

例

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

上記のコードでは、buffer グループを起動し、4つのファイルを非同期で保存するように設定します。その後、グループ定義を終了します（この時点で保存が開始されます）。
