バッファピーク

を使えば buffer_read()関数を使うと、与えられたbuffer から現在の「シーク」位置でデータを読むことができ、各データは読み書きのバイト数だけこの位置を進めて読まれます。しかし、あるデータを読むときに、現在のシーク位置を変更したくない場合があります。この関数に、buffer の ID と、そのbuffer 内の読み出しオフセット位置 (buffer の先頭から) と、読み出したいデータ型を指定するだけです。

注意：読み込むデータに対して誤ったデータ型を使用すると、誤った値が表示されます。

関数が成功すれば，データ型に応じた値（Real,string, boolean, etc...）が返され，失敗すれば単に0が返されます。

構文です。

buffer_peek(buffer, offset, type);

論証 タイプ 商品説明
buffer バッファID The index of the buffer to use.
offset リアル The offset position (in bytes) within the buffer to read the given data from.
type バッファデータ型定数 The type of data that is to be read from the buffer (see the list of constants here).

リターンです。

Real, Boolean, String or 0 (if it fails)

例

var red = buffer_peek(buff, 1, buffer_u8);
var green = buffer_peek(buff, 2, buffer_u8);
var blue = buffer_peek(buff, 3, buffer_u8);
image_blend = make_colour_rgb(red, green, blue);

上記のコードは、変数 "buff "にインデックスされたbuffer 内の3つの異なる位置から3つの値を取得し、それらの値を使ってインスタンスの画像ブレンドを設定します。
