バッファロード

を使用して保存したbuffer を読み込むために使用されます。 buffer_save()関数、およびインクルードファイルや外部で読み込まれたファイルです。これは，この関数が "grow"buffer (ここを参照)として，バイトアライメントを1にして作成した新しいバッファのIDを返します．このIDは変数に格納し，このbuffer に対する以降のすべての関数呼び出しで使用されるべきです．何らかの理由で読み込みに失敗した場合、この関数は -1 を返し、コンパイラの出力ウィンドウに読み込みに失敗した旨のメッセージが表示されます。

注意メモリリークを防ぐために、このように動的に作成されたリソースは、不要になった時点でメモリから削除することが重要です。作成したbuffer を使い終わったら、再度、以下を使用してリソースを解放してください。 buffer_delete().

プラットフォーム固有の注意事項

  • HTML5 では、ローカルストレージから読み込んだbuffers は base64 でデコードされます（base64strings として保存されるため）。しかし、Included Files や外部ファイルに対してこの機能を使用すると、base64 形式であることが期待されないため base64 デコードは使用されません。
  • HTML5では、すべてのIncluded FilesはUTF-8エンコードされたテキストとして保存されます。UTF-8エンコードされたバージョンのファイルは、（フォーマットの変更により）元のファイルとは異なるサイズを持つ可能性があるため、元のファイルと比較すると、読み込んだbuffer のサイズが増加する可能性があります。

構文です。

buffer_load(filename);

論証タイプ 商品説明
filename The name of the file to load from.

リターンです。

Buffer ID

例

player_buffer = buffer_load("Player_Save.sav");

上記のコードでは、以前に保存したbuffer をメモリにロードし、新しいbuffer を作成します。この新しいbuffer のインデックスは変数「player_buffer」に格納される。
