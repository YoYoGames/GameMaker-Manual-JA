buffer_get_surface

この関数を使うと、ある表面から与えられたbuffer に情報を書き込むことができます。buffer は以前に作成されている必要があり、書き込もうとするサーフェイスのデータを格納するのに十分な大きさの1バイトアラインのbuffer でなければなりません（不明な場合は、グロウバッファを使用するか、このページの一番下の例を参照してください）。

表面の情報は、（シーク位置に関係なく）常にbuffer の先頭に書き込まれますが、書き込み位置をbuffer の先頭から移動させるオフセット値（バイト単位）を選択することができます。

この関数は、buffer ターゲットでは、表面の各ピクセルをRGBAフォーマットでWindows に書き込むが、他のターゲットでは、OS や、デバイスによって異なる場合があることに注意する。

注：この関数はGameMaker のアップデート 2.3.1 以降で変更されています。以前のバージョンでは、定数 buffer_surface_copy を使用していましたが、現在は非推奨です。

構文です。

buffer_get_surface(buffer, surface, offset);

論証 タイプ 商品説明
buffer バッファID The index of the buffer to use.
surface サーフェスID The index of the surface to use.
offset リアル The data offset value (in bytes).

リターンです。

N/A

例

buffer = buffer_create(surf_width * surf_height * 4, buffer_fixed, 1);
buffer_get_surface(buffer, surface, 0);

このコードは、表面の正確なサイズに基づいて、1ピクセルあたり4バイト（R、G、B、A成分用）を格納する容量を持つ固定サイズのbuffer を作成し、そこに表面データをコピーします。
