キーボード入力

GameMakerでキーボードを扱う場合、押した状態や離した状態など、異なるキーボードの状態を認識するための様々な関数が用意されています。また、全てのキー入力をstring として保存するものや、最後に押したキーが何であったかを知ることができるもの、キーボードの状態を完全にクリアできるものなどがあります。

注意事項これらの関数は、Windows/Mac/Ubuntuのデスクトップ・プラットフォームのみを対象として設計されています。他のプラットフォームでは、組み込みの変数や定数が有効でない場合がありますし、携帯電話では多くの関数が動作しません。

キー（または複数のキー）からの入力文字は、それぞれUTF8コードという数値で定義されています。この値は、任意の文字に対して ord()関数がありますが、GameMakerには、最もよく使われるキーボードの特殊キーと特殊関数のための一連の constants もあります。一般的には、 ord() と keyboard_check*() の関数を組み合わせて、こんな感じで使うことになります。

if keyboard_check(ord("A"))
{
    hspeed = -5;
}

つまり、上記は「A」キーが押されているかどうかをチェックし、押されていればobject の水平速度を -5 に設定するということです。なお、この方法で ord() を使用すると、入力string が 1 文字だけで、0 から 9 までの数字または A から Z までの大文字のローマ字である場合にのみ正しく機能します。関数 ord() は完全な UTF8 値を返しますが、関数 keyboard_check*() はA から Z および 0 から 9 までしか検出しません。しかし、矢印キーを使いたい場合はどうでしょうか。あるいは、「shift」キーを使ってアクションを変更したい場合はどうでしょうか。その場合、GameMakerには vk_ という定数があり、 ord() の代わりに使うことができます（vk_ はvirtual key の略）。

を使用する際のちょっとした例です。 vk_ の定数です。

if keyboard_check_pressed(vk_tab)
{
    instance_create_layer(x, y, "Controllers", obj_Menu);
}

上記のコードは、「Tab」キーが押されたかどうかを検出し、押された場合はobject "obj_Menu" のインスタンスを作成します。

keyboard_lastchar もし、0〜9、A〜Z、またはVK定数以外のキー文字をチェックする必要がある場合は、例えば keyboard_* 変数の1つをチェックする必要があります。

var _key = keyboard_lastchar;
if ord(_key) == ord("ç")
{
    show_debug_message("ç key pressed");
    keyboard_lastchar = "";
}

キーボードでの作業時に使用できる機能は、次のとおりです。

注：画面上の仮想キーボードを使用する場合は、これらの機能は使用できません。

  • io_clear
  • keyboard_check
  • keyboard_check_pressed
  • keyboard_check_released
  • keyboard_check_direct
  • keyboard_clear
  • keyboard_set_map
  • keyboard_get_map
  • keyboard_unset_map
  • keyboard_set_numlock
  • keyboard_get_numlock

また、キー入力をシミュレートするための機能もあります。

  • keyboard_key_press
  • keyboard_key_release

最後に、以下の変数はGameMakerに組み込まれており、アクセスすることでキーボードの状態や入力に関する情報を取得することができます。

注：バーチャルキーボード使用時は keyboard_string変数がキーボード入力で更新されます。

  • keyboard_key
  • keyboard_lastkey
  • keyboard_lastchar
  • keyboard_string
