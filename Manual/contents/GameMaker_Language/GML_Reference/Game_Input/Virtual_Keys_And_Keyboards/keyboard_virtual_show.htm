キーボード_バーチャル_ショー

この関数は、ゲームを実行しているデバイスに仮想キーボードを表示するために使用することができます。この関数を呼び出す際には、最初の3つの引数にそれぞれ以下の定数のいずれかを指定する必要があります。

  • キーボードの種類を指定します。この定数は、仮想キーボードで利用可能なキーセットを設定するために使用されます。利用可能なキーボードタイプは以下の通りです。
仮想キーボードタイプ定数
キーボードタイプ 商品説明
kbv_type_default 現在のシステムにおけるデフォルトのキーボードタイプ。
kbv_type_ascii ASCII配列のみのキーボード。
kbv_type_url URL入力に最適化された通常のキーボード。通常、「.com」などのドメインキーや、「/」「.」キーが搭載されている。
kbv_type_email 電子メール入力用に最適化された通常のキーボードです。通常、"@"や"。"文字が使われています。
kbv_type_numbers 数字のみのキーボードで、通常はテンキーとして表示されます。
kbv_type_phone 電話パッドのキーボードです。通常は「*」と「#」のキーで数字のみ。
kbv_type_phone_name 電話番号と名前の両方を入力するために最適化されたキーボードです。通常、ASCIIキーボードに似ていますが、選択できる特殊文字が限られています。

  • リターンタイプ。この定数は、仮想キーボードのリターンキー/アクションキーに何を表示するかを設定するために使用されます。利用可能なリターンタイプは以下の通りです。
仮想キーボードのリターンタイプ定数
リターンタイプ 商品説明
kbv_returnkey_default 現在のシステムのデフォルトのリターンキータイトル。
kbv_returnkey_go リターンキータイトルを "Go "に設定する。
kbv_returnkey_google リターンキータイトルを "Google "に設定します。場合によっては一般的な検索アイコンに設定します。
kbv_returnkey_join リターンキータイトルを "Go "に設定する。
kbv_returnkey_next リターンキーのタイトルを "Next "に設定する。
kbv_returnkey_route リターンキータイトルを "Route "に設定する。
kbv_returnkey_search リターンキータイトルを "Search "に設定する。場合によっては一般的な検索アイコンに設定する。
kbv_returnkey_send リターンキータイトルを "送信 "に設定する。
kbv_returnkey_yahoo リターンキータイトルを "Yahoo "に、または場合によっては一般的な検索アイコンに設定します。
kbv_returnkey_done リターンキーのタイトルを "Done "に設定します。
kbv_returnkey_continue リターンキーのタイトルを "Continue "に設定する。
kbv_returnkey_emergency リターンキーのタイトルを「緊急通報」に設定する。

  • 自動大文字入力のタイプ。この定数は、仮想キーボードで入力された単語がどのように自動大文字入力されるかを決定するために使用されます。利用可能な自動大文字入力のタイプは以下の通りです。
仮想キーボードの自動大文字入力タイプ定数
オートキャピタリゼーションタイプ 商品説明
kbv_autocapitalize_none 自動大文字入力は無効です。
kbv_autocapitalize_words 単語は自動大文字になります。
kbv_autocapitalize_sentences 文章は自動大文字になります。
kbv_autocapitalize_characters すべての文字が大文字になります。

最後の引数は予測入力の有効・無効で、許可する場合は true に、そうでない場合は false に設定します。ただし、デバイス上のユーザーの設定に依存するため、許可したからといって必ず使用されるわけではないことに注意してください。予測入力モードでは、仮想キーボードは通常のGameMakerのキー入力イベントを発生させません。代わりに、最後に押された文字とキーボードstring 変数のみが更新されます。これは、キー押下を検出するための内部テキストフィールドの変化が、実際の仮想キーボードのキーによるものか、テキストの提案によるものかを検出することができないためです。このような場合、生のキー入力を読み取るのではなく、 keyboard_string の入力を読み取るようにしたいものです。

プラットフォームOSによって、ユーザーが手にするキーボードは異なる機能を持つことになり、ターゲットごとに使用する場合は以下のような注意点があることも知っておく必要があります。

  • UWP: UWPの実装は、 return_key_type, autocapitalization_type, predictive_text_enabled をサポートしていません。これらのパラメータを関数に渡してもエラーにはなりませんが、キーボードに効果がないだけです。

  • Androidです。Android では、キーボードタイプ kbv_type_phone_name はサポートされておらず、代わりに ASCII キーボードが表示されます。リターンキータイプは kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send または kbv_returnkey_done のみです。そのプラットフォームで他のリターンキータイプを使用した場合、システムは以下に示す代替リターンキーをデフォルトとして使用します。
使用できないリターンキー 交換用キー
kbv_returnkey_google
kbv_returnkey_yahoo		 kbv_returnkey_search
kbv_returnkey_join
kbv_returnkey_route
kbv_returnkey_emergency		 kbv_returnkey_go
kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV / FireTV。ASCIIキーボードでは、カスタムリターンキータイプはサポートされていません- デフォルトのリターンキーが常に表示されます。

  • tvOS: Predictive text and auto-capitalization are not supported ontvOS, and system events and the function keyboard_virtual_height() returns by keyboard height due to it spanning entire screen and due to keyboard rect dimension functions being disabled ontvOS, will default to the screen height.また、仮想キーボードが開いていない限り、物理キーボード（Bluetooth）のイベントはブロードキャストされないことに注意してください。

この関数を呼び出すと、システム非同期イベントが発生し、 async_load DS マップに以下のキー/バリューペアが入力されます。

  • "event_type" - 仮想キーボード機能によってトリガーされた場合、"virtual keyboard status" となります。

  • "screen_height" - 仮想キーボードの高さ (ピクセル単位)です。キーボードが見えない場合は、0 になります。

  • "keyboard_status" - キーボードの現在のステータスで、以下の文字列のうちの1つとして返されます。
    • "隠蔽"
    • "非表示"
    • "表示"
    • "可視"

シンタックス

keyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled);

論証 タイプ 商品説明
keyboard_type 仮想キーボードタイプ定数 Determines the keyset available on the virtual keyboard (see tables at the top).
return_key_type 仮想キーボードのリターンタイプ定数 Determines what is shown on the return/action key of the virtual keyboard (see tables at the top).
autocapitalization_type 仮想キーボードの自動大文字入力タイプ定数 Determines how/if the words typed via the virtual keyboard will be autocapitalized (see tables at the top).
predictive_text_enabled ブーリアン Set to true/false 、予測入力の有効/無効を設定します。

例

if input == false
{
    input = true;
    keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false);
}

上記のコードでは、与えられた変数が true に設定されていない場合、OS の仮想キーボードが表示されます。
