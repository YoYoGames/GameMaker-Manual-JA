virtual_key_add

画面領域への「タッチ」をキーボードイベントに対応させることができる機能です。つまり、ある領域を仮想キーに割り当てると、その領域へのタッチはすべて、割り当てたキーに対応するキーボード・イベントをトリガーするようになります。定義した各仮想キーは変数に割り当てることもでき、その変数を使って仮想キーの表示、非表示、削除を行うことができます。これらのキーは部屋ごとに割り当てられ、rooms を変更すると、GameMakerによって自動的に削除されます。

仮想キーの実際の位置は、room の位置ではなく、画面の位置を基準にしているので、x/y 値は画面上の絶対値です。つまり、ビューの使用や相対的なroom 座標を気にする必要がなく、単純にobject のCreate Eventでキーを定義し（バーチャルキーは部屋ごとに一度だけ定義すればよい）、Draw GUI Event でキーsprites を描き、GUI 層のサイズをスクリーンと同じにすることが可能です。

シンタックス

virtual_key_add(x, y, w, h, keycode);

論証 タイプ 商品説明
x リアル The x coordinate (left side) of the virtual key on the screen
y リアル The y coordinate (top side) of the virtual key on the screen
w リアル The width of the virtual key
h リアル The height of the virtual key
keycode 仮想キー定数(vk_*) Which keyboard key event should be triggered by touching this area

リターンです。

Virtual Key ID

例

global.Left = virtual_key_add(32, 32, 64, 64, vk_left);

上記のコードでは、画面上の（32, 32）に64x64ピクセルの正方形の仮想キーを作成し、タッチされるとvk_leftイベントが発生するようにし、この仮想キーのインデックスをグローバル変数に割り当てています。
