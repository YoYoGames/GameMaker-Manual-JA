os_check_permission

この関数を使うと、ユーザーからゲームに対して特定のパーミッションが与えられているかどうかをチェックすることができます。チェックするパーミッションはstring で、 "android.permission.<permission>" という形式で指定します。したがって、 RECORD_AUDIO のパーミッションをチェックするには、（例として）次のようにコールします。

os_check_permission("android.permission.RECORD_AUDIO");

この関数は、以下の3つの定数のいずれかを返し、許可の状態をゲームに伝えます。アプリのパーミッションの詳細については、Androidドキュメントを参照してください。

重要!本機能はAndroidターゲットのみです。

構文です。

os_check_permission(permission)

論証 タイプ 商品説明
permission 文字列 The permission to check (a string)

リターンです。

Permission State Constant

パーミッション状態定数
定数 商品説明
os_permission_granted これは、許可されたことを示します
os_permission_denied これは、許可が下りていないことを表しています
os_permission_denied_dont_request これは、携帯電話の設定によって許可がブロックされているか、ユーザーが以前にその要求を拒否して「二度と要求しない」を選択したことを示します。

例

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

上記のコードでは、OSの種類をチェックし、Android 、パーミッションのチェックを行い、「INTERNET」パーミッションがまだ付与されていなければ、それを要求しています。
