os_get_info

この関数は、ゲームが動作しているOSの詳細情報を DS Mapを、ゲームが動作している OS の詳細情報とともに返します。返される正確な情報はOSやデバイスによって異なるので、対象となるすべてのプラットフォームでこの関数をテストし、 show_debug_message() のような関数を使用して返された値を出力することをお勧めします。DSマップは自動的にメモリからクリアされないことに注意してください。 ds_map_destroy()関数を使用する必要があります。

HTML5以外のすべてのプラットフォームでは、マップは "is64bit" というキーを持ち、ランナーが 64 ビットモードで動作しているかどうかによって、 true または false となることに注意してください。

Windows / UWP

Windows Desktop とWindows UWP では、DX11 の追加データとして、DS マップに以下のキーが用意されています（extensions で便利です）。

  • udid:機械に固有の識別子
  • video_d3d11_device: DX11 デバイスへのポインタ
  • video_d3d11_context: DX11 コンテキストへのポインタ
  • video_d3d11_spawnchain: DX11 スワップチェーンへのポインター
  • video_adapter_vendorid:string アダプタのベンダ ID を含む。
  • video_adapter_deviceid:string アダプタのデバイス ID を含む
  • video_adapter_subsysid:string アダプタのサブシステム ID を持つ。
  • video_adapter_revision:string アダプタのリビジョン番号付き
  • video_adapter_description:string ビデオアダプタの記述あり
  • video_adapter_dedicatedvideomemory:string ビデオメモリサイズで
  • video_adapter_dedicatedsystemmemory:string システムメモリのサイズで（DX11 アダプタで使用）
  • video_adapter_sharedsystemmemory:string 共有システムメモリサイズで

マクロス

この関数は、macOS上でいくつかの限定されたOpenGL情報を返します。

  • udid:機械に固有の識別子
  • gl_vendor_string: GL_VENDOR from OpenGL, example -"Intel Inc.".
  • gl_version_string: GL_VERSION from OpenGL, 例 -"2.1 INTEL-16.2.16".
  • gl_renderer_string: GL_RENDERER OpenGLより、例 -"Intel(R) Iris(TM) Graphics 6100"

ウブントゥ

これには、macOS と同じ情報が含まれています。

PlayStation 4

ここで言及されている機能については、PS4のドキュメントを参照してください。

この機能は、PlayStation 4本体で以下のキーを返すものです。

  • display_safe_area_ratio: sceSystemServiceGetDisplaySafeAreaInfo() が返す浮動小数点値．
  • is_neo_mode:Neo(PS4 Pro)で動作している場合は 1 、通常のPS4で動作している場合は 0 となります。
  • enter_button_assign: sceSystemServiceParamGetInt(SCE_SYSTEM_SERVICE_PARAM_ID_ENTER_BUTTON_ASSIGN, &enterassign) が返す整数値 ; 値が 0 の場合、Circle ボタンが "assign" として使用され、 1 の場合、代わりに X ボタンが使用されることを意味します。

プレイステーション 5

この機能は、PlayStation 5本体で以下のキーを返すものです。

  • display_safe_area_ratio:PS4と同じ（上記参照）。
  • display_resolution: "HD"、"4K "または "不明"。
  • display_dynamic_range: "HDR"、"SDR "または "不明"
  • display_refresh_rate: "120Hz"、"60Hz "または "不明"

Xbox One

Xbox Oneで返される情報は、ほとんどWindows と同じですが、 video_adapter_* と udid のキーは0 です ( video_adapter_description は空ですstring "" を除く)。また、 video_d3d11_swapchain のキーもありません。

また、本プラットフォームでは、以下の情報を返します。

  • is_xbox_one_x:ゲームがXbox One Xで動作している場合は 1 、ベースのXbox Oneで動作している場合は 0 になります。

XboxシリーズX/S

XboxシリーズX/Sコンソールで返される情報は、Xbox Oneとほぼ同じですが、 video_d3d11_* キーがvideo_d3d12_* に置き換えられています。

ここでは、Xbox Oneに記載されているものに加え、このプラットフォームが返すキーをいくつか紹介します。

  • video_d3d12_cmdqueue:DX12コマンドキューを指す
  • video_d3d12_cmdlist:DX12コマンドリストをポイントする
  • video_d3d12_currentrt:現在の DX12 レンダリングターゲットを指す。

例

if (os_type == os_android)
{
    var _info = os_get_info();
    
    if (_info[? "android_tv"])
    {
        global.android_tv = true;
    }
}

上記のコードは、現在のOSがAndroid であるかどうかをチェックし、OS情報を取得し、返されたマップを読んで、ゲームがAndroid のテレビで動作しているかどうかをチェックします。その場合、カスタム変数 global.android_tv に true をセットします。
