os_is_paused

特に、iOS,Android,Windows Phone のデバイスでは、電話や他のアプリによってゲームが中断されることがあります。この機能を使って、デバイスが一時停止しているかどうかを確認することができます。

注目すべきは、この関数はデバイス上でコードブロックに含まれるコードを実行しますが、デバイスがバックグラウンドプロセスとしてゲームをcaches 、その直後にGameMakerが一時停止されることです。そのため、下記のコード例のような場合、キャッシュされた状態からゲームが再起動するまで、作成されるインスタンスは実際に表示されず、イベントも実行されません。つまり、 os_is_paused 関数でアラームを呼び出し、アラームイベント（デバイスの一時停止時に設定されるが、一時停止を解除するまで実行されない）で一時停止メニューや音楽の再開などを作成する方がよいでしょう。

この機能をWindows,Ubuntu (Linux), MacOS, HTML5 プラットフォームで使用する場合、ゲームのフォーカスが外れたときに true に1ステップだけトグルし、その後、ゲームのフォーカスが戻る（そして再び外れる）までは再び false に戻ることになります。これは、これらのプラットフォームでは、cache アプリやゲームがないため、これらがバックグラウンドで実行され続けるからです。

注：モバイル機器では、OSを一時停止すると、サウンドや音楽は停止します。適切な機能で特別に起動しない限り、再び起動することはありません。

構文です。

os_is_paused()

リターンです。

例

if os_is_paused()
{
    if (!instance_exists(obj_PauseMenu))
    {
        instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
    }
}

上記のコードでは、ゲームが一時停止されているかどうかをチェックし、一時停止されている場合は、object "obj_PauseMenu" のインスタンスを作成します（まだ存在しない場合）。
