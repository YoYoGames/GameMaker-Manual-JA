gml_release_mode

ゲームがコンパイルされる際、実行中にエラーチェックを行うための内部関数も含まれています。これらの内部チェックは、範囲、パラメータ、内部GML コンパイラの一般的な側面など、ゲームコードの多くの側面をカバーしており、以下のようなものです。 多くの場合、最終的なエラーメッセージを生成するものです。しかし、この関数を使うと、switch これらの内部チェックをオフにする（つまり、コンパイルされたゲームから完全に取り除く）ことができ、最終プロジェクトにスピードアップを与えることができます。

このため、ゲームコードの中で何か間違ったことをすると、コンパイルされたゲームが単にクラッシュしたり、非常に予期しない動作が発生する可能性があることに注意してください。たとえば、 DS Grid にアクセスした場合。 の座標が定義されたサイズから外れている場合、開発者モードではコンパイラーウィンドウのエラーメッセージが表示されますが、チェックが行われているため、ゲームの実行を継続することができます。しかし、リリースモードではこのようなチェックは行われないので、コードは次のようになります。 はNULLポインタか不正なポインタを返し、そのポインタはランダムなメモリーを指すことになります。そのため、最悪の場合、間違ったデータが返されたり、メモリ内のデータ、グラフィック、コードそのものを上書きしてしまう可能性があります。他の例として もし、色、位置、texture などのデータを、vertex フォーマットの作成時に定義した順序と異なる順序で供給すると、リリースモード時にゲームがクラッシュしたり、予期しない結果をもたらす可能性があります。 がオンになっている。

構文です。

gml_release_mode(flag);

論証タイプ 商品説明
flag Either true でリリースモードを、false で一般的な開発モードを設定します。

リターンです。

例

gml_release_mode(true);

上記のサンプルコードでは、コンパイル時にリリースモードを使用するように設定します。
