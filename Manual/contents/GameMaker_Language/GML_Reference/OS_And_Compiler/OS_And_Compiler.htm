OSとコンパイラ

クロスプラットフォームゲームを作成する場合、ゲームを実行しているデバイスが使用しているオペレーティングシステムの詳細を取得することが極めて重要になる場合があります。言語、バージョン、ネットワーク状態など、ゲームを実行するシステムに適応させ、エンドユーザーの体験を最高のものにするために、GameMakerには必要な情報を取得するための特別な機能が多数用意されています。

以下の関数と変数は、ゲームを実行しているオペレーティングシステムやブラウザに関する特定の情報を取得するために使用するために存在します。

  • os_browser
  • os_device
  • os_type
  • os_version
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

ある種の関数はゲーム実行時にプリコンパイルされます（つまり、最終的なゲームのコンパイル方法を定義するために使用されます）。また、いくつかの関数はゲームがコンパイルされたときに、runtime 環境に関する特定の詳細を取得するために使用されます。ここでは、これらの関数の全リストを見ることができます。

  • GM_build_date
  • GM_build_type
  • GM_version
  • GM_runtime_version
  • GM_project_filename（プロジェクト名
  • gml_release_mode
  • gml_pragma
  • parameter_count
  • parameter_string
  • environment_get_variable

Windows では、ゲーム実行時のWindows スレッドスケジューラの解像度はデフォルトで 1ms に設定されています。実行時にWindows のスレッドスケジューラの解像度を取得・変更するには、次の関数を使用し ます。

  • スケジューラ解像度の取得
  • スケジューラレゾリューションセット
