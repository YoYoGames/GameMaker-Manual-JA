os_check_permission

この機能により、特定のパーミッションを要求することができます。要求するパーミッションはstring で、 "android.permission.<permission>" という形式で指定します。したがって、 RECORD_AUDIO というパーミッションを要求するには (たとえば)

os_request_permission("android.permission.RECORD_AUDIO");

この関数は、システム非同期イベントをトリガーして、内蔵の async_loadDSマップには、以下のキーと値のペアが含まれる。

  • "type"を指定します。これはトリガーされたイベントのタイプで、値はstring "permission_request_result" となる。
  • "<permission>" です。これは、関数が要求したパーミッションをstring で表したもので、以下の定数のいずれかが含まれます。
定数 商品説明
os_permission_granted これは、許可されたことを示します
os_permission_denied これは、許可が下りていないことを表しています

なお、以下のパーミッションはGameMakerがネイティブでサポートしていますが、Googleが「危険」と判断したため、明示的に要求する必要があります（この機能がなくてもAndroid Game Optionsで要求できるパーミッションもあることに注意してください）。

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

アプリの権限について詳しくは、Androidドキュメントをご覧ください。

重要!本機能はAndroidターゲットのみです。

構文です。

os_request_permission(permission)

論証 タイプ 商品説明
permission 文字列 The permission to request (a string)

リターンです。

N/A

例

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

上記のコードでは、OSの種類をチェックし、Android 、パーミッションのチェックを行い、「INTERNET」パーミッションがまだ付与されていなければ、それを要求しています。
