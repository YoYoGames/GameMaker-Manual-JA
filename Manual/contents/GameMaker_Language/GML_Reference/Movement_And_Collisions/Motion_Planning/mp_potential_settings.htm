mp_potential_settings

mp_potential_ 関数は、この関数で変更可能ないくつかのパラメータを使用して動作します。全体として、このメソッドは次のように動作する。まず、ゴールに向かってまっすぐ進もうとし、パラメータ "ahead" で設定可能なステップ数だけ先を見る（デフォルト値は 3）。この値を小さくすると、前方が見えにくくなるため方向転換が遅くなり、大きくすると方向転換が早くなります。このチェックが衝突につながる場合は、理想的な直進方向からさらに左と右の方向を見るようになります。これはサイズ "rotstep "のステップで行われ（デフォルト値は10）、これを減らすとインスタンスはより多くの動きの可能性を得ますが、より多くの処理能力を必要とするのでより遅くなります。

パラメータ "maxrot "は、少し説明が難しいです。インスタンスは現在の移動方向を持ち、maxrot（デフォルト値は30）は、1ステップでその現在の方向から左右にどれだけ方向を変えることができるかを示します。つまり、例えばゴールに向かってまっすぐ進むことができても、この最大方向転換量に違反しない限り、そうすることができる。maxrot を大きな値にすると、インスタンスは各ステップで大きく方向を変えることができ、短いpath を見つけるのが容易になりますが、path はより醜く、自然でなくなります。この値を小さくすると、path は滑らかになりますが、より長い回り道をする可能性があります（時にはゴールを見つけるのに失敗することもあります）。最後に、ステップを作れないときの動作は、パラメータ「onspot」の値に依存する。onspotがtrue（デフォルト値）の場合、インスタンスはそのスポットで "maxrot "で示される量だけ回転し、falseの場合は全く動きません。falseに設定すると、車などの場合に便利ですが、インスタンスがpath を見つける可能性が低くなります。

構文です。

mp_potential_settings(maxrot, rotstep, ahead, onspot)

論証 タイプ 商品説明
maxrot リアル The number of degrees either side of the current direction that the instance can rotate in a step
rotstep リアル The number of degrees either side of the current direction that the instance can check for a collision
ahead リアル The number of steps ahead that the instance can check for a collision. Larger values are slower than lower ones.
onspot ブーリアン Use this to allow the instance to rotate on the spot when no path is found (true) or not (false)

リターンです。

N/A

例

mp_potential_settings(45, 5, 5, 0);

上記のコードでは、 mp_potential_ 関数で、インスタンスが 5 歩先を見て、1 歩ごとに 45 度方向を変え、障害物がある場合は現在の方向から左右に 5 度見て、衝突があった場合はその場で回転しない、という設定を使用するようにしました。
