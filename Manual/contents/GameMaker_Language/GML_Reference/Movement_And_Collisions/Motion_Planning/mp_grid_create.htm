mp_grid_create

この関数で、モーションプランニング関数用の mp_grid を作成することができます。この関数は、他のすべての MP グリッド関数のコールで使用するために、変数に格納されるべきインデックスを返します。x 座標と y 座標はグリッドの左上隅の位置を、hcells と vcells はグリッド内に作成する水平および垂直セルの数を、cell width と cell height はセルの大きさをピクセル単位で示す。異なる用途で必要であれば、複数のグリッド構造を同時に作成・管理することもできますが、各グリッドにはメモリが必要であり、セル数が多いほど必要なメモリ量も多くなることに注意してください。また、MP 関数はかなりプロセッサ負荷が高いので、頻繁に呼び出すほど、またセルの解像度が高いほど、パフォーマンスが低下する可能性が高くなることに注意してください。

構文です。

mp_grid_create(left, top, hcells, vcells, cellwidth, cellheight);

論証 タイプ 商品説明
left リアル Starting x coordinate of the mp_grid in the room
top リアル Starting y coordinate of the mp_grid in the room
hcells リアル Number of horizontal cells that the mp_grid will contain
vcells リアル Number of vertical cells that the mp_grid will contain
cellwidth リアル The width (in pixels) of each individual cell of the mp_grid
cellheight リアル The height (in pixels) of each individual cell of the mp_grid

リターンです。

MP Grid ID

例

global.grid = mp_grid_create(0, 0, room_width / 32, room_height / 32, 32, 32);

上記のコードでは、グローバル変数 "grid" を作成し、mp_grid を生成してそのインデックス (id) を変数に代入し、以降のすべての mp_grid 関数呼び出しで使用します。グリッドはroom をカバーし、セル解像度は 32x32 ピクセルである。これは、例えば、640x480 のroom の場合、グリッドは 300 個のセルを含むことを意味します：20 の水平セル (640 / 32) と 15 の垂直セル (480 / 32) を掛け合わせると 300 個のセルとなります。
