mp_potential_path

この関数は、インスタンスの現在の位置と向きから、引数 xgoal,ygoal で指定された位置までのpath を計算する。この関数は、すべてのインスタンス、または、指定されたインスタンスのみとの衝突を避けようとしながら、指定されたステップサイズを使用します。 というフラグが立っている。関数 mp_potential_step()は，これを達成するためにポテンシャルフィールドのステップを使用し，その方法は関数 mp_potential_settings(). 指定されたpath はすでに存在している必要があり、それは新しいpath によって上書きされます。この関数は，path が見つかったか（true），見つからなかったか（false）を返します．

path この関数が永遠に計算を続けることを避けるためには、1より大きな係数を指定する必要があります。この係数の倍数より短いスタートとゴールの間の距離が見つからない場合、この関数は停止し、失敗を報告します。4の係数 通常はこれで十分ですが、長い回り道が予想される場合は、もっと長くしてもよいでしょう。もし失敗してもpath は作成され、ゴールのおおよその方向に走りますが、ゴールにはたどり着けません。

注：この関数はインスタンスを移動させません。これはpath を設定するだけで、移動にはPath関数を使用する必要があります。

構文です。

mp_potential_path(path, xgoal, ygoal, stepsize, factor, checkall)

論証タイプ 商品説明
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
checkall Whether to check all instances (true) or just solid ones (false).

リターンです。

例

path = path_add();
mp_potential_path(path, obj_Player.x, obj_Player.y, 3, 4, 0);
path_start(path, 3, 0, 0);

上記のコードでは、新しいpath リソースを作成し、そのインデックスを変数 "path" に格納しています。そして、このコードによるpath とobject "obj_Player" の間に mp_potential_path を使ってobject を生成し、衝突をチェックします。 room 最後に、この関数が "obj_Player "への完全なルートを返せなかった場合でも、そのobject に沿ってpath を開始します。
