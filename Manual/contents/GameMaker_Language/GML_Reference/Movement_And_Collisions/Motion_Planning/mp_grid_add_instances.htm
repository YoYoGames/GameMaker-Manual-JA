mp_grid_add_instances

この関数は、インスタンスのコリジョンマスク（sprite プロパティまたは呼び出したインスタンスのマスクインデックスで決定）を使用して、MP グリッド内のセルを禁止または禁止でないとマークします。禁止セルは、どの経路探索関数でも横切ることができないようになっています。この関数では、精密な衝突を考慮するかどうかを指定することができ、この設定に応じてマークされる禁止セルが変化します。この画像は、この挙動を説明するものです。

上の2つのインスタンスは、 mp_grid_add_instances() 機能を使用して、precise を true に設定して MP グリッドに追加されています。ご覧のように、緑のインスタンスは、sprite のマスクが precise に設定されているため、「触れる」グリッドのマス目のみを禁じ手としてマークしています。しかし、2番目のインスタンスは他のマスもマークしていますが、そのマスは触れていないように見えます。これは、sprite のマスクが precise に設定されていないためです。つまり、関数で precise 設定を使用していても、sprite の bounding box しか考慮されないということです。これは、関数の precise 引数を false に設定した場合にも起こることです。precisesprite マスクを持つインスタンスでさえ、そのバウンディングボックスを基にグリッドに追加されることになります。

構文です。

mp_grid_add_instances(id, obj, prec);

論証 タイプ 商品説明
id MP Grid ID Index of the mp_grid that is to be used
obj オブジェクトのアセットまたはインスタンスID Object index, or instance id, of the instances to be added into the mp_grid
prec ブーリアン Whether the check is based on pixel-perfect collisions (true = slow) or its bounding box in general (false = fast).

リターンです。

N/A

例

mp_grid_add_instances(grid, obj_Wall, true);

上記のコードは、"obj_Wall" のすべてのインスタンスを、バウンディングボックスではなく、正確なコリジョンマスクを使用して、変数 "grid" でインデックスされた mp_grid に追加します。
