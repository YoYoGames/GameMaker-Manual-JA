mp_grid_add_cell

この機能により、与えられたMPグリッド内の個々のセルを「forbidden（禁止）」としてマークすることができます。つまり、path 検索関数が決してそのセルを横切らないようにすることができます。さて、通常は、MPグリッドのセル座標ではなく、x/yの部屋座標で作業します。そこで、この2つがどのように関連するかを示す次の画像について考えてみましょう。

ご覧のように mp_grid は一連のセルとして番号付けされており（左上の 0,0 から始まる）、このグリッドをroom の上に配置すると、room の座標（これも左上の 0,0 から始まる）とグリッド内のセルには直接的な相関があります。画像では、位置260,130のインスタンスがグリッドセル8,4に該当していることがわかります。グリッド解像度で数値を割って切り捨てるか、グリッド解像度の 2 乗（8、16、32 など）を使用している場合は、ビット演算子を使用すると、位置が該当するセルを簡単に数学的に計算することができます。なお、ビット演算の方が高速であり、以下の例では両方の方法を示しています。

構文です。

mp_grid_add_cell(id, h, v);

論証 タイプ 商品説明
id MP Grid ID Index of the mp_grid that is to be used
h リアル Horizontal position of the cell in the grid
v リアル Vertical position of the cell in the grid

リターンです。

N/A

例

with (obj_Wall)
{
    mp_grid_add_cell(grid, floor(x / 32), floor(y / 32));
}
with (obj_Door)
{
    mp_grid_add_cell(grid, x>>5, y>>5);
}

上記のコードは、すべての "obj_Wall" とすべての "obj_Door" が、変数 "grid" でインデックスされた mp_grid に、同等のセル位置を追加するようにします。この例の最初の部分では、object の x/y 座標をとり、グリッドの分解能で割ることによって行われます ( floor を使って値を整数として保ちます)。2番目の部分では (グリッドの分解能は2のべき乗なので) x,y 座標を5箇所右にビットシフトすることによって行われます。
