コリジョン矩形

Collision_rectangle は、最初の 4 つの引数 (x1,y1,x2,y2) を使って現在の 内の領域を定義し、次に "obj" 引数で定義されるroom 任意の object がその領域と衝突しているかどうかをチェックします。この衝突は正確にチェックするかしないか、また、コードを実行しているインスタンス自体をチェックするかしないかを選択することができます。この画像を考えてみましょう。

ここでは、真ん中のインスタンスがコリジョンレクタングルを使用して、ボールobjects をチェックします。さて、青いものは正確なバウンディングボックスを持っていません。ご覧のように、sprite が実際には矩形に触れていなくても、そのsprite のバウンディングボックスが collision_rectangle をオーバーラップするので、衝突は（関数内の precise オプションを true に設定しても）起こります。一方、緑のボールは、実際のsprite が矩形をオーバーラップしている場合にのみ衝突が考慮されます。正確な衝突を考慮するためには、object sprite と collision 関数の両方でprecise が on になっている必要があることを忘れないでください。また、この関数の戻り値には、衝突とみなされるインスタンスの ID を指定することができることにも注意が必要です。

object インデックスの代わりにインスタンスを指定することができます。 idで特定のインスタンスをチェックするか、インスタンスキーワード all, または other (イベントと現在のコードスコープによる) をチェックします。

構文です。

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

論証 タイプ 商品説明
x1 リアル The x coordinate of the left side of the rectangle to check.
y1 リアル The y coordinate of the top side of the rectangle to check.
x2 リアル The x coordinate of the right side of the rectangle to check.
y2 リアル The y coordinate of the bottom side of the rectangle to check.
obj オブジェクトのアセットまたはインスタンスID The object to check for instance collisions.
prec ブーリアン Whether the check is based on precise collisions (trueより遅い）、またはそのバウンディングボックス全般（false 、より速い）。
notme ブーリアン Whether the calling instance, if relevant, should be excluded (true) か、そうでないか (false) です。

リターンです。

Instance ID or noone

例

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if (inst != noone)
{
    instance_destroy(inst);
}

この短いコードは collision_rectangle を使ってroom の 50x, 50y（矩形の左上）から 200x, 200y（矩形の右下）までの範囲で "obj_ball" というobject のインスタンスをチェックするものです。戻り値は一時変数に格納され、その値がインスタンスIDであるか、またはキーワード「obj_ball」であるかをチェックします。 なし.もしそれが誰でも ない場合は、保存されているインスタンスIDを使用してobject を破棄します。
