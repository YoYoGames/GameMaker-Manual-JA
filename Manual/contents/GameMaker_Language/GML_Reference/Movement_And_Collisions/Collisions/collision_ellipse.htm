コリジョン_エリプス

Collision_ellipse は、最初の 4 つの引数 (x1,y1,x2,y2) を使って現在の 内の楕円の幅と高さを定義し、次に引数 "obj" で定義されているroom object がその領域と衝突しているかどうかをチェックします。この衝突は正確にチェックするかしないか、また、コードを実行しているインスタンス自体をチェックするかしないかを選択することができます。この画像を考えてみましょう。

ここでは、真ん中のインスタンスが衝突楕円を使用して、ボールobjects をチェックしています。さて、青いものは正確なバウンディングボックスを持っていません。ご覧のように、sprite が実際には楕円に触れていなくても、そのsprite のバウンディングボックスが collision_circle で定義された楕円の領域をオーバーラップするので、（関数内の precise オプションを true にしても）衝突は起こりえます。一方、緑のボールは、実際のsprite が定義された楕円をオーバーラップしている場合にのみ衝突が考慮されます。正確な衝突を考慮するためには、object sprite と collision 関数の両方でprecise が on になっている必要があることを忘れないでください。また、この関数の戻り値には、衝突とみなされるインスタンスの ID を指定することができることにも注意が必要です。

object インデックスの代わりにインスタンスを指定することができます。 idで特定のインスタンスをチェックするか、インスタンスキーワード all, または other (イベントと現在のコードスコープによる) をチェックします。

構文です。

collision_ellipse(x1, y1, x2, y2, obj, prec, notme);

論証 タイプ 商品説明
x1 リアル The x coordinate of the left side of the ellipse to check.
y1 リアル The y coordinate of the top side of the ellipse to check.
x2 リアル The x coordinate of the right side of the ellipse to check.
y2 リアル The y coordinate of the bottom side of the ellipse to check.
obj オブジェクトのアセットまたはインスタンスID The object to check for instance collisions.
prec ブーリアン Whether the check is based on precise collisions (trueより遅い）、またはそのバウンディングボックス全般（false 、より速い）。
notme ブーリアン Whether the calling instance, if relevant, should be excluded (true) か、そうでないか (false) です。

リターンです。

Instance ID or noone

例

if collision_ellipse(50, 50, 200, 100, obj_Player, false, true)
{
    instance_create_layer(obj_Player.x, obj_Player.y, "Effects", obj_Splash);
}

これはobject "obj_Player" の 50x, 50y と 200x, 100y の範囲にある楕円形のゾーンをチェックします。もしそのobject 、衝突があれば、obj_Playerのx/y座標に "obj_Splash "のインスタンスが生成されます。
