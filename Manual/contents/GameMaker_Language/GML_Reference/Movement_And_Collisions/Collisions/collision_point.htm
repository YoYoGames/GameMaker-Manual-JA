collision_point

Collision_point は、引数 x1,y1 で指定された点と、引数 "obj" で指定された のobject 任意のインスタンスとの衝突をチェックします。このチェックは精密でもそうでなくても構いませんが、精密衝突を有効にするには、チェックする またはインスタンスがその に対して精密衝突も有効でobject spriteなければなりません。そうでない場合は、デフォルトのチェックがバウンディングボックスに基づいて実行されます。次の画像は、この動作を示しています。

精密衝突を考慮するためには、object sprite と collision 関数の両方に、on としてマークされている必要があることに注意してください。また、関数の戻り値は、衝突とみなされたインスタンスのいずれかのidになる可能性があることにも注意が必要です。したがって、その時点で3つのインスタンスが重なっていれば、それらのidのいずれかが関数の戻り値になる可能性があります。

object インデックスの代わりにインスタンスを指定することができます。 idで特定のインスタンスをチェックするか、インスタンスキーワード all, または other (イベントと現在のコードスコープによる) をチェックします。

構文です。

collision_point(x, y, obj, prec, notme);

論証 タイプ 商品説明
x リアル The x coordinate of the point to check.
y リアル The y coordinate of the point to check.
obj オブジェクトのアセットまたはインスタンスID The object to check for instance collisions.
prec ブーリアン Whether the check is based on precise collisions (trueより遅い）、またはそのバウンディングボックス全般（false 、より速い）。
notme ブーリアン Whether the calling instance, if relevant, should be excluded (true) か、そうでないか (false) です。

リターンです。

Instance ID or noone

例

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10S;
}

ここでは、object "obj_Cursor" のコードを持つobject の位置のポイントをチェックしています。もしあれば、score変数に10を加算している。
