コリジョンラインリスト

この関数は collision_line()関数は、衝突しているインスタンスを一度に1つだけ検出するのではなく、複数のインスタンスを検出することだけが特徴です。チェックするラインの開始と終了の x/y 位置と、チェックするobject を指定します。また、チェックを正確に行うか (この場合、チェックするすべてのインスタンスが正確なコリジョンマスクを持つ必要があります) と、チェックに呼び出し側のインスタンスを含めるかどうかを設定できます。

DSのリストも渡すので id呼び出したインスタンスと衝突しているインスタンスの値は、与えられたリストの末尾に追加されます。この関数を呼び出す前にリストをクリアして、この関数の呼び出しによる結果のみを含むようにすることができます。また、線の中心から原点までの距離に基づいてインスタンスを並べるオプションもあります。この関数は、見つかったインスタンスの数を返し、何も見つからなかった場合は 0 を返します。

object インデックスの代わりにインスタンスキーワード all を指定すると、現在のroom にあるすべてのインスタンスをチェックすることができます。このインスタンスには、（ notme 引数に依存する）コードを実行しているインスタンスが含まれる場合があります。

構文です。

collision_line_list(x1, y1, x2, y2, obj, prec, notme, list, ordered);

論証 タイプ 商品説明
x1 リアル The x coordinate of the start of the line.
y1 リアル The y coordinate of the start of the line.
x2 リアル The x coordinate of the end of the line.
y2 リアル The y coordinate of the end of the line.
obj オブジェクトのアセットまたはインスタンスID The object to check for instance collisions.
prec ブーリアン Whether the check is based on precise collisions (trueより遅い）、またはそのバウンディングボックス全般（false 、より速い）。
notme ブーリアン Whether the calling instance, if relevant, should be excluded (true) か、そうでないか (false) です。
list DSリストID The DS list to use to store the IDs of colliding instances.
ordered ブーリアン Whether the list should be ordered by distance (true) か、そうでないか (false) です。

リターンです。

Real (The number of instances found to be in collision)

例

var _list = ds_list_create();
var _num = collision_line_list(x - 100, y, x + 100, y, obj_Enemy, false, true, _list, false);
if _num > 0
{
    for (var i = 0; i < _num; ++i;)
    {
        instance_destroy(_list[| i]);
    }
}
ds_list_destroy(_list);

上記のコードでは、インスタンスの呼び出し位置から左右100ピクセルのラインをチェックし、「obj_Enemy」のインスタンスとの衝突を確認します。衝突があった場合、事前に作成されたリストがループされ、衝突にあった各インスタンスが破壊されます。
