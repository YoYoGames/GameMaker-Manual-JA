ネットワーキング

GameMakerのネットワーク機能は、すべてのプラットフォームで業界標準となっているソケットをベースにしています。ソケットを使えば、簡単なservers 、クライアントとGML の両方を作成することができ、クライアント/サーバーモデルを使って一人用のゲームも作ることができます（後で簡単にマルチプレイを追加できるようにするためのものです）。

基本的に、ソケットは、ネットワーク上のポートを送受信、接続、リスンすることができるobject 。これらの機能は、インターネットが基づいているTCP/IP接続だけを扱います（より正確にはIPv4）。要するに、あるIPアドレス（例えば192.168.1.10）を使ったソケットを、別のIPアドレス（例えば192.168.1.11）を使ったソケットに接続することができるのです。いったん接続されると、これらの2つのソケットは、 network_send_* 関数 (たとえば network_send_packet()のような）関数と、それによって引き起こされる専用の非同期イベントを使ってデータを送り返すことができます。この非同期イベントは、受信したデータとその他の詳細を含むDS マップを生成します。完全な詳細については、ネットワーク非同期イベントのセクションを参照してください。

IPアドレスは、プログラムがマシンに入ってくるネットワークトラフィックをすべて読み込んで処理しなければならない代わりに、0から65535までの番号が振られたポートも処理する。これにより、各プログラムは、すべてのプログラムから送信されたデータではなく、興味のあるデータのパケットのみを取得することができる。これはまた、データ処理を真剣に削減するため、かなりのCPU時間を節約することができますので、GameMakerでは、ポートを「リッスン」することも可能です。つまり、server 、ソケットを作成し、それを特定のポートにバインドしてリッスンする。そして、そのソケットからの接続、切断、データ転送の通知を受けることができる。

データ「パケット」自体は、buffer から取り出したバイナリデータで構成されている。 このbuffer は、GameMaker の Buffer 関数を使って作成、操作することができる。

HTML5 をターゲットにしたプロジェクトを作成する場合、ブラウザの制限によりserver の作成関数を使用できず、Electron やNodeJS などを使ってこのプラットフォーム用に独自のserver を作成する必要があることに注意してください。また、ゲームがクロスプラットフォームで、モバイルまたはデスクトップデバイスで動作するserver からゲームの HTML5 インスタンスに接続することを想定している場合、server は Web Socket プロトコルを使用してのみ HTML5 インスタンスと通信できます（これは定数 network_socket_ws またはnetwork_socket_wss を使用して行われます - 詳細については、 を参照してください）。 network_create_server()を参照してください）。つまり、server は、HTML5 以外のターゲット (Android,Windows など) に接続するソケットと HTML5 ターゲットに接続するソケットの 2 つを異なるポートで作成しなければならない場合があります。
