network_send_raw

この関数を使うと、ネットワークを通じて「生の」データパケットを送信することができます。この関数は接続するソケット IDを受け取り、次に送信するデータを格納するバッファ ID(buffers についての詳細はリファレンス - バッファを参照)、最後にデータパケットのサイズ (バイト単位) を指定する必要があります。送信されるデータはGameMakerによってフォーマットされることはなく、受信側のデバイスはストリームとしてデータを受け取るので、自分で処理する必要がある。この関数は送信したデータのバイト数を返すが、送信に失敗した場合は0より小さい値を返す。

オプション引数

最後の引数はオプションであり、WebSocketでのみ使用されます。これにより、バイナリデータまたはテキストデータのどちらを送信するかを選択することができます。この引数には、どちらかの定数を指定することができます。

ネットワーク送信タイプ定数
定数 商品説明
network_send_binary バイナリーメッセージを送信する
network_send_text テキストメッセージを送信する

一部のプラットフォームの API は、WebSocket を使用する場合、テキストメッセージしか受け付けないため（Twitch など）、そのような場合はnetwork_send_text定数を使用することができます。この引数を指定しない場合、デフォルトでバイナリデータが送信されます。

構文です。

network_send_raw(socket, bufferid, size, [options]);

論証 タイプ 商品説明
socket ネットワークソケットID The id of the socket to use.
bufferid バッファID The id of the buffer to get the data from.
size リアル The size (in bytes) of the data.
options ネットワーク送信タイプ定数 OPTIONAL WebSocketでテキストデータとバイナリデータのどちらかを選択するために使用され、指定しない場合はバイナリデータが送信されます。

リターンです。

Real

例

buff = buffer_load("player_save.dat");
network_send_raw(sock, buff, buffer_get_size(buff));

上記の情報は、あらかじめ保存しておいたbuffer のデータをメモリにロードし、変数 "buff" に格納するbuffer の id を返す。この完全なbuffer は、変数 "sock "で識別されるソケットを使用して、ネットワーク上で生のデータパケットとして送信されます。
