gif_add_surface

この関数を使用すると、サーフェスからGIFファイルにフレームを保存することができます。GIFファイルID（関数 gif_open()) とサーフェスID (関数 surface_create()) と、フレーム間の遅延時間です。遅延時間は1/100秒単位で計算されますが、いったんGIFが作成されると、たとえばプロモーション目的で使用する場合、ブラウザによってフレーム遅延の解釈が若干異なる可能性があることに注意してください。この場合、GIF に描画される部分はデフォルトの (0,0) 位置ではなくオフセットから始まり、追加するサーフェスのセクションのみを選択できるようになります（オプションで）。量子化値は 0 から 3 までの整数で、0 は完全量子化、3 はゼロ量子化です (低い値ほど品質が落ちますが、より小さな GIF が作成されます)。デフォルトの品質である 2 と最高の品質である 3 の間にはあまり差がなく、3 を使用すると GIF の作成が大幅に遅くなるため、最大値を使用する場合は注意が必要です。

フレーム数やGIFのフレームサイズに組み込みの制限はありませんが、メモリを使いすぎるとこの関数は失敗して-1を返し、そうでない場合は0を返すことに注意してください。

構文です。

gif_add_surface(gif_index, surface, delay_time, [xoffset], [yoffset]);

論証 タイプ 商品説明
gif_index GIF ID The ID of gif to add the surface data to
surface サーフェスID The ID of the surface to use as the added frame
delay_time リアル The delay time for the frame (in 100ths of a second)
[xoffset] リアル OPTIONAL から追加されるサーフェスのX軸に沿ったオフセット。
[yoffset] リアル OPTIONAL から追加されるサーフェスのY軸に沿ったオフセット。
[quantization] リアル OPTIONAL 0〜3までの量子化量（0が最低品質、3が最高品質、デフォルトは2）

リターンです。

Real

例

if save_gif == true
{
    if count == 0
    {
        gif_image = gif_open(room_width, room_height);
    }
    else if count < 30
    {
        gif_add_surface(gif_image, application_surface, 6/100);
    }
    else
    {
        gif_save(gif_image, "GameCapture.gif");
        count = 0;
        save_gif = false;
    }
    count++;
}

上記のコードでは、アプリケーションの表面から30フレームを取り込んだGIF画像ファイルを作成し、保存します。
