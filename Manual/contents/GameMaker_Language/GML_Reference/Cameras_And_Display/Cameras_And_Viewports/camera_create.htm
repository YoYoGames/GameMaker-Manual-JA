カメラ作成

この関数は "空の "カメラ、つまりビューや投影の設定が何も定義されていないカメラを作成します。カメラを作成した後、さまざまな行列関数を使用して、ビューおよび投影行列を設定することができます。 camera_set_* ただし、少なくとも位置とビューサイズを指定しなければ、作成されたカメラは有効ではなく、ビューポートに適用するとおかしな結果になることに注意してください (object 以下、境界線、移動速度を設定しない場合、これらはRoom エディタでビューカメラのデフォルト値に設定されます)。

カメラは他のダイナミックリソースと同様に、作成時にメモリを占有することに注意してください。つまり、不必要にカメラを作成しないように注意し、不要なカメラは関数 camera_destroy().これは、カメラが global 変数に割り当てられた場合、room からroom まで持続することも意味します（インスタンス変数やローカル変数に割り当てられた場合も持続しますが、room を変更するとアクセスできなくなるので、メモリリークが発生します）。したがって、たとえば、新しいカメラを作成してグローバル変数に割り当て、各room の開始時に、必要なビューおよび投影マトリクスと使用するビューポートを設定することが可能です。

構文です。

camera_create();

リターンです。

Camera ID

例

view_camera[0] = camera_create();
var viewmat = matrix_build_lookat(640, 240, -10, 640, 240, 0, 0, 1, 0);
var projmat = matrix_build_projection_ortho(640, 480, 1.0, 32000.0);
camera_set_view_mat(view_camera[0], viewmat);
camera_set_proj_mat(view_camera[0], projmat);

上記のコードでは、新しいカメラを作成し、そのIDをview[0]に代入しています。その後、ビューマトリックスとプロジェクションマトリックスを作成し、ローカル変数に格納してから、カメラの設定に使用します。
