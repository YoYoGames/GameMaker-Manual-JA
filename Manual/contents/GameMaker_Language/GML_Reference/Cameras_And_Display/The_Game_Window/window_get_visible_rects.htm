window_get_visible_rects

この関数を使うと、添付の各ディスプレイ上で(x1,y1)から(x2,y2)で定義される矩形の重なり領域を求めることができます。この関数は，ディスプレイごとに 8 個の値を持つ配列を返します（つまり，ディスプレイが 2 台ある場合は，その配列は 0 ... 3]はoverlapx1, overlapy1, overlapx2, overlapy2 - このディスプレイの重なり領域を定義し，重ならない場合は 0,0,0 にセットされる - に対応し，[4 ... 7]の値は，このディスプレイの重なり領域に対応します． monitorx1, monitory1, monitorx2, monitory2 - 仮想ディスプレイ空間におけるディスプレイの座標。これは、保存されたウィンドウの位置が表示されるかどうかをテストするために使用できます（ユーザーが外部モニタを切断したか、または ウィンドウを画面外に移動させ、保存されたウィンドウの位置が有効でない状態になった）などがあります。

構文です。

window_get_visible_rects(x1, y1, x2, y2);

論証タイプ 商品説明
x1 The left edge of the rectangle to check
y1 The top edge of the rectangle to check.
x2 The right edge of the rectangle to check
y2 The bottom edge of the rectangle to check.

リターンです。

例

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length(display_data) / 8;

上記のコードでは、ディスプレイに関する情報を含む1次元配列を変数 display_data に生成し、見つかったアクティブなディスプレイの数を保持する変数 display_num を生成します。
