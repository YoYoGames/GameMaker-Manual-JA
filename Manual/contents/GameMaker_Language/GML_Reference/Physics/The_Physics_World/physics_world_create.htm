physics_world_create

objects の物理を定義しても、それを置く世界がなければ、GameMaker の伝統的なコリジョンシステムがデフォルトになってしまいます。したがって、この関数を使って、指定したroom の物理システムを作成する必要があります。 object イベント、またはroom 自体の作成コードで定義します (room エディタでコードを使用せずに定義することもできます)。この関数は物理学の「世界」をroom に関連付け、room で作成されたすべてのインスタンスは、次のような動作をします。 と定義された物理学と世界そのものです。

ピクセルサイズとメートルの比率を定義する引数があることにお気づきでしょう。 これは、物理関数は実世界の測定値を使用して動作するため、「ピクセルからメートルへのスケール」を使用して、ピクセル座標とピクセルサイズをこれらの実世界の測定値に変換する必要があるからです。あなたが使っているobjects の平均ピクセルサイズが、適切な大きさのシミュレーション物理学objects にほぼ変換されるまで、pixeltometrescale設定を調整したいと思います。注意すべきは 物理システムは、比率が大きくても小さくても動作しますが、objects の実測値が 0.1 メートル以上 50 メートル以下の場合に、最高のパフォーマンスを発揮することができます。

構文です。

physics_world_create(pixeltometrescale);

論証タイプ 商品説明
pixeltometrescale defines the ratio of pixels on screen to metres in the real world. A ratio of 32:1 will be specified as 1/32 (or 0.03125).

リターンです。

例

physics_world_create(1/32);

上記のコードは、現在のroom 、32ピクセルが1メートルに相当するスケール比の物理世界を作成します。
