physics_world_draw_debug

この関数はroom の物理世界システムの表現を描画し、物理シーンが正しく動作しないときにdebugging の手助けをします。たとえば、sprite が間違った位置や回転で描かれ、正しくない物理学のように見えることがあります。また、計算が正しくないこともあります。いずれにせよ、この関数は物理システムで起こっていることを描画することで、問題を発見し解決する手助けをします。これを行うには、システムのどの部分を描画するかを指示する必要がある。これは、さまざまな「フラグ」定数に格納されたビットから作成された値を渡すことで行われる。

物理デバッグフラグ
フラッグ 商品説明
phy_debug_render_aabb これは、room 軸に対する各フィクスチャの絶対的なバウンディング・ボックスを示します。
phy_debug_render_collision_pairs これは、現在コリジョン中のフィクスチャを表示します。
phy_debug_render_coms これは、部屋の中の各フィクスチャーの質量の中心を示すものです
phy_debug_render_core_shapes 部屋の中の什器を構成する基本的な形状を示す
phy_debug_render_joints これで、部屋の中のすべての什器の接合部がそれぞれ描画されます
phy_debug_render_obb これは、部屋の中の什器の相対的なバウンディングボックスを表示します。
phy_debug_render_shapes 部屋の中の什器を構成する実際の形状を示します。

そこで、これらのオプションのうちどれか2つを描画するには、その2つに対してビット単位の「または」（GameMakerでは｜で表します）を実行し、その結果を変数に格納して、関数に使用する必要があります。以下は、部屋にあるすべての備品の重心とジョイントだけを描画する関数で使用する変数を設定する方法の一例です。

flag = phy_debug_render_coms | phy_debug_render_shapes;

そうすると、こんな感じになります。

上記のコードでわかるように、画面には形状と重心だけが描かれます（その状態を表す色も描かれます - 詳しくはオンラインのBox 2D documentationをご覧ください）。同じように、デバッグしたい定数を表示することができます。ただ、それらをすべて一緒に "or "するだけですこの関数は、デバッグしたい定数より小さい深さのインスタンスの drawイベントでのみ呼び出さなければならない。

構文です。

physics_world_draw_debug(flag)

論証 タイプ 商品説明
flag 物理デバッグフラグ(s) The flag value for showing different debug features

リターンです。

N/A

例

flag = phy_debug_render_aabb | phy_debug_render_collision_pairs | phy_debug_render_obb;
physics_world_draw_debug(flag);

上記のコードは、現在の物理システム内で、絶対的バウンディングボックス、相対的バウンディングボックス、および衝突中のフィクスチャをすべて描画します。
