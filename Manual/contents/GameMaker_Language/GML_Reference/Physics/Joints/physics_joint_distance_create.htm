物理接合距離の作成

最も単純なジョイントのひとつが距離ジョイントで、2つのインスタンス上の2点間の距離が一定でなければならないというものです。距離ジョイントを指定する場合、2つのインスタンスはすでに作成され、フィクスチャが割り当てられているはずです。それから、2つのアンカーポイントをroom の座標で定義します。最初のアンカーポイントはインスタンス1に、2番目のアンカーポイントはインスタンス2に接続され、これらの点間の距離は距離拘束の長さを意味します。下の画像は、この仕組みを示しています。

room 特に、インスタンスをroom に配置するのではなく、ジョイントと同時にroom エディタで作成する場合は、アンカーポイントを定義する際に注意する必要があります。また、ジョイントはインスタンスのsprite のサイズや、インスタンスが接続しているフィクスチャとは無関係に作成されることも知っておく必要があります。したがって、距離ジョイントをインスタンスの原点以外の場所に作成しても、それはまだ有効であり、それが作成された位置に対して2つのインスタンスを相対的に拘束することになります。col" の値を true に設定すると、2つのインスタンスは相互作用して衝突することができますが、それは衝突イベントがある場合のみです。しかし、 false に設定すると、何があっても衝突することはありません。

構文です。

physics_joint_distance_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, col)

論証 タイプ 商品説明
inst1 インスタンスID The first instance to connect with the joint
inst2 インスタンスID The second instance to connect with the joint
w_anchor1_x リアル The first x coordinate for the joint, within the game world
w_anchor1_y リアル The first y coordinate for the joint, within the game world
w_anchor2_x リアル The second x coordinate for the joint, within the game world
w_anchor2_y リアル the second y coordinate for the joint, within the game world
col ブーリアン Whether the two instances can collide (true) or not (false)

リターンです。

Physics Joint ID

例

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id=instance_create_layer(x+300, y, "Instances", obj_Rudder);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_distance_create(id, o_id, x + 50, y, o_id.x - 50, o_id.y, 0);
physics_fixture_delete(mainFixture);

上記のコードでは、新しいフィクスチャを作成・定義し、"obj_Rudder "のインスタンスを作成しています。そして、フィクスチャはコードを実行しているインスタンスと新しく作成されたインスタンスに割り当てられ、それらの間にジョイントが作成されます。最後に、フィクスチャは不要になったので削除されます。
