physics_fixture_bind_ext

いったんフィクスチャを定義したら、それはインスタンスにバインドされなければなりません。これは、そのプロパティが、実際のフィクスチャそのものではなく、選択されたインスタンスに転送されることを意味し、すべてのプロパティが同じであるなら、一つのフィクスチャを複数のインスタンスにバインドすることができます。ターゲットにobject インデックスを指定すると、その時点でroom に存在するすべてのインスタンスがそのフィクスチャのプロパティを受け取ります(ただし、後から作成されたobject の新しいインスタンスは受け取りません)。また、特殊キーワードotherとall を使用することもできます。親を指定することもできます。object 、その親を持つすべての子インスタンスもフィクスチャを受け取ります。いったんフィクスチャが必要なすべてのインスタンスにバインドされたら、不要になったら削除することができます。そのフィクスチャのプロパティを持つインスタンスは影響を受けず、それらのプロパティを維持することができます。

通常、フィクスチャはインスタンスの原点に重心を置いてインスタンスにバインドされますが、これは常に必要なことではありません。 physics_fixture_bind()となります。）一つのフィクスチャーは一つのオフセットしかサポートしないことに注意。一つのフィクスチャーに複数のオフセットを追加することはBox2Dではサポートされないからだ。

この関数は、バインドされたフィクスチャに対してユニークな「id」値を返します。（フィクスチャそのものではありません）この値は、その後、関数を用いてインスタンスから物理プロパティを削除（「アンバインド」）するために使われます physics_remove_fixture().これにより、objects を破棄して再作成することなく、インスタンスから物理プロパティを追加および削除することができます。

NOTE: フィクスチャは不要になったら削除すべきです。削除しないとメモリリークが発生し、ゲームの動作が遅くなり、最終的にはクラッシュする可能性があるからです。

構文です。

physics_fixture_bind_ext(fixture, target, xoffset, yoffset)

論証 タイプ 商品説明
fixture 物理フィクスチャID The fixture that is to be bound
target インスタンスIDまたはオブジェクトアセット The target instance that is to receive the fixture (can be an instance id, an object id, other, or all)
xoffset リアル The offset along the x-axis
yoffset リアル The offset along the y-axis

リターンです。

Real

例

var fix, inst;
fix = physics_fixture_create();
physics_fixture_set_circle_shape(fix, 16);
physics_fixture_set_density(fix, 1.0);
inst = instance_create_layer(x, y, "Instances", genericBodyObject);
my_fix = physics_fixture_bind_ext(fix, inst, sprite_width / 2, -(sprite_height / 2));
physics_fixture_delete(fix);

上記のコードでは、フィクスチャーを作成し、そのインデックスを変数「fix」に代入しています。次に、フィクスチャーの形状と密度を定義してから、sprite の幅と高さに基づいたオフセットでインスタンスにバインドします。バインドされたフィクスチャーのインデックスは、変数 "my_fix" に格納されます。最後に、フィクスチャは不要になったので、メモリリークを防ぐために、フィクスチャ自体が削除される。
