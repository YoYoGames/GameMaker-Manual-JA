物理パーティクルデータ

この関数は、与えられたflags をチェックして、物理シミュレーションの各粒子に関する様々な情報を返します。使用されるbuffer は以前に関数 buffer_create()また、"grow "タイプである必要があり、そのサイズは予想される戻りデータのサイズとほぼ同じである。flags は以下に示す定数のいずれかを使って設定し、ビット演算 または"|" を使って、目的の情報を得るために一つのflag 値を作成することになる。

物理学 粒子データFlag 定数
定数 商品説明 データ型
phy_particle_data_flag_typeflags パーティクルのflags の値です。 buffer_u32
phy_particle_data_flag_position パーティクルのx、y位置。 2 x buffer_f32
phy_particle_data_flag_velocity 水平・垂直方向の速度です。 2 x buffer_f32
phy_particle_data_flag_colour 色とアルファ値（16進数）です。 buffer_f32
phy_particle_data_flag_category パーティクルカテゴリ（パーティクルまたはその所属グループを作成したときに定義したもの）。 buffer_u32

構文です。

physics_particle_get_data(buffer, flags)

論証 タイプ 商品説明
buffer バッファID The (previously created) buffer to use to store the data.
flags 物理 粒子データFlag 定数(s) The flags to use to extract data about specific particle types.

リターンです。

N/A

例

var count = physics_particle_count();
var flags = phy_particle_data_flag_position | phy_particle_data_flag_colour;
if (count > 0)
{
    var buffer = buffer_create(count * 12, buffer_grow, 4);
    physics_particle_get_data(buffer, flags);
    for (var n = 0; n < count; n++;)
    {
        var xx = buffer_read(buffer, buffer_f32);
        var yy = buffer_read(buffer, buffer_f32);
        var argb = buffer_read(buffer, buffer_u32);
        var alpha = (argb >> 24) & 255;
        draw_sprite_ext(sprBlob, 0, xx, yy, 1, 1, 0, c_green, alpha);
    }
    buffer_delete(buffer);
}

上記のコードでは、パーティクルの数を取得し、チェックするためのデータflags を変数に作成し、room にパーティクルがあるかどうかをチェックします。もしあれば、buffer を作成し、パーティクルデータを入れ、それをチェックし、パーティクル位置のsprite を描画するために使用します。
