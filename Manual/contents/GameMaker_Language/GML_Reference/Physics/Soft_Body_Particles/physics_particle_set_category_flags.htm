physics_particle_set_category_flags

この機能を使うと、パーティクルカテゴリのパーティクルflags を変更することができます。 category の値は、この関数で定義したものです。 physics_particle_create()または (グループを作成した場合) physics_particle_group_begin()であり、flags は以下の定数の組み合わせの戻り値である。

定数 商品説明
phy_particle_flag_water ソフトボディパーティクルのデフォルトのプロパティです。
phy_particle_flag_zombie ゾンビパーティクルは、このように他のすべてのフラグを立てて1ステップで破壊されるパーティクルです。
phy_particle_flag_wall これはパーティクルを静的と定義し、本質的に物理シミュレーションで不動のobject 、何が衝突しても固定された位置に留まるように作成します。密度を0に設定するよりも、このflag を使用する必要があります。
phy_particle_flag_spring スプリングパーティクルは、バネのように互いにくっついたような効果を生み出します。このパーティクル（flag ）で作られたパーティクルは、ペアで「接続」され、それぞれのパーティクルは、作成時に最も近くにいたパーティクルと接続されます。一度ペアになった粒子は「相手」を変えないので、外力に引っ張られるほど、その外力を取り除いたときに衝突する力が大きくなる。なお、対になった粒子がどんなに離れても、粒子間の結合が切れることはない。
phy_particle_flag_elastic 弾性体パーティクルは、物理シミュレーションで他の剛体と衝突すると変形したり、バウンドしたりすることがあります。
phy_particle_flag_viscous 粘性粒子とは、油のように「くっつきやすさ」「粘り強さ」を示す粒子のことです。粘性のある粒子は、より固まり、よりくっつきます。
phy_particle_flag_powder 粉体は、砂やホコリのような散乱効果を生み出します。
phy_particle_flag_tensile 引張粒子は、表面張力、つまり液体表面の張ったような曲面の効果を出すために使われます。例えば、水滴に見られるような表面張力を発生させるために使用されます。表面張力がなくなると、粒子は弾力性があるように跳ねますが、互いに引き合うようになります。その結果、粒子は弾みながらクラスターを形成する傾向があります。
phy_particle_flag_colourmixing カラーミキシングパーティクルは、衝突した他のパーティクルの色の一部を取り込みます。ただし、衝突する2つのパーティクルのうち1つだけが混色パーティクルの場合、もう1つのパーティクルは衝突前の色を保持します。

これらのflags はビットマスキングを使用して最終的な出力値を作成し、それをチェックして粒子カテゴリの異なる基本特性を設定します（基本特性は常に phy_particle_flag_water のものです）。例えば、表面張力を持つ粘性液体をシミュレートしたい場合、以下の例に示すように、ビットワイズまたは「｜」を使用して適切なビットをマスクオフします。このように、作成するパーティクルのカテゴリごとに異なるプロパティ（グローバルプロパティ以外）を設定することができます。

構文です。

physics_particle_set_category_flags(category, flags)

論証 タイプ 商品説明
category リアル The particle category to set.
flags 物理学 粒子Flag 定数(s) The flags to set on the particle category.

リターンです。

N/A

例

var flags = phy_particle_flag_water | phy_particle_flag_viscous | phy_particle_flag_tensile;
physics_particle_set_category_flags(1, flags);

上記のコードでは、flags の値を格納する変数を作成し、それを使ってカテゴリ値 1 で作成されたすべてのパーティクルのflags を設定します。
