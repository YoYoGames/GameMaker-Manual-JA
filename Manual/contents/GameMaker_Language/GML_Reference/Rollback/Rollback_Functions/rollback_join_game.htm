ロールバックジョインゲーム

説明： ゲームに参加しようとする。参加するためには、仮想のroom の ID を含む URL にいる必要があります。

room に参加した場合はtrue を返し、ゲームに参加していない場合はfalse を返します。

自分がすでに参加しているゲームに新しいプレイヤーが参加すると、「ロールバックイベント」イベントが発生します。詳しくは、「ロールバックイベント」を参照してください。

dry_run引数はオプションで、trueに設定すると実際にゲームに参加せずに関数を実行できるようになります。これは、実際にゲームに参加することなく、ゲームに参加できるかどうかを確認するために使用されます。

構文です。

rollback_join_game([dry_run]);

論証 タイプ 商品説明
dry_run ブーリアン Disabled by default (false).有効にすると（true）、この関数はゲームに参加せず、単にゲームに参加できる場合はtrueを返し、そうでない場合はfalseを返します。

このように、この関数を使用して、プレイヤーが招待されているかどうかだけをチェックすることができます。

リターンです。

Boolean

例

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_create_game(2, false);
}

上記のコードでは、プレイヤーobject を定義し、ゲームに参加しようとします。参加できなかった場合は、代わりに新しいゲームを作成します。最大プレイヤー数は2人で、Sync Testは無効になっています。

拡張例については、「マルチプレイヤーゲームを作成する」を参照してください。
