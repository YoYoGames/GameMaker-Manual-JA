rollback_define_player（ロールバック・プレイヤー）。

この関数は、マルチプレイヤーでプレイヤーに使用されるobject を定義します。これはオプションであり、使用された場合、システムは自動的に接続したプレイヤーのインスタンスを作成し、切断したプレイヤーのインスタンスを破棄します。

デフォルトでは、「Instances」レイヤーにプレイヤーのインスタンスが作成されます。このレイヤーはゲーム内に存在する必要があります。room 他のレイヤーを使用したい場合は、第2引数にその名前を渡します。

この関数を使った場合と使わない場合のゲーム作成については、「プレイヤーオブジェクトの定義」を参照してください。使用する場合、この関数はjoin/create関数の前に実行する必要があります。

この関数で作成された各インスタンスは、このページに記載されているインスタンス変数を取得します。

構文です。

rollback_define_player(object, [layer_name]);

論証 タイプ 商品説明
object オブジェクト・アセット The object that will be used for creating player instances.
layer_name 文字列 OPTIONAL プレイヤーインスタンスが作成されるレイヤーの名前。デフォルトは"Instances"。

リターンです。

N/A

例

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

上記のコードでは、プレイヤーobject を定義し、ゲームに参加しようとします。参加できなかった場合は、代わりに新しいゲームを開始します。最大プレイヤー数は2人で、Sync Testは無効になっています。
