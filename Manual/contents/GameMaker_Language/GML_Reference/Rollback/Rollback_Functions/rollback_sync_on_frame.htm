ロールバックシンクオンフレーム

この関数は、すべてのプレイヤーが現在のフレームで同期しているかどうかをチェックします。つまり、プレイヤーの状態は予測に基づくのではなく、プレイヤーから受け取った実際のデータに基づいています。そのような場合、true を返す。

現在のフレームでプレイヤーが同期していない場合、false を返し、すべてのプレイヤーの同期を試みる。

この関数でプレーヤーを同期させようとすると、フリーズすることがあります。Stepイベント内でこの関数が連続的に呼び出されないようにする必要があります。

代わりに、トグルとして使用します。ページの一番下にある例をご覧ください。

変数rollback_confirmed_frame は、すべてのプレーヤーが同期されたときに自動的に更新されます。

構文です。

rollback_sync_on_frame();

リターンです。

Boolean

例

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

このコードは、プレイヤーオブジェクトのStepイベント内で実行されます。

勝っているプレーヤーがいないか、現在のプレーヤーが勝利に必要なポイントを持っているかどうかをチェックします。

両方が真なら、そのプレイヤーは勝つべきということになる。しかし、その前に、すべてのプレイヤーが同期しているかどうかをチェックし、状態が正しいことを確認する。

すべてのプレイヤーが同期している場合、winning_player変数を設定します。この結果、最初の条件がfalse になり、rollback_sync_on_frame()が再び呼ばれないことが保証されます。

これは、関数が毎フレーム連続的に実行されるのではなく、トグルとして使用されることを意味します。
