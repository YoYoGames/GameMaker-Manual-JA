rollback_define_mock_input

この関数はSync Test中に使用され、指定されたプレーヤーに一時的な「モック」入力を割り当てることができます。これは、プレーヤーに定義されたコントロールとSync Test からのランダム入力をオーバーライドします。

使用方法については、モック入力を参照してください。

構文です。

rollback_define_mock_input(player_id, input_struct);

論証 タイプ 商品説明
player_id リアル The ID of the player to assign mock input to.
input_struct 構造 A struct containing input names as variables, and their assigned inputs as values.

リターンです。

N/A

例

rollback_define_input({
    fire: mb_left,
    interact: vk_space,
    left: ord("A"),
    right: ord("D")
});

rollback_define_mock_input(1, {
    fire: vk_control,
    interact: vk_shift,
    left: ord("J"),
    right: ord("L")
});

上記のコードでは、まずプレーヤーの入力を定義しています。次に、プレイヤー1用のモック入力を定義し、同じ入力名を維持したまま、異なる入力を割り当てています。こうすることで、2人がキーボードを使ってローカルにゲームをプレイすることができます。
