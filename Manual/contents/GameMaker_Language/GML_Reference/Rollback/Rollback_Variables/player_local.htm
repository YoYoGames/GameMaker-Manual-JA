プレイヤーローカル

を使用してプレーヤーを定義すると、システムは自動的にそのインスタンスを作成します。 rollback_define_player()objectを使用してプレーヤーを定義すると、システムは自動的に各プレーヤーにそのobject のインスタンスを作成します。そのobject の各インスタンスは player_local 変数を取得し、そのプレーヤーのインスタンスがローカルかどうかを知らせます。

rollback_define_player() を使用しない場合、この変数はobjects のいずれに対しても作成されません。

この変数は、どのプレイヤーがローカルであるかを示すために使用することができます。この変数をゲームロジックに影響を与えるような形で使用することはお勧めしません。ローカルとリモートのプレイヤーは同じであるべきです。

構文です。

player_local;

リターンです。

Boolean

例

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

上記のコードは Draw イベントで実行され、プレイヤー インスタンスがローカルであるかどうかをチェックします。その場合、インスタンスの上に矢印sprite を描き、どのインスタンスを操作しているかをプレイヤーに知らせます。
