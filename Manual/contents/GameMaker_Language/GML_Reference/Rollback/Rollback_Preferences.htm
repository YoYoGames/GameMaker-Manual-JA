ロールバック環境設定

各プレイヤーがキャラクターを選択できるようにしたり、そのプロパティをカスタマイズできるようにしたい場合があります。このようなプロパティは、「プレイヤー設定」として設定することができます。

プリファレンスを使うには？

まず、プリファレンスを有効にするために rollback_use_player_prefs()/Use Player Prefs.この機能で、すべてのプレイヤーのデフォルトのプリファレンスを任意に設定することができます。

この関数を呼び出すと、「自動スタート」が無効になります。つまり、すべてのプレイヤーが参加してもゲームは開始されず、手動で次のように開始する必要があります。 rollback_start_game()/ Gameを開始する。

ゲームが始まる前に、あるプレイヤーのプリファレンスを設定するには、そのプレイヤーのクライアントで rollback_set_player_prefs()/Set Player Prefsをそのプレイヤーのクライアントで呼び出すことで、プレイヤーの環境設定を行うことができます。設定した内容は、後で rollback_get_player_prefs()/Get Player Prefs、またはプレイヤーインスタンスの player_prefs 変数で読み取ることができます。

ロールバックイベント

プレイヤーが設定を変更すると、ゲーム内の全員がrollback_player_prefs Rollback Event を受信します。イベントが送信されるには、プレーヤーが完全に接続されている必要があります。

接続前に設定されていた場合、接続完了時に、接続前に設定されていた最後の設定が全プレイヤーに送信されます。

スタートコンディション

環境設定を有効にしていて、デフォルトの環境設定がない場合、全プレイヤーの環境設定がされていないとゲームは始まりません。

このような場合、アラームの中で rollback_start_game()を繰り返し呼び出すか、rollback_player_prefs Rollback Eventを受信するたびに呼び出すことができます。

どのような種類が使えますか？

プリファレンスは、数値、string 、配列など、どのような型でも構いませんが、変数のコレクションを格納する最も簡単な方法である構造体を使用することが推奨されます。
