ロールバックエラー

ロールバックシステムを使用する場合、特定のエラーが発生することがあります。これらのエラーはランナーエラーであるため、ゲームの実行中に表示されます。

次の表は、ロールバックシステムで作業しているときに表示される可能性のあるエラーとその説明の一覧です。また、特定のエラーメッセージごとに解決策を提供しています。

エラー 商品説明
プレイヤーを作成するためのレイヤーを取得できませんでした。rollback_define_player関数にレイヤー名を渡すか、Instancesレイヤーを作成する必要があります。 プレーヤーobject がrollback_define_player関数で定義されている場合、ロールバック システムはプレーヤーobject を、オプションで 2 番目のパラメーターとして渡すことができるレイヤー上に作成しようとします（定義されていない場合は「Instances」という名前のレイヤー上に作成します）。レイヤーがその名前で見つからなかった場合、このエラーが発生します。解決策は、プレーヤーを作成するレイヤー名を明示的に設定するか、レイヤー名を明示的に設定しない場合は、「Instances」という名前のレイヤーを作成することです。
マルチプレイヤーゲームを実行する場合、プレーヤーobjects を管理する必要があります。 playerobject がrollback_define_player関数で定義されており、playerobject が managed としてフラグされていない場合、このエラーが発生する。解決策は、Object Editor でプレーヤ（object ）を管理対象としてマークすることです。
player_id Nのユーザー情報の取得に失敗しました。 ロールバックシステムがserver からプレイヤーに関する情報を取得できなかった場合、このエラーが発生します。デベロッパー コンソールをチェックして、ブロックされたリクエストがあるかどうかを確認します。
ゲームに完全に接続する前にrollback_chatを呼び出すことができませんでした。rollback_game_infoイベントを待ってみてください。 プレイヤーがroom に参加するまで、チャット機能は利用できない。 解決策rollback_game_infoイベントを受信するまでチャットを無効にする（Rollback Eventsを参照）。
rollback_use_manual_start は rollback_create_game と rollback_join_game の前に呼び出す必要があります。 これらのエラーは、関数呼び出しが内部ステートマシン制約に違反した場合に発生します。エラーメッセージは解決策を示唆します。
