DSグリッド

DSグリッドは、基本的に 2次元の配列の一種である。グリッドの幅と高さを整数で設定すると、グリッド内の「セル」の総数（幅と高さの掛け算）が設定され、これらのセルにはさまざまなデータ値（実数またはstrings ）が格納されます。

この種の構造体は、そのx、yインデックス（x、y軸とも0から始まり、幅-1、高さ-1まである）を与えることで、グリッド内のセルの値を設定、取得することができます。しかし、特定のグリッド領域の値を設定したり、単一の値を追加したり、与えられた領域にわたる合計、最大、最小、平均値を取得することもでき、このデータ構造は利用できるものの中で最も多目的なものとなっています。

DSグリッドデータ構造にアクセスする場合、セル位置には常に整数値を使うようにしなければならず、そうしないと非整数のインデックスはすべてGameMakerによってフロアリングされてしまう。もし、そうでない場合は、チェックしたいインデックスを渡す前に丸め誤魔化しておく必要がある。

注意事項 すべての動的リソースと同様に、データ構造はメモリを消費するため、メモリリークによるゲームのスローダウンやクラッシュを防ぐために、不要になったら常に破棄する必要があります。

注意データ構造が存在するかどうかをチェックする必要がある場合は ds_exists()関数を使用します。

以下に示す特定の関数を除いて、DS グリッドの内容を追加したり変更したりするには、すべて式(アクセサと呼ばれる) を使用することができます。このアクセサは 2 次元配列に似ており、次のような構文になっています。

grid_index[# xpos, ypos]

Accessors のGML Overview ページに詳細な情報とサンプルが掲載されています。
