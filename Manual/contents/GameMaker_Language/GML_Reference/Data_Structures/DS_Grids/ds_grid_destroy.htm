ds_grid_destroy

この関数は、指定されたグリッドデータ構造をメモリから削除し、使用していたリソースを解放し、それに含まれるすべての値を削除する。この関数は、DS グリッドの使用を終了したときに必ず使用するようにし、メモリリークによるゲームのスローダウンやクラッシュを防止します。

重要!データ構造を作成するとき、それを識別するためのインデックス値は0から始まる整数値です。 これは、異なるタイプのデータ構造が同じインデックス値を持つことができることを意味します。 ds_exists()関数を使用してからアクセスします。また、インデックスは再利用されるため、破壊されたデータ構造のインデックス値が、その後新しく作成されたデータ構造で使用される可能性があることに注意してください。したがって、破壊後は必ずDSインデックスを保持していた変数を-1に設定することをお勧めします。

構文です。

ds_grid_destroy(index);

論証タイプ 商品説明
index This index of the grid to destroy.

リターンです。

例

if (lives == 0)
{
    ds_grid_destroy(Wall_Grid);
    Wall_Grid = -1;
    room_goto(rm_Menu);
}

上記のコードは、内蔵のグローバル変数「lives」の値をチェックし、それが0であれば、変数「Wall_Grid」でインデックスされたDSグリッドを破棄し、rooms.
