ds_set_precision

マップの検索やリストのソートなど、値を比較するとき、GameMakerは2つの値が等しいかどうかを判断しなければなりません。strings や整数の場合は明確ですが、実数の場合は浮動小数点の丸め誤差があるため、一見等しく見える数値も簡単に不等間隔になります。例えば、(5 / 3) * 3は 5 と等しくないということがあり得ます。これを避けるために、すべての実数関数で精度値が使用され、2つの数値の差がこの精度より小さいとき、それらは等しいとみなされます。この関数で変更しない限り、全てのデータ構造関数には0.0000001の精度がデフォルトで使用されます。

注：この精度はすべてのデータ構造で使用されるが，GML の他の比較では使用されない !

構文です。

ds_set_precision(prec);

論証 タイプ 商品説明
prec リアル The precision value (default 0.0000001)

リターンです。

N/A

例

ds_set_precision(0.0001);

上記のコードは、すべてのデータ構造関数のデフォルトの精度設定を変更します。
