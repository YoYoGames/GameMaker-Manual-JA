DSスタック

DS スタックは、いわゆる LIFO (Last-in-First-Out) データ構造で、DS キューに似ている。値をスタックにプッシュし、スタックからポップすることで、再び値を取り出すことができます。このとき、直近にスタックにプッシュされた値が、再びスタックからポップされる最初の値となります (コインのスタックを想像してみてください。追加した各コインは、下の残りのコインに到達する前に再び除去されなければなりません)。スタックは、割り込みを処理するときや、再帰的な関数を持つとき、あるいはゲームの初歩的なAIを構築するときにもよく使われます。

注：すべてのダイナミックリソースと同様に、データ構造はメモリを消費します。したがって、メモリリークによるゲームのスローダウンやクラッシュを防ぐために、不要になったら必ず破棄する必要があります。

注：データ構造が存在するかどうかを確認する必要がある場合は ds_exists()関数を使用します。

DSスタックを扱う機能として、以下のものが存在する。

  • ds_stack_create
  • ds_stack_destroy
  • ds_stack_clear
  • ds_stack_empty
  • ds_stack_size
  • ds_stack_copy
  • ds_stack_top
  • ds_stack_pop
  • ds_stack_push
  • ds_stack_read
  • ds_stack_write
