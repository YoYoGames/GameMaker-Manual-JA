ds_map_clear

この関数は、(以前に作成した) DS マップのすべてのキー/値ペアを消去しますが、 DS マップ自体をメモリから削除するわけではありません。そのためには、関数 ds_map_destroy().マップをクリアすると、マップに格納されているデータ構造の参照が解除され、メモリリークが発生することに注意してください。したがって、これを防ぐには、マップをクリアする前に、それを調べてすべてのデータ構造アイテムを手動で破棄する必要があります。この作業が不要なのは、マップ内のアイテムにDS リストや別のDS マップとしてのフラグをつけている場合のみで、 その場合は親マップが消去されるときにこれらのアイテムは破棄され (消去されない!)、自動的にメモリがクリーンアップされる。

構文です。

ds_map_clear(id);

論証タイプ 商品説明
id The id of the data structure to clear.

リターンです。

例

if global.new_game
{
    ds_map_clear(inventory);
}

上記の関数は、グローバル変数が真であるかどうかをチェックし、真であれば、インスタンス変数 "inventory" をインデックスとするDSマップのすべてのキー/値のペアをクリアします（ただし、破壊はされません）。
