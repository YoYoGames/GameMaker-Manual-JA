DSマップ

DSマップのデータ構造は非常に便利で、キーと 値のペアを保存することができます。例えば、RPGでヘルスポーションを10個、マナポーションを5個、ゴールドを100個持っているように、ゲームのキャラクターは様々なアイテム（キー）を大量に持ち、個々のアイテムに対してそれらの数（値）を持つことができます。マップはそのようなペアを一箇所にまとめて管理し、ペアをマップに追加したり、簡単な機能を使って特定のキーに対応する値を検索したりすることができます。しかし、マップを使う前に知っておくべきことがいくつかあります。つまり、あるキーを見つけるためには、マップ全体を繰り返し見る必要があります（これは非常に時間がかかります）。また、同じキーを2つ保持することはできませんし、1つのキーに2つの値を割り当てることもできません。

注：DSマップよりも構造体を使用することをお勧めします。類似の機能を持ち、使いやすく、自動的にガベージコレクションされるからです。

注意事項すべての動的リソースと同様に、データ構造はメモリを消費するため、メモリリークによるゲームのスローダウンやクラッシュを防ぐために、不要になったら常に破棄する必要があります。

以下に示す特定の関数とは別に、DS マップの内容を追加したり変更したりするために、式（アクセッサと呼ばれる）を使用することができます。このアクセサは 1 次元配列に似ており、次のような構文になっています。

map_index[? key]

Accessors のGML Overview ページに詳細な情報とサンプルが掲載されています。

DSマップを扱う機能として、以下のものがある。

  • ds_map_exists
  • ds_map_create
  • ds_map_add
  • ds_map_clear
  • ds_map_copy
  • ds_map_replace
  • ds_map_delete
  • ds_map_empty
  • ds_map_size
  • ds_map_find_first
  • ds_map_find_last
  • ds_map_find_next
  • ds_map_find_previous
  • ds_map_find_value
  • ds_map_keys_to_array
  • ds_map_values_to_array
  • ds_map_set
  • ds_map_read
  • ds_map_write
  • ds_map_destroy

DSマップの保存と読み込みには、さらに4つの関数が用意されています。これらの関数は、マップを難読化し、最終ファイルがデバイス間で転送できないようなファイル形式を使用して、ターゲットプラットフォームの安全な場所に保存します。

  • ds_map_secure_save
  • ds_map_secure_save_buffer
  • ds_map_secure_load
  • ds_map_secure_load_buffer

DSマップの用途の一つはJSONを扱うことであり、そのためにいくつかの特殊な関数がある。

  • ds_map_add_list
  • ds_map_add_map
  • ds_map_replace_list
  • ds_map_replace_map
  • ds_map_is_list
  • ds_map_is_map

注意これらの関数により、マップ内にリストやマップを追加することができますが、JSON以外では役に立ちません。また、ネストしたマップやリストは、ディスクへの書き込みや他の方法でアクセスしても正しく読み込まれません。
