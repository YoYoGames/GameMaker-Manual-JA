ds_map_destroy

DSマップは、作成時に割り当てられたメモリ内の領域を使用します。つまり、ゲーム実行時のエラーやメモリリーク、パフォーマンスの低下を防ぐために、DSマップが不要になったらこのメモリも解放する必要があります。この機能 は、まさにそれを行う。マップを破壊すると、そのマップに格納されているデータ構造の参照が解除され、メモリリークが発生することに注意してください。したがって、これを防ぐには、マップを破壊する前に、すべてのデータ構造項目を手動で破壊する必要があります。これを防ぐには この作業が必要ないのは、マップ内のアイテムをDSリストまたは別のDSマップとしてフラグを立てた場合のみで、その場合、これらのアイテムは破棄され、メモリがクリーンアップされます。 も自動で行います。

重要!データ構造を作成するとき、それを識別するためのインデックス値は0から始まる整数値です。 これは、異なるタイプのデータ構造が同じインデックス値を持つことができることを意味します。 ds_exists()関数を使用してからアクセスします。また、インデックスは再利用されるため、破壊されたデータ構造のインデックス値が、その後新しく作成されたデータ構造で使用される可能性があることに注意してください。 破棄した後は、DS のインデックスを保持していた変数を常に -1 に設定します。

構文です。

ds_map_destroy(id);

論証タイプ 商品説明
id The id of the map to destroy.

リターンです。

例

ds_map_destroy(inventory);
inventory = -1;

上記のコードは、変数 "inventory "にインデックスされたidを持つDSマップを破壊します。
