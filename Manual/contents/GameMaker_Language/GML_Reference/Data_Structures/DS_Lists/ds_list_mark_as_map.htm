ds_list_mark_as_map

この関数は、以前に作成した DS リスト内の指定した位置にDS マップを持つものとして "マーク" (あるいは "フラグ") を付ける。この機能は、JSONstrings をエンコードする際に必要となる (これは json_encode()この方法でマークされた項目は、親の DS リストが破棄されたときに自動的にガベージコレクションされる (破棄される) ため、データ構造をネストするときにも便利です。つまり、"親" リストを破棄する前に、手動でリストの内容を調べ、マークされたデータ構造を個別に破棄する必要がないのです。しかし、リストの位置を個別に削除した場合、そこに含まれるデータ構造は ガベージコレクションされないので、親リストの位置を削除する前に 適切な DS マップの破棄関数を呼び出さなければなりません。また、もしこの関数 ds_list_clear()の場合、マップと判定された項目はリストを消去する際に一緒に破棄されます。

また、マークされた位置がリスト内で変化した場合、「マーク」も一緒に移動します。例えば、リスト内の位置3をマークした後、その下にさらに2つのアイテムを挿入して位置5に移動しても、マップとしてのマークは残ります。

構文です。

ds_list_mark_as_map(id, pos);

論証タイプ 商品説明
id The id of the list to mark.
pos The position within the list to mark.

リターンです。

DS Map ID or -1 (if it fails)

例

var sub_map = ds_map_create();
ds_map_add(sub_map, "player", player_array);
ds_map_add(sub_map, "enemy", enemy_array);
ds_list_add(j_list, sub_map);
ds_list_mark_as_map(j_list, 0);

上記のコードでは、DS マップを作成し、それに 2 つのキーを設定し、それぞれに値の配列を格納しています。このマップは、与えられた DS リストに追加され、 後で正しくエンコードできるようにその位置が「マーク」されます。
