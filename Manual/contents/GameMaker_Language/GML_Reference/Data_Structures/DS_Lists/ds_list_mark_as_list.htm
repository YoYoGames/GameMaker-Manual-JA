ds_list_mark_as_list

この関数は、以前に作成した DS リスト内の指定した位置に、 別の DS リストを保持していることを "マーク" (あるいは "フラグ") するものである。この機能は、JSONstrings をエンコードする際に必要となる (これは json_encode()このようにマークされた項目は、親となる DS リストが破棄されるときに自動的にガベージコレクションされる (破棄される) ため、データ構造をネストするときにも便利です。つまり、"親" リストを破棄する前に、手動でリストの内容を調べ、マークされたデータ構造を個別に破棄する必要がないのです。しかし、リストの位置を個別に削除した場合、そこに含まれていた DS リストはガベージコレクションされないので、親リストの位置から項目を削除する前に 適切な DS リスト破棄関数を呼び出さなければなりません。また、この関数 ds_list_clear()の場合、リストとしてフラグが立っている項目は、リストがクリアされるときに、同様に破棄されます。

また、マークされた位置がリスト内で変化した場合、「マーク」も一緒に移動します。例えば、リスト内の位置3をマークした後、その下にさらに2つのアイテムを挿入して位置5まで移動しても、リストとしてのマークは残ります。

構文です。

ds_list_mark_as_list(id, pos);

論証タイプ 商品説明
id The id of the list to mark.
pos The position within the list to mark.

リターンです。

DS List ID or -1 (if it fails)

例

var j_list = ds_list_create();
var sub_list = ds_list_create();
ds_list_add(sub_list, health);
ds_list_add(sub_list, lives);
ds_list_add(sub_list, score);
ds_list_add(j_list, sub_list);
ds_list_mark_as_list(j_list, 0);

上記のコードでは、2つのDSリストを作成し、一方にグローバル変数から様々な値を入力しています。このリストは 2 番目のリストに追加され、後で正しくエンコードできるようにその位置が「マーク」されます。
