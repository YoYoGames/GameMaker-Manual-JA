DSリスト

DSリストは、情報を追加しながら順次格納していくデータ構造である（配列とよく似ている）。DSリストは非常に柔軟なデータ構造であり、リストの末尾に値を追加したり、途中に値を挿入したり、順序をシャッフルしたり（値をランダムにする）、昇順や降順に並べ替えたりすることが可能です。

注：DSリストよりも配列を使用することをお勧めします。同様の機能を持ち、使いやすく、自動的にガベージコレクションが行われるからです。

DSリストデータ構造にアクセスする場合、リストの位置には常に整数値を使うようにしなければならない。そうしないと、非整数のインデックスはすべてGameMakerによってフロアリングされることになる。もし、そうでない場合は、チェックしたいインデックスを渡す前に、丸め処理を自分で行う必要がある。

注：すべての動的リソースと同様に、データ構造はメモリを消費します。したがって、メモリリークによるゲームのスローダウンやクラッシュを防ぐために、不要になったら必ず破棄してください。

以下に示す特定の関数とは別に、DS リストの内容を追加したり変更したりするために、式（アクセッサと呼ばれる）を使用することができます。このアクセサは 1 次元配列に似ており、以下のような構文になっています。

list_index[| index]

Accessors のGML Overview ページに詳細な情報とサンプルが掲載されています。

DSリストを扱う機能として、以下のものがある。

  • ds_list_create
  • ds_list_destroy
  • ds_list_clear
  • ds_list_empty
  • ds_list_size
  • ds_list_add
  • ds_list_set
  • ds_list_delete
  • ds_list_find_index
  • ds_list_find_value
  • ds_list_insert
  • ds_list_replace
  • ds_list_shuffle
  • ds_list_sort
  • ds_list_copy
  • ds_list_read
  • ds_list_write

DS リストは、DS マップや他の DS リストを保持するために使われることもあり、 そのため、これらのデータ構造を保持するリスト内の位置を「フラグ」または 「マーク」するために使われるいくつかの特殊関数があります。一般に、これは必須ではありませんが、親リストが破壊されたときに マークされたデータ構造を自動的にガベージコレクションする (破壊する) という利点がありますし、また json_encode()関数は、リストとそのデータが適切にエンコードされるようにします。そのための関数が

  • ds_list_mark_as_list
  • ds_list_mark_as_map
  • ds_list_is_list
  • ds_list_is_map
