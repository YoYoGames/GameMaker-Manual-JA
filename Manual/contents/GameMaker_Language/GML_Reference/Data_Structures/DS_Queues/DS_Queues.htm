DSキュー

DSqueue はDS スタックと似ていますが、先入れ先出し (FIFO) 方式で動作します。queue に最初に入れられた値は、そこから最初に取り出される値でもある。これは、お店でお金を払うときのqueue のようなもので、queue に最初に入った人が最初にお金を払ってからお店を出る、と考えることができる。Queues は通常、インスタンスがまだ行う必要のあるアクションや行動 (scripts などの形式) を保存するために使われるが、他にも多くの使い道がある。

注：すべてのダイナミックリソースと同様に、データ構造はメモリを消費します。したがって、メモリリークによるゲームのスローダウンやクラッシュを防ぐために、不要になったら必ず破棄する必要があります。

注意：データ構造が存在するかどうかを確認する必要がある場合は ds_exists()関数を使用します。

DSキューを扱う機能として、以下のものが存在する。

  • ds_queue_create
  • ds_queue_destroy
  • ds_queue_clear
  • ds_queue_empty
  • ds_queue_size
  • ds_queue_dequeue
  • ds_queue_enqueue
  • ds_queue_head
  • ds_queue_tail
  • ds_queue_copy
  • ds_queue_read
  • ds_queue_write
