ds_queue_read

この機能を使うと、保存された DS キュー（以前、string として ds_queue_write()).まず、string を読み込むために新しい DS キューを作成する必要があります。もし、DS キューがすでに存在し、情報が保存されている場合は、読み込む前にクリアされます。この機能は、ゲームのセーブ/ロード機構を作成する際に非常に重要です。

なお、指定されたDSstring がGameMakerで書かれたものである場合。Studio 1.2.xruntime (またはそれ以前) の場合、その後string のフォーマットが変更されたため、オプション引数 "legacy" を指定して true に設定する必要があります。

構文です。

ds_queue_read(id, str [, legacy]);

論証 タイプ 商品説明
id DSQueue ID The id of the data structure to read into.
str 文字列 The string to read from.
legacy ブーリアン OPTIONAL true または のいずれか、または完全に省略することができます。 false

リターンです。

N/A

例

queue = ds_queue_create();
ini_open("save.ini");
var str = ini_read_string("Queues", "0", "");
if (str != "")
{
    ds_queue_read(queue, str);
}
ini_close();

上記のコードでは、queue を作成し、そのインデックスを変数 "queue" に格納しています。次に、 ini ファイルを開き、そこからstring を読み込む。最初にstring が空として返されないことを確認する。そして、このstring が新しく作成された DSqueue に読み込まれる。
