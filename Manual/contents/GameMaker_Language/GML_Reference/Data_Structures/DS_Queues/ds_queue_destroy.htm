ds_queue_destroy

この関数は、与えられたqueue データ構造をメモリから削除し、それが使用していたリソースを解放し、それが含んでいたすべての値を削除します。この関数は、ゲームのスローダウンやクラッシュを引き起こすメモリリークを防ぐため、DS キューの使用を終了したときに常に使用する必要があります。

重要!データ構造を作成するとき、それを識別するためのインデックス値は0から始まる整数値です。 これは、異なるタイプのデータ構造が同じインデックス値を持つことができることを意味します。 ds_exists()関数を使用してからアクセスします。また、インデックスは再利用されるため、破壊されたデータ構造のインデックス値が、その後新しく作成されたデータ構造で使用される可能性があることに注意してください。したがって、破壊後は必ずDSインデックスを保持していた変数を-1に設定することをお勧めします。

構文です。

ds_queue_destroy(id);

論証タイプ 商品説明
id The id of the data structure to remove.

リターンです。

例

if (lives == 0)
{
    ds_queue_destroy(AI_queue);
    AI_queue = -1;
    room_goto(rm_Menu);
}

上記のコードは、内蔵のグローバル変数「lives」の値をチェックし、それが0であれば、変数「AI_queue」でインデックスされたDSキューを破棄し、rooms を変更するものです。
