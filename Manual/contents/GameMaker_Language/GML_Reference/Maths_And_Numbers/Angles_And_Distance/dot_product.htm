ドットプロダクト

内積とは、2つのベクトルの角度関係を表す値で、2つのベクトルを取り、掛け合わせ、足し合わせることで求められる。ドットプロダクトという名称は、中央のドット"-"に由来する。 スカラー積はベクトルではなくスカラーであることを強調した名称である）。

実際の数式はこのように書くことができる。

つまり、2Dではベクトル a[x1,y1] と b[x2,2] の内積は x1x2 + y1y2 、GameMakerでのdot_productは次のように計算されます。

a · b = (x1*x2)+(y1*y2);

内積の不思議なところは、入力ベクトルが作る角度と関係があり、次のように表現できることだ。

a · b = (length of a) * (length of b) * cos(angle)

つまり、2つのベクトルの内積は、それらのベクトル間の角度の余弦に、それぞれのベクトルの長さを掛けたものになるのです。以下は、その説明のための画像です。

注：ベクターの詳細については、こちらをご覧ください。

任意の2つのベクトルから、その内積の結果との関係で述べられることがいくつかある。

  • aとbが垂直（互いに90度）であれば、 cos(ang) が0になるため、内積の結果は0になります。
  • aとbのなす角が90度より小さい場合、 cos(ang) 、内積は正（0より大きい）になります。
  • aとbの角度が90度より大きい場合、 cos(ang) 、内積は負（0より小さい）になります。

では、ゲームを作る私たちにとって、これはどういう意味を持つのでしょうか。この数学的関係は様々な状況で利用できますが、それを知るには実際のシナリオを作り、何が起こっているのかを自分で確認するのが一番です。最も簡単な方法のひとつは 例えば、プラットフォームゲームで敵の簡単な「視線」チェックを生成し、敵のベクトル法線の左右90°以内にプレイヤーがいれば「見える」ようにする方法です。

基本的には、敵からの法線ベクトル（方向と視線距離で構成される）を取得する必要があります。 そして、プレイヤーから敵へのベクトルを取得する必要があります。そして、これらのベクトルのdot_productを求め、その結果が正であれば、プレイヤーは見られており、負であれば見られていないことになります。これを実現するための実際のコードは は、以下の例のとおりです。

構文です。

dot_product(x1, y1, x2, y2)

論証タイプ 商品説明
x1 The x coordinate of the first vector.
y1 The y coordinate of the first vector.
x2 The x coordinate of the second vector.
y2 The y coordinate of the second vector.

リターンです。

例

var x1, y1, x2, y2;
x1 = lengthdir_x(1, image_angle);
y1 = lengthdir_y(1, image_angle);
x2 = o_Player.x - x;
y2 = o_Player.y - y;
if dot_product(x1, y1, x2, y2) > 0 seen=true else seen=false;

上記のコードでは、インスタンスの画像の角度を用いてベクトルを作成し、プレーヤーobject "o_Player" のベクトルを自分自身に取得します。最後に、この2つのベクトルの内積を計算し、それが0より大きい場合は変数 "seen" を true に、0 以下なら "false" に設定する。
