ゲームコントロール全般

これらの関数と変数は、ロード/セーブ、再起動など、すべてゲーム固有の機能に関連するものです。

以下のグローバルスコープの ビルトイン変数は、debugging 、問題が発生した場合にログファイルへの出力として使用できます。また、現在のビルドIDなどをユーザーに表示する際にも便利です。

  • game_id
  • game_save_id
  • game_display_name
  • game_project_name

一般的なゲームコントロールには、以下のような機能があります。

  • game_end
  • game_restart
  • game_load
  • game_load_buffer
  • game_save
  • game_save_buffer
  • game_get_speed
  • game_set_speed

ゲームのハイスコアを作成、管理するために、GameMakerは10ヶ所のグローバルなハイスコア配列を作成します。この配列にアクセスし、追加、変更することで、あまり手間なく独自のハイスコア・リストを作成することができます。この機能はゲームにローカルで、すべてのプラットフォームで動作するので、ファイル処理関数と一緒にこれらの関数を使えば、基本的なハイスコア表を簡単に保存、管理することができます。

内部ハイスコア・リストを扱う関数として、以下のものが存在する。

  • highscore_add
  • highscore_name
  • highscore_value
  • highscore_clear

GameMakerには、ハイスコアを画面に一覧で表示する機能もありますが、非常に基本的なもので、自分でハイスコア表を作る方が簡単で、表示方法もはるかに自由にコントロールできるので、何よりもdebugging 、これを使うことをお勧めします。

  • draw_highscore

最後に、マウスカーソルにsprite を割り当てることができる関数を紹介します。

  • cursor_sprite
