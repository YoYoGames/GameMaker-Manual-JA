デバッギング

ゲームを作っているとき、特定の時点におけるゲームの仕組みの詳細を把握したり、ある関数に特定のキー値を入力して、ゲームのさまざまな手法やアイデアを試したりすることが非常に重要になることがよくあります。そのため、GameMakerでは、デバッグモード（またはWindows ターゲット）中にメッセージを表示したり入力を求めたりするデバッグ関数を多数用意していますが、他のターゲットモジュール用にコンパイルすると、これらの関数は入力が解決されるまでゲームを閉じた状態（loop ）にするため、無視されることになります。

注意特定のプラットフォームで動作する場合もありますが、debugging 以外に使用することはできません。

  • debug_mode
  • debug_event
  • debug_get_callstack
  • exception_unhandled_handler
  • get_integer
  • get_string
  • show_error
  • show_message
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiled
  • fps
  • fps_real

GameMakerには、これらの専門的な機能とは別に、ゲームコードの記述時、コンパイル時、ゲーム実行時に表示されるエラーメッセージのリストが用意されています。これらのメッセージは、問題のある特定のエラーを指し示し、特定のobject またはscript 関数内の問題を突き止めるために使用することができ、ゲームのdebugging プロセスに大いに役立ちます。しかし、エラーメッセージが何を意味しているのか、必ずしも明らかではありません。

  • シンタックスエラー
  • コンパイラーエラー
  • ランナーエラー

また、ゲームのデバッグ方法については、「デバッガ」のセクションでさらに詳しく説明しています。
