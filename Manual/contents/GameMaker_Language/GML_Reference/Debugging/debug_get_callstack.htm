debug_get_callstack

この関数は、strings の配列を「コールスタック」として生成します。ここでは、現在のscript が最初にリストされ、次に、現在のscripts が実行されるために実行された他のすべてのscript がリストされます。正確なstring の形式は、選択したターゲットプラットフォームによって異なるが、たいていの場合、スクリプト/イベント名、コロン : 、行番号の順で、このような形式となる。

"gml_Script_script2:1"
"gml_Script_script1:32"
"gml_Script_script0:64"
"gml_Object_object0_Create_0:1"

この関数では、オプションの引数として、返されるコールスタックの最大深度を渡すことができる。この値は、現在のscripts から始まる返される配列に含まれるscript の数である。この引数が指定されない場合、完全なコールスタックが返されます。

コールスタックscripts をリストアップした後、返される配列の最後の位置には常に 0 が含まれることに注意してください。

構文です。

debug_get_callstack([maxdepth])

論証 タイプ 商品説明
[maxdepth] リアル OPTIONAL 返されるコールスタックの最大深度

リターンです。

Array

例

if debug_mode
{
    if (keyboard_check(vk_escape))
    {
        var _a = debug_get_callstack(4);
        for (var i = 0; i < array_length(_a); i++)
        {
            show_debug_message(_a[i]);
        }
    }
}

上記のコードでは、デバッグモードが有効かどうかを確認し、有効な場合は、キーが押されているかどうかを確認します。その場合、現在のコールスタックをコンソールに出力し、最大深度は4です。
