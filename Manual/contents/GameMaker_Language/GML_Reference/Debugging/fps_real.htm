fps_real

GameMakerでは、ゲームの実行速度を伝えるために、主に2つの方法があります。それは game speed(ゲームオプションで指定)と fps (frames per second)です。これらの値はよく混同されますが、基本的に一方はGameMakerが1秒間に完了することになっているゲームステップ数（ゲームスピード）、もう一方はGameMakerが実際に1秒間に完了しているCPUステップ数（実fps）です。この値は一般的にゲームスピードよりずっと高いですが、ゲームがより複雑になり設定したroom スピードを維持するための処理能力がより高くなると下がります。

この読み取り専用変数は、現在のfpsを整数値で返します。この関数はゲームの各ステップごとに一度だけ更新されるので、ある値から別の値に「飛ぶ」ように見えるかもしれませんが、これはごく普通のことであることに注意してください。

注：HTML5ターゲットでは、GameMakerはブラウザにタイミングと配信を依存するため、この変数は代わりにモニターのリフレッシュレートの倍数を返すことになる。

構文です。

fps_real

リターンです。

例

if debug_mode
{
    draw_text(32, 32, "FPS = " + string(fps_real));
}

上記のコードは、ゲームがデバッグモードであるかどうかを確認し、デバッグモードであれば、現在の実fpsを画面に表示します。
