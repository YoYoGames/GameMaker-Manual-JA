show_debug_overlay

この機能は、ゲームのテスト時に標準のデバッグオーバーレイをswitch でオン/オフするために使用でき、デフォルトでは無効になっています。デバッグオーバーレイは、実際のゲームウィンドウにグラフィック CPU/GPU 使用率バー、現在のリアル fps 値、texture スワップ数、頂点バッチ数（texture スワップ数と頂点バッチ数は決してゼロにはならず、通常は 2 か 3 の値を示すことに注意してください、なぜならroom とobjects が空でもGameMaker は描画とバッチを行う必要があります）。

このバーは分割されており、各セクションは1/60秒となっています。上の画像からわかるように、バーは様々な色で構成されています。

  • 緑色- 入力/出力処理（キーボード、マウス、ゲームパッド、ネットワークなど）
  • Red- ステップイベントの更新速度
  • 黄色- 描画イベントに必要な時間
  • オレンジ色-デバッグ更新時間、通常はデバッグモジュール使用時のみ表示されます。
  • 白- GPUレフトオーバー時間。これは、GPUがフレームのレンダリングを終了するのを待って、次のフレームを開始できるようにするために費やされる時間です。
  • Cyan- テキストのレンダリング時間です。
  • グレー- 各描画ステップの画面クリアに必要な時間
  • 青色-ガベージコレクタの実行に必要な時間
  • 暗赤色- GPUフラッシュ。GPUがメモリから画像を消去するのにかかる時間です。

この機能を使用すると、デバッグモードかどうかにかかわらずオーバーレイを追加することができ、この方法で、debugger を並行して実行することによるオーバーヘッドなしに、ゲームの効率的な実行と、利用可能なリソースの使用方法に関する視覚的な手がかりを得ることができるのです。

注：この機能は、HTML5ターゲットでは、他のすべてのターゲットと動作方法が異なるため、動作しません。

構文です。

show_debug_overlay(enable);

論証タイプ 商品説明
enable switch on (true)またはオフ(false) the debug overlay.

リターンです。

例

if global.debug
{
    show_debug_overlay(true);
}
else
{
    show_debug_overlay(false);
}

上記のコードは、グローバル変数の値によってデバッグのON/OFFを切り替えます。
