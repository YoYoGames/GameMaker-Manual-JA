ウェブとHTML5

GameMakerは、インターネットやserver と連携するための様々な機能を提供し、利用可能なすべてのターゲットプラットフォームで使用することができます。これらの関数は

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

HTML5ターゲットには、上記の一般的な機能とは別に、ターゲットとするゲームで使用するために特別に設計されたWeb機能がいくつかあります。

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

これらのHTML5ターゲットモジュールの機能とは別に、稀にhtml5gameフォルダーの中身が全く別の場所（URL）に格納されている場合に使用できる特別なJavaScript変数があります。通常、このpath は JavaScript ファイルにハードコードされており、JavaScript が呼び出された場所からの相対パスとなります。しかし、ゲームscript を呼び出す前に変数 g_GameMakerHTML5Dir を設定すれば、ゲームは新たに定義された場所から実行されます。例えば

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

また、gmcallback_ という接頭辞を使ってflag script 関数を作成すると、HTML5 用にプロジェクトをコンパイルするときに難読化されなくなることも特筆すべき点です。これは、上記の clickable_* 関数を使用する場合に非常に便利です (詳細については、こちらを参照してください)。
