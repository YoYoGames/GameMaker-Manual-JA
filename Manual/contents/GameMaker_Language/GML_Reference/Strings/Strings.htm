ストリングス

ゲームを作っていると、いつの間にかテキストを使う必要が出てきます。string GameMakerには、strings の挿入、string のコピー、strings の数字や文字の解析など、strings をさまざまに操作するための関数が揃っています。一般に、string は二重引用符 " " 内にテキストを追加することによってのみ作成でき、一重引用符strings は受け付けません。また、string を複数行に分割して、改行が改行であるかのようにGameMaker でレンダリングすることもできません（ただし、以下に説明するように、文字列リテラルの @ 識別子が使用されている場合を除きます）。

strings を作成する際には、エスケープ文字の使用に関する一定の規則があることに注意する必要があります。エスケープ文字とは、「\ 」という記号を前につけた文字のことです。たとえば、string の中に引用符を入れたい場合は、次のようになります。

str = "Hello\"World\"";

GameMakerは4バイト幅のUnicode文字も完全にサポートしており、標準の上限のUnicode文字（絵文字を含むがこれに限らない）をデコードおよびエンコードすることができる。Unicode 文字を扱うには、任意の Unicode リテラルの前に "\" を使用することができます。16 進数の数字を "u" の前に置き、例えば "\u00e2" は "á" のように、数字が Unicode 文字の番号になります。このようにUnicodeを扱う場合、GameMakerは"u"に続くすべての数字を解釈するという事実に注意する必要があります。例えば、"áa "と書きたい場合は、次のようにします。

"\u00e2\a"

または

"\u00e2\u61"

または

"\u00e2" + "a"

というのは、"\u00e2a"を使うだけで、実際にはユニコード文字 "ส" になってしまうからです（実質的には "\ue2a" になります）。

GameMakerは16進数のリテラルも扱うことができます。通常、"0x"に続く16進数の数字で、例えば"0xff"のように記述されます。GameMakerでは、これらの文字は "\x" の後に16進数で記述されます。これらのエスケープ文字と、その他の定義済みエスケープ文字は以下の表のとおりです。

定数 商品説明
\n 改行
\r キャリッジリターン
\b バックスペース(0x08)
\f フォームフィード(0x0c)
\t 水平タブ (0x09)
\v 垂直タブ(0x0b)
\\ バックスラッシュそのもの (0x5c)
\a アラート(0x07)
\u[Hex Digits] ユニコード文字の挿入
\x[Hex Digits] 16進文字列の挿入
\[Octal Digits] 8進数のユニコード文字を挿入する

注： Strings はフォームフィード、垂直タブなどをサポートしていますが、これはレンダリングがそうであると言う意味ではなく、strings を描くときにこれらの文字が無視される場合があります。

また、string 全体の前に @ 文字を置くことで、逐語的な文字列リテラルを作成することができます。

var test = @"
Line breaks
over multiple
lines
";

上記のコードでは、改行エスケープ文字が含まれているかのように、string を複数行にわたってレンダリングします。string @"Hello\World" リテラルは、以前のGameMaker バージョンstring リテラルと似ていますが、二重引用符または一重引用符を使用し、 @ シンボルを先頭に付けなければなりません。また、コード ファイル内で複数の行に分割することができ、エスケープ文字をサポートしません。ただし、このようにstring リテラルを使用する場合、string の一部として引用符を含めたい場合は、string を分割する必要があることに注意してください。

var test = @"Hello " + "\"" + @"World" + "\""

もう一つの注意点は、Unicode 文字 9647 (↪So_25AF) は、draw イベントでレンダリングするときに、指定したfont に欠けているグリフを置換するために使用されます。ですから、例えばfont に ° 記号がない場合、90° と書くと実際には 90▯ が生成されます。
