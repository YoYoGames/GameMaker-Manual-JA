クラウドファイルセーブ

この関数は、選択したクラウドサービスにファイルを保存するようコミットします。この関数は一意のID値を返し、それを適切な非同期イベントで使用して、クラウドサービスからの「コールバック」として返される DS マップを識別する必要があります。クラウドには1つの「データブロブ」しか保存できないため、ファイルにはゲームに必要なすべての情報が含まれていなければなりません。 cloud_string_save()関数を使用します)。説明文は、「Level2, Stage2」のように、保存内容を説明する短い情報（string ）である必要があります。

返される非同期データの詳細については、関数 cloud_synchronise().

構文です。

cloud_file_save(file, description);

論証タイプ 商品説明
string The file to be uploaded (as a string).
description A brief description of the data being stored.

リターンです。

Async Request ID

例

var t_str = "";
for (var i = 0; i < 10; i++;)
{
    t_str += string(global.Highscore[i]) + "|"
}
var file = file_text_open_write("Highscores.txt");
file_text_write_string(file, t_str);
file_text_close(file);
save_check = cloud_file_save("Highscores.txt", "Current Highscores");

上記のコードでは、グローバル配列「Highscores」に格納されている値からstring を作成し、このstring をローカルストレージ用のファイルに書き込んでいます。このファイルは、その後クラウドサービスに送信され、保存されます。
