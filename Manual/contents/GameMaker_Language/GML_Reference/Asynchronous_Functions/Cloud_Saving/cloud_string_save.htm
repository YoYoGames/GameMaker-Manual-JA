cloud_string_save

この関数は、選択したクラウドサービスにstring をコミットし、保管する。この関数は一意のID値を返し、それを適切な非同期イベントで使用して、クラウドサービスからの「コールバック」として返される DS マップを識別する必要があります。クラウドには1つの「データブロブ」しか保存できないため、string にはゲームに保存する必要があるすべての情報が含まれているはずです。この関数を再度実行すると、以前に保存した値がすべて上書きされます (同様に cloud_file_save()関数を使用します)。説明文は、「Level2, Stage2」のように、保存内容を説明する短い情報（string ）である必要があります。

返される非同期データの詳細については、関数 cloud_synchronise().

構文です。

cloud_string_save(string, description);

論証タイプ 商品説明
string The data string to be uploaded.
description A brief description of the data being stored.

リターンです。

Async Request ID

例

var t_str = "";
for (var i = 0; i < 10; i++)
{
    t_str += string(global.Highscore[i]) + "|"
}
save_check = cloud_string_save(t_str, "Current Highscores");
var file = file_text_open_write("Highscores.txt");
file_text_write_string(file, t_str);
file_text_close(file);

上記のコードでは、グローバル配列「Highscores」に格納された値からstring 、このstring 、クラウドサービスに書き込んで保存するとともに、ファイルに書き込んでローカルに保存しています。
