HTTP

ここでは、GameMakerで利用できる様々な非同期HTTP関数の一覧を示します。これらの関数は、その関数を持つすべてのインスタンスで非同期HTTPイベントを発生させます。

  • http_request
  • http_get
  • http_get_file
  • http_post_string

なお、上記 http_* の機能は、クロスドメインセキュリティの問題により、期待通りに機能しない場合があります。これは、あなたのserver へのリクエストや、ドメインをまたがるリソースのロードの試みがブロックされ、空白の結果や404エラーが返されるように見える場合があることを意味します。これを回避する方法の1つは、特定のドメインがあなたのserver にアクセスすることを許可するserver サイドの PHP を用意することです（これは、あなたのリソースを保護し、PHP 許可リストに含まれていないservers をブロックする方法でもあります）。

以下に、このために使用できるPHPコードの例を示します。

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
{
    header("Access-Control-Allow-Origin: *");
}

クロスオリジンタイプの決定や設定が重要な画像読み込みリクエストには、以下のような関数があります。

  • http_get_request_crossorigin
  • http_set_request_crossorigin

ほとんどの場合、これらの関数を使用する必要はありませんが、ゲームが安全なserver に保存されている場合（特定のassets にアクセスするには基本認証が必要で、ロード時にセキュリティエラーが発生する場合）、画像リクエストのクロスオリジン型を "use-credentials" に設定する必要があります（デフォルトの "anonymous" 設定とは対照的です）。
