http_set_request_crossorigin

この機能により、ファイルから画像を読み込む際に使用するクロスオリジンの種類を設定することができます。この関数は HTML5 プラットフォーム専用で、 sprite_add() 関数を使用してファイルからsprites を読み込む場合のみ有効です。use-credentials" に設定すると、読み込むpath に絶対的なsprite を指定する必要がなくなり、代わりに相対的な path を指定することになります（以下の例に示すように）。デフォルトでは、クロスオリジンタイプは "anonymous "に設定されています。

構文です。

http_set_request_crossorigin(origin_type);

論証タイプ 商品説明
origin_type The cross origin type to use (a string)

リターンです。

例

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
    http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

上記のコードでは、まず現在設定されているクロスオリジンタイプをチェックし、それが「use-credentials」でなければ「use-credentials」に設定し、ファイルからsprite を追加しています。
