push_get_next_local_notification（プッシュ・ゲット・ネクスト・ローカル・ノーティフィケーション）。

この関数は、queue の次のローカル プッシュ通知を表示するための一連のキー/値ペアを、事前に作成されたDS マップに入力します。キューに追加された通知がない場合は -1 を返し、そうでない場合は通知の ID を表す実数値を返します。このIDは、次に、関数 push_cancel_local_notification().を呼び出す必要があります。 push_get_first_local_notification()を使用してqueue の最初の通知を取得し、その後この関数を使用してそれを続行します。

注：この機能は、iOSおよびAndroidのターゲットモジュールに限定されます。

DSマップは以下のキーと値のペアを保持する。

  • "title" - 通知タイトル（iOS はタイトル情報を保存しないため、これはAndroidプラットフォームでのみ利用可能です）。
  • "message" - メッセージの本文
  • "data" - データパッケージの文字列

構文です。

push_get_next_local_notification(map);

論証タイプ 商品説明
map The DS Map を使用するためのIDです。

リターンです。

Push Notification ID or -1

例:

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

上記のコードは、 DS Map を作成し、queue で最初に利用可能な通知のデータを要求します。通知が存在する場合、マップは入力され、コードは次に特定のstring のために「データ」キーをチェックします。string が見つかった場合、通知はキャンセルされ、queue のすべてがチェックされるまで、次の通知をチェックするために移動します。
