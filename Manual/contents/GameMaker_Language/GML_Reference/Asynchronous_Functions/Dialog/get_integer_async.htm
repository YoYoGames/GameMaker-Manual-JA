get_integer_async

この関数はウィンドウを開き、メッセージと、ユーザーが値を入力するためのスペースを表示します（最初に提供されたデフォルト値が含まれます）。これは非同期関数なので、GameMakerは答えを待つ間、実行中のデバイスをブロックすることなく、通常通りイベントを実行し続けます。ユーザーがstring を入力し、「OK」ボタンを押すと、非同期の ダイアログイベントが発生し、そのイベントの間だけ、DS マップが変数 async_load.

このマップには、"id", "status", "value" の3つのキーが含まれます。 "id" は、関数が呼び出されたときに返された値で、「状態」は、「OK」ボタンが押された場合は true 、メッセージがキャンセルされた場合は false (すべてのターゲットプラットフォームがメッセージをキャンセルできるわけではないので適宜) のいずれかになります。最後に "value" は、ユーザーがstring として入力した整数を返します (入力がなかった場合は、空のstring "" が返されます)。

注意事項この機能は デバッグおよびテスト用であり、リリースされたゲームでは使用しないでください。そのためには、プレイヤーからの入力を受け取るためのカスタムユーザーインターフェースを作成し、ダイアログの外観や動作を完全に制御できるようにする必要があります。

構文です。

get_integer_async(string, default);

論証 タイプ 商品説明
string 文字列 The message to show in the dialog.
default リアル The default value.

リターンです。

Async Request ID

拡張された例。

メッセージを表示しているobject のマウスの左ボタンを押すイベント（例）には、次のようなコードがあります。

msg = get_integer_async("How old are you?", 0);

上記では、ユーザーにstring を入力し、「OK」を押すよう要求するメッセージを表示します。関数IDは変数「msg」に格納され、以下のように非同期ダイアログ・イベントで使用されます。

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        global.Age = ds_map_find_value(async_load, "value");
    }
}

上記のコードでは、返された DS Map の "id" キーと、変数 "msg" に格納されている値を照合しています。もし同じなら、（ウィンドウを閉じたりキャンセルしたのではなく）「OK」が押されたかどうかを調べ、もし true を返したら、返された整数からグローバル変数をセットします。
