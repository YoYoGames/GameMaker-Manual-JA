show_message_async

この関数では、ウィンドウを開き、関数内で定義したメッセージをユーザーに表示します。この関数は非同期関数であるため、GameMakerは回答を待つ間、実行中のデバイスをブロックせず、通常通りイベントを実行し続ける。ユーザーが "OK "ボタンを押すと、非同期的に ダイアログイベントがトリガーされ、そのイベントの間だけ、ds_map が変数 async_load.

id "は関数が呼ばれたときに返された値で、"status "は "Okay "ボタンが押されたときの true 、メッセージがキャンセルされたときの false （すべてのターゲットプラットフォームでダイアログのキャンセルができるわけではないので利用可能）である。

注意事項この機能は デバッグおよびテスト用であり、リリースされたゲームでは使用しないでください。そのためには、プレイヤーからの入力を受け取るためのカスタムユーザーインターフェースを作成し、ダイアログの外観や動作を完全に制御できるようにする必要があります。

構文です。

show_message_async(string);

論証 タイプ 商品説明
string 文字列 The message to show to the user.

リターンです。

Async Request ID

拡張された例。

メッセージを表示しているobject のマウスの左ボタンを押すイベント（例）には、次のようなコードがあります。

msg = show_message_async("You will now be taken to the store");

上記は、与えられたstring のメッセージを表示します。メッセージIDは変数 "msg "に格納され、以下のように非同期ダイアログイベントの中で使用されます。

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if (i_d == msg)
{
    if (ds_map_find_value(async_load, "status"))
    {
        url_open("https://play.google.com/store");
    }
}

上記のコードでは、返された DS Map の「id」キーを、変数「msg」に格納されている値と照合しています。もし同じなら、「OK」ボタンが押されたかどうか（ウィンドウを閉じたりキャンセルしたのではなく）確認し、もし true を返したら、urlを開きます。
