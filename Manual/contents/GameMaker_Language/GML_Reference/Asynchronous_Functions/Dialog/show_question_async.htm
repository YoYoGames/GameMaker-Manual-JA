ショー・クエスチョン・アシンク（show_question_async

この関数はウィンドウを開き、関数内で定義した質問をユーザーに表示します。これは非同期関数なので、GameMakerは回答を待っている間、実行中のデバイスをブロックせず、通常通りイベントを実行し続けます。この関数には "Yes "と "No "の2つのボタンがあり、ユーザーがどちらかを押すと、非同期のダイアログイベントが発生し、そのイベントの間だけ、DSマップが変数 async_load.

このマップには、"id"と"status"の2つのキーが含まれます。" id "は、関数が呼び出されたときに返された値で、" status "は、それぞれ "Yes" と "No" を表す true または false のいずれかになります。

注意事項この機能は デバッグおよびテスト用であり、リリースされたゲームでは使用しないでください。そのためには、プレイヤーからの入力を受け取るためのカスタムユーザーインターフェースを作成し、ダイアログの外観や動作を完全に制御できるようにする必要があります。

構文です。

show_question_async(string);

論証 タイプ 商品説明
string 文字列 The question to ask to the user.

リターンです。

Async Request ID

拡張された例。

メッセージを表示しているobject のマウスの左ボタンを押すイベント（例）には、次のようなコードがあります。

msg = show_question_async("Do you want to buy some armour for " + string(armour[0, 5]) + "coins?");

上記は、与えられたstring で質問を表示し、ユーザーに「はい」または「いいえ」のどちらかを押すように要求します。関数IDは変数「msg」に格納され、次のような非同期ダイアログイベントで使用されます。

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        coins -= armour[0,5];
        global.protection += armour[0,0];
    }
}

上記のコードでは、返された DS Map の「id」キーと、変数「msg」に格納されている値を照合しています。もし同じなら、2つのボタンのうちどちらかが押されたかどうかを調べ、もし true を返したら、変数から値を引き、グローバル変数にも値を追加します。
