get_string_async

この関数はウィンドウを開き、メッセージと、ユーザーがstring を入力するためのスペースを表示します (このスペースには、提供されたデフォルトのstring が最初に含まれます)。これは非同期関数なので、GameMakerは答えを待っている間、実行されているデバイスをブロックせず、通常通りイベントを実行し続ける。ユーザーがstring を入力して "Okay" ボタンを押すと、非同期の Dialog イベントが 発生し、そのイベントの間だけ DS マップが変数 async_load.

idは関数が呼び出されたときに返された値、statusは"Okay "ボタンが押されたときの true 、メッセージがキャンセルされたときの false （すべてのターゲットプラットフォームがメッセージのキャンセルを許可しないため該当する場合）である。最後に "result "は、ユーザーが入力したstring (または、何も入力されなかった場合は空のstring "")が返されます。

注意事項この機能は デバッグおよびテスト用であり、リリースされたゲームでは使用しないでください。そのためには、プレイヤーからの入力を受け取るためのカスタムユーザーインターフェースを作成し、ダイアログの外観や動作を完全に制御できるようにする必要があります。

構文です。

get_string_async(string, default);

論証 タイプ 商品説明
string 文字列 The message to show in the dialog.
default 文字列 The default string.

リターンです。

Async Request ID

拡張された例。

メッセージを表示しているobject のマウスの左ボタンを押すイベント（例）には、次のようなコードがあります。

msg = get_string_async("What's your name?","Anon");

上記では、ユーザーにstring を入力し、「OK」を押すよう要求するメッセージを表示します。関数IDは変数「msg」に格納され、以下のように非同期ダイアログ・イベントで使用されます。

var i_d = ds_map_find_value(async_load, "id");
if (i_d == msg)
{
    if (ds_map_find_value(async_load, "status"))
    {
        if (ds_map_find_value(async_load, "result") != "")
        {
            global.Name = ds_map_find_value(async_load, "result");
        }
    }
}

上記のコードでは、返された DS Map の「id」キーと、変数「msg」に格納されている値を照合しています。もし同じなら、（ウィンドウを閉じたりキャンセルしたのではなく）「OK」が押されたかどうかを調べ、もし true を返したら、string の「結果」をチェックして、グローバル変数を設定する前に空のstrings が返されていないことを確認します。
