ダイアログ

最近のパソコンや携帯端末は、ダイアログウィンドウが表示されたままゲームが停止することを嫌います。ゲームランナーが停止したと解釈し、エラーとみなしてアプリを早々に終了してしまうことがあるようです。GameMakerには 非同期ダイアログ機能があり、ユーザーからのフィードバックや入力を要求することができますが、ゲームランナーをブロックすることはありません。ユーザーが応答すると、ダイアログの非同期イベントが呼び出され、その結果を処理することができます。

注意: 例外として shop_leave_rating()を除いて、これらの関数は デバッグとテスト用にのみ使用され、リリースされたゲームでは使用されるべきではありません。そのためには、プレイヤーからの入力を受け取るためのカスタムユーザーインターフェースを作成し、ダイアログの外観や動作を完全に制御できるようにする必要があります。

以下は、利用可能なすべての非同期ダイアログ関数のリストです。

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
