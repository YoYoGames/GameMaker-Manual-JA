非同期関数

GameMakerの文脈では、非同期関数とは、Webserver に送信したり、ユーザーの入力を求めたり、あるいはデータのストリームを行いながら、実行中のゲームをブロックすることなくGameMakerの実行を継続させるものを指します。情報のコールバックを受け取ると、特別なAsync Eventが発生し、Async Eventが割り当てられたインスタンスがさらに関数を実行できるようになる（ただし、必ずしもそうである必要はない）。つまり、基本的には、ゲームやプロジェクトが応答を待つ間に停止することなく、外部のリソース（server 、ユーザーまたはデバイス）と通信するための方法です。

なぜこれが重要なのか？GameMakerはデータの送受信中も機能し続けることができるので、loop 、ゲームをブロックすることなくいろいろなことができるようになります。

  • プレイヤーがロードを待つことなく、デバイスからゲームにデータをストリームする
  • 正しい情報を受け取ったときだけ処理を行うコールバックイベントがある
  • ウェブサーバーと通信し、データを交換する

これにより、すべてがより流動的で控えめな方法で機能するようになります。非同期イベントは、それを持つすべての インスタンスに対してトリガーされることに注意してください。したがって、あるインスタンスで非同期関数を使用し、別のインスタンスで返信を処理する非同期イベント、あるいは他のさまざまなインスタンスを持つことができます。非同期イベントがどのように動作するかについては、こちら（非同期イベント）を参照してください。
