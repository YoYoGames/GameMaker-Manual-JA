UWPとXbox Live

ここでは、ターゲットプラットフォームがUWPである場合に使用する関数の概要を説明します。最初の関数のセットは、UWP アプリを作成する際に一般的なゲームの動作を制御するためのものです。

  • uwp_suspend
  • uwp_is_suspending
  • uwp_is_constrained
  • uwp_was_terminated
  • uwp_was_closed_by_user
  • uwp_show_help
  • uwp_license_trial_version
  • uwp_license_trial_user
  • uwp_license_trial_time_remaining
  • uwp_check_privilege

UWP では、以下の関数を使用して XBox Live API を使用したXBox One プラットフォーム用のゲームを作成することもできます。これらの関数は、UWP ゲーム オプションの[Enable XBox Live] オプションをオンにする必要があります。これらの関数の中には、ユーザーがXBox Liveにサインインしている必要があるものがありますが、GameMakerはプロジェクトの実行時に自動的にサインインしようとします。この "サイレント "サインインは、システム非同期イベントを生成し、処理を進める前に成功したかどうかを確認することができます。Xbox Oneコンソールのエクスポートとは異なり、XBox Live用のUWPエクスポートでは、一度にサインインできるのは一人のユーザーだけであることに注意してください。

重要!UWP ターゲットは、Creator Program の一環として設定された場合、XBox Live に対する最低限のサポートしかなく、一部の基本的なユーザーおよびアカウント機能に限定されます (設定の詳細については、こちらをご覧ください)。ここに記載されているすべての機能を使用するには、ID@XBoxアカウントが必要です。

UWPエクスポートターゲットで使用するためのXBox Liveの各種機能は、以下のセクションから検索できます。

  • ユーザーとアカウント
  • データの保存
  • 統計とリーダーボード
  • マッチメイキング
