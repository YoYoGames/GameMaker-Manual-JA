xboxlive_matchmaking_find

この機能は、XDPダッシュボードであらかじめ作成しておいたSession Template、matchmaking hopper、Secure Device Associationのテンプレートを使って、マルチプレイヤー・セッションを検索するものです。を呼び出すことで、特定のゲーム・パッドのユーザーIDを取得することができます。 xboxlive_user_for_pad()関数を使用します。この関数は一意のリクエスト ID 値を返し、それを使ってこの関数の正しいソーシャル非同期イベントを特定することができます。このイベントはセッションが作成されたときに起動され、DS Map を変数 async_loadには、以下のキーと値のペアが含まれます。

  • 「requestid" - 呼び出し関数によって返されたリクエストIDを含む。
  • "status" -string "session_created "を含み、どのような種類のイベントが発生したかを知らせます。
  • 「num_results" - セッションが見つかった場合は1以上、そうでない場合は0を格納。
  • "sessionid<index> " - セッションIDを格納、失敗の場合は-1(<index/gt;は0から "num_results" - 1までの値)。
  • "sessionOwner<index> " - セッションが見つかった場合、セッションホストのIDを含みます(<index>は 0 から "num_results" - 1 までの値です)。
  • 「error" - 成功した場合は0，失敗した場合は-1が格納されます．

構文です。

xboxlive_matchmaking_find(user_id, template, hopper, sdatemplate, [matchattributes]);

論証 タイプ 商品説明
user_id XboxユーザーID The user ID pointer to use
template 文字列 The name of the session template
hopper 文字列 The name of the matchmaking hopper
sdatemplate 文字列 The name of the secure device association template
matchattributes リアル OPTIONAL マッチの属性

リターンです。

Async Request ID

例

var userID = xboxone_user_for_pad(global.PadIndex);
result = xboxone_matchmaking_find(userID, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic");

上記は、与えられたゲームパッド上のユーザーのユーザーIDを取得し、そのユーザーのマッチメイキングセッションを探そうとするものです。
