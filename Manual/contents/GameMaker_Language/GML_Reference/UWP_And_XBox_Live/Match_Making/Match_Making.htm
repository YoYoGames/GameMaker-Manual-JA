Xbox LiveとUWPのマッチメイキング（マルチプレイ）には、Xboxデベロッパーポータルでのセッションテンプレートとマッチメイキングホッパーのセットアップによる、ちょっとしたコンフィギュレーションが必要です。GameMakerは、UWPターゲットを使用することでこれをサポートし、以下のことを可能にします。

  • マルチプレイヤーセッションの作成
  • マルチプレイヤーセッションを探す（または参加する
  • 関連するプレイヤーのセッションへの参加・離脱を通知し、そのネットワーク接続情報を受け取ることができます。
  • セッションホストが別のユーザーに移行したときに通知されるようになりました。
  • セッションに参加しているユーザーの一覧を取得する
  • セッションを終了する

マルチプレイのセッションを作成するには、どのセッションテンプレートを使用するかを指定する必要があります。これらのセッションテンプレートはXDPで定義されていますが、以下に例を示します。

Session Name: MatchSession
Contract Version: 107
Session Template (JSON text):
{
    "constants": {
        "system": {
            "version": 1,
            "maxMembersCount": 12,
            "capabilities": {
        },
            "memberInitialization": {
                "externalEvaluation": false,
                "membersNeededToStart": 1
            }
        },
        "custom": {}
    }
}

フィールドの説明は、XDKヘルプファイルの「セッションの概要」に記載されています。なお、現在のところ、GameMakerから動的にセッションパラメータを設定することはサポートしていません。

セッション・テンプレートと一緒に、セッションを検索できるようにマッチメイキング・ホッパーもセットアップする必要があります（これはXDPでも行われます）。ホッパーは、プレーヤーをマッチングするときに使用されるルールを定義するために使用されます。セッション・パラメータと同様、現在のところ、GameMaker.NETからホッパー・パラメータを動的に設定することはサポートしていません。Studioから動的に設定することはできません。詳細は、XDKヘルプの「SmartMatchマッチメイキング設定」を参照してください。

Xbox Liveのマッチメイキングでは、以下の機能が利用できます（接続等には、GameMakerの ネットワーク機能も必要ですのでご注意ください...）。

  • xboxlive_matchmaking_start
  • xboxlive_matchmaking_stop
  • xboxlive_matchmaking_create
  • xboxlive_matchmaking_find
  • xboxlive_matchmaking_session_get_users
  • xboxlive_matchmaking_join_session
  • xboxlive_matchmaking_session_leave

GameMakerには、ネットワーク上の他のプレイヤーをセッションに招待したり、他のプレイヤーのセッションへの招待を受けたりするためのマッチメイキング機能が追加されています。利用できる機能は以下の通りです。

  • xboxlive_matchmaking_send_invites
  • xboxlive_matchmaking_set_joinable_session
  • xboxlive_matchmaking_join_invite

プレイヤーが他のプレイヤーからの招待を受けると、次の2つのうちどちらかが起こります。

  • ゲームが現在実行中であれば、ソーシャル非同期イベントとしてasync_load以下のキー/値を含むDSマップ。
    • "id" - MATCHMAKING_INVITATIONに設定されます。
    • "invitation_id" - これは一意なセッションIDの値です。
    • "invitation_host" - セッションホストの一意のセッションID値です。
    ID と Host の値を保存し、必要なセッションを残して、関数 xboxlive_matchmaking_join_invite() を呼び出し、引数として招待 ID と招待 Host の値を Step イベントで渡す必要があります。
  • 現在ゲームが起動していない場合は、OSによって起動されます。上記と同じキーが設定された、同じタイプのSocial Asynchronous Eventが届きます。それらを変数に保存し、Xbox Liveマッチメイキングに接続し、最後に xboxlive_matchmaking_join_invite() を呼び出します。

xboxlive_matchmaking_join_invite() を呼び出すと、ユーザーはセッションに参加しようとし、このセッションで xboxlive_matchmaking_find() を呼び出したかのように、非同期ソーシャルイベントを受け取ります。
