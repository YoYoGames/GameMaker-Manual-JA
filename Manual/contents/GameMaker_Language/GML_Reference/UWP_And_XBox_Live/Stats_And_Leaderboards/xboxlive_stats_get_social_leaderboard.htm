xboxlive_stats_get_social_leaderboard（ソーシャルリーダーボード）。

この関数は、指定された統計値に対するランクのソーシャルリーダーボードを取得するために使用することができます。ユーザ ID を指定します (たとえば、関数 xboxlive_get_user()) 、統計情報string (xboxlive_stats_set_stat_* 関数で作成したときの定義)、そして取得したいリーダーボード情報についての詳細をいくつか指定します。ソーシャルリーダーボードを取得できるのは、int または real 統計情報のみで、string 統計情報は取得できません。また、flag 引数 "favourites_only" を true にすると、ユーザーが「お気に入り」としてマークしているフレンドのデータのみが結果に含まれることに注意してください。

重要!ソーシャルリーダーボードで使用する統計情報は、XDP/Windows Dev Centerで "Featured Stats "として登録する必要はありません。

この関数は、ソーシャル非同期イベントのトリガーとなるコールバックを生成します。このイベントは、特別な DS マップ async_loadを解析し、以下のキーを取得する必要があります。

  • 「id" - 定数を保持します。 achievement_stat_event
  • "event" -string "GetLeaderboardComplete" を保持します。
  • "userid" - リクエストに関連付けられたユーザー ID。
  • "error" - 成功した場合は 0、エラーが発生した場合は他の値。以下は、最も一般的に返されるエラーです。
    • -2145844848:不正なリクエスト (スタッツが有効なリーダーボードタイプでない、つまりstring)
  • "errormessage" - エラーメッセージがある場合は、string 。
  • "display_name" - プロバイダーダッシュボードで定義された、リーダーボードの一意のIDです。
  • "numentries" - あなたが受け取ったリーダーボードへのエントリーの数です。

DSマップの残りの部分にも、次のような形式のリーダーボードデータが含まれます（「N」はリーダーボードデータ内の位置で、0から「numentries」までです）。

  • "PlayerN" - プレーヤーの名前。"N" は、リーダーボードのエントリーリスト内での位置に対応する整数値です。
  • "PlayeridN" - プレーヤーのユニークなユーザーID、"N".
  • 「RankN" - リーダーボード内でのプレイヤー "N "の順位。
  • "ScoreN" - プレイヤー "N "のスコア。

構文です。

xboxlive_stats_get_social_leaderboard(user_id, stat, num_entries, start_rank, start_at_user, favourites_only);

stat 文字列 The stat (as string) to create the global leaderboard from
num_entries リアル The number of entries from the global leaderboard to retrieve
start_rank リアル The rank in the leaderboard to start from (use 0 if the "start_at_user" argument is set to true)
start_at_user ブーリアン Set to true はユーザーIDのランクから開始、それ以外は (引数 "start_rank" が 0 以外の場合は に設定される) false false
favourites_only ブーリアン Set to true をクリックすると、"お気に入り "としてマークされた友人だけを表示することができます。または、 それ以外の場合は、"お気に入り "をクリックします。 false

拡張された例。

以下に、この関数の使用方法の拡張例を示します。まず始めに、Room StartやCreate などのイベントでこの関数を呼び出します。

xboxlive_stats_get_social_leaderboard(user_id, "GlobalTime", 20, 1, false, false);

上記のコードは、ゲームのグローバルリーダーボードポジションのリストを取得するために呼び出され、ソーシャル非同期イベントのコールバックを生成し、次のように処理します。

if (async_load[? "id"] == achievement_stat_event)
{
    if (async_load[? "event"] == "GetLeaderboardComplete")
    {
        global.numentries = async_load[? "numentries"];
        for(var i = 0; i < numentries; i++;)
        {
            global.playername[i] = async_load[? "Player" + string(i)];
            global.playerid[i] = async_load[? "Playerid" + string(i)];
            global.playerrank[i] = async_load[? "Rank" + string(i)];
            global.playerscore[i] = async_load[? "Score" + string(i]);
        }
    }
}

上記のコードは、ソーシャル非同期イベントで返されたDSマップをチェックし、その「id」がチェック対象の定数と一致する場合、返されたリーダーボードデータによってイベントがトリガーされたかどうかをチェックしてからマップをループし、すべての異なる値をいくつかのグローバル配列に格納しています。
