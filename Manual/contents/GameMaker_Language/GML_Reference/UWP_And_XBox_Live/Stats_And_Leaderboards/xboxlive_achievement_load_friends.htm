xboxlive_achievement_load_friends

この関数は、ログインしているすべてのユーザーの友人に関する情報をserver にリクエストし、コールバックSocial Asynchronous Eventをトリガーします。 async_loadマップに関連するキーと値のペアを入力します。この DS Map のidキーは、正しいコールバックを識別するために使用されます（任意の非同期イベントに対して複数のトリガー関数が存在する可能性があります）。また、定数 xboxlive_achievement_friends_info と、各友人の他の多くのキー/値のペアが対になっています。マップの正確な内容は以下の通りです。

  • 「id" - この関数の場合、以下のようになります。 xboxlive_achievement_friends_info
  • 「FriendN" - 友人の名前。"N" は友人リスト内での位置を表す整数値です。
  • "FriendidN" - 友人の一意のユーザーID、"N"。

構文です。

xboxlive_achievement_load_friends();

拡張された例。

以下のコードは、プレイヤーが自分のゲームアカウントにログインした後に呼び出され、そのユーザーのフレンドのリストを取得します。

xboxlive_achievement_load_friends();

これは、ユーザーの友人に関する情報のリクエストを送信し、次のデータを含む特別な async_load DS マップで非同期コールバックを生成します。

var ident = ds_map_find_value(async_load, "id");
if ident == xboxlive_achievement_friends_info
{
    var numfriends = ds_map_find_value(async_load, "numfriends");
    global.numfriends = numfriends;
    for(var i=0; i < numfriends; i++;)
    {
        global.friendname[i] = ds_map_find_value(async_load, "Friend" + string(i));
        global.friendid[i] = ds_map_find_value(async_load, "Friendid" + string(i));
    }
}

上記のコードは、ソーシャル非同期イベントで返された DS Map をチェックし、その「id」がチェック対象の定数と一致した場合、loops 、マップを介して複数の配列にすべての異なる値を格納します。
