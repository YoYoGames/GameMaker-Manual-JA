xboxlive_stats_flush_user

この関数は、指定されたユーザーの統計データを統計マネージャからライブserver にフラッシュするために使用することができ、server が現在の値で最新であることを保証します。XBoxのドキュメントによると、XBoxはリクエストを制限し、XBox OSも約5分ごとに自動的に統計データをフラッシュするので、開発者はこれをあまり頻繁にコールしないように注意する必要があるとのことです。

注：ユーザーに設定されている統計が無効な場合（統計名にアルファベット以外の文字が含まれているなど）、ユーザーを削除するとエラーが返されることがあります。

この関数を使用するには、（例えば）関数 xboxlive_get_user()および優先度値（優先度が高い場合は1、低い場合は0となる）。この関数は，エラーが発生した場合やユーザIDが無効な場合は-1，関数が正常に呼び出された場合はその他の値を返します．また、この関数は、システム非同期イベントをトリガするコールバックを生成します。このイベントは、特別な DS マップ async_load を持ち、以下のキーについて解析される必要があります。

  • 「id" - 定数を保持します。 achievement_stat_event
  • "eventname" -string "StatisticUpdateComplete" を保持します。
  • "userid" - リクエストに関連付けられたユーザー ID。
  • "error" - 成功した場合は0、エラーが発生した場合は他の値を指定します。
  • "errormessage" - エラーメッセージがある場合は、string 。

統計は、ユーザーを統計マネージャから削除すると、自動的にフラッシュされることに注意してください。詳細は xboxlive_stats_remove_user()をご覧ください。

構文です。

xboxlive_stats_flush_user(user_id, priority);

論証 タイプ 商品説明
user_id XboxユーザーID The user ID (a pointer) to use
priority リアル The prority value (0 or 1)

リターンです。

Real

例

for(var i = 0; i < array_length(user_id); ++i;)
{
    xboxlive_stats_flush_user(user_id[i], 0);
}

上記のコードでは、loops 、保存されているユーザーアカウントIDを調べ、各ユーザーのstat managerデータをフラッシュしています。
