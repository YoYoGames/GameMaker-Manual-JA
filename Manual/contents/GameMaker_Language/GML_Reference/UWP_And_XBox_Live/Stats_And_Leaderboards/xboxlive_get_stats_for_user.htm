xboxlive_get_stats_for_user

この関数は、Xbox Liveservers から特定の統計情報についてのデータを取得するために使用することができます。この関数が返すユーザー ID を入力します。 xboxlive_get_user()次に、ゲームのサービス構成ID（XDPコンソールに表示されているもの）、最後に必要な統計情報です。最大14個の統計情報をリクエストできますが、リクエストの合計長さには制限があるため、最大統計情報数は統計情報名自体の長さに依存するため、14個の統計情報の結果が得られることを保証するものではありません。

この関数は、stat リクエストが送信された場合は 0 を、エラーがあった場合は -1 を返し、コールバックはシステム非同期イベントをトリガーします。このイベントは特別な DS マップ async_loadを解析して、次のキーを探します。

  • "event_type" - この関数によってイベントが発生した場合、string "stat_result" を保持する必要があります。

イベントタイプがこの機能に関連する場合、以下の追加キーも存在する。

  • "user" - 要求された統計情報のユーザーIDを保持します。

キーは要求されたstat名で、値はserver 上のそのstatの値である。 このユーザに対してstatが一度も作成されていない場合(設定するイベントを実行していないため)、そのstatに対する値が返されないことがあることに注意する。ds_map_exists()は、 async_load マップ内の統計情報を確認するために使用することができます)。

リクエストの長さが大きすぎるためにリクエストが失敗した場合、コンソール出力にエラーコードを記載したメッセージが表示されるはずです。

xboxlive_get_stats_for_user - exception occurred getting results - 0x80190190

この場合、非同期イベント DS マップには、値が "0" の "succeeded" キーがあるはずで、この場合は、一度の呼び出しでより少ない統計情報を取得しようとするはずです。

構文です。

xboxlive_get_stats_for_user(user_id, serviceconfig_id, statname1, …);

論証 タイプ 商品説明
user_id XboxユーザーID The user ID pointer.
serviceconfig_id 文字列 The service config file ID
statname1 (2, 3, etc...) ストリングス The stat names to retrieve the information for.

リターンです。

Real

例

var _uid = xboxlive_get_user(0);
var _configid = "00000000-0000-0000-0000-000000000000";
xboxlive_get_stats_for_user(xbuser, _configid, "GameProgress", "CurrentMode");

上記のコードでは、ユーザーIDを取得し、それを使ってそのユーザーの特定の統計情報についての情報を要求しています。
