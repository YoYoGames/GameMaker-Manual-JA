uwp_check_privilege

この関数を使用すると、指定されたユーザが選択された特権を持っているかどうかを確認することができます。また、attempt_resolution 引数を true に設定し、特権が有効でない場合、システムダイアログ (ゲームの中断) を開き、特権を取得するために必要なアカウントのアップグレードなどをユーザに促します。ユーザーが必要なオプションを有効にすると、この関数は真を返します。

この機能では、以下のいずれかを使用する必要があります。 UWP 特権定数を特権要求のために使用します。

UWP 特権定数
定数 商品説明
uwp_privilege_internet_browsing インターネット閲覧が許可されているか確認する
uwp_privilege_social_network_sharing ソーシャルネットワークへの共有が許可されているかどうかを確認する
uwp_privilege_share_kinect_content Kinectコントローラーの共有が許可されているか確認する
uwp_privilege_video_communications ビデオ通信が許可されているかどうかを確認する
uwp_privilege_communications インターネット通信の可否を確認する
uwp_privilege_user_created_content ユーザーが作成したコンテンツへのアクセスが許可されているかどうかを確認する
uwp_privilege_multiplayer_sessions オンラインマルチプレイヤーセッションが許可されているかどうかを確認する
uwp_privilege_sessions オンラインセッションが許可されているかどうかを確認する
uwp_privilege_fitness_upload フィットネスデータのアップロードが許可されているかどうかを確認する

この関数が呼び出されると、XBox One 、コールバックの結果が受信されると非同期システムイベントが発生し、特殊なDSマップを持つことになります。 async_load.そして、このマップを解析して、次のキーを探します。

  • 「event_type" - この関数によってイベントが発生した場合、"check_privilege_result"という値を保持する必要があります。

イベントタイプがこの関数に関連する場合、XBox上でこの関数が呼び出されたときに、以下の追加キーも表示されます。

  • 「result" - 以下のうちの1つまたは複数になります。 UWP 特典結果定数s（複合値）。
    UWP 特権結果定数
    定数 説明 値
    uwp_privilege_result_no_issue そのユーザーの権限に問題はありません 0
    uwp_privilege_result_purchase_required ユーザーは、アクセスするために何か追加のもの（通常はサブスクリプション）を購入する必要があります。 1
    uwp_privilege_result_aborted チェックが中止された 2
    uwp_privilege_result_banned 禁止されている 4
    uwp_privilege_result_restricted 通常、ペアレンタルコントロールによりアクセスが制限されている。 8

  • "privilege" - 関数で使用する上記の定数のいずれかになるので、複数回チェックする場合にどの権限を要求したかを確認することができます。

プロジェクトがXBox One 以外の UWP プラットフォームで実行される場合、非同期イベントはトリガーされず、代わりにこの関数は直ちに -1、0、または 1 のいずれかを返します。

  • -1 は、-1 ユーザーが権限を持っていないことを意味します。
  • 0は、この関数がユーザーに問い合わせることができなかったことを意味します。
  • 1は、そのユーザーが権限を持っていることを意味します。

構文です。

uwp_check_privilege(user_id, privilege, attempt_resolution);

論証 タイプ 商品説明
user_id XboxユーザーID The User ID pointer to check.
privilege UWP 特権定数 The privilege constant to check (see description, above).
attempt_resolution ブーリアン Enable attempting a resolution if set to trueまたは、 false に設定されている場合は無視します。

リターンです。

Real

例

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
{
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
}

上記は、ユーザーがサインインしているかどうかを確認し、サインインしている場合は、マルチプレイヤー・セッションのパーミッションを確認するものです。
