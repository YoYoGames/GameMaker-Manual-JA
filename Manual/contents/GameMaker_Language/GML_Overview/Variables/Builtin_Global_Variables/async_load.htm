非同期ロード

この変数はグローバルなスコープで、非同期イベントで使用されるときはDSマップを保持し、それ以外のときは-1するために使用されます。DS マップの実際の内容は、非同期イベントのコールバックの種類と、イベントの トリガーとして使われた関数に依存するので、可能なすべての DS マップの内容の詳細については、それらのイベントの個々のページを 参照してください。

構文です。

async_load;

リターンです。

DS Map ID

拡張された例。

sprite =  sprite_add("site.com/path/image.png", 16, true, true, 0, 0);

上記のコードは、外部URLからsprite を読み込むイベントの中で呼び出されることになります。これがImage LoadedAsynchronous Event のトリガーとなり、 async_load マップをパースすることになります。

if ds_map_find_value(async_load, "id") == sprite
{
    if (ds_map_find_value(async_load, "status") >= 0)
    {
        sprite_index = sprite;
    }
}

上記のコードでは、まず async_load マップの ID をチェックし、次にコールバックのステータスをチェックします。値が0以上であれば（成功を意味する）、コールバックの結果を使用して、インスタンスのsprite インデックスを新しく読み込まれた画像に設定します。
