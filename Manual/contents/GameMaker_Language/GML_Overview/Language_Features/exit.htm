出口

exit 文は次のような構文です。

exit;

exit は、単に現在のスクリプト関数の実行を終了させるだけです。 methodまたは event.ここでは、スコープによって若干の使い分けがあることに注意してください。

  • script 関数やメソッドの中で exit を使用すると、単に関数を終了して、関数を呼び出したコードに戻ります。
  • オブジェクトのイベント内のコードブロックで exit を使用すると、 exit が呼ばれた後に様々な別のコードブロックがあったとしても、イベント全体を終了させることができます。
  • 親イベントで exit を使用し、そのイベントを event_inherited()を子イベントobject で呼び出すと、親イベントだけが終了し、子イベント ( event_inherited() を呼び出したもの) は継続されます。

イベント内で使用する場合、 exit は通常、特定の条件が満たされた（または満たされなかった）ときに、インスタンスがそれ以上のコードを実行しないようにするために使用されます。以下のコードは、どのようなイベントでも使用できますが、この場合は衝突イベントでの使用例を示しています。

if (!visible)
{
    exit;
}

other.hp -= attack;
other.coins -= 4;
coins += 4;

上記のコードは、現在のインスタンスが表示されていないかどうかをチェックし、その場合はコードブロックを終了し、それ以外の場合は先に進んで残りのコードを実行しています。

注意：ゲームの実行を終了させるものではありません。そのためには関数 game_end().
