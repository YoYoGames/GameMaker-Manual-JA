コードの基本構造

コードはブロック単位で記述され、典型的なコードブロックはステートメントと呼ばれる一連の命令で構成されています。ステートメントはGameMakerによって解釈され、ゲーム内で何かを起こすために使用されます。その「何か」は、2と2を足して4にするような単純なものから、敵の体力が一定値以下になったら逃げさせるような複雑なものまであります。実際のプログラムの構造は、使用する関数によって大きく異なりますが、基本的にはこのような感じです。

<statement>;
<statement>;
...

ステートメントは、変数宣言の誤りを防ぎ、コードをきれいに整理するために「;」記号で区切る必要があり、変数宣言、式、特定の関数の呼び出しで構成することができます。また、次の条件の例のように、中括弧 {} を使ってステートメントをブロックとして「グループ化」して、一緒に実行することもできます。

if (<conditional>)
{
    <statement>;
    <statement>;
    ...
}

注 GameMaker Language は、中括弧 {} の代わりに begin と end も受け付けますが、これは一般的に最も一般的な方法とは言えません。

if (<conditional>)
begin
    <statement>;
    <statement>;
    ...
end

ここでは、GameMaker Script Editorで スクリプトとして作成したコードブロックを、より視覚的に表現しています。

ステートメント、式、条件文、関数など、さまざまな種類がありますが、これらはすべて、本書の後続のセクションで詳しく説明します。

注：プログラミングに慣れていない方は、先に『クイックスタートガイド』をご覧ください。
