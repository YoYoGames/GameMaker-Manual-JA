ランタイムファンクション

関数の一般的な定義は次のようなものです。

関数は入力と出力を持ち、出力は入力と関連している。

GameMakerでは、これは2つの異なるものに変換されますが、どちらも同じように機能します。

  • ランタイム関数-ランタイム関数とは、GameMaker言語(GML)の一部として提供されているものです。ランタイム関数は、GameMaker 言語に組み込まれており、ゲームの中で何かを実現するために使用することができます。GML runtime 関数は非常にたくさんあり、マニュアルの言語リファレンスセクションで説明されています。
  • スクリプト関数とメソッド スクリプト 機能と Mメソッド変数は、GameMaker 言語の基本構成要素 (GML) を使って自分で作った関数です。これらは、コードの一部としてruntime の関数を含むことができます。

上記のいずれの場合も、関数は関数名と、それに続く入力引数をカンマで区切った大括弧 () で囲んだ形式をとります（関数に入力引数がない場合は、大括弧だけを使用します）。このページでは、ランタイム関数（つまり、GameMaker に組み込まれてGameMaker Language を形成する関数）に焦点を当てていますが、以下に示す一般的なルールは、すべてのタイプの関数に適用されます。

ここでは、関数の構造の概略を説明します。

<function>(<arg0>, <arg1> ,... <argN>);

関数には、値を返して式の中で使えるものもあれば、次の2つのruntime 関数の例で示すように、単にコマンドを実行するものもあります。

// Destroy the calling instance
// This requires no arguments and returns nothing
instance_destroy();

// Get the distance from the current instance position to the mouse position
// This takes four arguments and returns a real value
dist = point_distance(x, y, mouse_x, mouse_y);

注意このマニュアルでは、関数の戻り値として "N/A "がある場合、その関数は操作の一部として値を返すことを意図していないことを意味します。これらの関数の戻り値を確認しようとすると、 undefined という値が返されます。

関数の戻り値が代入に使いたい値である場合もありますが、どんな関数も代入の左辺として直接使うことはできないので注意が必要です。例えば、インスタンスに速度を設定する次のようなコードはエラーになります。

instance_nearest(x, y, obj).speed = 0;

このコード例の式の戻り値は整数値（最も近いインスタンスのユニークなID値）なので、このように使用して必要なインスタンスのアドレスを適切に指定するには、括弧で囲む必要があります（詳細は「他のインスタンスにおける変数のアドレス指定」を参照してください）。上記のコードは、正しく次のように記述されます。

(instance_nearest(x, y, obj)).speed = 0;

//or more correctly still

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

マニュアルの言語リファレンスには、runtime 関数の一覧とその使用例、および関数の戻り値やトリガーとなるイベントなどに関する重要な情報が記載されています。ユーザー定義のscript 関数とメソッドの詳細については、「スクリプト関数」と「メソッド変数」のセクションを参照してください。
