コメントコード

他の人と共同作業をするとき、大きなプロジェクトを扱うとき、あるいは自分自身の参考のためにdebugging 、イベントコードにコメントを残すことは非常に重要です。scripts 。GameMakerでは、現在編集中のobject のイベントリストで、コードの一部やブロック全体について、メモやコメントを残すことができるさまざまな仕組みがあります。このようにして、自分自身や同僚にメモを残したり、特に難しいコードの部分を説明したり、あるいはただ、何かが何をするのか、あるいは将来的に何をさせたいのかについてのリマインダーを残したりすることができます。

簡単な1行コメントを残すには、まず本文の前に// 。例えば

//initialize variables
sr = 10;
hp = 100;

また、複数行のコメントを残して、謝意を表したり、コードの一部分を省略してdebugging 、あるいは将来的にコードをどのように変更したいかについてのメモを残すこともできます。このような場合は /* ... */のようにします。

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

コメントとは別に、特別な #region タグを使って、コードの一部をコードフォールディング用にマークすることもでき、各領域にコメントを含めることができます。この仕組みは、 #region タグを、任意のコメントテキストとともにコードのセクションの先頭に追加し、対応する #endregion タグをセクションの下部に追加すると、コードを「折りたたむ」、つまり、エディタで一度に見えるコードの行数を削減するためにコードを隠すことができるようになります。

実際のコードは次のようなものです。

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

最後に、オブジェクトエディタでイベントを扱う場合、イベントの最初に特別な "description" というコメントを付けると、イベントの大まかな内容を示すことができ、これがObject Editor のイベントウィンドウに表示されることをお伝えしておきます。このコメントは、コードエディターの最初の行に記述します。

/// @description <This Is An Event Description>

イベントウィンドウに表示されるようになります（例）。

Object Editor Preferencesから、すべてのイベントに対して作成されるデフォルトのコメントを定義できることに注意してください。

最後に、スクリプト関数を書いていて、それがIDE でどのように扱われるかを制御したい場合（たとえば、オートコンプリートやscript パラメータの表示など）、ここで説明するように、JSDoc記法を使用してコメントを書く必要があります。
